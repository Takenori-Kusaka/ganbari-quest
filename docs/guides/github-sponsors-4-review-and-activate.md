# GitHub Sponsors 設定ガイド (4/4) — 審査提出とスポンサーボタンの有効化

## 概要

全ての設定が完了したら、プロフィールを審査に提出し、通過後にリポジトリでスポンサーボタンを有効化します。

**所要時間**: 提出作業5分 + 審査待ち（通常6-8営業日、早ければ1日以内）

---

## 前提条件

審査を提出するには、以下の**全て**が完了している必要があります:

- [ ] ガイド 1/4: プロフィール作成（Short bio + Introduction + Featured work）
- [ ] ガイド 2/4: Tier 作成（最低1つ公開済み）
- [ ] ガイド 3/4: W-8BEN 税務書類の提出
- [ ] Stripe Connect アカウントの作成・銀行口座の登録
- [ ] 2FA（二要素認証）の有効化

---

## Step 1: 審査前の最終チェック

1. https://github.com/sponsors/Takenori-Kusaka/dashboard にアクセス
2. 左サイドバー上部 → **「Overview」**
3. 各セクションの横にチェックマーク（✓）が付いていることを確認:

```
✓ Profile details       — プロフィール設定済み
✓ Sponsor tiers         — Tier 作成済み
✓ Stripe Connect        — 決済アカウント接続済み
✓ Tax forms             — W-8BEN 提出済み
✓ Two-factor auth       — 2FA 有効
```

> **どれか1つでも未完了の場合**: 該当する項目をクリックして設定を完了してください。全てにチェックが付くまで審査は提出できません。

---

## Step 2: 審査に提出

1. Overview ページの下部、または右側に表示される **「Submit for review」** ボタンをクリック
2. 確認ダイアログが表示されたら **「Submit」** をクリック
3. ステータスが **「Under review」** に変わることを確認

---

## Step 3: 審査を待つ

| 項目 | 内容 |
|------|------|
| **通常の審査期間** | 6-8営業日（祝日・週末を除く） |
| **早い場合** | 1日以内（日本の開発者の実績あり） |
| **通知方法** | メールで審査結果が届く |
| **2週間以上待つ場合** | GitHub Support に問い合わせ |

### 審査を早めるコツ

- Introduction が詳しく書かれていること
- Featured work にアクティブなリポジトリが設定されていること
- GitHub プロフィール全般が充実していること（README、ピン留めリポジトリ等）

---

## Step 4: 審査通過後 — Payout 設定の確認

審査通過メールを受け取ったら、Payout（入金）設定を確認します。

1. ダッシュボード → **「Payouts」** または Stripe Connect ダッシュボード
2. 以下を確認:

| 項目 | 推奨設定 |
|------|---------|
| **入金先銀行口座** | メガバンク推奨（三菱UFJ、三井住友、みずほ） |
| **入金スケジュール** | 月次（Monthly）推奨 |

> **対応していない銀行**: ゆうちょ銀行、じぶん銀行は Stripe Connect 非対応です。メガバンクまたは対応しているネット銀行を指定してください。

---

## Step 5: リポジトリでスポンサーボタンを有効化

### 5-1: リポジトリ設定

1. https://github.com/Takenori-Kusaka/ganbari-quest にアクセス
2. **「Settings」** タブをクリック
3. 左サイドバー → **「General」**
4. ページをスクロールして **「Features」** セクションを探す
5. **「Sponsorships」** にチェックを入れる

> ここにチェックが入っていないと、FUNDING.yml が存在してもスポンサーボタンは表示されません。

### 5-2: FUNDING.yml の確認

`.github/FUNDING.yml` がデフォルトブランチ（main）に存在し、以下の内容であることを確認:

```yaml
github: [Takenori-Kusaka]
```

> 既に設定済みのはずですが、念のため確認してください。

---

## Step 6: ゴール設定（推奨）

スポンサーページに目標を表示し、進捗が見えるようにします。

1. ダッシュボード → **「Your goals」**
2. **「Create a goal」** をクリック
3. 以下のいずれかを設定:

| タイプ | 推奨設定 |
|--------|---------|
| **スポンサー人数目標** | `3 sponsors`（最初は達成しやすい目標） |
| **月間金額目標** | `$10/month`（小さく始める） |

4. **ゴールの説明**（任意）:

```
最初の3人のスポンサーを目標にしています。
サーバー運用費の一部をカバーし、プロジェクトの持続性を高めます。
```

> **ポイント**: 達成しやすい小さな目標から始めましょう。達成後に次の目標を設定できます。プログレスバーが表示されるため、支援者にとって「あと少し」という気持ちを引き出せます。

---

## Step 7: 最終確認

### スポンサーボタンの確認

1. https://github.com/Takenori-Kusaka/ganbari-quest にアクセス
2. リポジトリ名の右側に **ハートアイコン付きの「♡ Sponsor」ボタン** が表示されていることを確認
3. クリックするとスポンサーオプション（GitHub Sponsors リンク）が表示されることを確認

### スポンサーページの確認

1. https://github.com/sponsors/Takenori-Kusaka にアクセス
2. 以下が正しく表示されていることを確認:
   - [ ] Short bio（一行紹介文）
   - [ ] Introduction（詳細紹介文）
   - [ ] Featured work（ganbari-quest）
   - [ ] 月額 Tier 一覧（$1 / $5 / $10 / $25）
   - [ ] ワンタイム Tier 一覧（$5 / $15 / $100）
   - [ ] ゴール進捗バー

### 別アカウントでの確認（推奨）

- ログアウト状態、または別の GitHub アカウントでスポンサーページにアクセス
- 「Sponsor」ボタンをクリックし、支援フローが開始されることを確認
- （実際に課金する必要はありません。フローの開始が確認できれば OK）

---

## 設定完了！

全ての手順が完了すると、以下の状態になります:

```
✅ GitHub Sponsors プロフィールが公開されている
✅ スポンサーが月額/単発で支援を選択できる
✅ リポジトリに「Sponsor」ボタンが表示されている
✅ 支援金は Stripe Connect → 日本の銀行口座に入金される
✅ 日米租税条約により源泉徴収 0%
```

---

## 運用メモ

| 項目 | 内容 |
|------|------|
| **W-8BEN 再提出** | 3年ごとに必要。期限前に GitHub からメール通知あり |
| **確定申告** | 年間20万円超で申告義務。雑所得 or 事業所得として申告 |
| **Tier の変更** | 価格変更は不可。退役→新規作成で対応 |
| **README スポンサー掲載** | Tier 特典に含めた場合、手動で README を更新する |
| **GitHub 手数料** | 個人スポンサーからは 0%、組織スポンサーからは最大 6% |
| **Stripe 手数料** | 約 2.9% + $0.30 + 国際取引 1.5% |
