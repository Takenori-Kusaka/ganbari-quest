# Discord サーバー設定ガイド — がんばりクエスト

## 概要

このガイドに従って Discord サーバーを開設し、ユーザーコミュニティ・問い合わせ・スポンサー活動の基盤を整えます。  
設定が完了したら、各 Webhook URL を GitHub Actions 変数および AWS SSM パラメータに登録します。

---

## Step 1: サーバー作成

1. Discord にログインし、左サイドバー最下部の **「+」** ボタンを押す
2. **「自分のサーバーを作成する」**→ **「自分のグループや友人のために」**
3. サーバー名: `がんばりクエスト`
4. サーバーアイコン: ロゴ画像（`static/icons/icon-512.png`）をアップロード
5. **「作成」**

---

## Step 2: カテゴリとチャンネルの構成

以下の構成でカテゴリとチャンネルを作成します。  
各チャンネル名の横に **[閲覧]** / **[投稿]** / **[非公開]** を示します。

```
📢 お知らせ
  ├── #お知らせ           [全員: 閲覧のみ / 運営: 投稿可]
  └── #アップデート情報   [全員: 閲覧のみ / Bot投稿のみ]

💬 コミュニティ
  ├── #はじめに           [全員: 閲覧のみ / 運営: 投稿可]
  ├── #雑談               [全員: 閲覧・投稿可]
  └── #使い方・質問       [全員: 閲覧・投稿可]

📬 フィードバック
  ├── #機能要望           [全員: 閲覧・投稿可]
  └── #バグ報告           [全員: 閲覧・投稿可]

🌟 スポンサー
  └── #スポンサー限定     [スポンサーロール: 閲覧・投稿可 / 一般: 不可]

🔧 開発（運営内部）
  ├── #deploy-log         [非公開 / Botのみ投稿]
  └── #お問い合わせ受信   [非公開 / Botのみ投稿]
```

---

## Step 3: 各チャンネルの詳細設定

### 📢 #お知らせ

| 項目 | 設定値 |
|-----|-------|
| チャンネルトピック | 「がんばりクエストの重要なお知らせを投稿します。重要な変更・障害情報はここで確認できます。」 |
| 低速モード | 無効 |
| 権限 | @everyone: メッセージ送信 ❌ / メッセージ閲覧 ✅ |

使用目的: 障害情報・大幅な仕様変更・サービス終了予告など、重要事項の告知専用。

---

### 📢 #アップデート情報

| 項目 | 設定値 |
|-----|-------|
| チャンネルトピック | 「新機能・改善・バグ修正のリリースノートを自動投稿します。」 |
| 権限 | @everyone: メッセージ送信 ❌ / メッセージ閲覧 ✅ |

使用目的: GitHub Actions の **deploy.yml** から本番デプロイ成功時に自動投稿。  
→ このチャンネルの Webhook URL を `DISCORD_WEBHOOK_URL`（GitHub Actions vars）に設定する。

> **⚠️ 注意**: セキュリティのため「内部デプロイログ」は **#deploy-log**（非公開）に分けています。  
> このチャンネルの Webhook はユーザー向けの成功/失敗メッセージのみ投稿します。

## Step 4: ロールの作成

| ロール名 | 色 | 用途 |
|---------|---|------|
| 管理者 | 赤 | オーナー・運営スタッフ |
| スポンサー | 金 | GitHub Sponsors 支援者（手動付与 or Bot連携） |
| ユーザー | 青 | 一般コミュニティメンバー |

`@everyone` のデフォルト権限から「メッセージ送信」を外し、  
`ユーザー` ロールに送信を許可することで、初見スパムを防げます（任意）。

## Step 5: Webhook の作成方法

Webhook は **チャンネルごとに個別に作成**します。

### 手順（チャンネルごとに繰り返す）

1. 対象チャンネルを右クリック →「チャンネルを編集」
2. 左メニュー「連携サービス」→「ウェブフックを作成」
3. 名前: `がんばりクエスト Bot`（用途がわかる名前）
4. アバター画像: ロゴ画像を設定（任意）
5. 「ウェブフックのURLをコピー」でURLを取得 →　下表に記録

| 用途 | チャンネル | URL の保存先 |
|-----|---------|------------|
| ユーザー向けデプロイ通知 | `#アップデート情報` | GitHub Actions `vars.DISCORD_WEBHOOK_URL` |
| 内部デプロイログ | `#deploy-log` | GitHub Actions `vars.DISCORD_WEBHOOK_URL`（上と兼用可） |
| アプリ内問い合わせ | `#お問い合わせ受信` | AWS SSM `FEEDBACK_DISCORD_WEBHOOK_URL` |

> **セキュリティ**: Webhook URL は**シークレットに近い扱い**をしてください。  
> URL が漏洩すると第三者がチャンネルに任意投稿できます。  
> 漏洩した場合はすぐ「ウェブフックを削除」して再作成してください。

## Step 6: GitHub Actions 変数の登録

[リポジトリ Settings → Secrets and variables → Actions → Variables タブ](https://github.com/[your-org]/ganbari-quest/settings/variables/actions)

| 種別 | 変数名 | 値 | 備考 |
|-----|-------|---|------|
| **Variable** | `DISCORD_WEBHOOK_URL` | `#アップデート情報` or `#deploy-log` の Webhook URL | `vars.DISCORD_WEBHOOK_URL` として参照 |

> **Variables（vars）vs Secrets の使い分け**:  
> Webhook URL は Git ログ等には残りませんが、Actions の実行ログに表示される可能性があります。  
> `vars` でも機能しますが、漏洩リスクをより下げたい場合は `secrets.DISCORD_WEBHOOK_URL` に変更し、`deploy.yml` の参照箇所も更新してください。

---

## Step 7: AWS SSM パラメータの登録（本番環境用）

アプリの `FEEDBACK_DISCORD_WEBHOOK_URL` は SSM パラメータとして登録します。

```bash
aws ssm put-parameter \
  --name "/ganbari-quest/prod/FEEDBACK_DISCORD_WEBHOOK_URL" \
  --value "https://discord.com/api/webhooks/XXXXX/YYYYY" \
  --type "SecureString" \
  --region ap-northeast-1
```

NUC サーバー（ローカル）の場合は `C:\Docker\ganbari-quest\.env` に追記:

```env
FEEDBACK_DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/XXXXX/YYYYY
```

## Step 8: 動作確認

### アプリ内フィードバックの確認
1. アプリに親としてログイン
2. 「設定 → お問い合わせ」からフィードバックを送信
3. Discord の `#お問い合わせ受信`（非公開）に Embed が届くことを確認

### デプロイ通知の確認
1. main ブランチに push（または GitHub Actions を手動トリガー）
2. `#アップデート情報` or `#deploy-log` にデプロイ結果が届くことを確認

## Step 9: ユーザー向け告知（コミュニティを育てる）

## Step 10: スポンサー活動の設計（GitHub Sponsors 連携）

Discord の `#スポンサー限定` チャンネルを GitHub Sponsors の特典として訴求します。

### GitHub Sponsors tier 設定例

| Tier | 月額 | 特典 |
|-----|-----|-----|
| ☕ コーヒー一杯 | $5 | お名前を THANKS.md に掲載 |
| 🌟 サポーター | $15 | Discord `#スポンサー限定` チャンネルへの参加 + 機能要望への優先返答 |
| 💎 パトロン | $50 | 上記 + 開発ロードマップへの意見参加（月1回の相談 DM） |

スポンサー確認後、Discord で `スポンサー` ロールを手動付与（月1回の棚卸しを推奨）。  
自動化する場合は GitHub Sponsors Webhook + Discord Bot の連携が必要（将来対応）。
