# 活動マスタ拡充ガイドライン

Copilot / Claude Code が活動を追加・修正する際の判定基準ドキュメント。

---

## 1. カテゴリ定義（5カテゴリ）

| カテゴリ | 説明 | 具体例 |
|---------|------|-------|
| `うんどう` | 身体を動かす活動全般 | 体操、ランニング、水泳、部活動 |
| `べんきょう` | 学習・知識習得に関する活動 | ひらがな練習、ドリル、読書、テスト勉強 |
| `せいかつ` | 日常生活の習慣・お手伝い | 歯磨き、着替え、食事準備、掃除 |
| `こうりゅう` | 対人関係・社会性に関する活動 | 挨拶、友達と遊ぶ、チーム活動 |
| `そうぞう` | 創造性・表現に関する活動 | お絵かき、工作、楽器演奏、プログラミング |

**判定の優先順位**: 活動が複数カテゴリに該当する場合、主目的に基づいて判定する。
例: 「ダンス発表会」→ 主目的が表現なら `そうぞう`、体力づくりなら `うんどう`。

---

## 2. 年齢帯（gradeLevel）

| gradeLevel | 対象年齢 | ageMin | ageMax | 出典 |
|-----------|---------|--------|--------|------|
| `baby` | 0〜2歳 | 0 | 2 | 保育所保育指針 |
| `kinder` | 3〜5歳 | 3 | 5 | 幼稚園教育要領 |
| `elementary_lower` | 6〜9歳 | 6 | 9 | 小学校学習指導要領（低学年） |
| `elementary_upper` | 10〜12歳 | 10 | 12 | 小学校学習指導要領（高学年） |
| `middle_school` | 13〜15歳 | 13 | 15 | 中学校学習指導要領 |
| `high_school` | 16〜18歳 | 16 | 18 | 高等学校学習指導要領 |

**ageMin / ageMax の設定ルール**:
- `ageMin` は「この活動ができる最低年齢」を設定
- `ageMax` は「この活動の対象上限年齢」。上限がない場合は `null`
- 幅広い年齢で適用可能な活動（例: 読書）は、開始可能年齢のみ設定し `ageMax = null`

---

## 3. ポイント設計基準

| ポイント | 基準 | 具体例 |
|---------|------|-------|
| 3点 | 毎日の基本的な生活習慣 | 歯磨き、挨拶、早寝早起き |
| 5点 | 日常的な学習・活動 | ドリル、読書、外遊び、お手伝い |
| 10点 | 計画的努力が必要な活動 | スポーツ練習、調理実習、観察記録 |
| 15点 | 高い集中力・継続力が必要 | テスト勉強、部活動、ボランティア |
| 20点 | 特別な成果物を伴う活動 | 探究レポート発表、自由研究完成 |

**設計原則**:
- 難易度と所要時間に比例してポイントを設定
- 毎日継続すべき習慣は低ポイント（3〜5点）にし、ストリークボーナスで動機づけ
- 週1回程度の特別な活動は高ポイント（10〜15点）

## 4. source（出典）の設定

| source | 用途 |
|--------|------|
| `seed` | 初期データ（v1.0 の 15件） |
| `curriculum` | 学習指導要領に基づく活動 |

| `parent` | 保護者が手動追加した活動 |

## 5. subcategory の命名規約

サブカテゴリはカテゴリ内の細分類。ひらがな or 漢字で簡潔に。

| カテゴリ | subcategory 例 |
|---------|---------------|
| `うんどう` | 体操, 球技, 水泳, ランニング, 武道 |
| `べんきょう` | 国語, 算数, 理科, 社会, 英語, 読書, プログラミング |
| `せいかつ` | 清潔, 食事, 片付け, 料理, 安全 |
| `こうりゅう` | 挨拶, 協力, 思いやり, ボランティア |
| `そうぞう` | 絵画, 工作, 音楽, ダンス, 創作 |

---

## 6. アイコン選定ガイドライン

- 1活動1絵文字。活動内容を直感的に表すものを選ぶ
- 対象が子供のため、分かりやすさを優先
- 同カテゴリ内でアイコンが被らないようにする
- 使用中のアイコンは `seed.ts` を参照して重複を避ける

---

## 7. 活動追加時のチェックリスト

新しい活動を追加する際に確認すること:

1. **既存の類似活動がないか** — `seed.ts` の活動名一覧を確認
2. **カテゴリが適切か** — セクション1の定義に基づいて判定
3. **年齢帯が適切か** — 発達段階に合った ageMin/ageMax を設定
4. **ポイントが基準に沿っているか** — セクション3のポイント設計基準に準拠
5. **アイコンが重複していないか** — 既存活動のアイコンと被らない
6. **description が 200文字以内か** — バリデーション制約
