## 1. 本書の目的

がんばりクエストの Discord サーバーを**有料ユーザー向けコミュニティ・サポートチャネル**として運用するための設計を定義する。

サーバー構築・ロール設定・チャンネル作成は手動作業であり、本書はその作業手順と設計根拠を文書化したものである。

### 1.1 背景

- 現状：単一招待リンクで全員が同じアクセス権を持っており、有料ユーザー限定サポートの方針（19-プライシング戦略書 §3）と矛盾している
- 目標：ロールベースのアクセス制御により、プランに応じたサポート体験を提供する

### 1.2 方針

| ユーザー区分 | Discord アクセス | 根拠 |
|-------------|-----------------|------|
| 無料ユーザー | アクセス不可（将来的に read-only 領域を検討） | サポートコスト抑制 |
| スタンダードプラン | コミュニティ + 質問チャンネル | 19-プライシング戦略書 §3 |
| ファミリープラン | コミュニティ + 質問チャンネル + 優先対応 | 同上 |
| GitHub Sponsors | スポンサー限定チャンネル | OSS 支援者への特典 |

## 4. Webhook・外部連携

### 4.1 Webhook 一覧

| チャンネル | Webhook 用途 | GitHub Actions 登録先 | 種別 |
|-----------|-------------|---------------------|------|
| 📢 アップデート情報 | Gemini 生成リリースノート | Variable: `DISCORD_RELEASE_NOTES_WEBHOOK_URL` | Variable |
| 🔧 deploy-log | デプロイ成否通知 | Variable: `DISCORD_WEBHOOK_URL` | Variable |
| 📩 お問い合わせ受信 | アプリ内フィードバックフォーム | Secret: `FEEDBACK_DISCORD_WEBHOOK_URL` | Secret |
| 📩 お問い合わせ受信（メール） | support@ メール受信通知 | Secret: `DISCORD_WEBHOOK_SUPPORT` | Secret |

### 4.2 Webhook アーキテクチャ

```
┌────────────────────────────────────────────────────────────────────┐
│                    Webhook データフロー                              │
│                                                                    │
│  [GitHub Actions]                                                  │
│    ├─ deploy.yml notify job                                        │
│    │   └─ DISCORD_WEBHOOK_URL ──→ 🔧 deploy-log                   │
│    ├─ deploy.yml release-notes job                                 │
│    │   └─ DISCORD_RELEASE_NOTES_WEBHOOK_URL ──→ 📢 アップデート情報  │
│    ├─ weekly-report.yml                                            │
│    │   └─ DISCORD_WEEKLY_REPORT_WEBHOOK_URL ──→ (週次レポート)      │
│    └─ cost-audit.yml                                               │
│        └─ DISCORD_WEBHOOK_URL ──→ 🔧 deploy-log                   │
│                                                                    │
│  [AWS Lambda]                                                      │
│    └─ フィードバック API                                             │
│        └─ FEEDBACK_DISCORD_WEBHOOK_URL ──→ 📩 お問い合わせ受信      │
│                                                                    │
│  [AWS SES → SNS → Lambda]                                          │
│    └─ メール受信                                                    │
│        └─ DISCORD_WEBHOOK_SUPPORT ──→ 📩 お問い合わせ受信（メール）  │
│                                                                    │
│  [NUC ローカル]                                                     │
│    └─ .env FEEDBACK_DISCORD_WEBHOOK_URL                             │
│        └─ フィードバック API ──→ 📩 お問い合わせ受信                  │
└────────────────────────────────────────────────────────────────────┘
```

### 4.3 GitHub Actions 環境変数の設定

| 変数名 | 種別 | 設定場所 | 用途 |
|--------|------|---------|------|
| `DISCORD_WEBHOOK_URL` | Variable | GitHub Settings → Variables | デプロイ通知・コスト監査通知 |
| `DISCORD_RELEASE_NOTES_WEBHOOK_URL` | Variable | GitHub Settings → Variables | リリースノート投稿 |
| `DISCORD_WEEKLY_REPORT_WEBHOOK_URL` | Variable | GitHub Settings → Variables | 週次レポート |
| `FEEDBACK_DISCORD_WEBHOOK_URL` | Secret | GitHub Settings → Secrets | フィードバック送信先 |
| `DISCORD_WEBHOOK_SUPPORT` | Secret | GitHub Settings → Secrets | サポートメール受信通知 |

### 4.4 Lambda 環境変数への伝搬

```
GitHub Secret: FEEDBACK_DISCORD_WEBHOOK_URL
  → deploy.yml: -c feedbackDiscordWebhookUrl=${{ secrets.FEEDBACK_DISCORD_WEBHOOK_URL }}
  → CDK context → Lambda 環境変数: FEEDBACK_DISCORD_WEBHOOK_URL

GitHub Secret: DISCORD_WEBHOOK_SUPPORT
  → deploy.yml: -c discordWebhookSupport=${{ secrets.DISCORD_WEBHOOK_SUPPORT }}
  → CDK context → Lambda 環境変数: DISCORD_WEBHOOK_SUPPORT
```

### 4.5 NUC ローカル環境

NUC サーバーの `.env` に以下を設定:

```
FEEDBACK_DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/...
```

## 5. 招待リンク設計

### 5.1 現状の課題

- LP・法的ページに公開招待リンクが掲載されており、無料ユーザーも参加可能
- これは有料ユーザー限定サポートの方針と矛盾する（#405 で対応中）

### 5.2 招待リンクの運用方針

| 招待リンク種別 | 対象 | 配布方法 | 有効期限 |
|-------------|------|---------|---------|
| 有料ユーザー向け | スタンダード/ファミリープラン契約者 | アプリ内（ライセンス画面）に表示 | 無期限 |
| スポンサー向け | GitHub Sponsors 支援者 | GitHub Sponsors ダッシュボードで案内 | 無期限 |
| 公開招待リンク | 無し | LP・法的ページからは削除（#405） | — |

### 5.3 アプリ内招待リンク表示（案A: 静的リンク方式）

有料プラン契約後に表示される招待リンクは、Discord サーバーの「無期限・回数無制限」の招待リンクを使用する。

- 表示場所: `/admin/license` ページ内、プラン情報の下部
- 表示条件: `licenseStatus === 'active'`（有料プラン契約中のみ）
- ロール付与: 手動（参加後に管理者が「有料ユーザー」ロールを付与）

> **将来的な改善案（案B: Bot 連携）**
>
> Discord Bot を導入し、以下を自動化することを検討:
> - Stripe Webhook でプラン契約を検知 → Bot がロールを自動付与
> - 解約時にロールを自動剥奪
> - ただし Bot 開発・ホスティングのコストが発生するため、ユーザー数が増えてから検討

### 5.4 LP・法的ページからの公開リンク削除

#405 で対応。以下の方針:

- LP の「Discord コミュニティ」セクションからの直接招待リンクを削除
- 代わりに「有料プランに加入すると Discord コミュニティに参加できます」の案内に変更
- フッターの Discord アイコンリンクも削除

| 通知トリガー（運用設計書） | Discord チャンネル | 実装状況 |
|-------------------------|-------------------|---------|

| 新規課金 | （未割当） | 未実装 |
| プラン変更 | （未割当） | 未実装 |
| 解約 | （未割当） | 未実装 |
| 退会申請 | （未割当） | 未実装 |
| 退会完了（データ削除） | （未割当） | 未実装 |
| メンバー追加 | （未割当） | 未実装 |
| 問い合わせ（アプリ内） | 📩 お問い合わせ受信 | **実装済み** |

## 8. セキュリティ考慮事項

### 8.1 個人情報保護

| 対策 | 詳細 |
|------|------|
| 子供の個人情報 | チャンネルルールで投稿禁止を明記（#はじめに 固定メッセージ） |
| Webhook URL | GitHub Secrets / SSM Parameter Store で管理。コードにハードコードしない |
| 招待リンク | 公開ページからは削除。有料ユーザーのみアプリ内で確認可能 |

### 8.2 スパム対策

| 対策 | 詳細 |
|------|------|
| @everyone 投稿権限 | 無効（管理者のみ） |
| メッセージ送信 | 有料ユーザーロール以上のみ |
| 認証レベル | Discord サーバー設定で「メール認証済み」を要求 |
| AutoMod | Discord 組み込みの AutoMod を有効化（NGワード・スパムリンク検出） |

## 10. 将来の拡張計画

| 項目 | 概要 | 優先度 | 前提条件 |
|------|------|--------|---------|
| Bot によるロール自動付与 | Stripe Webhook 連携で有料ユーザーロールを自動付与/剥奪 | 中 | ユーザー数増加 |
