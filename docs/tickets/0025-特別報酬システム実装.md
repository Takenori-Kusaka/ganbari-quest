`Backlog`

### 概要

実績システム（#0024）でカバーしきれない主観的・第三者的な評価に対して、親が手動で特別報酬ポイントを付与できるシステムを実装する。通常のポイントとは別枠で管理し、子供にとって「特別な成果」として可視化する。

### 背景・動機

「テストで満点をとった」「漢字検定に合格した」「運動会で1位をとった」など、システムが自動判定できない成果に対して、親が表彰としてポイントを付与したい。ただし**親のUXが煩雑だと使われなくなる**ため、ワンタップ〜2タップで付与できる簡便さが重要。

### 前提条件

- #0022 認証・ユーザー管理基盤（親認証が必要）
- #0018 親用管理画面（UIの配置先）

### ゴール

- [ ] `special_rewards` テーブル
  ```
  id, child_id, granted_by (user_id), title, description, points,
  icon, category, granted_at
  ```
  - category: `academic`（学業）, `sports`（運動）, `social`（社会性）, `creative`（創作）, `life`（生活）, `other`（その他）
- [ ] `src/lib/server/db/special-reward-repo.ts`
- [ ] `src/lib/server/services/special-reward-service.ts`
  - `grantSpecialReward(childId, grantedBy, data)` — 特別報酬付与
  - `getChildSpecialRewards(childId, options)` — 履歴取得
  - `getRewardTemplates()` — よく使うテンプレート取得
- [ ] `src/routes/api/v1/special-rewards/[childId]/+server.ts` (GET, POST)
- [ ] `src/lib/domain/validation/special-reward.ts`
- [ ] 特別報酬テンプレート機能
  - よく使う報酬をプリセット化（テスト満点、資格取得、大会入賞 等）
  - 親がカスタムテンプレートを追加可能
- [ ] ポイント台帳連携（`point_ledger` に `special_reward` タイプで記録）
- [ ] 親用管理画面UI
  - ワンタップ付与: 子供選択 → テンプレート選択 → 確定
  - カスタム付与: タイトル・ポイント・アイコン入力
  - 履歴確認
- [ ] ユニットテスト
- [ ] 統合テスト

### 技術方針

- **親のUXを最優先**: 3タップ以内で付与完了を目標
- プリセットテンプレートで入力コストを最小化
- ポイントは通常ポイントに加算されるが、`special_reward` タイプで区別可能
- 子供画面では特別報酬の受取演出を表示（#0026/kinder で実装）
- テンプレートは `settings` テーブルに JSON で保存、または専用テーブル
