# #0116 活動表示スタイルのカスタマイズ機能（UXリサーチ先行）

## ステータス: Ready
## 優先度: 中
## 種別: UXリサーチ + 新機能

---

## 概要

親管理画面から、子供画面の活動一覧の**表示スタイル**をカスタマイズできる機能を追加する。カードサイズ、表示件数、レイアウト方式（グリッド列数）、スクロール方向等を子供ごとに調整可能にすることで、年齢・端末・利用状況に応じた最適な表示を実現する。

**本チケットは実装の前にUXリサーチフェーズを設け、業界のデザインガイドラインや競合UIの調査結果に基づいて設計方針を策定した上で実装に進む。**

### 課題

| 課題 | 詳細 |
|------|------|
| **カードが小さすぎる場合がある** | 現在のグリッド最小幅70px・アスペクト比4:5で、小さい端末ではタップしづらい。特に2〜3歳児には75px以上のタップターゲットが推奨される |
| **活動数が多すぎて埋もれる** | カテゴリに十数件並ぶと目的の活動を見つけにくい。不要な活動を非表示にする手段がない |
| **端末の向きに最適化されていない** | 横置きタブレット（1280×800）前提のレイアウトで、縦置きスマホでは1カテゴリの表示が窮屈 |
| **年齢による最適サイズが異なる** | Baby（0-2歳）は120pxタップターゲット、Kinder（3-5歳）は80px、小学生は44pxで十分。現在はCSS変数で定義されているが実際の活動カードグリッドには反映されていない |
| **個人差への対応不足** | 同じ年齢でも指の大きさや操作習熟度は子供ごとに異なる |

### 現状の実装

| 項目 | 現在値 | 出典 |
|------|--------|------|
| グリッド定義 | `grid-cols-[repeat(auto-fill,minmax(70px,1fr))]` | CategorySection.svelte |
| カードアスペクト比 | `aspect-[4/5]`（4:5） | ActivityCard.svelte |
| カード最小高さ | `min-h-[60px]` | ActivityCard.svelte |
| アイコンサイズ | `lg`（3rem=48px） | ActivityCard.svelte |
| テキストサイズ | `text-[10px]` | ActivityCard.svelte |
| カテゴリ間ギャップ | `gap-1`（8px） | CategorySection.svelte |
| Baby タップターゲット | `--age-tap-size: 120px`（CSS変数のみ、グリッドには未反映） | app.css |
| Kinder タップターゲット | `--age-tap-size: 80px`（CSS変数のみ） | app.css |
| 表示件数制限 | なし（全件表示） | — |
| スクロール方向 | 縦スクロール（全体） | — |
| 子供別表示設定 | なし | — |

#### C. レイアウトパターンの検証

以下のレイアウトパターンそれぞれのメリット・デメリットを比較分析する。

| # | パターン | 特徴 | 想定ユースケース |
|---|---------|------|----------------|
| 1 | **固定グリッド（現行）** | `auto-fill, minmax(70px, 1fr)` で自動列数 | 多活動表示、一覧性重視 |
| 2 | **大カード + 横スクロール** | 1行にカード3〜4枚、横スワイプで続き表示 | 幼児向け、タップ領域大 |
| 3 | **リスト表示** | 1行1活動、アイコン+テキスト横並び | 活動名の可読性重視 |
| 4 | **カルーセル** | カテゴリごとに横スライド、中央フォーカス | 探索的操作、ゲーム的UI |
| 5 | **アコーディオン + 省略** | カテゴリ折りたたみ、「もっとみる」で展開 | 活動数が多い環境 |
| 6 | **ページネーション** | カテゴリごとにページ分割（←→ナビ） | 画面遷移ベースの階層設計 |

## Phase 1: カスタマイズ設定の実装

**※ Phase 0 のリサーチ結果に基づいて詳細設計を確定する。以下は暫定案。**

### データモデル

#### 方式: `children` テーブルへの設定カラム追加

```sql
ALTER TABLE children ADD COLUMN display_config TEXT DEFAULT '{}';
-- JSON形式で柔軟に設定を保持
```

#### 設定スキーマ（暫定）

```typescript
// src/lib/domain/display-config.ts
interface DisplayConfig {
  /** レイアウトモード */
  layout: 'grid' | 'large-grid' | 'list' | 'carousel';

  /** カードサイズプリセット */
  cardSize: 'small' | 'medium' | 'large' | 'extra-large';

  /** カテゴリごとの表示件数（0 = 無制限） */
  itemsPerCategory: number;

  /** 折りたたみ表示（表示件数超過分を「もっとみる」で隠す） */
  collapsible: boolean;

  /** アイコンサイズ倍率（0.8〜2.0） */
  iconScale: number;

  /** テキスト表示（活動名を表示するか） */
  showText: boolean;
}
```

#### カードサイズプリセット（暫定値 — リサーチ後に確定）

| プリセット | グリッド最小幅 | アスペクト比 | アイコンサイズ | テキスト | 想定年齢 |
|-----------|-------------|-------------|-------------|---------|---------|
| `small` | 60px | 4:5 | 2rem (32px) | 8px | 小学生〜 |
| `medium` | 80px | 4:5 | 3rem (48px) | 10px | Kinder（現行とほぼ同じ） |
| `large` | 110px | 1:1 | 4rem (64px) | 12px | Kinder ゆったり |
| `extra-large` | 140px | 1:1 | 5rem (80px) | 14px | Baby / 運動能力に配慮 |

### デフォルト値（年齢別自動設定）

| 年齢層 | layout | cardSize | itemsPerCategory | collapsible |
|-------|--------|----------|-----------------|-------------|
| Baby (0-2) | `large-grid` | `extra-large` | 6 | true |
| Kinder (3-5) | `grid` | `medium` | 0 (無制限) | false |
| Elementary (6-8) | `grid` | `small` | 8 | true |
| Upper (9+) | `grid` | `small` | 10 | true |

### 親管理画面UI

```
/admin/children/:id/display
  └── 表示スタイル設定
       ├── レイアウト
       │   ├── ◉ グリッド（標準）              ← ビジュアルプレビュー付き選択
       │   ├── ◯ 大きいグリッド（幼児向け）
       │   ├── ◯ リスト表示
       │   └── ◯ カルーセル
       ├── カードサイズ
       │   ├── [S] [M] [L] [XL]               ← タップでサイズ切り替え
       │   └── プレビュー表示（実際のカード見本）
       ├── 表示件数
       │   ├── カテゴリあたり表示数: [__6__] 件
       │   ├── ☑ 超過分を「もっとみる」で折りたたむ
       │   └── ☐ すべて表示（スクロール）
       ├── アイコンサイズ
       │   └── [──●──────] 1.0x               ← スライダー (0.8〜2.0)
       ├── テキスト表示
       │   ├── ◉ 表示する
       │   └── ◯ アイコンのみ
       └── [プレビュー]                         ← 設定変更をリアルタイムプレビュー
            └── [保存]
```

### リアルタイムプレビュー

設定画面内に、現在の設定で活動カードがどう見えるかの**ライブプレビュー**を表示する。サンプル活動3〜5件を表示し、設定変更が即座に反映されるようにする。これにより「保存 → 子供画面で確認 → 戻って再調整」の往復を不要にする。

## Phase 2: 子供画面への適用

### CategorySection の動的レイアウト

```svelte
<!-- CategorySection.svelte（改修イメージ） -->
<script lang="ts">
  let { displayConfig }: { displayConfig: DisplayConfig } = $props();

  const gridClass = $derived(() => {
    const minWidth = {
      small: '60px',
      medium: '80px',
      large: '110px',
      'extra-large': '140px'
    }[displayConfig.cardSize];
    return `grid grid-cols-[repeat(auto-fill,minmax(${minWidth},1fr))]`;
  });
</script>

<div class={gridClass} style="--icon-scale: {displayConfig.iconScale}">
  {#each visibleItems as activity}
    <ActivityCard {activity} showText={displayConfig.showText} />
  {/each}
</div>

{#if displayConfig.collapsible && hiddenCount > 0}
  <button onclick={toggleExpand}>
    {expanded ? 'とじる' : `あと ${hiddenCount} こ みる`}
  </button>
{/if}
```

### ActivityCard のサイズ適応

```svelte
<!-- ActivityCard.svelte（改修イメージ） -->
<button
  class="w-full rounded-[var(--radius-md)] border-2 transition-all"
  style="aspect-ratio: {aspectRatio};"
>
  <CompoundIcon
    icon={icon}
    size={iconSize}
    style="transform: scale(var(--icon-scale, 1));"
  />
  {#if showText}
    <span class="font-bold line-clamp-2" style="font-size: {textSize};">
      {name}
    </span>
  {/if}
</button>
```

## 注意事項

- **リサーチ先行**: Phase 0 のリサーチレポートが完了し、方針が決定するまで Phase 1 に進まないこと
- **既存ガイドラインの尊重**: `docs/reference/幼児 子供向けデザインガイドライン.md` に記載のタップターゲットサイズ（Baby: 75px以上、Kinder: 60px以上）を下回る設定は警告表示する
- **パフォーマンス**: 表示件数を0（無制限）にした場合でも、カテゴリ30件以上ではパフォーマンス劣化しないこと
- **CSS変数の活用**: 既存の `--age-tap-size`, `--age-icon-size` を実際のグリッドレイアウトに反映させる（現在はCSS変数定義のみで未活用）
- **段階的改善**: すべてのレイアウトパターンを一度に実装せず、リサーチ結果で最も有効と判断された1〜2パターンから着手する
