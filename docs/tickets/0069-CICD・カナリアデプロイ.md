### 概要

GitHub Actionsベースのci/CDパイプラインを構築し、加重ルーティングによるカナリアデプロイ（段階的リリース）を実現する。

### 背景・動機

SaaS運用では安全なデプロイが不可欠。新機能を段階的にユーザーに展開し、問題があれば即座にロールバックできる仕組みが必要。

### 対応方針

#### CI/CDパイプライン

```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]

jobs:
  test:
    # Vitest + svelte-check + Biome lint

  build:
    needs: test
    # SvelteKit build + Docker image

  deploy-canary:
    needs: build
    # 25%のトラフィックに新バージョンをデプロイ

  promote:
    needs: deploy-canary
    # 手動承認後、100%に昇格
    environment: production
```

#### カナリアデプロイ（加重ルーティング）

**CloudFront + Lambda@Edge方式:**
```
CloudFront
  └── Lambda@Edge (viewer-request)
       └── Cookie "canary=true" の有無で振り分け
            ├── 25% → 新バージョン（v2）
            └── 75% → 現行バージョン（v1）
```

**API Gateway方式（推奨）:**
```
API Gateway ステージ
  ├── v1 (75%) → Lambda v1
  └── v2 (25%) → Lambda v2
```
- API Gatewayのカナリアリリース機能を利用
- 1週目: 25%、問題なければ2週目: 100%

#### ロールバック

```
1. CloudWatch アラーム（エラー率 > 1%）が発火
2. 自動ロールバック（Lambda エイリアスを前バージョンに切替）
3. Slack/Discord通知
4. 原因調査・修正
```
