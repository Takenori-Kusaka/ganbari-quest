### 概要

親管理画面のポイント変換機能を拡張する。現在は500P単位の固定ボタンのみだが、以下の2つの変換方法を追加する。

1. **自由入力変換**: 1円=1ポイントで、任意の金額を手入力して変換
2. **領収書OCR変換**: カメラで領収書を撮影し、合計金額をAI（Gemini Vision）で読み取って変換

### 背景・動機

現在のポイント変換は500P単位の固定選択肢しかなく、実際の支出に基づいた柔軟な変換ができない。子供のがんばりポイントを実際のお小遣いや買い物に変換する際、「お店で648円のものを買った」など端数を含む金額に対応できない。領収書撮影機能により、買い物の実績に基づいた正確なポイント変換が可能になる。

### 対応方針

#### Phase A: 自由入力変換UI

**バリデーション変更**:
- `point.ts`: 自由入力モード用のスキーマを追加（1P以上、残高以下の整数）
- `+page.server.ts`: 変換モード（`preset`/`manual`）に応じたバリデーション分岐
- プリセットボタンは従来通り残す（クイック操作用）

**UI変更** (`+page.svelte`):
```
[既存プリセットボタン: 500 / 1000 / 1500 / 2000]

--- または ---

[自由入力フィールド] [P] ← 数値入力（1P単位）
  └─ 入力補助: 残高表示、全額変換ボタン
  └─ リアルタイム: 「= ○○円」の換算表示
```

**サービス層**:
- `convertPoints()` は金額制約を緩和（1P以上、残高以下）
- `point_ledger.description` に変換方法を含める

#### Phase B: 領収書OCR変換

**カメラ撮影**:
- `navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment' } })` でリアカメラ起動
- 撮影プレビュー + 再撮影ボタン
- フォールバック: `<input type="file" accept="image/*" capture="environment">` （iOS/Android対応）

**OCR処理**:
- 撮影画像をGemini Vision API（既存のGemini連携を活用）に送信
- プロンプト例:
  ```
  この領収書/レシートの合計金額（税込）を数値のみで回答してください。
  合計金額が複数ある場合は最終合計を回答してください。
  金額が読み取れない場合は "ERROR" と回答してください。
  ```
- レスポンスから数値を抽出

**確認UI**:
```
[撮影した領収書のプレビュー画像]
読み取り金額: [1,280] 円  ← 編集可能
  └─ 「この金額で変換する」ボタン
  └─ 「金額を修正する」（手動修正可能）
  └─ 「やり直す」（再撮影）
```

**APIエンドポイント**:
- `POST /api/v1/points/ocr-receipt` — 画像を受け取り、Gemini Visionで金額を返す
- レスポンス: `{ amount: number, confidence: string, rawText: string }`

#### DB変更

スキーマ変更なし。`point_ledger.description` フィールドで変換方法を区別:
- プリセット: `"500ポイントをおこづかいにかえました"`
- 手動入力: `"648ポイントをおこづかいにかえました（手動入力）"`
- 領収書: `"1280ポイントをおこづかいにかえました（領収書読み取り）"`

### 技術メモ

#### Gemini Vision API（既存連携）

プロジェクトには既にGemini API連携がある（`docs/reference/gemini_image_generation_guide.md`）。
画像認識にはマルチモーダル入力を使用:

```typescript
const response = await model.generateContent([
  { text: "この領収書の合計金額（税込）を数値のみで回答してください。" },
  { inlineData: { mimeType: "image/jpeg", data: base64Image } }
]);
```

#### カメラAPI フォールバック戦略

| 環境 | 方法 |
|------|------|
| モダンブラウザ | `navigator.mediaDevices.getUserMedia` |
| iOS Safari | `<input type="file" capture="environment">` |
| デスクトップ | ファイル選択のみ（カメラなし） |

#### セキュリティ考慮

- 領収書画像はサーバに一時保存のみ（OCR後に削除）
- Gemini APIへの送信はサーバサイドで行う（APIキーをクライアントに露出しない）
- 画像サイズ上限: 5MB
