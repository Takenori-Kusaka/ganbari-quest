# 0055 活動記録時の効果音が鳴らない

## Status: Done


### 概要

活動を記録したときに効果音（record-complete等）が鳴らなくなっている可能性がある。

### 調査結果

#### 基盤は正常
- 音声ファイル: `static/sounds/` に9種類配置済み（tap, record-complete, point-gain, achievement-unlock, level-up, error, special-reward, omikuji-roll, omikuji-result）
- Docker: `static/` はSvelteKitビルド時にbuild outputに含まれるため本番配信可能
- soundService: Web Audio API ベースで正しく実装
- baby/kinder の `+page.svelte` で `soundService.play()` 呼び出し確認済み

#### 考えられる原因
1. **ブラウザAutoplay Policy**: AudioContextはユーザー操作前に作成するとsuspended状態になる。`ensureContext()` がフォームsubmit時に呼ばれているが、タイミングの問題でブロックされる可能性
2. **プリロード失敗**: `soundService.preload()` がネットワークエラー等でサイレント失敗している可能性
3. **設定状態**: localStorage の `sound-enabled` フラグがfalseになっている可能性
4. **Docker環境固有**: 音声ファイルのMIMEタイプ設定やパスの問題

### 確認手順

1. ブラウザのDevToolsコンソールでエラー確認
2. Network タブで `/sounds/record-complete.mp3` がロードされているか確認
3. `localStorage.getItem('ganbari-sound-enabled')` の値確認
4. AudioContext の state 確認（suspended/running）

### 対応方針

- DevToolsで原因特定後、修正方針を決定
- soundService.play() にエラーログ追加を検討
- ensureContext() の呼び出しタイミング見直し

### 成果・結果

- `src/lib/ui/sound/sound-service.ts` — Web Audio API ベースの効果音システム（`ensureContext()` で AudioContext suspended 状態の自動 resume 対応）
- `src/lib/ui/sound/play-sound.ts` — 再生ヘルパー（フォーム submit 時の `ensureContext()` 呼び出し）
- `src/lib/ui/sound/sound-state.svelte.ts` — 効果音ON/OFF状態管理
- `static/sounds/` に9種類の音声ファイル配置済み（tap, record-complete, point-gain, achievement-unlock, level-up, error, special-reward, omikuji-roll, omikuji-result）
- baby/kinder の両ホーム画面で `soundService.play()` 呼び出し確認済み
- AudioContext Autoplay Policy 対応（`ensureContext()` でユーザー操作時にコンテキスト resume）
