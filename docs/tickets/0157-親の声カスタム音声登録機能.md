# #0157 親の声・カスタム音声登録機能

## Status: Waiting
## 優先度: 中
## 種別: 新機能

---

## 概要

親が録音したボイスメッセージ（例：「やったね！」「すごい！」）をアプリに登録し、
子供が活動を完了した際にその声が再生されるようにする。

**既存のショップ効果音（system sound）はすべての家族が共通で使うもの**だが、
この機能は**家族固有の音声**であり、ごほうびショップとは別の概念として設計する。

---

## モチベーション

現在のアプリは機能的には充実しているが、**どの家庭で使っても同じ体験**になっている。  
子育てアプリとして最も価値があるのは「自分の家族だけの体験」である：

- 「お母さんの声で褒めてくれる」 → 市販のゲームや汎用アプリでは絶対に実現できない
- 「お父さんが録音した特別なかけ声」 → 子供が頑張るモチベーションになる
- 誕生日に録音したメッセージを1年間使う、など家族の思い出にもなる

このようなパーソナライズは**がんばりクエストの競合優位性** にもなる。

---

## 機能設計

### 機能の全体像

```
[親管理画面 / 子供設定]
  └── 📢 カスタムボイス管理
        ├── 録音ボタン（マイク入力）     ← ブラウザ MediaRecorder API
        ├── ファイルアップロード          ← MP3 / WAV / M4A を選択
        ├── 登録済みボイス一覧＋再生確認
        └── アクティブボイスの選択

[子供ホーム画面 / 活動完了時]
  └── アクティブボイスが設定されている場合 → 親の声を再生
        ※ 設定がない場合 → ショップの選択サウンドを再生（既存動作）
```

### ボイスの種類（拡張を想定した設計）

Phase 1 では「活動完了時」のみ対応。将来的に複数のシーンを想定：

| シーン | 再生タイミング |
|-------|--------------|
| `complete` | 活動記録完了時（短い褒め言葉「やったね！」等） ← **Phase 1 対象** |
| `levelup` | レベルアップ時（将来） |
| `login` | ログイン時の挨拶（将来） |

### 子供ごとの設定

ボイスは子供ごとに独立管理。  
「上の子には厳しめの声、下の子には優しい声」のように使い分け可能。

## データ設計

### 新規テーブル: `child_custom_voices`

```sql
CREATE TABLE child_custom_voices (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  child_id    INTEGER NOT NULL REFERENCES children(id) ON DELETE CASCADE,
  scene       TEXT NOT NULL DEFAULT 'complete',  -- 'complete' | 'levelup' | 'login'
  label       TEXT NOT NULL,                     -- "お母さんの声" など親が付けた名前
  file_path   TEXT NOT NULL,                     -- ストレージキー (uploads/voices/{childId}/{uuid}.mp3 等)
  public_url  TEXT NOT NULL,                     -- サーブ用 URL (/uploads/voices/...)
  duration_ms INTEGER,                           -- 再生時間 (ms)
  is_active   INTEGER NOT NULL DEFAULT 0,        -- 1 = このシーンで有効
  created_at  TEXT NOT NULL DEFAULT (CURRENT_TIMESTAMP),
  tenant_id   TEXT NOT NULL
);

CREATE INDEX idx_child_custom_voices_child ON child_custom_voices(child_id, scene);
```

### `children` テーブルの変更

変更なし。`child_custom_voices` から JOIN してアクティブボイスを取得する。

## API 設計

### ボイスアップロード
```
POST /api/v1/children/:childId/voices
  Content-Type: multipart/form-data
  Body: { file: <audio file>, scene: 'complete', label: 'お母さんの声' }
  Response: { id, publicUrl, durationMs }
```

### ボイス一覧取得
```
GET /api/v1/children/:childId/voices?scene=complete
  Response: { voices: [{ id, label, publicUrl, durationMs, isActive }] }
```

### アクティブ切替
```
PATCH /api/v1/children/:childId/voices/:voiceId
  Body: { isActive: true }
  Response: { success: true }
```

### ボイス削除
```
DELETE /api/v1/children/:childId/voices/:voiceId
  Response: { success: true }
```

### 録音 UI（ブラウザ MediaRecorder）

```
[● 録音開始] → [■ 停止] → [▶ 確認再生] → [💾 保存]

最大録音時間: 10秒
対応フォーマット: webm/mp4 (ブラウザで録音) / mp3・wav・m4a (ファイルアップロード)
```

**注意**: iOS Safari では `getUserMedia` + MediaRecorder が限定的。  
`audio/mp4` (AAC) を primary、`audio/webm` をフォールバックとして扱う。

## 技術設計

### ファイルストレージ

既存のアバター画像アップロード（`uploads/avatars/`）と同じパターンを踏襲:

```
ストレージキー: uploads/voices/{childId}/{uuid}.mp3
サーブURL:     /uploads/voices/{childId}/{uuid}.mp3
サーバーロジック: src/routes/api/v1/children/[id]/voices/+server.ts
ファイル配信:   src/routes/uploads/voices/[childId]/[filename]/+server.ts
```

セキュリティ:
- ファイルサイズ上限: **5MB**（約10秒の音声は 1MB 以下）
- MIME タイプ許可リスト: `audio/mpeg`, `audio/mp4`, `audio/wav`, `audio/webm`, `audio/ogg`
- tenantId チェック（所有者以外はアクセス不可）
- ファイル名はサーバー側で UUID に変換（ユーザー入力のファイル名を使わない）

### クライアント側での再生

子供画面のレイアウト `src/routes/(child)/+layout.svelte` でアクティブボイスのパスを取得し、  
既存の `soundService.setCustomRecordSound()` の仕組みを活用する：

```typescript
// src/routes/(child)/+layout.svelte
$effect(() => {
  // 既存: ショップのカスタム効果音
  // 変更: 親のカスタムボイスがあればそちらを優先
  const customVoicePath = data.avatarConfig?.customVoicePath ?? null;
  const shopSoundPath = data.avatarConfig?.customSoundPath ?? null;
  soundService.setCustomRecordSound(customVoicePath ?? shopSoundPath);
});
```

`avatarConfig` に `customVoicePath` を追加し、`avatar-service.ts` で取得・付与する。

- [ ] ファイルサイズ超過・MIME タイプ不正のアップロードはエラーを返す
