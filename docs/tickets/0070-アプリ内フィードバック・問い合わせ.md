### 概要

アプリ内から問い合わせやフィードバックを送信できる機能を実装する。ユーザーがアプリ使用中に気軽に改善提案や不具合報告を行えるようにする。

### 背景・動機

ユーザーとのコミュニケーションチャネルがアプリ内にあることで、フィードバック収集率が大幅に向上する。外部リンクだけでは離脱が多い。

### ゴール

- [x] 親管理画面にフィードバックフォーム設置
- [x] Discord/GitHub Issues への連携リンク
- [x] アプリ内バグ報告機能（スクリーンショット付き）
- [x] フィードバック集計ダッシュボード（管理者用）

### 対応方針

#### フィードバック機能設計

**親管理画面（設定ページ）:**
```
設定
├── アカウント管理
├── 通知設定
├── フィードバック送信 ← NEW
│   ├── カテゴリ選択（機能要望/バグ報告/その他）
│   ├── テキスト入力
│   ├── スクリーンショット添付（任意）
│   └── 送信
├── よくある質問（FAQ）
├── Discordコミュニティ → 外部リンク
├── 利用規約
├── プライバシーポリシー
└── バージョン情報
```

**実装方式:**
1. **シンプル方式（推奨）**: フォーム → Lambda → Discord Webhook
   - 最小コスト
   - Discordチャンネルに投稿される
   - 返信もDiscord上で対応
2. **GitHub Issues連携**: フォーム → GitHub API → Issue自動作成
3. **メール方式**: フォーム → SES → 管理者メール

#### 子供画面での表示

子供画面にはフィードバック機能は設置しない（操作の複雑化を避ける）。
代わりに、親管理画面への導線として最小限の情報を設定ページに表示。
