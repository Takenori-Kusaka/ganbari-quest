# 親管理画面ヘッダーへのロゴ適用

## Status: Done

### GitHub Copilot 対応: 可能

> #0108 で作成したロゴアセット（`static/logo-compact.svg` / `static/logo.svg`）が、親管理画面のヘッダータイトルにまだ適用されていない。現在はシンボルマーク（favicon.svg）＋プレーンテキストで構成されており、正式ロゴデザインに置き換える。

### 概要

`src/routes/(parent)/admin/+layout.svelte` のヘッダーは、`<Logo variant="symbol" size={28} />` でシールドアイコンのみを表示し、隣に `<h1>がんばりクエスト 管理</h1>` を平文テキストで並べている。

#0108 で設計・作成されたロゴアセット（`static/logo-compact.svg`）と `Logo.svelte` コンポーネントは既に実装済みであるため、これをヘッダータイトルに適用する。

### 現状

```svelte
<!-- src/routes/(parent)/admin/+layout.svelte -->
<div class="flex items-center gap-2">
    <Logo variant="symbol" size={28} />
    <h1 class="text-lg font-bold text-gray-700">がんばりクエスト 管理</h1>
</div>
```

| 要素 | 現状 | 問題 |
|------|------|------|
| ロゴ | `variant="symbol"` → `/favicon.svg`（28px シールドアイコン） | ワードマーク未表示 |
| テキスト | `<h1>` プレーンテキスト | ブランドデザインと一致しない |

### ゴール

- [x] ヘッダーのロゴ部分を `<Logo variant="compact" />` に変更し、`/logo-compact.svg`（ワードマーク入り正式ロゴ）を表示
- [x] プレーンテキストの「がんばりクエスト」部分を削除（ロゴ画像にワードマークが含まれるため重複を避ける）
- [x] 「管理」の表記を必要に応じてロゴの右隣にバッジ／テキストとして残す（ユーザーが管理画面であることを認識できるよう）
- [x] レスポンシブ対応：モバイル幅でもヘッダーが崩れないこと
- [x] ダークモード等で視認性を損なわないか確認

### 修正イメージ

```svelte
<!-- After -->
<div class="flex items-center gap-2">
    <Logo variant="compact" />
    <span class="text-xs font-medium text-gray-400 border border-gray-300 rounded px-1.5 py-0.5">管理</span>
</div>
```

> **補足**: `Logo.svelte` の `compact` バリアントのデフォルトサイズは `width: 160, height: 24`。ヘッダー高さに合わせて `size` prop を調整すること。
