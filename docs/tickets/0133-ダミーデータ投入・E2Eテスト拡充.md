### 概要

認証・マルチテナント機能の動作確認用にダミーデータ（テストユーザー・テスト家族グループ）を投入し、E2E テストを拡充する。

### ダミーデータ設計

#### テスト家族グループ

```
家族グループ A（テナント: t-test-family-a）
├── owner: test-owner@example.com (PW: TestOwner123!)
├── parent（共用）: test-shared@example.com (PW: TestShared123!)
├── child（ティーン）: test-teen@example.com (PW: TestTeen123!)
├── CHILD#1: 太郎（8歳）
├── CHILD#2: 花子（5歳）← test-teen に紐付け
├── 活動データ（各子供に 30日分）
├── コンボデータ
└── ライセンス: GQ-TEST-AAAA-0001（月額、有効）

家族グループ B（テナント: t-test-family-b）
├── owner: test-owner-b@example.com (PW: TestOwnerB123!)
├── CHILD#1: 一郎（7歳）
├── 活動データ（10日分）
└── ライセンス: GQ-TEST-BBBB-0001（年額、有効）
```

#### テスト用ライセンスキー

```
GQ-TEST-0000-0001: unused（テスト用未使用キー）
GQ-TEST-0000-0002: expired（期限切れテスト用）
GQ-TEST-0000-0003: revoked（無効化テスト用）
```

### E2E テストシナリオ

#### 認証フロー

| # | シナリオ | 期待結果 |
|---|----------|----------|
| 1 | owner がログイン | → /admin にリダイレクト |
| 2 | parent（共用）がログイン | → /admin/select-child に遷移 |
| 3 | child がログイン | → /child/{id} にリダイレクト |
| 4 | 無効パスワードでログイン | → エラー表示 |
| 5 | ログアウト | → /auth/login にリダイレクト |

#### テナント分離

| # | シナリオ | 期待結果 |
|---|----------|----------|
| 6 | 家族A の owner が家族B の子供データにアクセス | → 403 |
| 7 | 家族A の child が家族A の他の子供データにアクセス | → 403 |
| 8 | 家族A の owner が自分のテナントデータを取得 | → 正常取得 |

#### サインアップ

| # | シナリオ | 期待結果 |
|---|----------|----------|
| 9 | 有効なライセンスキーでサインアップ | → テナント作成 + ログイン |
| 10 | 無効なライセンスキーでサインアップ | → エラー |
| 11 | 使用済みライセンスキーでサインアップ | → エラー |

| # | シナリオ | 期待結果 |
