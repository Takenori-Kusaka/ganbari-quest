# APIエラーハンドリング改善

### ステータス

`Backlog`

---

### 概要

API→WebUI間のエラーハンドリングを見直し、ユーザーが自分で対処可能なエラーメッセージを表示する仕組みを構築する。

### 背景・動機

現状の問題:
1. **APIエラーがそのまま表示される**: `VALIDATION_ERROR: name is required` のような技術的メッセージがユーザーに見える
2. **対処方法が不明**: エラーが表示されても、ユーザーは何をすればよいかわからない
3. **エラー種別の区別がない**: ネットワークエラー、入力エラー、サーバーエラーが同じ見た目で表示される

現在のエラーコード体系（errors.ts）:
- `VALIDATION_ERROR` (400), `CANCEL_EXPIRED` (400), `ALREADY_RECORDED` (409)
- `ALREADY_CLAIMED` (409), `INSUFFICIENT_POINTS` (400), `INVALID_PIN` (401)
- `UNAUTHORIZED` (401), `LOCKED_OUT` (429), `NOT_FOUND` (404), `INTERNAL_ERROR` (500)

**要件4**: APIエラーハンドリングと実装方針の見直し

### ゴール

#### API側（errors.ts 拡張）

- [ ] エラーコードごとにユーザー向けメッセージ（`userMessage`）を定義
- [ ] エラーレスポンスに `userMessage` フィールドを追加
- [ ] エラーの深刻度（`severity`）を追加: `info` | `warning` | `error` | `critical`
- [ ] 対処方法（`action`）を追加: `retry` | `fix_input` | `contact_admin` | `none`

#### エラーレスポンス新形式

```json
{
  "error": {
    "code": "INSUFFICIENT_POINTS",
    "message": "Not enough points for conversion",
    "userMessage": "ポイントが足りません。もう少し活動を記録してから再度お試しください。",
    "severity": "warning",
    "action": "none",
    "details": {
      "required": 100,
      "available": 45
    }
  }
}
```

#### WebUI側（共通エラー表示コンポーネント）

- [ ] `$lib/ui/components/ErrorAlert.svelte` を作成
  - severity に応じた色分け（info=青, warning=黄, error=赤）
  - userMessage を大きく表示
  - 対処方法をサブテキストで表示
  - リトライボタン（action=retry の場合）
- [ ] 既存の `form?.error` 表示箇所を `ErrorAlert` コンポーネントに置換
- [ ] fetch API呼び出し箇所にエラーハンドリングヘルパーを導入
- [ ] ネットワークエラー（fetch失敗）用のフォールバックメッセージ

#### エラーメッセージ定義

| コード | userMessage | action |
|--------|------------|--------|
| VALIDATION_ERROR | 入力内容に問題があります。内容を確認してもう一度お試しください。 | fix_input |
| CANCEL_EXPIRED | キャンセル期限を過ぎています。管理者に相談してください。 | contact_admin |
| ALREADY_RECORDED | この活動は既に記録済みです。 | none |
| ALREADY_CLAIMED | このボーナスは既に受け取り済みです。 | none |
| INSUFFICIENT_POINTS | ポイントが足りません。もう少し活動を記録してから再度お試しください。 | none |
| INVALID_PIN | PINが正しくありません。もう一度入力してください。 | fix_input |
| UNAUTHORIZED | ログインが必要です。管理画面からログインしてください。 | fix_input |
| LOCKED_OUT | 連続で間違えたため、しばらくログインできません。時間をおいてお試しください。 | retry |
| NOT_FOUND | お探しのデータが見つかりませんでした。 | none |
| INTERNAL_ERROR | システムに問題が発生しました。しばらくしてからお試しください。 | retry |

### 技術方針

#### errors.ts の拡張

```typescript
interface ErrorDefinition {
  status: number;
  userMessage: string;
  severity: 'info' | 'warning' | 'error' | 'critical';
  action: 'retry' | 'fix_input' | 'contact_admin' | 'none';
}

const ERROR_DEFINITIONS: Record<ErrorCode, ErrorDefinition> = {
  VALIDATION_ERROR: {
    status: 400,
    userMessage: '入力内容に問題があります。内容を確認してもう一度お試しください。',
    severity: 'warning',
    action: 'fix_input',
  },
  // ...
};
```

#### ErrorAlert.svelte

```svelte
<script lang="ts">
  let { error, onRetry }: { error: ApiError; onRetry?: () => void } = $props();
  const colorMap = { info: 'bg-blue-50 border-blue-300', warning: 'bg-yellow-50 border-yellow-300', error: 'bg-red-50 border-red-300' };
</script>

<div class="border rounded-lg p-4 {colorMap[error.severity]}">
  <p class="font-bold">{error.userMessage}</p>
  {#if error.action === 'retry' && onRetry}
    <button onclick={onRetry}>もう一度試す</button>
  {/if}
</div>
```
