### 概要

子供が学校・保育園に行く前に、必要な持ち物をチェックできるテンプレート型チェックリスト機能を実装する。
全アイテムチェック完了でポイントを付与する。

### 背景・動機

- 子供が自分で準備をする習慣を身につけるゲーミフィケーション
- 忘れ物を減らし、親の「確認した？」の手間を軽減
- 曜日・イベントで持ち物が変わる現実に対応する必要がある

### 設計方針

**テンプレート型 + ワンオフ追加の2層構造**

#### データモデル

```
checklist_templates（テンプレートマスタ）
├── id, childId, name, icon
├── createdAt, updatedAt
│
checklist_template_items（テンプレートアイテム）
├── id, templateId, name, icon
├── frequency: 'daily' | 'weekday:月' | 'weekday:火' | ... | 'event'
├── direction: 'bring' | 'return' | 'both'（持参/持帰/往復）
├── sortOrder
│
checklist_logs（チェック実施記録）
├── id, childId, templateId, checkedDate
├── completedAll: boolean（全完了フラグ）
├── pointsAwarded: number
├── createdAt
│
checklist_log_items（個別チェック記録）
├── id, logId, itemName
├── checked: boolean
├── checkedAt
│
checklist_overrides（日付別の追加/除外）
├── id, childId, targetDate
├── action: 'add' | 'remove'
├── itemName, icon
```

#### 曜日・イベント対応の仕組み

1. **毎日アイテム** (`frequency: 'daily'`): ハンカチ、ティッシュ、水筒 etc.
2. **曜日アイテム** (`frequency: 'weekday:月'`): 体操服は月曜持参
3. **持帰アイテム** (`direction: 'return'`): 金曜に上履き持帰 → 月曜に持参
4. **ワンオフ** (`checklist_overrides`): 遠足の日だけリュック・お弁当追加
