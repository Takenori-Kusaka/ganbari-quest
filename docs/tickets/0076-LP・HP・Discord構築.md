### 概要

サービスの公式HP兼LP（ランディングページ）をGitHub Pagesで構築し、ユーザーコミュニティとしてDiscordサーバーを開設する。

### 背景・動機

ユーザーがサービスを発見し、理解し、問い合わせできる場が必要。GitHub Pagesなら無料でホスティングでき、デモサイト（#0060）と統合できる。Discordはコミュニティ運営の定番で無料。

### 対応方針

#### LP/HP構成（規定書 4.2 準拠）

```
/ (トップ)
├── ファーストビュー
│   ├── キャッチコピー: 「子供の『がんばり』をゲームに変える」
│   ├── ヒーロー画像/GIFアニメ
│   └── CTA: 「無料で始める」「デモを試す」
├── 機能紹介セクション
│   ├── ゲーミフィケーション（ポイント・レベル・実績）
│   ├── 年齢帯別UI
│   ├── 家族共有
│   └── 管理画面
├── スクリーンショットギャラリー
├── 料金プラン
├── FAQ
├── ユーザーの声（運用開始後に追加）
└── フッター
    ├── GitHub | Discord | Twitter/X
    ├── 利用規約 | プライバシーポリシー | 特商法表記
    └── Copyright

/demo     → デモアプリ（#0060）
/docs     → ドキュメント・マニュアル
/status   → サービスステータスページ
/legal    → 法的文書
```

#### Discord構成

```
がんばりクエスト Discord
├── #お知らせ（announcement）
├── #はじめに（rules + FAQ）
├── #雑談
├── #機能要望
├── #バグ報告
├── #開発進捗
├── #スポンサー限定（Sponsors Tier特典）
└── #管理者チャンネル（非公開）
```

#### ステータスページ

**選択肢:**
1. **Upptime（推奨）**: GitHub Actions ベースの無料ステータスモニター。GitHub Pagesにデプロイ。
2. **自前実装**: Lambda で定期的にヘルスチェック → S3に結果を書き出し
3. **外部サービス**: UptimeRobot（無料枠あり）

**推奨**: Upptime（GitHub リポジトリで管理、完全無料）
