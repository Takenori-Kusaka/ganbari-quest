# #0117 親管理画面に子供の誕生日設定を追加

## ステータス: Ready
## 優先度: 高
## 種別: バグ修正（機能欠落）

---

## 概要

DBスキーマ（`children.birth_date`）には誕生日カラムが存在するにもかかわらず、親管理画面の**子供の追加・編集フォームに誕生日の入力フィールドがない**。このため、誕生日に依存する以下の機能がすべて機能しない状態にある:

- 誕生日イベント・振り返り機能（#0046 `birthday-service.ts`）
- 年齢の自動更新（現在は手動で `age` を設定しており、誕生日を迎えても自動更新されない）
- 年齢に基づく活動フィルタリングの精度（`age` の手動設定に依存）

### 影響範囲

```
DB: children.birth_date ✓ カラム存在（nullable TEXT）
  ↓
リポジトリ: insertChild() / updateChild() ❌ birth_date パラメータなし
  ↓
サービス: addChild() / editChild() ❌ birth_date パラメータなし
  ↓
サーバーアクション: +page.server.ts ❌ formData から birth_date を取得していない
  ↓
UI: +page.svelte ❌ 誕生日入力フィールドなし
  ↓
結果: birthday-service.ts は存在するが、birth_date が NULL のため常に誕生日イベント無し
```

### 5. 誕生日 → 年齢の自動計算（オプション）

誕生日が設定されている場合は `age` を自動計算するロジックを追加する。

```typescript
function calculateAge(birthDate: string): number {
  const birth = new Date(birthDate);
  const today = new Date();
  let age = today.getFullYear() - birth.getFullYear();
  const monthDiff = today.getMonth() - birth.getMonth();
  if (monthDiff < 0 || (monthDiff === 0 && today.getDate() < birth.getDate())) {
    age--;
  }
  return age;
}
```

- 誕生日設定時に `age` フィールドを自動更新する
- 誕生日設定済みの場合、年齢フィールドは読み取り専用にする（自動計算優先）
- 誕生日未設定の場合は、従来通り年齢の手動入力を許可する

- **E2Eテストへの影響**: `tests/e2e/global-setup.ts` のテスト子供データに `birthDate` を追加する必要あり
