# #0081 コンボボーナスシステム

## ステータス: Done
## 優先度: 中
## 種別: 機能追加（ゲーミフィケーション強化）

---

## 概要

同日に複数の活動を記録した際にボーナスポイントを付与する「コンボ」システムを追加する。
同一カテゴリ内での連続記録（カテゴリコンボ）と、複数カテゴリにまたがる記録（クロスカテゴリコンボ）の2種類を実装する。

## 背景

現在のボーナス要素はストリーク（連続日数）のみ。1日の中で複数タスクをこなす動機付けが弱い。
コンボシステムにより「今日はもう1つやろう！」という意欲を促進する。

## 要件

### 1. カテゴリコンボ（同一カテゴリ内）

同日に同じカテゴリの異なる活動を複数記録するとボーナス:

| コンボ数 | 名称 | ボーナスポイント |
|---------|------|-----------------|
| 2種類 | ダブル | +2P |
| 3種類 | トリプル | +5P |
| 4種類以上 | スーパー | +10P |

- 同じ活動の複数回実行はカウントしない（異なる活動IDのみ）
- カテゴリごとに独立にカウント

### 2. クロスカテゴリコンボ（複数カテゴリ横断）

同日に異なるカテゴリの活動を記録するとボーナス:

| カテゴリ数 | 名称 | ボーナスポイント |
|-----------|------|-----------------|
| 2カテゴリ | にとうりゅう | +3P |
| 3カテゴリ | さんみいったい | +8P |
| 4カテゴリ | スーパーヒーロー | +15P |
| 5カテゴリ | パーフェクト | +30P |

- 既存の週次評価ボーナス（3カテゴリ+5P, 4カテゴリ+10P, 5カテゴリ+20P）との関係を整理
  - 週次ボーナスはそのまま維持（週間を通したバランス報酬）
  - コンボは「1日の中での即時報酬」として差別化

### 3. コンボ演出

- 活動記録の結果ダイアログにコンボ情報を表示
  - 「ダブルコンボ！ +2P」のようなメッセージ
  - コンボ名称と絵文字での演出（例: 「にとうりゅう！ +3P」）
- 効果音の追加（コンボ発生時）

### 4. コンボ履歴

- `pointLedger` に `type: 'combo_bonus'` で記録
- `description` にコンボ種別を記載

## 技術設計

### バックエンド

- `src/lib/server/services/combo-service.ts`（新規）
  - `checkAndGrantCombo(childId, date)`: その日のコンボ状態を判定し、未付与分のボーナスを付与
  - 活動記録API（`/api/v1/activities/record`）の成功レスポンスにコンボ情報を含める

- コンボ判定ロジック:
  ```
  1. 当日の activityLogs を取得
  2. カテゴリ別にユニークな activityId をカウント
  3. カテゴリコンボ: 各カテゴリで2種類以上あればボーナス計算
  4. クロスカテゴリコンボ: 活動したカテゴリ数でボーナス計算
  5. 既に付与済みのコンボボーナスを差し引いて差分のみ付与
  ```

### フロントエンド

- 記録結果ダイアログにコンボ表示を追加
- baby/kinder 両方のホーム画面で対応

### DB変更

- スキーマ変更なし（`pointLedger` の `type` に新しい値 `'combo_bonus'` を使用）
