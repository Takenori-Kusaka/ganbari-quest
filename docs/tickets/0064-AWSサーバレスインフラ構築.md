### 概要

現行のDocker/NUC構成から、AWSフルサーバレスアーキテクチャに移行する。月額無リクエスト時1,000円未満を目標とする。

### 背景・動機

SaaS展開には自動スケーリング・高可用性のインフラが必要。個人開発の費用制約から、フルサーバレスで固定費を最小化する。WAFは初期導入しない（費用対効果が悪い）。

### 対応方針

#### アーキテクチャ構成

```
[ユーザー]
    │
    ▼
[CloudFront] ─── [S3: 静的ファイル (SvelteKit client)]
    │
    ▼
[API Gateway (HTTP API)]
    │
    ├── [Lambda Authorizer (Cognito JWT検証)]
    │
    ▼
[Lambda (SvelteKit SSR + API)]
    │
    ├── [DynamoDB (メインDB)]
    ├── [S3 (アバター画像・バックアップ)]
    └── [Cognito (認証)]

[Route 53] → CloudFront
[CloudWatch] → Lambda/API Gateway ログ（最小保持）
```

#### SvelteKit → Lambda 変換

**選択肢:**
1. **svelte-adapter-aws** (コミュニティ製): SvelteKit をLambda@Edge + S3に変換
2. **adapter-node + Lambda Web Adapter**: Node.jsサーバーをLambda互換にラップ
3. **adapter-node + API Gateway + Lambda**: 標準的なNode.js Lambda

**推奨**: 選択肢2（Lambda Web Adapter）
- AWS公式のWeb Adapterを使用
- SvelteKitのadapter-nodeをそのまま利用可能
- SSR対応、ストリーミング対応
- Cold start: ~500ms（SnapStart非対応だが許容範囲）

```dockerfile
FROM public.ecr.aws/awsguru/aws-lambda-web-adapter:latest AS adapter
FROM node:22-alpine
COPY --from=adapter /lambda-adapter /opt/extensions/
COPY build/ /app/
WORKDIR /app
CMD ["node", "index.js"]
```

#### SQLite → DynamoDB 移行設計

**シングルテーブル設計（推奨）:**

| PK | SK | データ | 用途 |
|----|-----|-------|------|
| TENANT#t1 | PROFILE | {name, plan, ...} | テナント情報 |
| TENANT#t1#USER#u1 | PROFILE | {name, age, ...} | 家族メンバー |
| TENANT#t1#USER#u1 | ACT#2026-03-06#a1 | {points, ...} | 活動記録 |
| TENANT#t1 | ACHIEVEMENT#ach1 | {unlocked, ...} | 実績 |

**GSI:**
- GSI1: `SK` + `PK`（逆引き用）
- GSI2: `type` + `createdAt`（タイプ別時系列クエリ）

#### WAFなしのセキュリティ対策

| 対策 | 実装 | コスト |
|------|------|--------|
| Lambda Authorizer | Cognito JWT検証 | 無料（Lambda実行費のみ） |
| API Gatewayスロットリング | リクエスト/秒の上限設定 | 無料 |
| CloudFrontジオブロック | 日本国内のみ許可（初期） | 無料 |
| DynamoDB RCU/WCU制限 | オンデマンドの上限設定 | 無料 |
| Cognito MFA | TOTP or SMS（任意） | SMS以外無料 |
| CloudWatch異常検知 | Lambda実行数の異常アラート | 最小構成 |

#### IaC選定

**推奨: AWS CDK (TypeScript)**
- TypeScriptプロジェクトとの親和性
- 高レベル抽象化でインフラ定義が簡潔
- `cdk deploy` で一括デプロイ
