# 誕生日イベントと振り返り機能

### ステータス

`Done`

---

### 概要

子供の誕生日に特別イベントを発生させ、1年の振り返りフォーム・次年度の抱負設定・特別ポイント付与を行う機能を実装する。

### 背景・動機

子供の成長において「1年の節目」は重要な振り返りの機会。誕生日をトリガーとして:
- 1年間の活動実績を振り返る
- 健康面の自己評価でポイント付与（「元気に過ごせた」という自己肯定）
- 次の1年の目標を立てる習慣を育む
- 目標を立てたこと自体にポイントを付与し、自己計画力を評価

これにより、年1回の大きな節目イベントとしてシステム内に「成長の記録」が蓄積される。

**要件9**: 誕生日イベント・振り返り・抱負機能

### ゴール

#### 誕生日検知・イベント発火

- [ ] children テーブルに `birthdate` カラムを追加（DATE型）
- [ ] 日次バッチ or ログイン時に誕生日を検知するロジック
- [ ] 誕生日当日〜3日間の猶予期間内に振り返りイベントを表示
- [ ] 年齢に応じた基本ポイント自動付与（例: 年齢 × 100pt）

#### 1年の振り返りフォーム

- [ ] 過去1年間のステータス変化サマリー表示（カテゴリ別の成長量）
- [ ] 過去1年間の活動回数・合計ポイントのサマリー
- [ ] 健康チェックリスト（選択式）:
  - 「大きなけがをしなかった」
  - 「風邪をあまりひかなかった」
  - 「たくさん外で遊んだ」
  - 「好き嫌いなく食べられた」
  - 「早寝早起きができた」
- [ ] 健康チェックの各項目にポイント付与（1項目あたり50pt等）

#### 次年度の抱負設定

##### 文字入力可能な子供（小学生以上）
- [ ] 自由入力テキストで抱負を記載（200文字以内）
- [ ] 抱負を設定したことに対してポイント付与（200pt等）

##### 文字入力困難な子供（幼児）
- [ ] 選択式UI: 各ステータスカテゴリ（ちから、かしこさ等）に対して
  - 「すごくがんばる」「がんばる」「ふつう」の3段階
- [ ] 選択で抱負を設定したことに対してポイント付与（100pt等）

#### データ保存・閲覧

- [ ] `birthday_reviews` テーブルを追加
  - `id`, `child_id`, `year`, `age_at_review`
  - `health_checks` (JSON: チェック項目と結果)
  - `aspiration_text` (自由入力抱負)
  - `aspiration_categories` (JSON: カテゴリ別目標)
  - `points_earned` (付与ポイント合計)
  - `created_at`
- [ ] 過去の振り返り・抱負を閲覧できる「おもいで」ページ
- [ ] 管理画面から振り返り履歴の確認

### 技術方針

#### DBスキーマ追加

```sql
-- childrenテーブルに birthdate 追加
ALTER TABLE children ADD COLUMN birthdate TEXT;

-- 誕生日レビュー記録
CREATE TABLE birthday_reviews (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  child_id INTEGER NOT NULL REFERENCES children(id),
  review_year INTEGER NOT NULL,
  age_at_review INTEGER NOT NULL,
  health_checks TEXT NOT NULL DEFAULT '{}',    -- JSON
  aspiration_text TEXT,
  aspiration_categories TEXT DEFAULT '{}',     -- JSON
  base_points INTEGER NOT NULL DEFAULT 0,
  health_points INTEGER NOT NULL DEFAULT 0,
  aspiration_points INTEGER NOT NULL DEFAULT 0,
  total_points INTEGER NOT NULL DEFAULT 0,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(child_id, review_year)
);
```

#### 誕生日検知フロー

```
ログイン時 or ページロード時
  → children.birthdate と今日の日付を比較
  → 誕生日当日〜+3日 かつ 今年のreviewが未作成
  → 誕生日イベントモーダルを表示
  → 振り返り → 抱負 → ポイント付与 → 完了演出
```

#### ポイント配分案

| 項目 | ポイント |
|------|---------|
| 誕生日基本ボーナス | 年齢 × 100 |
| 健康チェック（1項目あたり） | 50 |
| 健康チェック全項目達成ボーナス | 100 |
| 抱負設定（自由入力） | 200 |
| 抱負設定（選択式） | 100 |
