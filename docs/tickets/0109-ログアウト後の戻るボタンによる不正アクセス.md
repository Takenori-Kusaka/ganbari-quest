# #0109 親管理画面ログアウト後のブラウザ戻るボタンによる不正アクセス

## Status: Done
## 優先度: 高
## 種別: セキュリティバグ修正

---

## 概要

親管理画面からログアウトした後、ブラウザの「戻る」ボタンを押すと親管理画面が再表示される。子供が親の管理画面にアクセスできてしまうセキュリティ上の問題。

## 再現手順

1. 親管理画面にPINでログインする
2. 管理画面内で何ページか遷移する
3. ログアウトする → `/` にリダイレクトされる
4. ブラウザの「戻る」ボタンを押す
5. **問題**: 親管理画面が表示される（認証済みページの内容がそのまま見える）

## 根本原因

### 原因1: Cache-Control ヘッダー未設定

`hooks.server.ts` で `/admin` 配下のレスポンスに `Cache-Control` ヘッダーを設定していない。ブラウザは認証済みページをキャッシュし、「戻る」ボタン押下時にサーバーへのリクエストなしにキャッシュから復元する。

```typescript
// hooks.server.ts L58-60（現状）
const response = await resolve(event);
// → Cache-Control ヘッダーなし → ブラウザがページをキャッシュ
return response;
```

### 原因2: Back/Forward Cache（bfcache）

モダンブラウザはパフォーマンス最適化のため bfcache を使用する。`Cache-Control` ヘッダーがないページは bfcache に保存され、「戻る」操作でサーバーへの再リクエストなく表示される。

Lighthouse でも `"Page didn't prevent back/forward cache restoration"` の警告が出ている。

### 原因3: ログアウト時のクライアント状態未クリア

ログアウト API（`/api/v1/auth/logout`）は以下のみ実施:
- DB のセッショントークン削除
- Cookie の削除
- 302 リダイレクト

ブラウザの履歴やキャッシュには介入していない。

## 修正方針

### 修正1: `/admin` ページにキャッシュ防止ヘッダーを設定（必須）

`hooks.server.ts` で認証が必要なページのレスポンスに `Cache-Control` を追加:

```typescript
const response = await resolve(event);

// 認証が必要なページはブラウザキャッシュを禁止
if (path.startsWith('/admin') || path === '/login') {
    response.headers.set('Cache-Control', 'no-cache, no-store, must-revalidate');
    response.headers.set('Pragma', 'no-cache');
    response.headers.set('Expires', '0');
}

return response;
```

これにより:
- `no-store`: ブラウザがページをキャッシュしない
- `no-cache`: キャッシュ使用前にサーバーに再検証を要求
- `must-revalidate`: 期限切れキャッシュの利用を禁止
- `Pragma: no-cache`: HTTP/1.0 クライアント向け互換

### 修正2: bfcache からの復元時に認証チェック（推奨）

bfcache からの復元を検知してクライアント側で認証確認する:

```typescript
// (parent)/admin/+layout.svelte に追加
$effect(() => {
    const handlePageShow = (event: PageTransitionEvent) => {
        if (event.persisted) {
            // bfcache からの復元時は認証状態を再確認
            fetch('/api/v1/auth/status').then(res => {
                if (!res.ok) {
                    window.location.href = '/login';
                }
            });
        }
    };
    window.addEventListener('pageshow', handlePageShow);
    return () => window.removeEventListener('pageshow', handlePageShow);
});
```

### 修正3: ログアウト後の履歴操作（補助的）

ログアウト後に `replaceState` で履歴を書き換え、戻るボタンで認証ページに遷移しないようにする:

```typescript
// ログアウト後のリダイレクト先で
window.history.replaceState(null, '', '/');
```
