### 概要

日本市場を前提とした事業計画書を策定する。市場規模、原価構造、料金プラン、損益分岐点を根拠ある数字で設計し、目標顧客数を設定する。

### 背景・動機

個人開発からSaaS展開する上で、家計を圧迫しない持続可能な事業モデルを設計する必要がある。AWSサーバレスの原価を正確に把握し、現実的な料金設定を行う。

### 対応方針

#### 市場規模算出（概算手法）
```
TAM（最大市場規模）:
  日本の0-12歳人口: 約1,200万人
  子育て世帯数: 約1,000万世帯
  デジタルサービス利用率: 約60%
  → TAM: 600万世帯

SAM（獲得可能市場）:
  教育・習慣化アプリ利用意向: 約15%
  → SAM: 90万世帯

SOM（現実的な初期ターゲット）:
  → SOM: 1,000-10,000世帯（初年度目標）
```

#### 原価構造（AWSサーバレス）

| コスト項目 | 固定/変動 | 無リクエスト時 | 100家庭 | 1,000家庭 |
|-----------|---------|-------------|---------|----------|
| Route 53 (ドメイン) | 固定 | $0.50 | $0.50 | $0.50 |
| CloudFront | 変動 | $0.00 | ~$1.00 | ~$5.00 |
| S3 | 変動 | ~$0.10 | ~$0.50 | ~$2.00 |
| Lambda | 変動 | $0.00 | ~$0.50 | ~$3.00 |
| API Gateway | 変動 | $0.00 | ~$0.50 | ~$3.00 |
| DynamoDB | 変動 | $0.00 | ~$1.00 | ~$5.00 |
| Cognito | 変動 | $0.00 | $0.00 | $0.00 |
| CloudWatch | 固定 | ~$1.00 | ~$2.00 | ~$5.00 |
| **合計** | | **~$2 (~300円)** | **~$6 (~900円)** | **~$24 (~3,600円)** |

※ Cognito: 月50,000MAUまで無料
※ Lambda: 月100万リクエスト無料枠あり
※ DynamoDB: オンデマンドモードで最小利用

#### 損益分岐点シミュレーション

前提:
- ベーシックプラン: 480円/月
- 有料化率: 5%（フリーミアム→有料）
- ストア手数料: 15%（初年度Small Business Program）
- 月額固定費: 1,000円（AWS最小構成）

```
必要有料ユーザー数 = 固定費 / (単価 × (1 - 手数料率))
= 1,000円 / (480円 × 0.85)
= 1,000 / 408
≈ 3ユーザー（月額固定費のみの場合）

変動費込み（100家庭規模）:
= (1,000 + 900) / 408
≈ 5ユーザー

変動費込み（1,000家庭規模）:
= (1,000 + 3,600) / 408
≈ 12ユーザー
```

→ 極めて低い損益分岐点。サーバレスの利点。

#### KPI目標（初年度）

| KPI | 目標 |
|-----|------|
| MAU | 500 |
| 有料ユーザー | 25 |
| MRR | 12,000円 |
| Churn Rate | < 5%/月 |
| NPS | > 30 |
