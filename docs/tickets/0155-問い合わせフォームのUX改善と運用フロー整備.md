# #0155 問い合わせフォームのUX改善と運用フロー整備

## Status: Done
## 優先度: 中
## 種別: 機能改善 / UX

---

## 背景と方針

### 現状

- `/admin/settings` にフィードバックフォームが存在する（カテゴリ + 本文のみ）
- Discord Webhook 送信は**実装済み**（`FEEDBACK_DISCORD_WEBHOOK_URL` 環境変数）
- 送信成功メッセージ: 「フィードバックを送信しました。ありがとうございます！」
- 返信先メールアドレスの入力フィールドがない
- ユーザーが返信を受け取れるかどうか何も示唆されていない

### 問題

がんばりクエストのターゲットユーザーは「子供を持つ親」。  
Discord に不慣れなユーザーが多いため、**Discord のサーバーに直接誘導するのが困難**。  
一方、管理側は Discord で一元管理したい（スレッド・リアクション・FAQ化が容易）。

### 解決方針

```
ユーザー側: シンプルな問い合わせフォーム（アプリ内完結）
                ↓
管理側:     Discord Webhook で受信・返信・FAQ化
```

- **ユーザーは Discord を意識しない** — アプリ内フォームだけで完結
- **返信先メールアドレス（Optional）** を入力すれば、メールで返信される可能性があることを示唆
- **Discord 上で質問/回答を蓄積** → 適宜 HP の FAQ セクションに転記（手動運用）

## 変更スコープ

### 1. フォームにオプション項目「返信先メールアドレス」を追加

| フィールド | 変更 |
|-----------|------|
| カテゴリ（機能要望/バグ報告/その他） | 変更なし |
| 本文（最大1000文字） | 変更なし |
| **返信先メールアドレス** | **新規追加（Optional）** |

バリデーション:
- 入力された場合: RFC 5321 準拠のメール形式チェック（`type="email"` + サーバー側バリデーション）
- 空欄の場合: スキップ（送信可能）

### 2. 送信成功メッセージの変更

#### 現在
```
フィードバックを送信しました。ありがとうございます！
```

#### 変更後（メールアドレスなし）
```
お問い合わせありがとうございます。
今後の参考とさせていただきます。
```

#### 変更後（メールアドレスあり）
```
お問い合わせありがとうございます。
今後の参考とさせていただきます。
ご入力いただいたメールアドレスに、内容によってはご返信する場合があります。
```

### 3. Discord Webhook ペイロードへのメールアドレス追加

現在の Embed に `reply_to` フィールドを追加する。

```json
{
  "embeds": [{
    "title": "📬 機能要望",
    "description": "...",
    "fields": [
      { "name": "テナント", "value": "...", "inline": true },
      { "name": "送信者", "value": "...", "inline": true },
      { "name": "返信先", "value": "user@example.com または なし", "inline": true }
    ]
  }]
}
```

返信先メールアドレスが入力されている場合は Discord のスレッドで直接確認できるため、
Discord 上でのコミュニケーションで完結しつつメール対応も可能になる。

### `+page.server.ts` の変更

```typescript
sendFeedback: async ({ request, locals }) => {
  const form = await request.formData();
  const category = ...;
  const text = ...;
  const replyEmail = form.get('email')?.toString()?.trim() ?? '';

  // メールアドレスのバリデーション（入力された場合のみ）
  if (replyEmail && !isValidEmail(replyEmail)) {
    return fail(400, { feedbackError: 'メールアドレスの形式が正しくありません' });
  }

  // Discord Webhook ペイロードに reply_to を追加
  fields: [
    { name: 'テナント', value: tenantId, inline: true },
    { name: '送信者', value: email, inline: true },
    { name: '返信先', value: replyEmail || 'なし', inline: true },
  ]
}

// メールバリデーション（シンプルな正規表現、インジェクション対策済み）
function isValidEmail(email: string): boolean {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email) && email.length <= 254;
}
```

## 運用フロー（実装外: 手動運用）

```
[アプリ内フォーム送信]
       ↓
[Discord #問い合わせ チャンネルに通知]
       ↓（管理者が対応）
┌──────────────────────────────────────┐
│ A. メールアドレスあり                │
│    → Discord スレッドで内容確認      │
│    → 必要に応じてメール返信          │
│                                      │
│ B. メールアドレスなし                │
│    → Discord でリアクション管理のみ  │
│    → 返信不要（匿名フィードバック扱い）│
└──────────────────────────────────────┘
       ↓（FAQ化対象の場合）
[Discord の Q&A  より抜粋]
       ↓
[HP の FAQ セクションに転記（手動）]
```

## セキュリティ考慮事項

- メールアドレスはサーバー側でも正規表現バリデーション（最大254文字）
- Discord Webhook に送信する内容は Embed の `field.value` として渡すため XSS リスクなし
- メールアドレスは Discord のサーバー管理者のみ参照可能（外部に公開されない）
- フォーム本文の既存の1000文字制限は継続

## 備考

- `FEEDBACK_DISCORD_WEBHOOK_URL` 環境変数が未設定の場合はログのみ（既存動作と同じ）
- Cognito モードでは `locals.identity.email` が自動設定されているが、返信先は**ユーザーが明示的に入力したもの**のみ使用する（デフォルトで Cognito メールを転送しない）
