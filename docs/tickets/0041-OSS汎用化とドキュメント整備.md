# OSS汎用化とドキュメント整備

### ステータス

`Done`

---

### 概要

日下家専用のハードコードを排除し、不特定多数の家庭がカスタマイズして利用できるOSSとして整備する。初期セットアップウィザードまたはドキュメントにより、任意の家庭構成でシステムを立ち上げられるようにする。

### 背景・動機

現在のシステムは日下家の子供5名がseed.tsにハードコードされており、PINも固定値`1234`で初期化される。OSSとしてリリースするためには:
- 家族固有データの除去（子供名、人数、年齢等）
- 初期セットアップフローの整備（対話的 or ドキュメント）
- 環境構築手順の詳細ドキュメント化
- ライセンス・コントリビューションガイドの整備

が必要。

**要件1**: OSS汎用化

### 技術方針

#### seed.ts の変更方針

```typescript
// Before: 日下家固有
const childrenData = [
  { nickname: 'ゆうきちゃん', age: 4, ... },
  ...
];

// After: 空（初回起動時にWeb UIから登録）
// activitiesとachievementsのマスタデータのみseed
// childrenテーブルは空のまま起動
```

#### セットアップウィザード

- `/setup` ルートを追加（childrenテーブルが空の場合のみアクセス可能）
- ステップ1: 子供の登録（名前、生年月日、テーマ、UIモード）
- ステップ2: 管理PIN設定
- ステップ3: 完了→ホーム画面へリダイレクト
