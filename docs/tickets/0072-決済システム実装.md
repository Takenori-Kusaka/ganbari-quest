### 概要

Google Play / Apple App Store のアプリ内課金を主軸とし、Web版向けにStripe決済も提供する。RevenueCat等のサブスクリプション管理SaaSを活用し、実装コストを最小化する。

### 背景・動機

メイン顧客（子育て世帯）はモバイルアプリ経由でのアクセスが中心。ストアの決済基盤を活用することで信頼性と利便性を確保する。Web版は補助的な位置付けで、優先度は低め。

#### RevenueCat活用

**RevenueCatを使う理由:**
- Google Play / Apple Store / Stripe を統一APIで管理
- レシート検証を自動化
- サブスクリプション分析ダッシュボード
- 無料プラン: MAU $2,500収益まで無料

**実装:**
```typescript
// Capacitorプラグイン
import { Purchases } from '@revenuecat/purchases-capacitor';

// 初期化
await Purchases.configure({ apiKey: 'xxx' });

// 商品取得
const offerings = await Purchases.getOfferings();

// 購入
const result = await Purchases.purchasePackage({ aPackage: offering.current.monthly });

// サーバーサイドでの検証
// RevenueCat Webhook → Lambda → DynamoDBにプラン情報更新
```

#### Web版決済（Stripe）

- 優先度: 低（モバイル版が安定してから）
- Stripe Checkout Session API
- サブスクリプション管理: Stripe Customer Portal
- Webhook → Lambda → プラン情報更新

#### ライセンスキー方式（Web版アクセス制御 — #0066 で詳細設計済み）

```
1. ストア or Stripe で購入後、Webhook 経由でライセンスキー自動発行
2. ユーザーにメール + 画面でライセンスキーを通知
3. サインアップ時にライセンスキー入力 → テナント作成 → 利用開始
4. 買い切り (mode: 'payment') / サブスク (mode: 'subscription') 両対応
5. ライセンス状態遷移・ DynamoDB スキーマ・コスト分析は #0066 を参照
```

→ **注意:** #0072 は決済フローの実装に集中し、ライセンス管理のデータ設計・状態遷移・認証統合は #0066 に一元化済み。

#### 決済に関する法的対応

- 特定商取引法表記（#0073）
- 資金決済法: サブスクリプション（役務の対価）のため非該当
- 消費税: 課税事業者になった場合はインボイス対応
