# ステータスmax値の年齢スケーリング

## Status: Done


### 概要

ステータス（つよさ）の最大値を固定100ではなく、子供の年齢に応じて動的にスケーリングする仕組みを導入する。

### 背景・動機

現在のステータスは0〜100の固定範囲でクランプされている:
```typescript
const newValue = Math.max(0, Math.min(100, currentValue + changeAmount));
```

しかし、0歳から20歳まで毎日1ptの活動を続けただけでも累計7,300ptになる。固定max=100では:
- 早期にカンスト（100到達）して成長実感がなくなる
- 年齢が上がっても天井が変わらず、長期的なモチベーション維持が困難
- グラフ表示上、全カテゴリが100に張り付いて差が見えなくなる

max値を年齢に応じてスケールさせることで、常に「伸びしろ」がある状態を維持する。

**要件8**: ステータスmax値の年齢スケーリング

### 技術方針

#### 年間最大増加量の算出

```
1日あたり最大活動数 = 年齢帯別の活動カテゴリ数 × 平均ポイント
- kinder (3-5歳): 約10カテゴリ × 平均2pt = 20pt/日
- elementary_lower (6-9歳): 約15カテゴリ × 平均3pt = 45pt/日
- elementary_upper (10+歳): 約20カテゴリ × 平均3pt = 60pt/日

年間最大 = 日あたり最大 × 365
- 3-5歳: ~7,300pt/年
- 6-9歳: ~16,425pt/年
- 10+歳: ~21,900pt/年
```

実際にはカテゴリ別に分散するため、各カテゴリのmax増加量はさらに細分化が必要。

#### max値テーブル設計（丸め値）

| 年齢 | カテゴリ別年間max概算 | 累積max（丸め） |
|------|----------------------|----------------|
| 1歳 | - | 50 |
| 2歳 | - | 100 |
| 3歳 | ~1,500/カテゴリ | 200 |
| 4歳 | ~1,500/カテゴリ | 350 |
| 5歳 | ~1,500/カテゴリ | 500 |
| 6歳 | ~3,000/カテゴリ | 800 |
| 7歳 | ~3,000/カテゴリ | 1,100 |
| 8歳 | ~3,000/カテゴリ | 1,500 |
| 9歳 | ~3,000/カテゴリ | 2,000 |
| 10歳 | ~4,500/カテゴリ | 2,500 |
| ... | ... | ... |
| 18歳 | ~4,500/カテゴリ | 8,000 |

※ 上記は概算。実際のmax値は活動マスタのポイント分布から正確に算出する。

#### 実装変更箇所

1. **status-service.ts**: `Math.min(100, ...)` → `Math.min(getMaxForAge(age), ...)`
2. **status表示コンポーネント**: バーの幅を `value / maxForAge * 100%` に
3. **レベルテーブル**: `minAvg` を絶対値ではなく `maxForAge` に対する割合に変更
4. **偏差値計算**: ベンチマークのmean/stdDevも年齢別max値を考慮

#### DB変更

```sql
-- status_max_scales テーブル追加
CREATE TABLE status_max_scales (
  age INTEGER PRIMARY KEY,
  max_value INTEGER NOT NULL,
  updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```
