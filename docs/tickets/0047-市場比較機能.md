# 市場比較機能

### ステータス

### 概要

子供のステータスを世の中の同年齢の一般水準と比較し、秀でている/劣っている領域を可視化する機能を実装する。

### 背景・動機

もともとの企画要件にあった市場比較機能が未実現。現在のstatus-service.tsには偏差値計算ロジックとmarket_benchmarksテーブルが既に存在するが、以下が不足:
1. ベンチマークデータの充実（現在のデータの精度・網羅性の検証）
2. 子供用画面・管理画面での比較表示UI
3. ベンチマーク更新フロー

世の中の幼児・未成年向け人材力診断サービスの手法を参考に、本プロジェクト独自の比較指標を構築する。

**要件10**: 市場比較・ベンチマーク機能

### 現状のデータ品質の問題

#### 問題1: 全データが根拠なしの推定値

`scripts/add-benchmarks.cjs` に投入された50件のベンチマークデータ:
- `source` が全件 `'推定値'` — 公的統計・研究データの参照なし
- 例: 3歳・うんどう mean=15.0, std_dev=5.0 → 何を基準にした数値か不明

#### 問題2: 値の成長パターンが非現実的

各年齢の mean が単純な線形増加:
| 年齢 | うんどう | べんきょう | せいかつ |
|------|---------|-----------|---------|
| 3歳 | 15 | 10 | 20 |
| 6歳 | 80 | 60 | 90 |
| 9歳 | 220 | 180 | 250 |
| 12歳 | 400 | 360 | 480 |

実際の子供の発達は非線形（急成長期とプラトーがある）であり、
この線形増加パターンは発達心理学的に妥当ではない。

#### 問題3: std_dev の設定根拠が不明

標準偏差も同様に推定値であり、年齢ごとの個人差のばらつきを
反映していない。実際には低年齢ほど個人差が大きいカテゴリや、
年齢とともに差が広がるカテゴリがある。

### 技術方針

#### 既存リソースの活用

status-service.ts に既に実装されているもの:
```typescript
// 偏差値計算
deviationScore = ((value - mean) / stdDev) × 10 + 50

// 星評価
stars = { 5: >=65, 4: >=58, 3: >=50, 2: >=42, 1: <42 }

// キャラクタータイプ
characterType = { hero: >=55, normal: >=45, ganbari: <45 }
```

#### ベンチマークデータ構造（移行済みスキーマ）

```sql
CREATE TABLE market_benchmarks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  age INTEGER NOT NULL,
  category_id INTEGER NOT NULL,
  mean REAL NOT NULL,
  std_dev REAL NOT NULL,
  source TEXT,             -- ← ここにデータ出典を記載すべき（現状は全件 '推定値'）
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE UNIQUE INDEX idx_benchmarks_age_category ON market_benchmarks (age, category_id);
```

#### レーダーチャート実装

- SVGベースで自前実装（外部ライブラリ不要）
- 各カテゴリの偏差値を正規化してプロット
- 平均ライン（偏差値50）と子供の値を重ねて表示

### 作業メモ

- 偏差値計算は既に実装済み（status-service.ts）なので、主にUI実装とデータ整備が中心
- ベンチマークデータの「正確さ」よりも「相対的な位置づけのわかりやすさ」を優先
- 年1回のベンチマーク更新はseed.tsの更新 + `npm run db:seed` で対応
- **注意**: 「推定値」のままでも機能としては動作するが、ユーザーに「世の中との比較」として提示する以上、最低限の根拠は必要
