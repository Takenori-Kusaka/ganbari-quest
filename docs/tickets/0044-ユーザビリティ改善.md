# ユーザビリティ改善

### ステータス

`Done`

---

### 概要

本番運用で得られた知見とユーザーフィードバックをもとに、全体的なユーザビリティを向上させる。

### 背景・動機

本番運用開始に伴い、日常的に使う中で以下のユーザビリティ課題が浮上している。子供（3歳〜14歳）と親の両方が快適に使えるよう、段階的に改善する。

**要件5**: ユーザビリティの積極的改善

### ゴール

#### ナビゲーション・導線

- [ ] パンくずリスト or ページタイトルの統一表示（親管理画面の各ページ）
- [ ] 管理画面のサイドバー or タブナビゲーションで現在地を明示
- [ ] 子供切り替え画面(`/switch`)からの戻り導線を改善

#### パフォーマンス・体感速度

- [ ] ページ遷移時のスケルトンローディング表示

#### アクセシビリティ

#### 管理画面

- [ ] 活動一覧の検索・フィルタ機能（174件+26件=200件の活動から探しやすく）

- [ ] 設定変更時の確認ダイアログ追加

### 技術方針

- トースト通知: Ark UIの`Toast`コンポーネントを `$lib/ui/primitives/Toast.svelte` にラップして利用
- ローディング: SvelteKitの `$navigating` store を活用したグローバルローディングバー
- スケルトン: `$lib/ui/components/Skeleton.svelte` を共通コンポーネントとして作成

### 作業メモ

- 優先度順: フォーム送信防止 > 完了フィードバック > ローディング > その他
- 子供画面のUXは年齢帯別（baby/kinder）で異なるアプローチが必要
