# ユーザビリティ改善

### ステータス

`Done`

---

### 概要

本番運用で得られた知見とユーザーフィードバックをもとに、全体的なユーザビリティを向上させる。

### 背景・動機

本番運用開始に伴い、日常的に使う中で以下のユーザビリティ課題が浮上している。子供（3歳〜14歳）と親の両方が快適に使えるよう、段階的に改善する。

**要件5**: ユーザビリティの積極的改善

### ゴール

#### ナビゲーション・導線

- [ ] パンくずリスト or ページタイトルの統一表示（親管理画面の各ページ）
- [x] 管理画面のサイドバー or タブナビゲーションで現在地を明示
- [ ] 子供切り替え画面(`/switch`)からの戻り導線を改善
- [ ] 操作完了時のフィードバック統一（トースト通知コンポーネント）

#### フォーム・入力

- [x] フォーム送信中のローディング表示（二重送信防止）— `submitting` 状態管理 + `disabled` 属性で実装済み
- [x] 活動記録時の確認ダイアログにアニメーション追加（子供の達成感を演出）— `animate-bounce-in` 等で実装済み
- [ ] 数値入力フィールドのステッパー（+/-ボタン）追加

#### パフォーマンス・体感速度

- [x] ページ遷移時のスケルトンローディング表示
- [x] 画像（アバター等）の遅延読み込み（loading="lazy"）
- [ ] Service Worker でオフライン時の基本的なフォールバック表示

#### アクセシビリティ

- [ ] Lighthouse Accessibility 95→100点を目指す（残り5%の改善項目調査）
- [ ] キーボードナビゲーションの動作確認・修正

#### 管理画面

- [x] 活動一覧の検索・フィルタ機能（174件+26件=200件の活動から探しやすく）
- [ ] 子供ごとの最終活動日時表示（ダッシュボード）
- [ ] 設定変更時の確認ダイアログ追加

### 技術方針

- トースト通知: Ark UIの`Toast`コンポーネントを `$lib/ui/primitives/Toast.svelte` にラップして利用
- ローディング: SvelteKitの `$navigating` store を活用したグローバルローディングバー
- スケルトン: `$lib/ui/components/Skeleton.svelte` を共通コンポーネントとして作成

### 作業メモ

- 優先度順: フォーム送信防止 > 完了フィードバック > ローディング > その他
- 子供画面のUXは年齢帯別（baby/kinder）で異なるアプローチが必要

### 成果・結果

**実装済み:**
- フォーム送信中のローディング表示（`submitting` 状態 + `disabled` + `animate-spin`）
- 確認ダイアログのアニメーション（`animate-bounce-in`、💮はなまるスタンプ等）
- 画像の遅延読み込み（`loading="lazy"`）

**追加実装（2026-03-16）:**
- ナビゲーションプログレスバー（`NavigationProgress.svelte`）— ページ遷移時にインジケーター表示
- スケルトンローディング（`Skeleton.svelte` + `skeleton-block` CSSクラス）— 子供・管理画面の両レイアウトに統合
- 管理画面タブナビゲーションに現在地ハイライト（`aria-current="page"` + 青背景）— 実装済み
- 活動一覧の検索・フィルタ機能（テキスト検索 + カテゴリフィルタ）— 実装済み
