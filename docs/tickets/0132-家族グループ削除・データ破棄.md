### 概要

owner が解約リクエストした場合、または全ユーザーが離脱した場合に、家族グループの全データを削除するフロー。30日の猶予期間を設ける。

### トリガー条件

| トリガー | 開始者 |
|----------|--------|
| owner が「解約」ボタンを押す | owner |
| owner/parent が「削除申請」を出す | owner/parent |
| 全メンバーがテナントから離脱 | システム自動 |

### フロー

```
1. トリガー発生
2. テナントステータスを grace_period に変更
3. 全メンバーに通知メール送信
4. 猶予期間（30日）:
   - データは参照可能（読み取り専用）
   - 新しいデータの追加・編集は不可
   - 解約キャンセル可能（owner のみ）
5. 猶予期間満了後:
   - DynamoDB: TENANT#{id} 配下の全アイテム削除
     - CHILD#, ACTIVITY#, COMBO#, SETTING, LICENSE 等
   - S3: tenant/{id}/ 配下の全オブジェクト削除
     - アバター画像等
   - Cognito: メンバーの custom:tenantId をクリア
     - ユーザーアカウント自体は残す（再サインアップ可能）
   - Stripe: サブスクリプションをキャンセル
6. 削除完了メール送信
```

### 猶予期間中の制御

```typescript
// hooks.server.ts or ミドルウェア
if (context.tenantStatus === 'grace_period') {
  // 書き込み系 API はブロック
  if (event.request.method !== 'GET') {
    throw error(403, 'テナントは解約手続き中です。読み取り専用モードです。');
  }
}
```

### データ削除バッチ

```
実行方法: Lambda スケジュール（EventBridge 日次実行）

1. grace_period かつ expiresAt < now() のテナントを検索
2. テナント配下の全データを BatchWriteItem で削除
3. S3 オブジェクト削除
4. Cognito ユーザー属性クリア
5. Stripe サブスクキャンセル
6. テナントステータスを deleted に変更
7. 削除完了メール送信（SES）
8. CloudWatch にログ出力
```
