### 概要

Web版、Android版、iOS版を提供し、どの環境からでもアクセスできるようにする。PWAをベースとし、Capacitorでネイティブラッパーを作成する方式を採用する。

### 背景・動機

子育て世帯の主要デバイスはスマートフォン/タブレット。App Store/Google Playでの配布がメイン顧客へのリーチに必須。Web版は補助的な位置付け。

### 対応方針

#### 技術スタック

```
[SvelteKit Web App (SSR/SPA)]
         │
    ┌────┴────┐
    ▼         ▼
  [PWA]    [Capacitor]
    │         │
    │    ┌────┴────┐
    │    ▼         ▼
    │  [Android]  [iOS]
    │    │         │
    ▼    ▼         ▼
  Web  Google    Apple
  版   Play     App Store
```

#### PWA化

1. `vite-plugin-pwa` でService Worker生成
2. `manifest.json` 設定
   - `display: standalone`
   - アイコン（192x192, 512x512）
   - テーマカラー
3. オフライン対応
   - 静的アセットのキャッシュ
   - オフライン時のフォールバックページ
   - オンライン復帰時のデータ同期

#### Capacitor統合

```bash
npm install @capacitor/core @capacitor/cli
npx cap init ganbari-quest com.ganbariquest.app
npx cap add android
npx cap add ios
```

**ネイティブ機能の利用:**
- プッシュ通知（`@capacitor/push-notifications`）
- カメラ（アバターアップロード用）
- ハプティクス（タップフィードバック）
- App Badge（未記録活動数表示）

#### ストア公開

**Google Play:**
- 開発者アカウント: $25（一回のみ）
- AABビルド
- ストア掲載情報（スクリーンショット、説明文）
- プライバシーポリシーURL必須
- 対象年齢設定（ファミリー向けポリシー準拠）

**Apple App Store:**
- Apple Developer Program: $99/年
- Xcode でビルド・署名
- App Review Guidelines 準拠
- 子供向けカテゴリの場合、追加審査あり

#### Web版アクセス制御

**方式案:**
1. ~~証明書ベース~~（運用コスト高い）
2. **ライセンスキー方式**: アプリ内購入後にライセンスキー発行→Web版でキー入力
3. **OAuth + プラン検証**: ログイン時にプラン情報確認、Web版は有料プランのみ
4. **アプリ版限定**: Web版は提供せず、PWA/ネイティブのみ

**推奨**: 方式3（OAuth + プラン検証）
- Web版は全機能利用可（有料プランのみ）
- 無料プランはアプリ版のみ
- 技術的にはCognito JWTのカスタムクレームでプラン判定
