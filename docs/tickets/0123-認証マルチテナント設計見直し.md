### AUTH_MODE 設計

| AUTH_MODE | 認証 | テナント | DB | 想定環境 |
|-----------|------|---------|-----|---------|
| `local` | なし（全ルート許可） | 単一（暗黙 'local'） | SQLite | NUC, Docker, LAN内 |
| `cognito` | Email/Password + MFA | マルチテナント | DynamoDB | AWS Lambda, インターネット公開 |

**ローカル版（AUTH_MODE=local）:**
- ログイン画面なし。アクセス時に即座に管理画面/子供画面を表示
- Identity: `{ type: 'local' }` (常に認証済み扱い)
- Context: `{ tenantId: 'local', role: 'owner', licenseStatus: 'none' }`
- 現行の PIN 認証は廃止（LAN 内アクセス制限で十分）

**AWS版（AUTH_MODE=cognito）:**
- Cognito Email/Password + MFA
- Identity: `{ type: 'cognito', userId, email }`
- Context: `{ tenantId, role, licenseStatus }`

### ロール設計

| ロール | 内部名 | 権限 | ログイン後の画面 |
|--------|--------|------|-----------------|
| 親 | owner | 管理画面フルアクセス + ライセンス管理 | /admin |
| 共用 | parent | 管理画面アクセス + ライセンス管理 | /admin（子供選択画面経由） |
| 子供 | child | 子供画面のみ | /child/{childId} |

**共用アカウント:**
- 1つの Email/Password を家族全員で共有するアカウント
- ロールは `parent`（管理画面アクセス可）
- ログイン後に子供選択画面を表示（どの子供として使うか選択）
- 親が操作する場合はそのまま管理画面へ

```

```

### 決済連携設計（Stripe）

**選定理由:** Stripe Japan（日本法人）。コンビニ決済・PayPay対応で日本ユーザーにフレンドリー。個人事業主利用可。手数料 3.6%。

**購入フロー:**
```
1. ユーザーが料金プランページで「購入」ボタン
2. SvelteKit → Stripe Checkout Session 作成 → Stripe ホスティングページにリダイレクト
3. ユーザーが決済（クレカ / コンビニ / PayPay）
4. Stripe Webhook: checkout.session.completed
5. Lambda: ライセンスキー生成 → DynamoDB に保存 → SES でメール送信
6. ユーザーがライセンスキーを使ってサインアップ
```

**サブスク更新フロー:**
```
1. Stripe が自動課金（月額/年額）
2. Webhook: invoice.paid → ライセンス有効期限延長
3. 課金失敗: invoice.payment_failed → 猶予期間 → ライセンス停止
```
