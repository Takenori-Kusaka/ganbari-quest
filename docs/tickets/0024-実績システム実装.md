### 概要

子供が大きな節目を達成した際に「実績解除」として特別なポイントと称号を付与するシステムを実装する。実績は客観的に達成が確認できるもの（システムが自動判定可能な条件）に限定する。

### 背景・動機

日々の活動記録に加えて、中長期的な目標達成を可視化・報酬化することで、子供のモチベーションを高める。例えば「連続7日記録」「全カテゴリ制覇」など、マイルストーン的な達成感を演出する。

### ゴール

- [ ] `achievements` テーブル（実績マスタ）
  ```
  id, name, description, icon, category, condition_type, condition_value,
  bonus_points, rarity, sort_order, created_at
  ```
  - condition_type: `streak_days`, `total_activities`, `category_complete`, `level_reach`, `total_points` 等
  - rarity: `common`, `rare`, `epic`, `legendary`

  id, child_id, achievement_id, unlocked_at

  - UNIQUE(child_id, achievement_id)

  - `checkAndUnlockAchievements(childId)` — 全条件をチェックし未解除の実績を解除
  - `getChildAchievements(childId)` — 解除済み実績一覧
  - `getAllAchievements()` — 全実績一覧（進捗率表示用）
- [ ] `src/routes/api/v1/achievements/[childId]/+server.ts` (GET)
- [ ] `src/lib/domain/validation/achievement.ts` — 実績条件の定義・判定ロジック
- [ ] 活動記録時に自動チェック（activity-log-service から呼び出し）
- [ ] シードデータ: 初期実績マスタ（10〜20件）
  - はじめての記録（初回活動記録）
  - 3日連続（連続3日記録）
  - 7日連続（連続7日記録）
  - 30日連続（連続30日記録）
  - カテゴリマスター（1カテゴリ全活動記録）
  - 全カテゴリ制覇（全5カテゴリを1日で記録）
  - レベル5到達
  - レベル10到達（最高レベル）
  - 100ポイント達成
  - 1000ポイント達成
- [ ] 実績解除時のポイント付与（point_ledger に `achievement` タイプで記録）
- [ ] ユニットテスト
- [ ] 統合テスト

### 技術方針

- 実績チェックは活動記録の `recordActivity` 内で自動実行
- 新規解除があった場合、レスポンスに `unlockedAchievements` を含める
- 子供画面での解除演出は #0026（kinder）/ #0029（baby）で実装
- 条件判定はストラテジーパターンで拡張可能にする
