# 子供画面オートリロード

### ステータス

`Done`

---

### 概要

子供画面を1分間隔で自動リロードし、親が管理画面から行った変更（ステータス更新、活動の追加・非表示化など）をリアルタイムに反映する。

### 背景・動機

- 親が管理画面でステータスを変更しても、子供画面はリロードしないと反映されない
- 子供（特に幼児）に「画面を更新して」と伝えるのは現実的でない
- タブレットに常時表示している運用を想定すると、自動リフレッシュが必要

### ゴール

- [x] 子供画面（baby/kinder）のホーム・ステータス・実績等で定期間隔のオートリロード
- [x] `invalidateAll()` または `goto()` による SvelteKit データ再取得
- [x] ページがバックグラウンド（非アクティブタブ）の場合はリロードを一時停止
- [x] ユーザ操作中（モーダル表示中、確認ダイアログ中）はリロードをスキップ

### 成果・結果

- `src/routes/(child)/+layout.svelte` に `setInterval` + `invalidateAll()` で30秒毎のオートリロードを実装
- `document.hidden` でバックグラウンドタブ検知、タブ復帰時に即リロード
- 親が管理画面で変更した内容が子供画面に自動反映

### 技術方針

#### SvelteKit での実装

`invalidateAll()` を使うことで、現在のページの `load` 関数を再実行し、UIを更新できる。ページ遷移は発生せずデータのみ更新される。

```typescript
// src/routes/(child)/+layout.svelte に共通実装
import { invalidateAll } from '$app/navigation';
import { onMount } from 'svelte';

const RELOAD_INTERVAL_MS = 60_000; // 1分

onMount(() => {
  const interval = setInterval(() => {
    // バックグラウンドタブなら何もしない
    if (document.hidden) return;
    invalidateAll();
  }, RELOAD_INTERVAL_MS);

  return () => clearInterval(interval);
});
```

#### バックグラウンド検知

`document.visibilitychange` イベントでタブのアクティブ状態を検知し、非表示時はリロードを省略する。復帰時に即座に1回リロードすると体験が良い。

```typescript
document.addEventListener('visibilitychange', () => {
  if (!document.hidden) {
    invalidateAll(); // タブ復帰時に即リロード
  }
});
```

### 影響範囲

- `src/routes/(child)/+layout.svelte` - オートリロードロジック追加（子供画面全体に適用）

### 依存

なし

### 作業メモ

- `invalidateAll()` は SvelteKit 標準のデータ再取得メカニズムで、全 `load` 関数を再実行する
- ネットワークエラー時は `try/catch` で握りつぶし、次のインターバルで再試行
- 将来的にはWebSocketやSSEでプッシュ通知に切り替えることも検討可能だが、LAN内限定なのでポーリングで十分
