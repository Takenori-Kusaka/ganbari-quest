## 背景と問題分析

### 現在の「きせかえショップ」の限界

現在の着せ替えショップ（#0086 で実装済み）は 19 アイテム（背景 8、フレーム 6、エフェクト 5）を揃え、ポイント消費 → 購入 → 装備の仕組みは完全に動作している。しかし **本質的な UX 上の問題** がある：

| 問題点 | 詳細 |
|--------|------|
| **表示が小さすぎる** | アバターは Header の 36px 円にしか表示されない。背景グラデーションやフレーム色の違いは、この小ささではほぼ判別不能 |
| **体験に繋がらない** | 背景色が変わっても子供の日常操作（記録→達成→ポイント獲得）の体験は一切変わらない。装飾は「持っている」だけで、使っている実感がない |
| **達成の瞬間が全員同じ** | 活動の記録完了時に再生される効果音（`record-complete.mp3`）・視覚演出（✅ bounce-in + ポイント pop）は全子供が同一。レベルや所持アイテムに関係なく変化しない |
| **ポイントの使い道が弱い** | ポイントを貯める → ショップでアバター装飾を買う、のサイクルが「頑張った結果を体感できる報酬」になっていない |

### 子供の心理と報酬設計

子供のモチベーションを維持するのに効果的な報酬は、**自分の行動に対するフィードバックが変化すること**である：

1. **音が変わる** → 「頑張ったらかっこいい音が鳴るようになった！」
2. **演出が変わる** → 「達成したら花火が出るようになった！」
3. **見た目が変わる** → 「アバターが光っている！」（現在の着せ替え）

現在は 3 のみ実装されているが、インパクトの大きさは **1 > 2 >> 3** である（日々何度も体験するのが効果音と演出であり、アバターのフレーム色ではない）。

## 提案: 「ごほうびショップ」への進化

現在の着せ替えショップを廃止するのではなく、**「ごほうびショップ」として拡張**し、子供が日常的に体験するフィードバック自体をカスタマイズできるようにする。

#### 技術設計

- 新しい MP3 ファイルを `static/sounds/custom/` に配置
- `avatar_items` テーブルに `category: 'sound'` のアイテムを追加（既存スキーマで対応可能）
- `child_active_avatars` テーブルに `soundId` カラムを追加（既存の bgId/frameId/effectId と同じパターン）


## 注意事項

- **効果音の素材**: フリー素材 or AI 生成で用意する。ライセンスに注意
- **パフォーマンス**: 演出アニメーション（特に Canvas 系）は低スペック端末でも 60fps を維持できること
- **既存アイテムは削除しない**: 着せ替え（アバター装飾）はそのまま残す。あくまでカテゴリ追加による拡張
- **段階的リリース**: Phase 1（きろくおん）を先行実装し、子供の反応を見てから Phase 2 に進む
- **ポイントバランス**: 日常的に貯まるポイント量に対して適切な価格設定になっているか要検証

## 設計判断の根拠

現在の着せ替えショップが「意味がない」のではなく、**報酬の体験が弱い** ことが問題。子供が毎日何度も体験する「記録完了の瞬間」のフィードバックをカスタマイズ可能にすることで：

1. **日常的な達成感の強化** → 好きな音・演出で記録が報われる
2. **ポイントの価値向上** → 「貯めたら体験が変わる」という実感
3. **継続モチベーション** → 「次はあの演出が欲しい」という目標設定
4. **自己表現** → 自分だけの達成スタイルを作れる

これはゲーミフィケーションにおける **Process Reward（プロセス報酬）** の原則に沿っており、単なるコスメティック報酬（見た目のみ）よりも行動継続に効果が高い。
