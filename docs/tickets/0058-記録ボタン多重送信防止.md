# 0058 記録ボタンの多重送信防止

## Status: Done


### 概要

活動記録ボタン押下から完了表示までのタイムラグ中に連打されると、APIリクエストが多重送信される。ローディング状態を挟んで防止する。

### ゴール

- [x] `use:enhance` のコールバック開始時に `submitting = true` を設定
- [x] submitting中はボタンをdisabledにし、ローディングスピナー表示
- [x] baby/kinder の両ホーム画面に適用

### 対応方針

- `use:enhance` のコールバック開始時に `submitting = true` を設定
- submitting中はボタンをdisabledにし、ローディングスピナー表示
- baby/kinder の両ホーム画面に適用
- チェックリストのtoggle操作にも適用

### 成果・結果

- `src/routes/(child)/baby/home/+page.svelte` — `submitting` 状態管理 + ボタン `disabled` 属性で連打防止
- `src/routes/(child)/kinder/home/+page.svelte` — 同上
- ローディングスピナー（`animate-spin`）表示で処理中を視覚的にフィードバック
- チェックリストは `use:enhance` のデフォルト動作で自然に二重送信を抑制
