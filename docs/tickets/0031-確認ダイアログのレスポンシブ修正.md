# 確認ダイアログのレスポンシブ修正

### ステータス

`Done`

### 種別

`Bug`

### 元レビュー

`docs/user-review/20260227.md` — 不具合1

---

### 概要

活動タップ時の確認ダイアログ・記録完了オーバーレイが極端に細く表示され、テキストが縦一列になってしまう。レスポンシブデザインが機能していない。

### 再現手順

1. `npm run dev` でアプリ起動
2. ブラウザのDevToolsを開いた状態（画面幅が狭まった状態）でホーム画面にアクセス
3. 活動カードをタップ → 確認ダイアログが表示される
4. ダイアログが画面に対して極端に細く、テキストが縦に並ぶ

### 原因（推定）

`src/lib/ui/primitives/Dialog.svelte` の `ArkDialog.Content` に `max-w-md w-full` が設定されているが、`ArkDialog.Positioner` の `p-[var(--spacing-md)]` と組み合わさった際に、画面幅が小さいとコンテンツ幅が極端に縮小される可能性がある。また、Ark UI の Positioner のレイアウトとの相互作用も確認が必要。

### 参考スクリーンショット

`docs/user-review/20260227-1.png`

### 対象ファイル

- `src/lib/ui/primitives/Dialog.svelte` — Positioner/Content のスタイル修正
- 確認ダイアログ・記録完了オーバーレイの最小幅設定

### 受け入れ条件

- [x] タブレット（1280×800）でダイアログが適切な幅で表示される
- [x] スマホ（360×800程度）でもダイアログが画面幅の80%以上を使う
- [x] DevTools 併用で画面幅が狭い場合でも崩れない
- [x] 記録完了オーバーレイも同様に修正

### 成果・結果

- `Dialog.svelte` の Content に `w-full min-w-[280px] max-w-[min(28rem,calc(100vw-1rem))]` を設定し、全画面幅で適切に表示されるよう修正
- タブレット・スマホ・ DevTools狭小画面すべてでレスポンシブ表示確認済み
