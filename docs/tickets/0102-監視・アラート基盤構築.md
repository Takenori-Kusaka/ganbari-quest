### 概要

SaaS運用に必要な監視・アラート基盤を構築する。CloudWatch Alarms + SNS による障害検知・通知、外部からの死活監視、CloudWatch Dashboard による可視化を **追加費用 0円（AWS無料枠内）** で実現する。

### 背景・動機

現状の設計（#0064, #0068, #0078）では CloudWatch ログの保管のみ言及されており、**アラームによる異常検知→自動通知** の仕組みが存在しない。個人開発で常時ダッシュボードを見張ることは不可能であり、異常時にメール/Discordで即座に気付ける仕組みが必須。

#0075 SLA（99.5%/月 = 月間3.6時間ダウン許容）を達成するには、障害発生から15分以内に気付く仕組みが必要（#0078 障害対応フローの前提条件）。

### 対応方針

#### 1. SNS 通知基盤

全てのアラームの通知先となる SNS トピックを 1 つ作成。

```
SNS Topic: ganbari-quest-ops-alerts
  ├── Email: 運用者のメールアドレス
  └── (将来) Discord Webhook → Lambda 経由
```

- 初期はメール通知のみ（$0）
- Discord 通知は #0078（運用体制）で Lambda 関数を追加する形で拡張可能

#### 2. CloudWatch Alarms（10 枠内で優先配置）

| # | アラーム名 | メトリクス | 閾値 | 期間 | 優先度 |
|---|----------|-----------|------|------|--------|
| 1 | Lambda-Errors | Lambda Errors | ≥ 3 回 | 5分間 | P0 |
| 2 | Lambda-Throttles | Lambda Throttles | ≥ 1 回 | 5分間 | P0 |
| 3 | ApiGw-5xx | API Gateway 5XXError | ≥ 5 回 | 5分間 | P0 |
| 4 | ApiGw-4xx-Spike | API Gateway 4XXError | ≥ 50 回 | 5分間 | P1 |
| 5 | Lambda-Duration | Lambda Duration | ≥ 10,000ms (p99) | 5分間 | P1 |
| 6 | Lambda-ConcurrentExec | Lambda ConcurrentExecutions | ≥ 50 | 1分間 | P1 |
| 7 | DynamoDB-ThrottledReq | DynamoDB ThrottledRequests | ≥ 1 回 | 5分間 | P1 |
| 8 | DynamoDB-SystemErrors | DynamoDB SystemErrors | ≥ 1 回 | 5分間 | P0 |
| 9 | CloudFront-5xx | CloudFront 5xxErrorRate | ≥ 5% | 5分間 | P0 |
| 10 | (予備) | — | — | — | — |

**注意:**
- 無料枠は「基本アラーム」10個。高解像度（10秒間隔）は有料なので使用しない
- 評価期間は 5分（=300秒）で統一。1分間隔にすると高解像度メトリクスが必要になり有料化のリスクがある
- 1つ予備枠を残し、運用開始後に必要なアラームを追加する余地を確保

#### 3. CloudWatch Dashboard


- 無料枠のダッシュボード 3 枠のうち 1 枠を使用
- CDK で `cloudwatch.Dashboard` として定義し、コード管理する

#### 4. 外部死活監視（UptimeRobot）

| 設定 | 値 |
|------|-----|
| URL | `https://ganbari-quest.com/api/health` |
| 監視間隔 | 5分（無料プランの最短） |
| アラート通知先 | メール（同じ運用メール） |
| タイムアウト | 30秒 |

- CloudWatch は AWS 内部の監視。外部からユーザー視点で到達性を監視するため。
- AWS 側で障害が起きても通知できる経路として UptimeRobot を併用する
- AWS 障害時に CloudWatch 自体が動かない可能性があるため、外部監視は保険として重要

#### 5. /api/health の SaaS 対応

現行の `/api/health` は SQLite チェックのみ。SaaS 移行後は DynamoDB ＋ 追加チェックが必要。

```typescript
// SaaS版 /api/health
export const GET: RequestHandler = async () => {
  const checks = {
    dynamodb: await checkDynamoDB(),   // GetItem でテスト読み取り
    version: APP_VERSION,              // package.json の version
    region: process.env.AWS_REGION,
    uptime: process.uptime(),
  };
  const healthy = checks.dynamodb;
  return json(
    { status: healthy ? 'ok' : 'degraded', ...checks },
    { status: healthy ? 200 : 503 }
  );
};
```
