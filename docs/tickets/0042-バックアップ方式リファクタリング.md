# バックアップ方式リファクタリング

### ステータス

`Done`

---

### 概要

バックアップ機能を「ローカルバックアップ作成」（コア）と「外部転送」（プラグイン）の2層に分離し、汎用的なバックアップ基盤を構築する。Google Driveバックアップはオプショナルプラグインとして再実装する。

### 背景・動機

現在のバックアップは以下の2スクリプトで構成:
- `scripts/backup-db.cjs`: ローカルバックアップ（SQLite `.backup()`使用、最新10世代保持）
- `scripts/backup-to-gdrive.cjs`: Google Driveへの転送

OSSとして公開する場合、Google Driveは全ユーザーが使えるわけではない。以下の方式が考えられる:
1. **ローカルバックアップのみ**（メイン機能）→ ユーザーが自由に後処理
2. **AWS S3 / Azure Blob Storage** への転送
3. **NAS / 共有フォルダ** へのコピー
4. **Google Drive** への転送（現在の実装）

よって、コア機能としてはローカルバックアップの作成までを責務とし、外部転送はフック（post-backup script）で拡張可能にする。

**要件3**: バックアップ方式の汎用化

### ゴール

#### コア機能

- [ ] `scripts/backup-db.cjs` をメインのバックアップスクリプトとして整備
  - バックアップ先ディレクトリを環境変数 `BACKUP_DIR` で指定可能にする（デフォルト: `data/backups/`）
  - 保持世代数を環境変数 `BACKUP_RETENTION` で指定可能にする（デフォルト: 10）
  - バックアップ完了後のフックスクリプトパスを環境変数 `BACKUP_POST_HOOK` で指定可能にする
- [ ] バックアップ完了時にフックスクリプトを実行する仕組みを追加
  - フックにはバックアップファイルのパスが引数として渡される
  - フックが未設定の場合はスキップ（ローカルバックアップのみ）
- [ ] Web UI（管理画面 > 設定）から手動バックアップ実行ボタンを追加
- [ ] バックアップ履歴（日時、ファイルサイズ、転送先）をDBに記録

#### Google Driveプラグイン

- [ ] `scripts/hooks/gdrive-upload.cjs` としてGoogle Drive転送をフック化
  - 引数: バックアップファイルパス
  - 環境変数: `GDRIVE_CLIENT_ID`, `GDRIVE_CLIENT_SECRET`, `GDRIVE_REFRESH_TOKEN`, `GDRIVE_FOLDER_ID`
- [ ] `BACKUP_POST_HOOK=node scripts/hooks/gdrive-upload.cjs` で有効化

#### ドキュメント

- [ ] `docs/backup-guide.md` を作成
  - ローカルバックアップの設定方法
  - Google Driveフックの設定方法（OAuth2セットアップ含む）
  - カスタムフック（S3, NAS等）の作成方法
  - cron/タスクスケジューラでの自動化手順

### 技術方針

```
[Core] backup-db.cjs
  ├── SQLite .backup() でローカルコピー作成
  ├── 古いバックアップの自動削除（BACKUP_RETENTION世代）
  ├── バックアップ履歴をDBに記録
  └── BACKUP_POST_HOOK が設定されていれば実行
        │
        ├── [Hook] gdrive-upload.cjs  ← Google Drive転送
        ├── [Hook] s3-upload.sh       ← AWS S3転送（ユーザー自作例）
        └── [Hook] cp-to-nas.sh       ← NASコピー（ユーザー自作例）
```
