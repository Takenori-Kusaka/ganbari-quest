# #0108 ブランドアイデンティティ・ロゴ・Favicon 統一デザイン

## ステータス: Ready
## 優先度: 中
## 種別: デザイン改善

---

## 概要

現在の Favicon やアプリ内のビジュアル要素は、「子供の成長を助けるサービス」というコンセプトを適切に表現できていない。SaaS展開やログイン画面の整備に向けて、ブランドアイデンティティを統一的に設計し、Favicon・ロゴ・各画面での一貫したビジュアル表現を確立する。

## 現状の問題

### 1. Favicon が不適切

- 現在: 青い角丸四角に黄色い星＋顔（`static/favicon.svg`）
- 「子供の成長を助ける」サービスイメージと結びつかない
- 16x16px で視認性が低い

### 2. ロゴが存在しない

- ロゴ専用の画像ファイル・コンポーネントが一切ない
- ログイン画面では 👨‍👩‍👧 絵文字で代替している
- ヘッダーにもロゴ要素がない

### 3. ブランドガイドラインが未整備

- UI設計書（06-UI設計書.md）にカラーパレット・タイポグラフィはあるが、ロゴ・ビジュアルアイデンティティの定義がない
- 各画面のアイコン使用が絵文字ベースで統一感に欠ける

### 4. 画面ごとの不統一

| 画面 | 現在のビジュアル | 問題 |
|------|----------------|------|
| ブラウザタブ | 星キャラの Favicon | サービス内容が伝わらない |
| ログイン画面 | 👨‍👩‍👧 絵文字 + ひらがなタイトル | ロゴ不在、プロダクト感が弱い |
| 子供ホーム画面 | アバター + ポイント表示 | ブランド要素なし |
| 親管理画面 | テキストのみのヘッダー | ブランド要素なし |
| PWA インストール時 | Favicon がそのまま使われる | 専用のアプリアイコンが必要 |

## 設計方針

### ブランドコンセプト

**「がんばりクエスト」= 子供の成長を冒険に変える**

| 要素 | 方向性 |
|------|-------|
| キーワード | 成長・冒険・がんばり・達成・家族 |
| トーン | 温かい・楽しい・信頼感・やさしい |
| ターゲット印象 | 子供: わくわく / 親: 安心・信頼 |

### ロゴデザイン要件

#### メインロゴ
- 「がんばりクエスト」のサービス名を含むロゴタイプ
- シンボルマーク + ワードマークの組み合わせ
- 子供の成長・冒険・達成感を象徴するモチーフ
  - 候補: 芽吹き🌱、星⭐、旗🚩、山の頂上、盾（クエスト）、上昇する矢印
- カラー: 既存のデザイントークンと調和（primary blue + accent yellow/green）

#### バリエーション
| 種別 | 用途 | サイズ |
|------|------|-------|
| フルロゴ | ログイン画面、ホームページ | 横幅 200px〜 |
| コンパクトロゴ | ヘッダー、ナビゲーション | 横幅 120px〜 |
| シンボルマーク | Favicon、PWAアイコン、OGP | 32〜512px |
| モノクロ版 | 印刷、低解像度環境 | 各サイズ |

### Favicon 要件

- 16x16, 32x32, 180x180（Apple Touch）, 192x192, 512x512（PWA）
- シンボルマークを単独で使用
- 小サイズでも識別可能なシンプルな形状
- SVG + PNG の両形式

### 各画面への適用

| 画面 | 適用内容 |
|------|---------|
| `src/app.html` | Favicon・OGPメタタグの更新 |
| `(parent)/login/+page.svelte` | ロゴ表示（絵文字 → メインロゴに置換） |
| `src/lib/ui/components/Header.svelte` | コンパクトロゴの追加（子供画面） |
| `(parent)/+layout.svelte` | 親管理画面ヘッダーへのロゴ表示 |
| `static/manifest.json` | PWA アイコンの更新 |
| ホームページ（将来） | フルロゴの使用 |

- [ ] Favicon が各ブラウザ（Chrome, Safari, Firefox）で正しく表示されること
- [ ] PWA インストール時にアプリアイコンが正しく表示されること
- [ ] ログイン画面でロゴが適切に表示されること
- [ ] ダークモード/ライトモードでロゴの視認性が確保されていること

## 注意事項

- デザイン制作にはFigma等のデザインツールまたはAI画像生成の利用を検討
- 既存のカラーパレット（UI設計書のデザイントークン）との調和を優先
- 子供向け画面と親向け画面で同一ロゴを使いつつ、周囲のトーンで差別化
- SaaS展開時のホームページ（LP）デザインとの一貫性を意識
