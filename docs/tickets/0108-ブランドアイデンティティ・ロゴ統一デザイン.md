# #0108 ブランドアイデンティティ・ロゴ統一デザイン【デザイン再設計】

## Status: Ready
## 優先度: 高
## 種別: デザイン改善

---

## 概要

現在の Favicon やアプリ内のビジュアル要素は、「子供の成長を助けるサービス」というコンセプトを適切に表現できていない。SaaS展開やログイン画面の整備に向けて、ブランドアイデンティティを統一的に設計し、Favicon・ロゴ・各画面での一貫したビジュアル表現を確立する。

## 現状の問題

### 1. Favicon が不適切

- 現在: 青い角丸四角に黄色い星＋顔（`static/favicon.svg`）
- 「子供の成長を助ける」サービスイメージと結びつかない
- 16x16px で視認性が低い

### 2. ロゴが存在しない

- ロゴ専用の画像ファイル・コンポーネントが一切ない
- ログイン画面では 👨‍👩‍👧 絵文字で代替している
- ヘッダーにもロゴ要素がない

### 3. ブランドガイドラインが未整備

- UI設計書（06-UI設計書.md）にカラーパレット・タイポグラフィはあるが、ロゴ・ビジュアルアイデンティティの定義がない
- 各画面のアイコン使用が絵文字ベースで統一感に欠ける

### 4. 画面ごとの不統一

| 画面 | 現在のビジュアル | 問題 |
|------|----------------|------|
| ブラウザタブ | 星キャラの Favicon | サービス内容が伝わらない |
| ログイン画面 | 👨‍👩‍👧 絵文字 + ひらがなタイトル | ロゴ不在、プロダクト感が弱い |
| 子供ホーム画面 | アバター + ポイント表示 | ブランド要素なし |
| 親管理画面 | テキストのみのヘッダー | ブランド要素なし |
| PWA インストール時 | Favicon がそのまま使われる | 専用のアプリアイコンが必要 |

## 設計方針

### ブランドコンセプト

**「がんばりクエスト」= 子供の成長を冒険に変える**

| 要素 | 方向性 |
|------|-------|
| キーワード | 成長・冒険・がんばり・達成・家族 |
| トーン | 温かい・楽しい・信頼感・やさしい |
| ターゲット印象 | 子供: わくわく / 親: 安心・信頼 |

### ロゴデザイン要件

#### メインロゴ
- 「がんばりクエスト」のサービス名を含むロゴタイプ
- シンボルマーク + ワードマークの組み合わせ
- 子供の成長・冒険・達成感を象徴するモチーフ
  - 候補: 芽吹き🌱、星⭐、旗🚩、山の頂上、盾（クエスト）、上昇する矢印
- カラー: 既存のデザイントークンと調和（primary blue + accent yellow/green）

#### バリエーション
| 種別 | 用途 | サイズ |
|------|------|-------|
| フルロゴ | ログイン画面、ホームページ | 横幅 200px〜 |
| コンパクトロゴ | ヘッダー、ナビゲーション | 横幅 120px〜 |
| シンボルマーク | Favicon、PWAアイコン、OGP | 32〜512px |
| モノクロ版 | 印刷、低解像度環境 | 各サイズ |

### Favicon 要件

- 16x16, 32x32, 180x180（Apple Touch）, 192x192, 512x512（PWA）
- シンボルマークを単独で使用
- 小サイズでも識別可能なシンプルな形状
- SVG + PNG の両形式

### 各画面への適用

| 画面 | 適用内容 |
|------|---------|
| `src/app.html` | Favicon・OGPメタタグの更新 |
| `(parent)/login/+page.svelte` | ロゴ表示（絵文字 → メインロゴに置換） |
| `src/lib/ui/components/Header.svelte` | コンパクトロゴの追加（子供画面） |
| `(parent)/+layout.svelte` | 親管理画面ヘッダーへのロゴ表示 |
| `static/manifest.json` | PWA アイコンの更新 |
| ホームページ（将来） | フルロゴの使用 |

## ⚠️ デザイン再設計の背景（なぜ Done にできないか）

### 現行デザインのマーケティング評価

#### 現在の `favicon.svg` / `logo.svg` — **不合格の理由**

| 評価軸 | 現状 | 問題 |
|--------|------|------|
| **第一印象** | 青い盾に星 | セキュリティソフト・VPN・保険アプリに見える。子育て・ゲーム・成長と結びつかない |
| **アプリカテゴリの伝達** | 伝わらない | アイコンだけを見て「子供の習慣化アプリ」と想像できる人はほぼゼロ |
| **競合との差別化** | なし | 盾+星の組み合わせはありふれており、AppStore/PlayStore で埋もれる |
| **子供への訴求** | 弱い | 「わくわく感」「冒険感」がない。子供が「これ使いたい！」と思う要素がない |
| **親への信頼感** | 曖昧 | 盾は「守る」イメージだが、教育・成長・楽しさとの結びつきが弱い |
| **フォント品質** | 問題あり | `Hiragino Maru Gothic Pro` は macOS 専用。Windows・Androidでは別フォントで描画される。意図したやさしい丸ゴシックが全環境で再現されない |
| **小サイズ視認性** | 不十分 | 盾内部に星を描くと 16px では潰れて何のアイコンかわからない |
| **独自性・記憶定着** | 低い | Duolingo のフクロウ、ClassDojo のモンスターキャラのような「このアプリといえばこれ」がない |

#### ベンチマーク比較

| アプリ | シンボル | なぜ機能しているか |
|--------|---------|-----------------|
| Duolingo | 緑のフクロウ | 愛着・習慣・学習を一体化。フクロウ = 知性 + かわいさ |
| Khan Academy Kids | シンプルなキャラ | 子供が「友達」に感じられるキャラクター |
| ClassDojo | カラフルなモンスター | 子供が喜ぶ非現実的なキャラ。個性・多様性を表現 |
| Forest | 木が育つ | アプリのコアバリュー（成長・継続）をそのままシンボルに |
| Habitica | RPGキャラクター | ゲーミフィケーションの本質を視覚化 |

**がんばりクエストが目指すべき方向性**: Forest（木の成長）と Habitica（RPG）の中間。
「子供が育つ = 木が育つ / クエストを進める」を一目で伝えるシンボル。

### Phase 2: デザイン評価・比較基準

生成された各候補を以下の評価軸で採点（5段階）し、最高スコアを採用する。
複数候補で同点の場合は主要ターゲット（P1: 田中ゆかり）視点での印象で最終選定する。

| # | 評価軸 | 内容 | 重み |
|---|--------|------|------|
| 1 | **第一印象の訴求** | 見た瞬間に「子供の成長アプリ」と伝わるか | ×2 |
| 2 | **独自性・記憶への定着** | 他のアプリと混同されないか。1週間後に思い出せるか | ×2 |
| 3 | **子供の感情的反応** | 子供（5〜10歳）が「かわいい」「やってみたい」と感じるか | ×1.5 |
| 4 | **親の信頼感** | 親が「うちの子に使わせていいアプリ」と感じるか | ×1.5 |
| 5 | **小サイズ視認性** | 16px（ブラウザタブ）〜32px（スマホ通知）で識別可能か | ×1 |
| 6 | **競合との差別化** | Duolingo・ClassDojo・Habitica と並んで存在感があるか | ×1 |
| 7 | **カラーパレット整合性** | 既存デザイントークン（青/黄/白）と調和するか | ×1 |

#### ワードマークのフォント問題への対処

現行の `logo.svg` / `logo-compact.svg` は `Hiragino Maru Gothic Pro` 等のシステムフォントを参照しているため、**Windows・Android では意図しないフォントで描画される**。以下のいずれかで対処すること:

| 対処法 | 概要 | 推奨度 |
|--------|------|--------|
| **フォントを `<path>` に変換** | テキストをアウトライン化し `d=""` のパスとして埋め込む | ◎（最も確実） |
| **Google Fonts（Noto Sans JP / M PLUS Rounded 1c）を `@font-face` で SVG 内に埋め込む** | Base64 エンコードして SVG 内に `<style>` で定義 | ○ |
| **画像化（PNG）でロゴを管理する** | SVG のフォント問題を回避 | △（拡大時に画質劣化） |

**推奨**: テキスト部分を Inkscape または Figma 等でパスに変換してから SVG として出力。
