### 概要

未登録ユーザーがサインアップ前にアプリの全機能を体験できるデモモードを実装する。デモ用の家族・子供データがプリセットされ、操作は可能だがAPIは全てダミー応答を返し、実データは一切記録されない。

### 背景・動機

- SaaS版への登録前に実際の操作感を確認したいユーザー向けのコンバージョン施策
- 子供向けアプリであり、親が「安全で適切か」を事前確認できることが重要
- 競合サービスの多くがデモ/トライアルを提供しており、比較検討時に不利にならないため

### ゴール

- [x] デモモードのルーティング（`/demo` 以下で全子供画面・親画面にアクセス可能）
- [x] デモ用データプロバイダ（家族1組 + 子供4名（baby/kinder/lower/teen各1名）+ 活動・ログ・実績・アバター等のプリセット）
- [x] デモ用モックサービス層（全APIがダミー応答を返す、書き込み系はno-op + 成功レスポンス）
- [x] デモモード表示バナー（「これはデモです。データは保存されません」の常時表示）
- [x] LP に「デモを体験する」CTAボタン追加
- [x] デモからサインアップへの導線（デモ操作中に「本番で使ってみる」ボタン）

### 対応方針

#### Phase 1: デモデータ基盤

1. `src/lib/server/demo/` にデモ専用データプロバイダを作成
   - `demo-data.ts`: 静的なデモ用家族・子供・活動ログ・実績データ
   - `demo-service.ts`: 全サービスのモック実装（読み取り系はデモデータを返す、書き込み系はno-op）

2. デモ用家族データ:
   ```
   家族: がんばり家
   ├── 親: がんばり太郎（管理者）
   ├── 子供1: がんばりベビー（1歳・babyモード）レベル3
   ├── 子供2: がんばりキッズ（5歳・kinderモード）レベル12
   ├── 子供3: がんばりジュニア（8歳・lowerモード）レベル28
   └── 子供4: がんばりティーン（13歳・teenモード）レベル52
   ```

3. 各子供に以下のデモデータをプリセット:
   - 活動ログ（直近30日分のダミー記録）
   - ポイント残高・レベル・経験値
   - 称号（獲得済み + 未獲得）
   - ステータス5軸の値
   - アバター（一部購入済みアイテム）
   - コンボ状態（連続記録日数）
   - デイリーミッション
   - チェックリスト
   - キャリアプラン（teenのみ）

#### Phase 2: デモルーティング

1. `src/routes/demo/` 以下にデモ専用レイアウトを作成
   - `+layout.server.ts`: 認証不要、デモコンテキストをセット
   - `+layout.svelte`: デモバナー表示 + サインアップ導線

2. 子供画面の既存ルート構造をデモでも使えるようにする
   - 方法A: `/demo/(child)/[childId]/` として子供画面をミラー（コンポーネント再利用）
   - 方法B: 既存ルートに `?demo=true` パラメータで分岐

3. 親管理画面もデモで閲覧可能にする
   - `/demo/(parent)/` でデモ用管理画面を表示

#### Phase 3: LP連携

1. LP に「デモを体験する」ボタン追加
2. デモ操作中のバナーに「本番で使ってみる → サインアップ」導線

### 技術的考慮事項

- **パフォーマンス**: デモデータは静的JSONからの読み込み。DB不要
- **セキュリティ**: デモモードではDB書き込み・外部API呼び出しを一切行わない
- **コード重複**: 既存コンポーネントを最大限再利用。デモ専用コンポーネントは最小限
- **SEO**: `/demo` はnoindexで検索エンジンに不要なインデックスを防止
