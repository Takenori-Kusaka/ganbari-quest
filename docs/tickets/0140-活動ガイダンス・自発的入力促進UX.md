# 活動ガイダンス・自発的入力促進UX

## Status: Done


### 概要

「いつ・どのタイミングで活動ボタンを押せばいいのか分からない」という課題と「デイリーミッション達成率0%」の問題を解消する。

**根本方針**: 子供が「ヒントを探しに行く」UIではなく、**見れば分かる・光っていれば押す** という受動的UXで自発入力を促す。子供に認知負荷を与えない。

---

### 背景・動機

#### ユーザーフィードバック

> 「親がどういうときにどの活動を入力していいのかを具体的に指示し、入力させた経験がないと、
> 子供が活動への入力を行わない。またそれ以外の活動についてはどのようなケースで押すのか
> わからず、自発的な入力が期待できない」

#### 現状の問題と根本原因

**問題1: デイリーミッション達成率0%**

現在のミッションセクション表示形式（テキストリスト）:

```
⬜ 🤸 たいそう
⬜ 📖 えほん
⬜ 🍽️ しょっきはこび
```

→ **文字主体のリストで4歳には読めない**。さらにミッションリストとその下の活動グリッドが視覚的に無関係に見える。子供はリスト（上）とボタン（下）を2つの別物として認識し、両者が連動していることを理解できない。

**問題2: 「いつ押すか」ブリッジの欠如**

```
現状: 活動カード → ???無関係??? → 子供が押す（起きない）
目標: 活動カード + 「はみがきのあとに！」常時表示 → 子供が文脈を理解して押す
```

このブリッジ（コンテキストトリガー）を親が口頭で全活動に教えるのは現実的でない。アプリが代行する必要がある。

#### 当初案（v1）が機能しない理由

| 当初の提案 | 問題点 |
|-----------|-------|
| 「？」ヒントボタン | 「困ったらボタンを押して確認する」はメタ認知が必要。4歳は困ると親を呼ぶか諦める。能動的なヘルプ探索は発生しない。 |
| 「さいきんやってないよ！」バッジ | ひらがな読み取り前提。否定的フレーミングで幼児に刺さらない。「いつ押すか」の根本問題を解決しない。 |
| ホーム上部のおすすめセクション | ミッションセクションと同じ「別リスト」構造の繰り返し。ミッション達成率0%の問題を再現するだけ。 |

#### 既存インフラ（再利用可能）

- `activities.description`: スキーマ存在・シード値投入済み。子供画面に**一度も表示されていない**
- `daily_missions` テーブル + `daily-mission-service.ts`: ミッション判定インフラ完備
- `ActivityCard.svelte`: props追加で対応可能
- `findRecentActivityIds(childId, sinceDate, tenantId)`: dormant活動の特定に再利用可（将来）

### UXデザイン詳細

#### Phase 1: ミッションカードの視覚強調

**現状（問題）**:
```
┌─────────────────────────────┐
│ 🎯 きょうのミッション    1/3 │ ← このリストは子供に伝わらない
│  ⬜ 🤸 たいそう            │
│  ⬜ 📖 えほん              │
│  ⬜ 🍽️ しょっきはこび      │
└─────────────────────────────┘

[たいそう] [ボール]  [水泳]    ← グリッド内カードは普通の見た目
[えほん]  [ずけい]  [さんすう]   ミッションかどうか視覚的に不明
```

**修正後（目標）**:
```
┌─────────────────────────────┐
│ 🎯 きょうのミッション    1/3 │ ← リストは親の確認用として残す
│  ⬜ 🤸 たいそう            │
│  ⬜ 📖 えほん              │
│  ⬜ 🍽️ しょっきはこび      │
└─────────────────────────────┘

[✨たいそう✨] [ボール]  [水泳]   ← ミッションカードが光っている！
[✨えほん✨]  [ずけい]  [さんすう]  子供が直感: 「光ったやつを押す」
```

**CSSイメージ**:
```css
/* isMission=true のとき */
.card-mission {
  border: 2px solid gold;
  box-shadow: 0 0 12px rgba(255, 200, 0, 0.6);
  animation: pulse-gold 2s ease-in-out infinite;
}
@keyframes pulse-gold {
  0%, 100% { box-shadow: 0 0 8px rgba(255, 200, 0, 0.4); }
  50%       { box-shadow: 0 0 20px rgba(255, 200, 0, 0.8); }
}
```

**ミッションクリア専用演出**: 通常の `+5P やったね！` ではなく `⭐ ミッションクリア！ +ボーナスP ⭐` を表示。毎回の達成に特別感を与え、「光っているボタン = 今日やること」の学習を強化する。

#### Phase 2: インライントリガーヒント

**カード表示イメージ（`triggerHint` あり）**:
```
┌──────────────┐
│     🦷        │
│  はみがき     │
│              │
│ はみがきのあと│ ← 2〜3語・小さいが常時見える
│     に！     │
└──────────────┘
```

**設計原則**:
- **子供が何もしなくても見える**（ボタン不要・ダイアログ不要）
- **2〜3語以内**（4歳の視野・注意スパンに合わせる）
- **文体**: `〜のあとに！` `〜したら！` `〜ができたら！` など行動完了トリガーを口語で
- ヒントがない場合: 表示しない（既存カードへの影響ゼロ）

**`description` の扱い**: 指導要領由来の記述で大人向け文体（「時と場に応じた適切な言葉遣い」）。子供への直接表示には不適。親管理画面のみで参照情報として表示する。

### 考慮事項・注意点

- **高さの整合性**: `triggerHint` の有無でカード高さが変わるとグリッドが崩れる。CSS Grid `align-items: stretch` か全カード固定高さで対処。
- **アニメーション負荷**: `box-shadow` pulse は GPU負荷が低い。`transform: scale` との組み合わせで視覚的インパクトを高める。ミッション3枚のみが光るため全体負荷は問題なし。
- **triggerHint の文字数上限**: カード内30字程度。「朝のはみがきが終わったら」より「はみがきのあとに！」が適切。親向けに記入ガイドを表示する。
- **DBスキーマ変更**: pull 時は `npx drizzle-kit push` 必須。`CLAUDE.md` の手順（stop → migrate → build → up）に従うこと。
