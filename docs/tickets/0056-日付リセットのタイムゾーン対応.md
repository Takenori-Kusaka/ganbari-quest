# 0056 日付リセットのタイムゾーン対応（UTC→JST）

## Status: Done


### 概要

活動の「一日リセット」がUTC基準で判定されているため、日本時間(JST)の00:00〜09:00の間に日付が前日扱いになる。日本時間でリセットされるように修正する。

### 問題の詳細

`todayDate()` が `new Date().toISOString().slice(0, 10)` でUTC日付を返しているため、JSTとの9時間のズレが発生する。

**具体例:**
- JST 2026-03-06 02:00 = UTC 2026-03-05 17:00
- `todayDate()` → `"2026-03-05"` (前日扱い)
- → 今日の活動記録が「昨日」として扱われる

### 影響範囲

| 機能 | ファイル | 問題 |
|------|---------|------|
| 活動記録の「今日」判定 | `src/lib/domain/validation/activity.ts:73` | `toISOString()` がUTC |
| ログインボーナス | `src/lib/server/services/login-bonus-service.ts:18` | 同上 |
| 実績の連続日数計算 | `src/lib/server/services/achievement-service.ts` | UTC基準 |
| チェックリスト | `src/routes/(child)/checklist/+page.server.ts:5` | `getDate()` (ローカル時間) |
| チェックリスト管理 | `src/routes/(parent)/admin/checklists/+page.server.ts:19` | 同上 |

**矛盾**: ページサーバーの一部は `getDate()`（ローカル時間）、サービス層は `toISOString()`（UTC）を使っており、不整合がある。

### 対応方針

#### 方針: JST固定の `todayDateJST()` を一元化

1. **`src/lib/domain/date-utils.ts` を新規作成**
   - `todayDateJST(): string` — JSTの今日の日付を `YYYY-MM-DD` で返す
   - `toJSTDateString(date: Date): string` — 任意のDateをJST日付文字列に変換
   - 実装: `new Date(Date.now() + 9 * 60 * 60 * 1000).toISOString().slice(0, 10)` または `Intl.DateTimeFormat('ja-JP', { timeZone: 'Asia/Tokyo' })`

2. **全 `todayDate()` を `todayDateJST()` に置換**
   - `activity.ts` のバリデーション層
   - `login-bonus-service.ts`
   - `achievement-service.ts`
   - `checklist/+page.server.ts` (2箇所)

3. **Docker環境変数にTZ追加**（補助策）
   - `docker-compose.yml` に `TZ: Asia/Tokyo`
   - `.env.example` に `TZ=Asia/Tokyo` 追記

4. **テスト追加**
   - `todayDateJST()` のユニットテスト
   - 既存テストでタイムゾーンモック確認

### 影響を受ける時間帯

- **JST 00:00〜08:59**: この間に日付判定がズレる（UTC前日扱い）
- 修正後は日本時間の0時でリセットされる

### 成果・結果

- `src/lib/domain/date-utils.ts` を新規作成、`todayDateJST()` と `toJSTDateString()` を実装
- JST固定で9時間オフセット対応
- 全 `todayDate()` 呼び出しを `todayDateJST()` に置換:
  - `src/lib/domain/validation/activity.ts`
  - `src/lib/server/services/login-bonus-service.ts`
  - `src/lib/server/services/achievement-service.ts`

  - `src/routes/(parent)/admin/checklists/+page.server.ts`
- `docker-compose.yml` に `TZ=Asia/Tokyo` 環境変数追加
- `tests/unit/domain/date-utils.test.ts` にユニットテスト追加済み
