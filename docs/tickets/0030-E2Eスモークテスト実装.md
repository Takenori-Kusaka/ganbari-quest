### 概要

Playwright を使ったE2Eスモークテストを実装し、アプリが正常に起動・動作することを自動検証する。DB初期化（スキーマ反映＋シード投入）からアプリ起動、主要画面の表示確認までを一気通貫でテストする。

### 背景・動機

- `npx drizzle-kit push` でスキーマ反映時に「no such column」エラーが発生（DB が古いスキーマのまま）
- 多数のチケットを連続実装してきたが、ブラウザでの統合的な動作確認が不十分
- 今後の開発でもリグレッションを早期発見するため、E2Eスモークテストの基盤が必要

### 前提条件

- Playwright (`@playwright/test@1.58.2`) インストール済み
- `playwright.config.ts` 設定済み（tablet/mobile プロジェクト、baseURL: localhost:5173）
- `tests/e2e/` ディレクトリ存在（`.gitkeep` のみ）
- 参考: `docs/reference/Playwright Windows Setup Guide.md`

### 技術方針

- `playwright.config.ts` の `webServer` 設定で `npm run dev` を自動起動
- テスト用 DB は `globalSetup` で初期化（DB なければ作成、あればシード確認+activity_logsクリア）
- ヘッドレスモード（Chromium）で実行、tablet (Desktop Chrome) と mobile (Pixel 7) の2プロジェクト
- mobile は tablet 完了後に実行（`dependencies: ['tablet']`）— SQLite 並列アクセス回避
- スクリーンショットは失敗時のみ取得（既存設定通り）
- 活動記録テストは並列テスト競合に対応（複数活動リトライヘルパー）

### 成果・結果

- 全52テスト（26ケース × tablet/mobile 2プロジェクト）グリーン
- 実行時間: 約2分
- サービス層のバグ修正: `recordActivity` の UNIQUE constraint 違反をハンドル（TOCTOU 対策）
