### 概要

法的ページ（利用規約・プライバシーポリシー・SLA・特定商取引法）をWebアプリ（SvelteKit）からHP（GitHub Pages LP）側に移設し、内容・デザインの全面見直しを行う。

### 背景・動機（ユーザー指摘事項）

1. **表示場所の問題**: 法的ページはHP（LP）側のリンクとして提供すべき。現在のWebアプリ内配置では、タイトルクリック時にWebアプリに誘導される（HPに誘導されるべき）
2. **プライバシーポリシーの表形式が不適切**: 表の罫線が表示されず読みづらい。法的文書で表形式は妥当でない
3. **デザイン不統一**: 利用規約とプライバシーポリシーで表示フォーマット・デザインが異なる
4. **特定商取引法ページ不要**: 個人事業主として届出をしていない現時点では記載不要
5. **サポートメールアドレス問題**: `support@ganbari-quest.com` は未取得で利用不可
6. **プラン情報のハードコーディング**: プラン変更時に法的文書も修正が必要になるリスク
7. **立ち位置が不明確**: 契約開始時に一読するリンクとしての位置づけが曖昧

### ゴール

- [x] 法的ページをHP（`site/`）側にHTML化して移設
- [x] プライバシーポリシーから表形式を廃止し、文章・リスト形式に統一
- [x] 全法的ページのデザインをLP（`site/index.html`）と統一
- [x] 特定商取引法ページを削除（個人事業主届出後に再作成）
- [x] `support@ganbari-quest.com` をGitHub Issuesリンクに差し替え
- [x] プラン情報のハードコーディングを除去し、汎用的な表現に変更
- [x] ヘッダーのロゴリンクをHP（LP）に誘導するよう修正
- [x] LP フッターのリンクをHP側法的ページに変更
- [x] SvelteKit側の `/legal/*` ルートを削除

### 対応方針

1. `site/` ディレクトリ配下に `terms.html`, `privacy.html`, `sla.html` を作成
2. LP（`site/index.html`）と統一されたデザイン（ヘッダー・フッター共通）
3. 法的文書の内容を見直し:
   - 表形式 → 文章・リスト形式に変換
   - 具体的な金額・プラン名 → 汎用的な表現（「料金ページをご覧ください」等）
   - サポートメール → GitHub Issues / お問い合わせフォーム
4. SvelteKit側の `src/routes/legal/` を削除
5. LP フッターリンクを相対パスに変更

### 残課題・次のアクション

- サポート用メールアドレスの取得（独自ドメインのメール設定）
- 個人事業主届出後の特定商取引法ページ再作成
- Stripe課金開始時のプラン詳細ページ作成
