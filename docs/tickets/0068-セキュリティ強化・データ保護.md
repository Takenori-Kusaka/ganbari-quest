### 概要

SaaS提供に必要なセキュリティ対策を実装する。データの暗号化、アクセス制御、監査ログ等を整備し、プライバシーポリシー（#0074）に準拠したデータ管理を実現する。

### 背景・動機

子供のデータ（活動記録、顔写真等）を扱うため、特に高いセキュリティ水準が求められる。WAFは費用面で初期導入しないが、Lambda AuthorizerやAPI Gatewayの設定で適切にAPIを保護する。

### 対応方針

#### セキュリティレイヤー

```
[CloudFront]
  ├── HTTPS強制（TLS 1.2+）
  ├── ジオブロック（日本国内限定、初期）
  ├── セキュリティヘッダ（CSP, HSTS, X-Frame-Options）
  └── リファラーチェック（任意）

[API Gateway]
  ├── スロットリング（100 req/sec デフォルト）
  ├── リクエストバリデーション
  └── CORS設定（オリジン制限）

[Lambda Authorizer]
  ├── Cognito JWT検証
  ├── テナントID抽出・注入
  ├── ロールベースアクセス制御（RBAC）
  └── レートリミット（ユーザー単位）

[DynamoDB]
  ├── 保存時暗号化（AWS managed key）
  ├── テナントキー分離（PKベース）
  └── RCU/WCU上限設定

[S3]
  ├── サーバーサイド暗号化（SSE-S3）
  ├── バケットポリシー（CloudFrontのみ許可）
  ├── パブリックアクセスブロック
  └── ライフサイクルポリシー（アバター90日保持等）
```

#### WAFなしでの防御策

| 脅威 | 対策 | 実装場所 |
|------|------|---------|
| DDoS | CloudFront標準DDoS保護（Shield Standard） | CloudFront |
| SQLインジェクション | DynamoDB使用のため対象外 | - |
| XSS | SvelteKitの自動エスケープ + CSPヘッダ | CloudFront + App |
| CSRF | SvelteKitのCSRF保護（Origin検証） | App |
| ブルートフォース | Cognito失敗回数制限 + Lambda Authorizerレートリミット | Cognito + Lambda |
| APIスクレイピング | API Gatewayスロットリング + ユーザー単位レートリミット | API Gateway |
| 不正アクセス | JWT検証 + テナント分離 | Lambda Authorizer |

#### 子供データの特別な保護

- アバター画像は署名付きURL（有効期限15分）でのみアクセス
- 子供の個人情報（名前・年齢）は表示名のみ保存（実名不要の設計）
- 活動記録は統計データとしてのみエクスポート可能（個別ログは非公開）
- 退会時のデータ完全削除（GDPR/個人情報保護法準拠）

#### 監査ログ

```
[CloudWatch Logs]
  ├── API Gateway: アクセスログ（最小保持: 30日）
  ├── Lambda: アプリケーションログ（最小保持: 14日）
  ├── Cognito: 認証イベント（最小保持: 90日）
  └── DynamoDB: TTLベースの操作ログテーブル
```

※ CloudWatch Logsの費用を抑えるため、保持期間は最小限に設定
