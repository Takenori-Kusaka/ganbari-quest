# 確認ダイアログの子供向けUX改善

## Status: Done

### 種別

`Enhancement`

### 元レビュー

`docs/user-review/20260227.md` — 気になった点4, 5

### 概要

活動タップ時の確認ダイアログについて、2つのUX課題がある。

1. **確認ダイアログ自体の是非**: 幼児に「きろくする？」という確認操作が伝わるのか。安全性（誤タップ防止）と子供の操作性のバランス
2. **モーダル外タップ時の動作**: 現状はモーダル外タップでキャンセルできる（Ark UI の `closeOnInteractOutside={closable}`）。子供にはボタンで明示的に操作させたほうがいいか

### 現状

- 活動カードタップ → 確認ダイアログ表示（「きろくする？」「きろく！」「やめる」）
- ダイアログ外をタップすると閉じる（キャンセル扱い）
- `Dialog.svelte` で `closeOnInteractOutside={closable}` が設定済み

### 検討事項

**確認ダイアログの代替案:**
- A) 現状維持: ダイアログで確認 → 安全だが操作ステップが多い
- B) ダイアログ廃止: 1タップで即記録 + 取り消しボタン表示 → シンプルだが誤タップリスク
- C) ロングプレス: 長押しで記録 → 誤タップ防止＋ダイアログ不要だが幼児には難しい
- D) スワイプ記録: カードをスワイプして記録 → 遊び要素あるが操作難易度高い

**モーダル外タップ:**
- 子供はダイアログの「外」という概念が分かりにくい可能性
- 明示的な「やめる」ボタンのみでキャンセルさせる方が安全かも
- → `closeOnInteractOutside={false}` に変更を検討

### 対象ファイル

- `src/routes/(child)/kinder/home/+page.svelte` — ダイアログの open/close ロジック
- `src/lib/ui/primitives/Dialog.svelte` — closeOnInteractOutside の設定

### 受け入れ条件

- [x] 確認ダイアログの方針を決定（A/B/C/D）— A案（ダイアログ維持）+ 外タップ無効化を採用
- [x] モーダル外タップの動作を決定 — `closable={false}` でボタンのみで操作
- [x] 選択した方針に基づいて実装 — 「きろる？」→「きろく！」「やめる」フロー
- [x] E2E テストを通過

### 成果・結果

- `Dialog.svelte` に `closable={false}` を設定し、子供がダイアログ外タップで閉じられないように対応
- 確認ダイアログは A案（維持）を採用し、明示的なボタン操作のみで記録/キャンセル
