# おみくじ演出デザインリニューアル

## Status: Waiting

### GitHub Copilot 対応: 可能

> ログインボーナスの「おみくじ」表示が、現状では単純なグラデーション背景のカードにテキストを載せただけのデザインで、おみくじらしさ・ワクワク感が不足している。和風のおみくじ体験を演出するデザインにリニューアルする。

### 概要

現在の `OmikujiOverlay.svelte` は Tailwind のグラデーション背景 + 大きなテキストだけの表示で、子供が「おみくじを引いた！」という体感を得にくい。実際のおみくじ（縦長の紙・筒から引く動き）や神社の雰囲気を取り入れたビジュアルに刷新し、演出の満足度を上げる。

### 背景・動機

- 現状のデザインは四角いカードにランク名とポイントが表示されるだけで、「おみくじ」というモチーフが視覚的に伝わらない
- 子供向けアプリとして、毎日のログイン時にワクワク感を演出することがリテンション向上に直結する
- 音声演出（`omikuji-roll` / `omikuji-result`）は実装済みだが、ビジュアル演出が追いついていない

### 現状の課題

1. **おみくじ感のなさ**: 単なるグラデーション四角カードで、おみくじの紙や筒のモチーフがない
2. **抽選演出が地味**: 赤い四角が Y 軸回転するだけで、おみくじを引く体験になっていない
3. **ランク間の差別化が弱い**: 背景色とパーティクル絵文字だけの違いで、大大吉と末吉の「特別感の差」が小さい
4. **和風テイストの欠如**: タイトル「🎋 きょうのうんせい 🎋」以外に和風要素がない

### ゴール

- [ ] **おみくじモチーフのビジュアル**: 縦長の札・紙のデザインに変更（角丸＋枠線＋和紙風テクスチャなど）
- [ ] **抽選演出の改善**: おみくじ筒を振る → 札が出てくるようなアニメーション演出
- [ ] **ランク別演出の差別化**: 大大吉は金色の光彩・紙吹雪、末吉は控えめな演出など、ランクごとの特別感に明確な差をつける
- [ ] **和風デザイン要素の追加**: 背景パターン（麻の葉、青海波など）、和風フォント風の装飾、色使いの和風化
- [ ] **レスポンシブ対応**: 現行同様、各端末サイズで適切に表示されること
- [ ] **アクセシビリティ維持**: アニメーション苦手なユーザー向け `prefers-reduced-motion` 対応
- [ ] **既存テストの維持**: E2E テスト（ログインボーナス関連）が引き続きパスすること

### デザイン方針案

#### 抽選フェーズ（revealed = false）
- おみくじの筒（円筒形）から札が飛び出すアニメーション
- または筒を振る → 一本の棒（おみくじ棒）が抜け出る演出
- 背景にうっすら神社の鳥居シルエットや桜の花びらなど

#### 結果表示フェーズ（revealed = true）
- 縦長の「おみくじ札」デザイン（和紙風テクスチャ、墨文字風のランク表示）
- ランクに応じた装飾レベル:
  - **大大吉**: 金色の光彩、紙吹雪パーティクル、背景全体が華やか
  - **大吉**: 紫＋金の装飾、光のエフェクト
  - **中吉〜小吉**: 落ち着いた色味の装飾
  - **吉〜末吉**: シンプルで上品な表示
- ポイント表示は札の下部に自然に配置

### 技術メモ

- CSS のみで実現できる範囲を優先（Canvas/WebGL は不要）
- 和紙風テクスチャは CSS グラデーション + ノイズで表現可能
- パーティクルは CSS animation + 疑似要素で軽量実装
- `prefers-reduced-motion` 時はアニメーションをスキップし即時表示
