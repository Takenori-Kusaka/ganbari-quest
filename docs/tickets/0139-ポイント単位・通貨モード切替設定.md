### 概要

親の教育方針に応じて「ポイント（P）」という抽象的な単位を廃し、円・ドルなどの実通貨単位で直接表示できるようにする。活動を記録した瞬間に「○○円もらった」という体験を仮想化し、お金の価値・稼ぎ・貯蓄の概念を自然に学べる環境を提供する。

### 背景・動機

現在のアプリは「ポイント＝仮想通貨、後で円に変換」という2段階の仕組みだが、次のようなニーズが存在する:

- **金銭教育重視派**: ポイントという抽象概念を挟まず、「お手伝い1回=10円」という直感的な報酬体験を子どもに与えたい
- **お小遣い管理派**: 実際のお小遣い帳と連動させたいため、円建てでアプリを使いたい
- **グローバル世帯**: 外貨での教育体験（ドル・ユーロ等）をしたいケースも存在する

一方で「ポイントのほうがゲーム感があっていい」というユーザーもいるため、**設定で切り替え可能**にする。

### 設計方針

#### 設定項目（2つ）

| キー（settings テーブル） | 型 | デフォルト | 説明 |
|--------------------------|-----|----------|------|
| `point_unit_mode` | `'point' \| 'currency'` | `'point'` | ポイントモード or 通貨モードの切替 |
| `point_currency` | `'JPY' \| 'USD' \| 'EUR' \| 'GBP' \| 'AUD' \| 'CAD'` | `'JPY'` | 通貨モード時の通貨種別 |
| `point_rate` | 数値文字列（例: `"1"`） | `"1"` | 1P = N 通貨単位（例: `"0.1"` なら 1P = 0.1円）|

#### 通貨定義

```typescript
export const CURRENCY_DEFS = {
  JPY: { symbol: '円', prefix: false, decimals: 0, flag: '🇯🇵' },
  USD: { symbol: '$',  prefix: true,  decimals: 2, flag: '🇺🇸' },
  EUR: { symbol: '€',  prefix: true,  decimals: 2, flag: '🇪🇺' },
  GBP: { symbol: '£',  prefix: true,  decimals: 2, flag: '🇬🇧' },
  AUD: { symbol: 'A$', prefix: true,  decimals: 2, flag: '🇦🇺' },
  CAD: { symbol: 'C$', prefix: true,  decimals: 2, flag: '🇨🇦' },
} as const;
```

#### 表示変換ルール

```
通貨モード + JPY + rate=1 の場合:
  100P  → ¥100 または 100円
  「活動完了！+5P」→「活動完了！+5円」

通貨モード + USD + rate=0.01 の場合:
  100P  → $1.00
  「活動完了！+5P」→「活動完了！+$0.05」

ポイントモード（デフォルト）:
  変更なし。1P = 1P 表示のまま
```

#### レート換算関数

```typescript
// src/lib/domain/point-display.ts
export function formatPointValue(
  points: number,
  mode: PointUnitMode,
  currency: CurrencyCode,
  rate: number,
): string {
  if (mode === 'point') return `${points.toLocaleString()}P`;
  const value = points * rate;
  const def = CURRENCY_DEFS[currency];
  const formatted = value.toFixed(def.decimals);
  return def.prefix
    ? `${def.symbol}${Number(formatted).toLocaleString()}`
    : `${Number(formatted).toLocaleString()}${def.symbol}`;
}
```

#### 変換ページ（ポイント→お小遣い）の扱い

- **ポイントモード**: 現状通り「○○P → ○○円」変換
- **通貨モード（JPY）**: 既に円建てのため「○○円を渡しました」として記録。変換ページを「支払い記録」として再解釈
  - 「ポイントを変換」→「金額を渡す（記録）」にページタイトル変更のみ
  - 内部ロジック（`point_ledger` 書き込み）は変更なし

#### 設定値の読み込み方

設定は既存の `settings` KVS テーブル（`getSetting` / `setSetting`）に追加する。
サーバーサイドでロードし `+layout.server.ts` で全ページに渡す（親管理画面レイアウト）。

```typescript
// (parent)/+layout.server.ts（既存に追加）
const pointUnitMode = await getSetting('point_unit_mode') ?? 'point';
const pointCurrency = await getSetting('point_currency') ?? 'JPY';
const pointRate = parseFloat(await getSetting('point_rate') ?? '1');
return { ..., pointSettings: { mode: pointUnitMode, currency: pointCurrency, rate: pointRate } };
```

子供画面側も `(child)/+layout.server.ts` で同様にロードし、表示に使用する。

### 実装の境界と注意事項

- **DBスキーマ変更なし**: `point_ledger` / `activities` 等のポイント数値カラムは変更しない。あくまで「表示ラベル」の変換のみ
- **内部演算はすべてP単位で維持**: `basePoints`・`balance` はポイント整数のまま。換算は表示直前に行う
- **`1P = 1P` の論理は壊さない**: 変換レートが変わっても、獲得・消費の内部ロジックは変更不要
- **ごほうびショップの価格は親が設定したポイント数**: 通貨モードでも「何円（の活動量）が必要か」の換算表示に留める
- **将来拡張**: テナント対応後は `tenantId` 毎に設定を持てるよう、`setSetting(key, value, tenantId)` の引数は現在も存在するため対応可能
