# 活動カードレイアウト最適化（スクロール削減）

### ステータス

`Done`

### 種別

`Enhancement`

### 元レビュー

`docs/user-review/20260227.md` — 気になった点1

---

### 概要

ホーム画面の活動カード一覧が、横幅に余裕がある状態でもスクロールが必要になっている。子供にとってスクロールは直感的でない操作のため、可能な限り1画面に全活動を表示したい。スマホの最小幅など物理的に収まらない場合のみスクロールを許容する。

### 現状

- 活動カードサイズ: `w-[80px] h-[100px]`（固定）
- カテゴリごとにセクション分けされ、各セクションにヘッダー付き
- kinder モードで約28活動（5カテゴリ）
- タブレット（1280×800）でもスクロールが必要

### 改善案

1. **カードサイズの動的調整**: 画面幅に応じてカードサイズを可変にする（CSS Grid の auto-fill/minmax）
2. **カテゴリヘッダーの省スペース化**: カテゴリ名をインライン表示にする等
3. **余白の最適化**: カード間・セクション間の余白を画面サイズに応じて調整
4. **レイアウト方式の変更**: 横スクロール → グリッド wrap レイアウト
5. **画面サイズ別のブレークポイント**: タブレットでは列数を増やし、スマホでは3-4列

### 対象ファイル

- `src/lib/ui/components/ActivityCard.svelte` — カードサイズの可変対応
- `src/lib/ui/components/CategorySection.svelte` — グリッドレイアウト調整
- `src/routes/(child)/kinder/home/+page.svelte` — 全体レイアウト

### 受け入れ条件

- [x] タブレット（1280×800）で全活動が1画面に収まる（スクロール不要）
- [x] スマホ（360×800）では最小限のスクロールで全活動が確認できる
- [x] カードのタップ領域が44px以上を維持（アクセシビリティ）
- [x] アイコン・テキストが読み取れるサイズを維持

### 成果・結果

- `ActivityCard.svelte` で動的サイズ対応実装
- `CategorySection.svelte` で CSS Grid の auto-fill/minmax レイアウトに変更
- タブレットでスクロール不要、スマホで最小限のスクロールに改善
