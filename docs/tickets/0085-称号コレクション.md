# #0085 称号コレクション

## ステータス: Done
## 優先度: 低
## 種別: 機能追加（ゲーミフィケーション強化）

---

## 概要

特定の条件を達成すると獲得できる「称号」を追加し、子供が好きな称号をプロフィールに設定できる機能を実装する。

## 背景

現在のレベル称号（はじめのぼうけんしゃ、がんばりルーキー等）はレベルに紐づいて自動変更される。
それとは別に、プレイスタイルや実績に応じた「選べる称号」を追加することで：
- コレクション欲を刺激
- 自分だけのキャラクター感を演出
- 特定の活動パターンを促進

## 要件

### 1. 称号の種類（例）

**カテゴリ特化称号（偏差値65以上で解放）:**
- うんどうマスター（うんどう偏差値65+）
- べんきょうの天才（べんきょう偏差値65+）
- せいかつの達人（せいかつ偏差値65+）
- こうりゅうの王（こうりゅう偏差値65+）
- そうぞうの魔法使い（そうぞう偏差値65+）

**行動パターン称号:**
- れんぞくの鬼（ストリーク30日達成）
- コンボマスター（パーフェクトコンボ10回）
- ミッションハンター（デイリーミッション30日コンプリート）
- 朝活ヒーロー（7時前の記録が50回）

**レアリティ別:**
- common: 基本称号
- rare: やや難しい条件
- epic: 高難度
- legendary: 極めて困難

### 2. 称号選択UI

- つよさ画面またはプロフィール画面から設定
- 獲得済み称号の一覧表示（カード形式）
- 未獲得称号はシルエット表示 + 解放条件ヒント
- 選択した称号がホーム画面の名前横に表示

### 3. DBスキーマ

```sql
CREATE TABLE titles (
  id INTEGER PRIMARY KEY,
  code TEXT NOT NULL UNIQUE,
  name TEXT NOT NULL,
  description TEXT,
  conditionType TEXT NOT NULL,
  conditionValue TEXT,
  rarity TEXT DEFAULT 'common',
  icon TEXT
);

CREATE TABLE childTitles (
  id INTEGER PRIMARY KEY,
  childId INTEGER NOT NULL,
  titleId INTEGER NOT NULL,
  unlockedAt TEXT NOT NULL,
  UNIQUE(childId, titleId)
);
```

- `children` テーブルに `activeTitleId` カラム追加

## 技術設計

- 実績システム（`achievements`）の拡張として実装可能
- 称号判定ロジックは実績判定と共通化
