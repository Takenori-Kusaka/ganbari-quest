# トップページ訴求力リニューアル（マーケティング改善）

### ステータス
`Done`

### GitHub Copilot 対応: 可能

> 現在の `site/index.html` トップページは「できること」の機能リストが中心で、OSS プロジェクトの README ページに近い構成になっている。訪問者が「これは私の課題を解決してくれる！」と瞬時に感じられるよう、ペルソナの痛みから始まる感情共鳴型のコピー・構成に全面リニューアルする。

### 背景・動機

#### 現状の課題

現トップページ (`site/index.html`) の問題点:

| 問題 | 詳細 |
|------|------|
| **機能説明先行** | ヒーローの次がいきなり「できること」機能リスト。訪問者の「共感」が得られる前に説明が始まる |
| **痛みへの共鳴がない** | 「夕方の宿題バトル」「シール帳3週間で挫折」など、親が日常で感じているシーンが一切登場しない |
| **抽象的なベネフィット** | 「成長の見える化」「習慣づくり」は正しいが、具体的なシーンがないため訴求力が弱い |
| **社会的証明の欠如** | 導入事例・体験談・数値的な実績がない（現時点では仕方ない面もあるが） |
| **差別化が薄い** | 競合アプリと並べたときに「なぜがんばりクエストなのか」が伝わらない |

#### 目指すユーザー体験

訪問者がファーストビューを見た瞬間に「あ、これ私のことだ」と感じ、
スクロールするごとに「まさにこれが欲しかった」「子供に試したい」という気持ちが高まる構成。

### ペルソナと共鳴シーン（設計素材）

#### メインターゲット: 田中ゆかり（35歳・共働き・子供2人）

- **朝シーン**: 何度呼んでも着替えない。「はやくして！」が1日の始まり
- **夕方シーン**: 「宿題やった？」→「まだ〜」→親子バトル。毎日同じことの繰り返し
- **週末シーン**: お手伝いを頼んでもなかなか動かない。ゲームなら自分からやるのに
- **継続失敗**: 100均のシール帳を買ってみたが3週間で飽きた。スタンプカードも同じ
- **お小遣い悩み**: お小遣いをあげたいが、「がんばった分だけ」を明確にする手段がない

#### 刺さる「問いかけ」方向性

以下はあくまで方向性・参考であり、**そのままコピーとして使うのはNG**。
実際のコピーライティングは感情に寄り添いつつ、押しつけがましくない表現にすること。

- 「声をかけなくても自分から動いてくれたら…」という望みを訴求
- 「ゲームには集中できるのに」という親の気持ちに共鳴
- 「頑張った分だけ評価してあげたい」というお小遣いの悩み
- 「シール帳・スタンプカードを試したことがある」という経験への共感

### ゴール

#### コンテンツ・コピー

- [x] **ファーストビュー刷新**: 機能説明ではなく「親の日常の悩み」から始まるヒーローコピーに変更
- [x] **Before/After セクション追加**: がんばりクエスト導入前後の夕方の様子を対比で描く
- [x] **具体的シーン演出**: 「こんなシーンに心当たりありませんか？」ブロック（3〜4シーン）を Features の前に挿入
- [x] **感情的ベネフィットの明文化**: 「成長の見える化」ではなく「お子さまの成長に、自信を持って『すごいね』と言える」のように感情まで言語化
- [x] **競合差別化メッセージ**: 「シール帳・スタンプカードとどう違うか」を端的に示すセクション
- [x] **社会的証明準備**: ユーザーレビュー・導入実績枠を設置（初期はダミー or リクルーティング誘導）
- [x] **FAQ をペイン起点に**: 「料金は？」よりも「本当に効果がありますか？」「子供が飽きないか心配」などの不安に先に答える構成

#### 構成変更

- [x] セクション順序を「機能→使い方」から「共感→解決策→機能→証拠→CTA」に変更
- [x] ヘッダーナビも「できること」から感情的なラベルへ検討（「悩みを解決」など）
- [x] CTA ボタンのコピーを見直し（「無料で試してみる」「うちの子に合うか試す」など）

### 推奨ページ構成（変更後）

```
hero            ← ペルソナの悩みを射抜く一言 + 感情的なサブコピー
                  「いつも声かけしてる、あなたへ。」みたいなニュアンスの方向
pain-scenes     ← 「こんな経験ありませんか？」3〜4の具体的親・子シーン
solution         ← がんばりクエストが解決する仕組みの概要（機能ではなく体験として）
before-after    ← 導入前後の典型的な1日を対比で描写
features        ← （現行）機能カード（感情的ベネフィットも添える）
age-modes       ← （現行）年齢対応（ほぼそのままで可）
social-proof    ← ユーザーの声 / 数値実績（初期はリクルート枠）
faq             ← 不安起点の Q&A に変更
cta-bottom      ← 最終 CTA + ハードル低めのコピー
setup           ← SaaS/セルフホスト（現行・変更は軽微）
footer          ← （現行）
```

### コピーライティング原則

1. **痛みから始める**: 解決策より先に「あなたの課題を理解している」を示す
2. **具体的な時間・場所・感情**: 「夕方6時、またお片付けの声かけで疲れた…」のように5W1H で描写
3. **ベネフィットの深堀り**: 機能(feature) → 利点(advantage) → 恩恵(benefit) → 感情(feeling) の順で掘り下げる
4. **One Thing Rule**: 1セクション = 1メッセージ。詰め込まない
5. **You ファースト**: 「子供の成長を見える化します」より「お子さまの成長が、親のあなたに届く」
