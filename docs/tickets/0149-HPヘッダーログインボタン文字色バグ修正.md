# HP ヘッダーのログインボタン文字色バグ修正

### ステータス
`Done`

### GitHub Copilot 対応: 可能

> `site/index.html` のヘッダーナビゲーションで、ログインボタン（青背景）のテキスト色が灰色になっており、ボタン上の文字が視認できない。

### 概要

ヘッダーの「ログイン」ボタンは `.btn-primary` クラスで青背景（グラデーション）を持つが、テキスト色が白 (`#fff`) ではなく灰色 (`var(--gray-500)`) になっているため、ボタン上の文字がほぼ見えない状態になっている。

### 原因

CSS セレクタの詳細度（Specificity）の競合。

```css
/* 詳細度: 0,1,1 — クラス + 要素セレクタ */
.header-nav a {
  color: var(--gray-500);  /* ← こちらが勝ってしまう */
}

/* 詳細度: 0,1,0 — クラスセレクタのみ */
.btn-primary {
  color: #fff;  /* ← 上書きされてしまっている */
}
```

`.header-nav a` の詳細度（0,1,1）が `.btn-primary`（0,1,0）より高いため、ログインボタンのリンクにも灰色が適用されてしまっている。

### 修正方針

`.header-nav .btn-primary` のようにより詳細度を上げた形で `color: #fff` を上書きするか、`.header-nav a` の対象からボタンクラスを除外する。

#### 修正案 A（推奨）: `.header-nav a` の対象から btn クラスを除外

```css
/* Before */
.header-nav a { color: var(--gray-500); }

/* After */
.header-nav a:not(.btn) { color: var(--gray-500); }
```

**メリット**: 将来 `.btn-outline` も置く場合に対応できる。根本的な解決。

#### 修正案 B: 詳細度の高いオーバーライドを追加

```css
.header-nav .btn-primary { color: #fff; }
```

**メリット**: 変更量が最小。**デメリット**: 場当たり的。
