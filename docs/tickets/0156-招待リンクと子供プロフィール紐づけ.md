# 招待リンクと子供プロフィールの紐づけ機能

### ステータス
`Ready`

### GitHub Copilot 対応: 可能

> #0129 で招待リンクの発行・承認フローは実装済みだが、招待されたユーザーと親管理画面で管理している子供プロフィールの紐づけが未実装。招待リンク経由でログインした子供が、どの子供プロフィールのデータを閲覧・操作できるかを決定する仕組みが必要。

### 背景・動機

現在の問題点:

1. **childId が未使用**: `invite` エンティティに `childId` フィールドは存在するが、`acceptInvite()` で無視されている
2. **招待作成 UI に子供選択がない**: 管理画面から招待リンクを発行する際、どの子供用の招待かを選択する UI がない
3. **Child エンティティに userId がない**: 子供プロフィールとユーザーアカウントの関連付けフィールドが存在しない
4. **両方向のユースケース未対応**:
   - ケース A: 先に子供を管理画面で追加 → その子供用の招待リンクを発行
   - ケース B: 先に招待リンクを発行 → 承認後に子供プロフィールと紐づけ

### ゴール

#### データモデル

- [ ] `Child` エンティティに `userId?: string` フィールドを追加（招待承認済みユーザーとの紐づけ）
- [ ] DB スキーマ更新（SQLite: マイグレーション、DynamoDB: 属性追加）

#### 招待作成フロー

- [ ] 招待リンク作成 UI に「対象の子供」セレクトボックスを追加（任意選択）
- [ ] 子供を選択した場合、`createInvite()` に `childId` を渡す
- [ ] 子供未選択の場合は汎用招待（承認後に紐づけ）

#### 招待承認フロー

- [ ] `acceptInvite()` で `childId` が指定されている場合、Child の `userId` に承認ユーザーの ID をセット
- [ ] `childId` が未指定の場合、承認後に子供選択画面を表示（またはオーナーが後から紐づけ）

#### アクセス制御

- [ ] 子供ロールでログインした場合、自分の `userId` に紐づいた `Child` プロフィールのデータのみ閲覧可能
- [ ] `hooks.server.ts` または認証ミドルウェアで `context.childId` を `userId` から解決

#### ケース A: 子供追加済み → 招待

1. 親が `/admin/children` で子供プロフィールを作成
2. 親が `/admin/members` で招待リンクを発行（対象の子供を選択）
3. 子供（または共有端末）が招待リンクを開いてアカウント作成
4. 承認時に自動で Child.userId がセットされる

#### ケース B: 招待先行 → 子供紐づけ

1. 親が `/admin/members` で汎用招待リンクを発行
2. ユーザーが招待リンクを開いてアカウント作成・承認
3. 親が `/admin/members` で承認済みユーザーと子供プロフィールを紐づけ
