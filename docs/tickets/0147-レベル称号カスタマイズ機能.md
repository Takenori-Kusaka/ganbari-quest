# レベル称号カスタマイズ機能

## Status: Waiting

### GitHub Copilot 対応: 可能

> ステータス管理画面に表示されるレベル称号（「はじめのぼうけんしゃ」「がんばりルーキー」等）は LEVEL_TABLE にハードコードされたデフォルト値しか使えない。親が子供に合った称号名にカスタマイズできるようにする。

### 概要

現在、Lv.1〜Lv.10 の称号は `src/lib/domain/validation/status.ts` の `LEVEL_TABLE` 定数で固定されており、すべてのテナント・子供で同一の称号が表示される。家庭ごとに子供の好みや親の方針に合わせた称号名を設定できるようにすることで、ゲーミフィケーションのモチベーション効果を高める。

### 背景・動機

- 称号は子供のモチベーション維持に直結する要素だが、現在は全ユーザー共通のデフォルト値のみ
- 「うちの子は恐竜が好きだから恐竜っぽい称号にしたい」「英語混じりの称号にしたい」などの個別ニーズに応えられない
- 既存の実績称号（titles テーブル）はコレクション用の別システムであり、レベル称号のカスタマイズとは異なる

### 設計方針

#### DB スキーマ

```sql
CREATE TABLE level_titles (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  tenant_id TEXT NOT NULL,
  level INTEGER NOT NULL,         -- 1〜10
  custom_title TEXT NOT NULL,     -- カスタム称号名
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(tenant_id, level)
);
```

- テナント単位で管理（子供個別ではなく家庭単位）
- レコードが存在しないレベルは LEVEL_TABLE のデフォルトを使用

#### 称号解決フロー

```
calcLevel(avgStatus)
  → { level: 4, title: 'つよつよチャレンジャー' }  // デフォルト

resolveTitle(level, tenantId)
  → DB に level_titles(tenant_id, level=4) があれば → カスタム称号
  → なければ → LEVEL_TABLE のデフォルト
```

#### 親管理画面 UI

- ステータス管理画面内に「称号カスタマイズ」ボタンまたはセクションを追加
- Lv.1〜10 の一覧を表示し、各行にテキスト入力欄
- 空欄時はデフォルト称号をプレースホルダーとして表示
- 「保存」「デフォルトに戻す」ボタン
- バリデーション: 最大20文字程度、空文字不可（空欄ならデフォルト扱い）

### 技術メモ

- 既存の `titles` テーブル（実績称号コレクション）とは別物。混同しないこと
- テナント単位の設定とすることで、同一家庭の兄弟で称号が統一される
- `getChildStatus()` の戻り値 `levelTitle` に反映する形で統合すれば、UI 側の変更は最小限
- DynamoDB 版も必要な場合は `src/lib/server/db/dynamodb/` に対応リポジトリを追加
