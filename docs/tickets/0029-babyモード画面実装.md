### 概要

0-2歳児向けの baby モード画面を実装する。テキストは使用せずアイコンのみ、超大タップターゲット（120px）、音声フィードバック中心の極めてシンプルなUIを構築する。

### 背景・動機

0歳の息子向けに、将来的に使えるbabyモードを用意する。babyモードは実質的に「親が子供の活動を代理記録する」ためのUIであり、子供自身が操作するというよりは親が片手で素早く記録できることが重要。ただし将来的に1-2歳になった際に子供自身がタップして楽しめる要素も入れる。

### ゴール

- [ ] `src/routes/(child)/baby/` 配下に全画面を実装
  - [ ] `baby/home/+page.svelte` — 超大アイコングリッド（120px）
  - [ ] `baby/history/+page.svelte` — 簡易履歴（アイコン + 日付のみ）
  - [ ] `baby/+layout.svelte` — baby専用レイアウト
- [ ] baby モード専用デザイン
  - タップターゲット: 120px 以上
  - テキスト: なし（アイコンのみ）
  - 背景: やさしいパステルカラー
  - アニメーション: 大きくバウンス、ゆっくり
  - 角丸: 24px 以上（丸に近い）
- [ ] baby 専用 CSS テーマ変数
  - `--baby-tap-size: 120px`
  - `--baby-radius: 24px`
  - `--baby-animation-duration: 0.6s`
- [ ] 音声フィードバック
  - 全タップに操作音
  - 記録完了時に褒め音声（ファンファーレ）
  - ページ表示時にウェルカム音
- [ ] 活動記録: 1タップで即記録（確認ダイアログなし）
  - 記録完了後は大きなアニメーション + 効果音
  - 誤タップ取消は画面端に小さく配置（親用）
- [ ] BottomNav: アイコンのみ（テキストなし）、3項目（ホーム、きろく、きりかえ）
- [ ] Header: ニックネーム + ポイント表示のみ（シンプル）
- [ ] `$lib/features/baby/` に baby 固有コンポーネントを配置
- [ ] 既存テスト全パス維持

### 技術方針

- kinderモードとは完全に別のレイアウト・コンポーネント
- 共通の Ark UI プリミティブ + +page.server.ts のロードロジックは再利用
- CSS カスタムプロパティで baby テーマを定義
- スクロール最小化（1画面に収まるグリッド）
- ステータス画面は baby では不要（親用管理画面で確認）
