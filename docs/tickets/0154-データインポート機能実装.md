# データインポート機能実装（#0114 Phase 2）

### ステータス
`Ready`

### GitHub Copilot 対応: 可能

> #0114 でデータエクスポート（Phase 1）は実装済みだが、インポート機能（Phase 2）は未実装のまま残っている。本チケットでインポート機能を完成させる。

### 背景・動機

#0114「データインポート・エクスポート機能」は Phase 1（エクスポート）のみ完了し、Phase 2（インポート）と Phase 3（差分マージ）は未着手。エクスポートした JSON データを別環境（例: ローカル→AWS、端末移行）に取り込む手段がないため、データポータビリティが片方向にしか機能していない。

### 現状

| 機能 | 状態 |
|------|------|
| JSON エクスポート（全データ一括） | ✅ 実装済み（`/api/v1/export`） |
| JSON インポート（全データ復元） | ❌ 未実装 |
| 差分マージ（既存データとの統合） | ❌ 未実装 |
| インポート UI | ❌ 未実装 |

### ゴール

#### Phase 2: インポート（全データ復元）

- [ ] `POST /api/v1/import` エンドポイント実装
  - エクスポート JSON を受け取り、全エンティティを復元
  - バリデーション: JSON スキーマの検証、必須フィールドの存在確認
  - トランザクション: 全て成功 or 全てロールバック
- [ ] インポート前の確認画面（データ件数の表示、既存データへの影響警告）
- [ ] 管理画面（`/admin/settings`）にインポート UI を追加
  - ファイル選択 → プレビュー → 確認 → 実行
- [ ] インポート時の ID 再採番（エクスポート元と異なる環境でも衝突しない）
- [ ] エラーハンドリング（不正 JSON、バージョン不一致、データ整合性エラー）

#### Phase 3: 差分マージ（将来検討）

- [ ] 既存データとの差分検出
- [ ] マージ戦略の選択（上書き / スキップ / 手動選択）
