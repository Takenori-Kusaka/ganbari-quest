### 概要

親管理画面の「こども管理」(`/admin/children`) と「ステータス管理」(`/admin/status`) が、どちらも「子供を選択して情報を見る・編集する」という同じメンタルモデルの画面でありながら、操作体系がバラバラで混乱を招いている。これらを統一的な UX に再設計する。

### 背景・動機

#### 現状の問題

現在、子供に関する情報の閲覧・編集が**2つの画面に分散**しており、親にとって直感的でない:

**画面 A: こども管理 (`/admin/children`)**
- カード一覧の「編集」ボタン → インライン展開でニックネーム / 年齢 / テーマカラーを編集
- カードクリック → `?id={childId}` で**下部に詳細セクションが展開**（ステータスバー・活動記録・実績・誕生日ふりかえり）
- 詳細セクションのステータスバーは**閲覧のみ**（編集不可）
- アバターのアップロード / AI 生成も詳細セクションで実行

**画面 B: ステータス管理 (`/admin/status`)**
- タブ形式で子供を選択
- カテゴリ別の**スライダーでステータス値を編集**
- ベンチマーク（年齢別平均・標準偏差）の編集

#### UX 上の混乱

| 問題 | 詳細 |
|------|------|
| **同じ操作パターンが異なる結果を生む** | どちらの画面も「子供を選んで詳細を見る」操作だが、片方は閲覧、もう片方は編集 |
| **「編集」の意味が不統一** | こども管理の「編集」ボタンは基本情報のみ。ステータス編集は別画面に行く必要がある |
| **ステータスの二重表示** | こども管理の詳細にも `/admin/status` にもステータスが表示されるが、片方は閲覧専用、もう片方は編集可能。親はどちらで修正すればいいか迷う |
| **情報の分断** | 子供の全体像（基本情報 + ステータス + 活動履歴 + 実績）を一画面で把握しながら必要な項目を編集する、という自然なフローが実現できない |
| **ナビゲーションの認知負荷** | サイドバーの「こども」と「ステータス」が別メニューで、親は使い分けを覚える必要がある |

### ゴール

- [ ] 子供の基本情報 / ステータス / 活動履歴を **1 つの統一されたプロファイル画面** で閲覧・編集できる
- [ ] 「閲覧モード」と「編集モード」の UI 上の区別を明確にする
- [ ] ナビゲーションの項目数を削減し、認知負荷を下げる
- [ ] 既存の機能（インライン編集・スライダー・ベンチマーク）は維持する
- [ ] 既存の E2E テスト・ユニットテストを更新する

---

### 提案: 統一プロファイルページへの再設計

#### 案 A: タブ式プロファイル（推奨）

`/admin/children` を子供一覧 + 子供プロファイル画面に統合。子供選択後にタブで情報を切り替える。

```
/admin/children              → 子供一覧（カードグリッド）
/admin/children?id={childId} → 子供プロファイル（タブ切替）
```

**タブ構成:**

| タブ | 内容 | 編集操作 |
|------|------|---------|
| 📋 基本情報 | ニックネーム・年齢・テーマカラー・アバター | インライン編集（現行踏襲） |
| 📊 ステータス | カテゴリ別ステータス + スライダー編集 | スライダー（現行の `/admin/status` から移植） |
| 📝 活動記録 | 最近の活動ログ一覧 | 閲覧のみ |
| 🏆 実績 | 解放済み / 未解放の実績一覧 | 閲覧のみ |
| 🎂 ふりかえり | 誕生日ふりかえり履歴 | 閲覧のみ |

**メリット:**
- 子供の全体像を 1 画面で把握できる
- 編集可能なタブと閲覧専用タブが明確に分離
- `/admin/status` ページを廃止でき、ナビゲーションがシンプルに

**デメリット:**
- ベンチマーク管理（年齢別・全子供共通）の配置先を別途検討する必要がある

#### 案 B: ステータス画面への統合

現在の `/admin/status` にプロファイル情報を統合する案。ステータス編集を軸にし、基本情報の編集もそこに含める。

**メリット:** 変更量が少ない
**デメリット:** 「ステータス管理」という名前が全体を表さなくなる

#### 案 C: 段階的改善（最小変更）

画面構成は変えず、以下の UX 改善のみ行う:

1. こども管理の詳細セクションにあるステータスバーに「📊 ステータスを編集する →」リンクを追加
2. ステータス管理ページに「基本情報を見る →」リンクを追加
3. 閲覧専用のステータスバーと編集可能スライダーの視覚的な違いを明確にする

**メリット:** 変更量が最小
**デメリット:** 根本的な分断は解消しない

### ベンチマーク管理の扱い

ベンチマークは特定の子供ではなく**年齢別の基準値**を管理する機能であり、子供プロファイルとは性質が異なる。

| 配置案 | 詳細 |
|--------|------|
| `/admin/settings` に移動 | 全体設定としての位置づけ。子供管理から分離 |
| `/admin/children` 内に「ベンチマーク設定」タブ追加 | 子供一覧の下部に全体設定として配置 |
| 現状維持（独立ページ） | `/admin/benchmarks` として独立させる |
