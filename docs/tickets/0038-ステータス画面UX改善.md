# ステータス画面UX改善

### ステータス

`Done`

---

### 概要

親管理画面のステータス変更画面において、スライダー操作時に現在の数値が視認しづらい問題を改善する。

### 背景・動機

現在のステータス変更画面ではrangeスライダーの右横に数値が表示されているが、スライダー操作中に変更後の値がリアルタイムで反映されない（フォーム送信後にページリロードで反映）。ユーザーはスライダーを動かしながら現在値を確認したいが、操作中の値が見えないため直感的でない。

**要件7**: スライダーの下に現在の数値も表示する

### ゴール

- [ ] rangeスライダーに `oninput` バインドを追加し、操作中の値をリアルタイム表示する
- [ ] スライダー下部に現在値を大きめのフォントで表示する（例: `42.5 / 100`）
- [ ] 変更前の値と変更後の値の差分が視覚的にわかるようにする（色変化等）
- [ ] モバイルでのタッチ操作でも数値が見やすいサイズを確保する

### 対象ファイル

- `src/routes/(parent)/admin/status/+page.svelte`

### 作業メモ

- 現状: `<input type="range">` + 右横の `{stat.value}` 表示
- Svelte 5 の `$state` でローカル変更値を追跡し、`oninput` で更新するパターンを採用
