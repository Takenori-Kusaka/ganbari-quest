### 概要

子供の年齢帯（baby/kinder/lower/upper/teen）に応じたUI表示を実現するため、`children` テーブルに `ui_mode` カラムを追加し、年齢帯別のルーティング構造を構築する。

### 背景・動機

4歳児と0歳児では必要なUI/UXが根本的に異なる。タップターゲットサイズ、テキスト表現、アニメーション量、情報密度すべてが年齢帯によって変わるため、共通ページの条件分岐ではなく年齢帯別のルート分離が必要。

### 技術方針

- URL構造: `/kinder/home`, `/baby/home` 等（年齢帯がURLに含まれる）
- `(child)/+layout.server.ts` で子供選択 → ui_mode判定 → 適切なサブルートへリダイレクト
- 共通レイアウト（Header, BottomNav）は `(child)/+layout.svelte` に残す
- `data-age-tier` 属性で年齢帯固有のCSS変数を設定
- BottomNav の `items` prop に年齢帯プレフィックス付きパスを渡す
- Ark UI プリミティブは全年齢帯で共有、スタイルのみ年齢帯別に変更

### 作業メモ

- 個別の年齢帯CSSファイル（`age-themes/baby.css` 等）ではなく、`app.css` 内の `[data-age-tier]` セレクタで統一管理する方針に変更
- ファイル移動は `git mv` で実施（履歴保持）
- `parent()` パターンは親レイアウト `(child)/+layout.server.ts` に到達するため、移動後もコード変更不要
- `$types` は SvelteKit が自動生成するため、ファイル移動の影響なし
- ルート `+page.server.ts` は Cookie から子供取得して uiMode 判定するため `load` シグネチャに `cookies` 追加
