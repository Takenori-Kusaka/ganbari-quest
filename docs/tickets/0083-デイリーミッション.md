# #0083 デイリーミッション

## Status: Done
## 優先度: 中
## 種別: 機能追加（ゲーミフィケーション強化）

---

## 概要

毎日ランダムに3つの「きょうのミッション」を提示し、全達成で大ボーナスを付与するデイリーミッション機能を追加する。

## 背景

現在、子供は活動一覧から自由に選んで記録するが、「今日何をやるか」のガイダンスがない。
デイリーミッションにより：
- 「何をやればいいかわからない」状態を解消
- 普段やらないカテゴリの活動にチャレンジする動機づけ
- 全ミッション達成の達成感

## 要件

### 1. ミッション生成ロジック

- 毎日0時（JST）にリセット
- 子供1人あたり3つのミッションを自動生成
- 生成ルール:
  - 3つは異なるカテゴリから選出（最大3カテゴリ）
  - 年齢・学年レベルに適した活動のみ
  - 前日と同じ組み合わせにならないようシャッフル
  - 非表示（isVisible=false）の活動は除外

### 2. ミッション表示UI

- ホーム画面（baby/kinder）の上部に「きょうのミッション」カード
- 各ミッション: アイコン + 活動名 + チェックマーク（達成済み/未）
- 全達成時: 特別な演出 + 「ミッションコンプリート！」メッセージ

### 3. 報酬

| 達成数 | ボーナス |
|--------|---------|
| 1/3達成 | なし（通常の活動ポイントのみ） |
| 2/3達成 | +5P |
| 3/3達成（コンプリート） | +20P |

- `pointLedger` に `type: 'daily_mission'` で記録

### 4. DBスキーマ

```sql
CREATE TABLE dailyMissions (
  id INTEGER PRIMARY KEY,
  childId INTEGER NOT NULL,
  missionDate TEXT NOT NULL,      -- 'YYYY-MM-DD'
  activityId INTEGER NOT NULL,
  completed INTEGER DEFAULT 0,    -- 0/1
  completedAt TEXT,
  FOREIGN KEY (childId) REFERENCES children(id),
  FOREIGN KEY (activityId) REFERENCES activities(id),
  UNIQUE(childId, missionDate, activityId)
);
```

### 5. 連携

- 活動記録時にミッション該当チェック → 自動で completed=1 に更新
- コンプリートボーナスは3つ目の達成時に自動付与
