## 概要

つよさ画面のレーダーチャートで、各軸のカテゴリ名（「うんどう」「べんきょう」等）・星評価・トレンドアイコンが見切れて読めない不具合を修正する。

## 現象

- レーダーチャートの5軸ラベルが親コンテナまたは SVG 描画領域からはみ出してクリップされ、途中までしか表示されない
- baby / kinder 両モードで再現

## 原因の調査ポイント

### SVG viewBox の余白

#0082 の不具合改修で `viewBox` に `padding = size * 0.2` の余白を追加済みだが、ラベル＋星評価＋トレンドアイコンの3行分には不十分な可能性がある。

確認項目:
- `labelRadius`（`size * 0.47`）でラベル位置を計算しているが、ラベルのテキスト幅・高さを含めた実領域が viewBox に収まっているか
- 日本語テキスト（ひらがな5文字）の実幅が `text-anchor` の配置とあわせて viewBox 外に出ていないか
- 下部ラベルの `dy` オフセット（+16）にさらに星（+14）・トレンド（+28）が加算され、viewBox の下端を超えていないか

### 親コンテナによるクリップ

- SVG 自体の viewBox が正しくても、親の `div` に `overflow: hidden` が効いている場合にクリップされる
- `max-width` 指定が小さすぎる場合に縮小されすぎてラベルが読めない可能性

## 修正方針

1. **padding の拡大**: `size * 0.2` → 必要に応じてさらに拡大（`size * 0.3` 等）
2. **labelRadius の調整**: ラベル位置をチャート外周からさらに外側に（テキスト3行分を考慮）
3. **ラベル配置ロジックの見直し**: 各頂点ごとに `dy` を個別設定し、上頂点は上方向に、下頂点は下方向に十分なオフセット
4. **SVG の `overflow: visible`**: SVG 要素に `overflow="visible"` を追加し、viewBox 外のラベルも描画可能にする（代替案）
5. **親コンテナの確認**: `overflow: hidden` があれば除去
