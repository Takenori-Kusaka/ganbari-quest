### 概要

個人開発の生命線であるコスト管理を自動化する。AWS Budgets による月額予算アラート、Cost Anomaly Detection による異常検知、ECR ライフサイクルポリシーによるイメージ蓄積防止、CloudWatch Logs の保持期間最適化を **追加費用 0円** で実現する。

### 背景・動機

#### コストアラートが存在しない

#0064 でコスト試算（月額192円〜）を記載しているが、**実際に異常課金が発生した場合に気付く仕組みがない**。DDoS やクローラによる Lambda 大量起動、API Gateway の意図しないトラフィック急増で課金が月数万円に跳ね上がるリスクがある。個人開発では致命的。

#### ECR イメージが無制限に蓄積する

#0069（CI/CD）および現行の `deploy.yml` では、デプロイのたびに `github.sha` タグ付きの Docker イメージが ECR に push される。ライフサイクルポリシーが未設定のため、イメージが無制限に蓄積する。

**蓄積コスト試算:**
| 期間 | デプロイ回数 | イメージ総容量 | ECR費用（$0.10/GB/月） |
|------|-----------|-------------|---------------------|
| 3ヶ月 | 60回 | ~12GB | ~$1.2（≈180円） |
| 6ヶ月 | 120回 | ~24GB | ~$2.4（≈360円） |
| 1年 | 240回 | ~48GB | ~$4.8（≈720円） |

※ 1イメージ ≈ 200MB、月20回デプロイと仮定
→ **1年放置するとサーバレス本体（月75円）より ECR ストレージの方が高くなる**

#### CloudWatch Logs の保持期間が未設定

デフォルトの CloudWatch Logs 保持期間は**無期限**。Lambda と API Gateway のログが蓄積し続け、5GB 無料枠を超えるとストレージ費用が発生する。

### 予算制約

| 項目 | 費用 | 無料枠 |
|------|------|--------|
| AWS Budgets | $0 | 2 予算/アカウント |
| Cost Anomaly Detection | $0 | 完全無料 |
| ECR ライフサイクルポリシー | $0 | 無料機能（費用削減効果あり） |
| CloudWatch Logs 保持設定 | $0 | 設定自体は無料 |
| **合計** | **$0/月** | **むしろ費用を削減する** |

### 対応方針

#### 1. AWS Budgets（月額予算アラート）

```
Budget: ganbari-quest-monthly
  ├── 月額予算: $5（≈750円）
  ├── アラート1: 実績 50% 到達時（$2.5）→ メール通知（情報）
  ├── アラート2: 実績 80% 到達時（$4.0）→ メール通知（警告）
  └── アラート3: 予測 100% 超過見込み → メール通知（緊急）
```

**なぜ $5 か:**
- 通常運用: ~$0.50/月（Route 53 のみ）
- 100家庭利用: ~$0.50/月（無料枠内）
- $5 は通常の10倍 → これを超えたら明らかに異常

**アラートの段階設計:**

| 段階 | 閾値 | アクション |
|------|------|-----------|
| 情報 | $2.5（50%） | メール通知のみ。確認・記録 |
| 警告 | $4.0（80%） | 原因調査を開始。API Gateway スロットリング確認 |
| 緊急 | 予測超過 | 即座に原因特定。必要に応じ API Gateway を無効化 |

#### 2. Cost Anomaly Detection

```
Anomaly Monitor: ganbari-quest
  ├── 対象: AWS アカウント全体
  ├── 異常検知: ML ベース（自動学習）
  └── 通知: $1 以上の異常増加時にメール
```

- AWS が過去の利用パターンを学習し、急激な費用増加を自動検知
- Budgets の予算ベースアラートと併用することで、「じわじわ増える」「急に跳ねる」両方をカバー

#### 3. ECR ライフサイクルポリシー

```json
{
  "rules": [
    {
      "rulePriority": 1,
      "description": "直近10個のタグ付きイメージを保持",
      "selection": {
        "tagStatus": "tagged",
        "tagPrefixList": ["v", "sha-"],
        "countType": "imageCountMoreThan",
        "countNumber": 10
      },
      "action": { "type": "expire" }
    },
    {
      "rulePriority": 2,
      "description": "未タグイメージは1日で削除",
      "selection": {
        "tagStatus": "untagged",
        "countType": "sinceImagePushed",
        "countUnit": "days",
        "countNumber": 1
      },
      "action": { "type": "expire" }
    }
  ]
}
```

**保持数の根拠:**
- 10イメージ = 約2GB → ECR費用 ~$0.20/月（≈30円）で固定
- ロールバック用に直近10世代あれば十分（約2週間分）
- 11個目以降は自動削除 → 蓄積が止まる

#### 4. CloudWatch Logs 保持期間

| ロググループ | 保持期間 | 理由 |
|------------|---------|------|
| Lambda 関数ログ | 14日 | デバッグに必要な最小期間 |
| API Gateway アクセスログ | 7日 | トラフィック分析は短期で十分 |
| CDK デプロイログ | 3日 | デプロイ直後の確認のみ |

**無期限 → 14日にすることで:**
- 1年後の Logs ストレージ: ~0GB（自動削除）vs 無期限だと数GB以上が蓄積
- 5GB 無料枠を超えないことを保証

#### 5. CDK 定義

```typescript
// infra/lib/cost-management-stack.ts
export class CostManagementStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props: CostManagementProps) {
    // AWS Budgets
    new budgets.CfnBudget(this, 'MonthlyBudget', {
      budget: {
        budgetLimit: { amount: 5, unit: 'USD' },
        budgetType: 'COST',
        timeUnit: 'MONTHLY',
      },
      notificationsWithSubscribers: [/* ... */],
    });

    // ECR Lifecycle Policy
    ecrRepo.addLifecycleRule({ maxImageCount: 10, tagStatus: ecr.TagStatus.ANY });
    ecrRepo.addLifecycleRule({ maxImageAge: cdk.Duration.days(1), tagStatus: ecr.TagStatus.UNTAGGED });

    // CloudWatch Logs Retention
    lambdaLogGroup.applyRemovalPolicy(/* ... */);
    // Note: retention は Lambda 構築時に LogRetention で設定
  }
}
```

**注意:** AWS Budgets は us-east-1 リージョンでの作成が必要な場合がある（CDK でクロスリージョン対応を確認すること）

|------|----------|------------|
| ECR ライフサイクル | $0 | ~$50（≈7,500円）削減 |
