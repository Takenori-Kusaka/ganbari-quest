### 概要

GitHub Pagesで動作するデモサイトを構築し、操作チュートリアル兼LP（ランディングページ）兼HPとして公開する。

### 背景・動機

ユーザーがアプリの操作感を事前に体験でき、サービスの価値を理解できる場が必要。GitHub Pagesなら無料でホスティングでき、同一リポジトリで管理できる。

### ゴール

- [ ] GitHub Pagesでホスティングされるデモサイト構築
- [ ] インタラクティブなチュートリアル（操作ガイド）
- [ ] LP機能（サービス紹介・CTA・機能説明）
- [ ] レスポンシブ対応（モバイルファースト）
- [ ] PWAとしてのインストール案内

### 対応方針

#### アーキテクチャ
```
gh-pages ブランチ or /docs ディレクトリ
├── index.html          # LP（サービス紹介）
├── demo/               # デモアプリ（モックデータ）
│   ├── index.html
│   └── ...
├── tutorial/           # チュートリアル
│   ├── getting-started.html
│   └── ...
├── assets/
│   ├── screenshots/    # スクリーンショット
│   └── gifs/          # 操作GIF
└── docs/              # マニュアル（後でHP統合）
```

#### 技術選定候補
1. **SvelteKit (adapter-static)**: 既存コードベースを流用し、モックデータで動作するデモ版をビルド
2. **Astro / VitePress**: ドキュメントサイト向けSSGで軽量に構築
3. **純粋HTML + Tailwind**: 最もシンプル、依存なし

**推奨**: SvelteKit adapter-static でデモアプリ部分を実際のUIで体験可能にしつつ、LPページも同フレームワークで構築

#### デモの仕様
- モックデータ（ローカルストレージ or 静的JSON）で動作
- API呼び出しなし（完全クライアントサイド）
- 子供画面のみデモ可能（管理画面はスクリーンショット紹介）
- データリセット機能付き

#### LP要素（規定書 4.2 ランディングページ設計書準拠）
1. **ファーストビュー**: キャッチコピー + アプリスクリーンショット + CTA
2. **機能紹介**: 3-5の主要機能をGIFアニメ付きで説明
3. **年齢帯別UI紹介**: baby/kinder/lower の画面例
4. **導入方法**: セルフホスト or SaaS
5. **料金プラン**: Phase 3で追加
6. **FAQ**: よくある質問
7. **フッター**: GitHub, Discord, ライセンス
