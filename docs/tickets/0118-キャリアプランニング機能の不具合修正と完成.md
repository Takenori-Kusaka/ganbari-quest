# #0118 キャリアプランニング機能の不具合修正と完成

## ステータス: Done
## 優先度: 高
## 種別: バグ修正（機能不全）

---

## 概要

チケット #0048 は「Done」ステータスだが、**実際にはキャリアプランニング機能が動作しない状態**で放置されている。コードは全層（DB・リポジトリ・サービス・API・UI）が実装済みであるにもかかわらず、以下の根本的な問題により子供が一切プランを作成できない。

### 根本原因

#### 問題1: 職業分野マスタデータ未投入

`career_fields` テーブルは `create-tables.ts` で作成されるが、**マスタデータの投入は `scripts/add-career-tables.cjs` を手動実行しなければ行われない**。`seed.ts` にも Docker エントリーポイント（`docker-entrypoint.sh`）にもキャリア関連のシードが含まれていない。

```
create-tables.ts → career_fields テーブル作成 ✅
seed.ts → career_fields へのデータ投入 ❌ ← ここが抜けている
docker-entrypoint.sh → seed.ts を実行 → career_fields は空のまま
```

結果として、`CareerFieldSelector` に渡される `fields` 配列が空になり、子供画面では「しょうらい、なにになりたい？ すきなおしごとをえらんでね！」の下に**選択肢が何も表示されない**。

#### 問題2: 年齢フィルタが低年齢を排除

全30職業の `minAge` が最低6歳に設定されているため、4歳の子供の場合:

```typescript
// career-repo.ts
findCareerFieldsByAge(age: 4)
→ WHERE min_age <= 4
→ 結果: 0件  ← 全職業がminAge=6以上のため全滅
```

現在の利用者（おじょうさま 4歳）はたとえマスタデータが投入されていても職業が1件も表示されない。

#### 問題3: 表記ゆれ

| 画面 | 表記 | 問題 |
|------|------|------|
| 親管理画面 子供一覧 | 「ステータス画面」 | 子供画面では「つよさ」と表記 |
| 親管理画面 キャリア | 「ステータス画面の『みらいのゆめ』から」 | 子供からは「つよさ」→「みらいのゆめ」の導線 |
| BottomNav | 「つよさ」 | ステータス画面のこと |

親が「ステータス画面」と案内されても、子供画面では「つよさ」というラベルなので、どこから行けるのか分かりにくい。

## 修正内容

### 修正1: seed.ts に職業分野マスタデータを追加

```typescript
// src/lib/server/db/seed.ts に追加

// キャリア職業分野マスタデータ
const careerFieldsData = [
  { name: 'かがくしゃ', description: '実験や研究で新しい発見をする', icon: '🔬', relatedCategories: '[2,5]', minAge: 4 },
  { name: 'おいしゃさん', description: '病気の人を助けて元気にする', icon: '🏥', relatedCategories: '[2,3]', minAge: 4 },
  // ... 30職業
];
```

### 修正2: minAge の見直し

4歳児でも「おおきくなったらなにになりたい？」は定番の質問。低年齢の子供にも使えるよう、minAge を見直す。

| 対応方針 | 内容 |
|---------|------|
| **基本職業（20件）** | `minAge: 4` に引き下げ（おいしゃさん、しょうぼうし、ケーキやさん等は幼児の定番） |
| **専門職業（10件）** | `minAge: 6` or `8` を維持（エンジニア、プログラマー、デザイナー等） |

### 修正3: 表記ゆれの統一

| 修正箇所 | 修正前 | 修正後 |
|---------|--------|--------|
| 親管理画面キャリアページ | 「子供がステータス画面の」 | 「子供が『つよさ』画面の」 |
| 親管理画面（その他参照箇所） | 「ステータス画面」 | 「つよさ画面」 or 「ステータス（つよさ）画面」 |

### 修正4: Docker エントリーポイントでのシード確認

`docker-entrypoint.sh` で初回起動時に `seed.ts` が実行されるが、**既存環境**ではDBが既に存在するためシードが再実行されない。既存環境向けにマイグレーションスクリプトを提供する。

```bash
# 既存環境向け
node scripts/add-career-tables.cjs data/ganbari-quest.db
```

### 修正5: 空状態のUX改善

`CareerFieldSelector` に渡される `fields` が空の場合に「おしごとがみつかりません」ではなく、適切なメッセージを表示する。

```svelte
{#if data.careerFields.length === 0}
  <div class="empty-state">
    <span class="text-4xl">🌟</span>
    <p>おしごとをじゅんびちゅう...</p>
    <p class="text-xs">おとうさん・おかあさんにおねがいしてね</p>
  </div>
{:else}
  <CareerFieldSelector ... />
{/if}
```
