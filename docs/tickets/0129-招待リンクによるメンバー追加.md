### 概要

親（owner/parent）が招待リンクを発行し、同一家族グループにメンバーを追加するフロー。招待されたユーザーはリンク経由でサインアップし、自動的に同じテナントに参加する。

### フロー

```
1. owner/parent が /admin/members で「メンバー招待」ボタン
2. 招待オプション選択:
   - ロール: parent（共用アカウント）/ child（ティーン）
   - 紐付け先の子供プロフィール（child ロールの場合）
3. 招待リンク生成（有効期限: 7日）
   - URL: https://ganbari-quest.com/auth/invite/{inviteCode}
4. owner/parent がリンクを共有（LINE, メール等）
5. 招待された人がリンクにアクセス
   - 既存アカウントあり → ログインしてテナントに参加
   - 既存アカウントなし → サインアップ画面（ライセンスキー不要）
6. テナントに参加（指定ロールで Membership 作成）
```

### DynamoDB 設計

```
INVITE#{inviteCode} / META
  - inviteCode: string（UUID）
  - tenantId: string
  - invitedBy: string（userId）
  - role: 'parent' | 'child'
  - childId?: number（child ロールの場合）
  - status: 'pending' | 'accepted' | 'expired' | 'revoked'
  - createdAt: string
  - expiresAt: string
```

### 制約

- 1ユーザーは1つの家族グループにのみ所属可能
- 招待された人が既に別の家族グループに属している場合はエラー
- 招待リンクは最大使用回数1回（使い捨て）

### 完了条件

1. owner/parent が招待リンクを生成できる
2. 招待リンク経由で新規ユーザーがテナントに参加できる
3. 既存ユーザーが招待リンクでテナントに参加できる
4. 既に他テナントに属するユーザーは参加できない
5. 期限切れの招待リンクは使用できない
