# ステータス表示のリデザイン — 児童心理学に基づくフィードバック設計

## Status: Ready

### 優先度: 高
### 種別: UX改善 / 設計見直し

---

## 概要

当初は「★評価の算出基準（偏差値 vs スコア割合）の不一致修正」として起票されたが、
児童心理学の学術リサーチを経て、**問題の本質は★評価の算出方法ではなく、
子供画面に他者比較情報を表示していること自体**であると判明した。

本チケットでは、学術知見に基づいて子供画面と親画面のフィードバック設計を根本から見直す。

---

## 学術的根拠

詳細は `docs/reference/child-psychology-ux-research.md` を参照。

### 核心的な知見

1. **5歳未満は社会的比較能力がほぼ未発達** (Butler, 1998; Child Development)
   - 幼児に他者比較情報を与えることは発達段階に合致しない
   - 偏差値のような相対的位置づけは認知的に処理不能 (Ruble et al., 1994)

2. **他者比較は固定マインドセットを助長する** (Dweck, 2006)
   - 「みんなと比べてどうか」→ 能力は固定的という信念
   - 「昨日の自分より成長」→ 能力は伸びるという信念（成長マインドセット）

3. **有形報酬（★、ポイント）は内発的動機を低下させうる** (Deci et al., 1999; メタ分析)
   - 「統制的報酬」（これをしたらもらえる）→ 内発的動機低下
   - 「情報的報酬」（こんなにできるようになった）→ 有能感向上

4. **プロセス褒めが長期的に最も効果的** (Gunderson et al., 2013; 縦断研究)
   - 1-3歳時の褒め方が7-8歳時のマインドセットを予測

5. **親への情報提示は「強み→成長余地→サポート方法」の順序** (Glascoe, 1997)
   - 偏差値の単一数値は親の不安を不必要に増幅 (Zuckerman & Halfon, 2003)

## 現状の問題

### 子供画面（ステータス）
- 偏差値ベースの★評価 = **他者比較** → 固定マインドセット助長リスク
- スコア表示（31.3/350）= **抽象的な数値** → 幼児には理解不能
- ★とスコアの不一致 → 親が見ても混乱

### 親管理画面（ステータス管理）
- 偏差値の数値表示 → 親の不安増幅リスク
- 「みんなとくらべて」のラベル表示は情報的価値あり（親向けとしては適切）
- ただし、偏差値よりも「同年齢の目安範囲」（帯表示）がより適切

---

## 設計方針: B案ベース + 子供画面からの他者比較完全排除

### 子供画面: 自己比較型フィードバックに移行

**表示すべき:**
- レーダーチャート（形状で得意分野を可視化、★は撤去）
- 自己比較メッセージ（「まえよりのびたよ！」）
- プログレス表現（成長の可視化）

**排除すべき:**
- ★★★☆☆ のような他者基準の評定
- 偏差値、パーセンタイル
- 「みんなとくらべて」ラベル
- 抽象的な数値スコア

### 親画面: スコア割合ベース★ + 目安範囲の帯表示

**表示する:**
- ★評価（スコア割合ベース、スコアと直感一致）
- 自己比較の数値（先週比 +3 等）
- 同年齢の目安範囲（帯表示、折りたたみ）
- 偏差値ラベル（「みんなとおなじくらい」、帯表示内に統合）

## 技術的な変更点

### `calcStars()` の変更

```typescript
// Before: 偏差値ベース
export function calcStars(deviationScore: number): number { ... }

// After: スコア割合ベース
export function calcStars(value: number, maxValue: number): number {
  const pct = maxValue > 0 ? (value / maxValue) * 100 : 0;
  if (pct >= 80) return 5;
  if (pct >= 60) return 4;
  if (pct >= 40) return 3;
  if (pct >= 20) return 2;
  return 1;
}
```

### 技術メモ

- `calcStars` は現在 `deviationScore: number` のみ引数。変更後は `value, maxValue` を受け取る
- `getComparisonLabel(deviationScore)` は親画面のみで使用する形に変更
- `calcDeviationScore()` とベンチマーク仕組みは削除しない（親画面の帯表示で使用）
- `status-service.ts` 内の `stars` 計算箇所は一箇所に集約済み
