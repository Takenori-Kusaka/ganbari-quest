# #0089 ごほうび表示が何度も繰り返される不具合

## Status: Done
## 優先度: 高
## 種別: バグ修正

---

## 概要

親が子供に付与した「ごほうび（特別報酬）」の表示オーバーレイが、子供のホーム画面を開くたびに・操作するたびに何度も表示されてしまう不具合を修正する。

## 現象

- 親の管理画面からごほうびを付与する
- 子供のホーム画面でごほうびオーバーレイが表示される（正常）
- オーバーレイを閉じた後も、画面遷移や活動記録のたびに再びオーバーレイが表示される（不具合）
- baby / kinder 両モードで再現

## 原因分析

### `$effect` の再トリガー問題

kinder / baby 両方のホーム画面で以下の `$effect` がごほうびチェックを行っている：

```typescript
$effect(() => {
  if (data.latestReward && !bonusClaiming && !omikujiOpen) {
    checkSpecialReward();
  }
});
```

- `$effect` は依存する値（`data`, `bonusClaiming`, `omikujiOpen`）が変化するたびに再実行される
- 活動記録後の `invalidateAll()` → `data` オブジェクトが更新 → `$effect` 再起動 → `checkSpecialReward()` が再呼出
- `checkSpecialReward()` 内の `localStorage` チェックは存在するが、以下のタイミング問題がある：
  - `seenRewardId`（インメモリ状態）は `handleRewardClose()` でしかセットされない
  - `invalidateAll()` より先に `$effect` が走ると、`localStorage` 書き込み前に再チェックが走る可能性
  - ページナビゲーション（SvelteKit のクライアントサイドルーティング）で `$effect` がリマウントされ `seenRewardId` が `null` にリセットされる場合がある

### localStorage キーの共有問題

- baby と kinder で同一の `localStorage` キー（`ganbari-last-seen-reward`）を使用
- 子供が複数いる場合、子供 A のごほうび ID が子供 B のキーを上書きする可能性がある

## 修正方針

### 案1: サーバーサイドで既読管理（推奨）

- `special_rewards` テーブルに `shown_at` カラムを追加
- ごほうびオーバーレイを閉じた際にサーバーにリクエストを送り `shown_at` を更新
- `+page.server.ts` で `shown_at IS NULL` のもののみ `latestReward` として返す
- クライアント側の `localStorage` / `$effect` 依存をなくし、サーバーが唯一の真実源

### 案2: クライアントサイドの修正（暫定）

- `$effect` 内で早期に `localStorage` チェックを行い、表示済みならスキップ
- `localStorage` キーを子供 ID ごとに分離（`ganbari-last-seen-reward-{childId}`）
- `checkSpecialReward()` の呼び出しを `$effect` ではなくページ初回マウント時のみに限定（`onMount` 相当の `$effect` で `untrack` を使用）

## 対象ファイル

- `src/routes/(child)/kinder/home/+page.svelte` — `$effect` / `checkSpecialReward` ロジック
- `src/routes/(child)/baby/home/+page.svelte` — 同上
- `src/lib/server/services/special-reward-service.ts` — 案1の場合、既読更新ロジック追加
- `src/lib/server/db/schema.ts` — 案1の場合、`shown_at` カラム追加
