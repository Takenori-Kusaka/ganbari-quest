# 活動の複数回実行対応

### ステータス

`Done`

---

### 概要

現在の活動記録は1日1回を前提としているが、はみがきや部屋の掃除など1日に何回やっても良い活動がある。活動マスタに「複数回実行可能」属性を追加し、該当する活動は1日に複数回記録できるようにする。UIも複数回実行を直感的に表現する。

### 背景・動機

- はみがき（朝・昼・夜）、掃除など1日複数回行うのが自然な活動がある
- 現在は `activity_logs` テーブルの `UNIQUE(childId, activityId, recordedDate)` 制約により1日1回に制限
- 子供のやる気を活かすために、複数回やったことを正しく記録・評価したい

### 技術方針

#### DBスキーマ変更

```sql
ALTER TABLE activities ADD COLUMN allowMultiple INTEGER NOT NULL DEFAULT 0;
```

#### UNIQUE制約の対応

現在の制約:
```
UNIQUE(childId, activityId, recordedDate)
```

方針: 制約自体は残し、`allowMultiple = 1` の活動については `recordActivity()` のロジックで重複チェックをスキップする。SQLiteでは既存のUNIQUE制約を簡単にDROPできないため、`INSERT OR IGNORE` の代わりに事前チェックのロジックを変更する。

代替案: `recordedDate` を `recordedAt`（タイムスタンプ）に変更し、日付+連番で区別する。

#### 複数回実行時のルール

| 項目 | 1回目 | 2回目以降 |
|------|-------|----------|
| ベースポイント | 付与 | 付与 |
| ストリークボーナス | 付与 | なし（1日1回） |
| ステータス更新 | 付与 | 付与 |
| 実績判定 | トリガー | トリガー |

#### seedで `allowMultiple = 1` にすべき活動の候補

**せいかつ（生活習慣）:**
- はみがきした / はみがき・せいけつ（朝昼夜）
- てあらい・うがいした（毎食前後）
- おかたづけした（都度）
- おへやをそうじする / 部屋を掃除する（朝夕）
- テーブルをふく（毎食後）
- おさらあらい / 皿洗い（毎食後）

**べんきょう（学習）:**
- どくしょ（20ぷん）/ 読書（30分以上）（複数セッション可）

**うんどう（運動）:**
- たいそう・ストレッチ（朝夕）

#### UI設計

**活動カード（記録済み + 複数回可能）:**
```
┌──────────────────┐
│  🪥               │
│  はみがきした      │
│  ✅ +2            │  ← バッジで回数表示
│  5pt × 3回        │
└──────────────────┘
```

- 1回目: 通常のスタンプ表示（✅）
- 2回目以降: スタンプの右上にバッジ「+N」を表示
- ボタンはdisableにせず、タップで追加記録可能
- 確認ダイアログ: 「もう1かいきろくする？（きょう N かいめ）」

**活動カード（記録済み + 1日1回制限）:**
- 従来通りスタンプ表示 + ボタンdisable
