# #0110 開発環境を Vite+ 統合ツールチェーンへ移行

## ステータス: Ready
## 優先度: 低
## 種別: 開発基盤改善

---

## 概要

現在の開発ツールチェーン（Vite 7 + Vitest + Biome）を Vite+（vite-plus）に移行し、ビルド・テスト・リント・フォーマットを単一CLIに統合する。設定ファイルの削減、ビルド高速化、開発体験の向上が目的。

**参考資料**: [Vite+ 完全調査レポート](../research/Vite+%20完全調査レポート：OSS化の概要と開発環境移行プランニング.md)

## 現状の開発ツールチェーン

| 用途 | 現在のツール | バージョン | 設定ファイル |
|------|------------|-----------|------------|
| 開発サーバー / ビルド | Vite | ^7.3.1 | `vite.config.ts` |
| テスト | Vitest | ^3.0.0 | `vite.config.ts` 内 |
| リント / フォーマット | Biome | ^1.9.0 | `biome.json` |
| 型チェック | svelte-check + tsc | ^4.3.6 | `tsconfig.json` |

**管理する設定ファイル**: `vite.config.ts`, `biome.json`, `tsconfig.json`, `playwright.config.ts`

## 移行後のツールチェーン

| 用途 | 移行後のツール | 設定 |
|------|--------------|------|
| 開発サーバー / ビルド | Vite 8 + Rolldown | `vite.config.ts` に統合 |
| テスト | Vitest（Vite+ 統合） | `vite.config.ts` に統合 |
| リント | Oxlint | `vite.config.ts` に統合 |
| フォーマット | Oxfmt | `vite.config.ts` に統合 |
| 型チェック | tsgolint | `vp check` で統合実行 |

**削減される設定ファイル**: `biome.json`（Oxlint/Oxfmt設定は `vite.config.ts` に統合）

## 移行フェーズ

### Phase 1: Vite 8 へのアップグレード

Vite+ 移行の必須前提条件。Vite 8 は Rolldown（Rust製バンドラー）に統一されており、単独でもビルド 1.6〜7.7 倍高速化の恩恵がある。

**作業内容:**

```bash
npm install vite@8 @sveltejs/vite-plugin-svelte@latest --save-dev
```

**確認事項:**
- [ ] `vite dev` で開発サーバーが正常起動すること
- [ ] `vite build` でプロダクションビルドが成功すること
- [ ] CJS パッケージ（better-sqlite3 等）のインポートに問題がないこと
- [ ] `esbuild.*` オプション使用箇所があれば `oxc.*` に書き換え
- [ ] SvelteKit アダプター（adapter-node）が Vite 8 に対応していること
- [ ] Tailwind CSS の Vite プラグイン（`@tailwindcss/vite`）が動作すること

**リスク:**
- `better-sqlite3` は CJS ネイティブモジュールのため、Rolldown の CJS インターロップに注意
- 問題発生時は `legacy.inconsistentCjsInterop: true` で一時回避

### Phase 2: Vite+ CLI の導入と `vp migrate` 実行

**作業内容:**

```powershell
# Vite+ CLI グローバルインストール（Windows）
irm https://vite.plus/ps1 | iex

# プロジェクトへの移行（インタラクティブ）
vp migrate
```

`vp migrate` が自動実行する処理:
1. `vite-plus` パッケージのインストール
2. `vite` インポートを `vite-plus` に書き換え
3. `vitest` インポートを `vite-plus/test` に書き換え
4. `package.json` スクリプトの更新
5. エディタ設定ファイル生成

### Phase 3: Biome 設定の Oxlint/Oxfmt への移行

現在の `biome.json` の設定を `vite.config.ts` に統合する。

**Biome → Oxlint ルール対応:**

| Biome ルール | Oxlint 対応ルール |
|-------------|------------------|
| `style/noNonNullAssertion: warn` | `@typescript-eslint/no-non-null-assertion: warn` |
| `style/useConst: error` | `prefer-const: error` |
| `suspicious/noExplicitAny: error` | `@typescript-eslint/no-explicit-any: error` |
| `suspicious/noRedeclare: warn` | `no-redeclare: warn` |
| `suspicious/noAssignInExpressions: warn` | `no-cond-assign: warn` |
| `correctness/noUnusedVariables: warn` | `no-unused-vars: warn` |
| `correctness/noUnusedImports: warn` | `unused-imports/no-unused-imports: warn` |

**Biome → Oxfmt フォーマット対応:**

| Biome 設定 | Oxfmt 設定 |
|-----------|-----------|
| `indentStyle: "tab"` | `useTabs: true` |
| `lineWidth: 100` | `printWidth: 100` |
| `quoteStyle: "single"` | `singleQuote: true` |
| `semicolons: "always"` | `semi: true`（デフォルト） |
| `trailingCommas: "all"` | `trailingComma: "all"` |

**移行後の `vite.config.ts` イメージ:**

```typescript
import { sveltekit } from '@sveltejs/kit/vite';
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite-plus';

export default defineConfig({
  plugins: [tailwindcss(), sveltekit()],

  // リント設定（旧 biome.json → linter）
  lint: {
    rules: {
      'prefer-const': 'error',
      '@typescript-eslint/no-non-null-assertion': 'warn',
      '@typescript-eslint/no-explicit-any': 'error',
      'no-redeclare': 'warn',
      'no-cond-assign': 'warn',
      'no-unused-vars': 'warn',
      'unused-imports/no-unused-imports': 'warn',
    },
    overrides: [
      {
        files: ['**/*.svelte'],
        rules: {
          'no-unused-vars': 'off',
          'unused-imports/no-unused-imports': 'off',
          'prefer-const': 'off',
        },
      },
    ],
    ignore: ['node_modules', '.svelte-kit', 'build', 'coverage', 'test-results', 'scripts/', 'infra/', 'cdk.out'],
  },

  // フォーマット設定（旧 biome.json → formatter）
  fmt: {
    useTabs: true,
    tabWidth: 2,
    printWidth: 100,
    singleQuote: true,
    trailingComma: 'all',
  },

  // テスト設定（既存を維持）
  test: {
    include: ['tests/unit/**/*.test.ts', 'tests/integration/**/*.test.ts'],
    environment: 'jsdom',
    globals: true,
    coverage: {
      provider: 'v8',
      reporter: ['text', 'html', 'lcov'],
      include: ['src/lib/**/*.ts', 'src/lib/**/*.svelte'],
      exclude: ['src/lib/**/*.d.ts', 'src/lib/**/index.ts'],
      thresholds: { lines: 80, functions: 80, branches: 75, statements: 80 },
    },
  },
});
```

### Phase 4: package.json スクリプトの更新と検証

**スクリプト対応表:**

| 用途 | 現在 | 移行後 |
|------|------|-------|

| ビルド | `vite build` | `vp build` |

| E2E テスト | `playwright test` | `playwright test`（変更なし） |
| DB操作 | `drizzle-kit *` | `drizzle-kit *`（変更なし） |

**AGENTS.md / CLAUDE.md の更新:**
- コマンド一覧を `vp` ベースに更新

## 削減されるもの

| 削除対象 | 理由 |
|---------|------|
| `biome.json` | Oxlint/Oxfmt 設定が `vite.config.ts` に統合 |
| `@biomejs/biome` (devDependency) | Oxlint/Oxfmt に置換 |
| `vitest` (devDependency) | `vite-plus` に内包 |

## リスクと対策

| リスク | 影響度 | 対策 |
|-------|-------|------|
| Vite+ が Alpha（v0.1.x）で不安定 | 高 | `package.json` でバージョンをピン留め（`^` なし）。問題発生時は Vite 8 + Biome に即時ロールバック |
| Windows サポートが不完全（WSL 推奨） | 中 | 開発は WSL 上で実施。CI/CD（GitHub Actions）は Linux なので問題なし |
| better-sqlite3 の CJS インターロップ | 中 | Phase 1 で事前検証。`legacy.inconsistentCjsInterop: true` で回避 |
| Svelte ファイルの Oxlint 対応 | 中 | `.svelte` ファイルの overrides 設定で lint ルールを調整 |
| Oxfmt のフォーマット差異 | 低 | CI で差分チェック。微小な差異は許容 |
| 破壊的変更の頻発 | 高 | Alpha 期間中はアップデートを慎重に、変更ログを毎回確認 |

## ロールバック手順

移行後に問題が発生した場合:

```bash
# 1. vite-plus をアンインストール
npm uninstall vite-plus

# 2. 元のツールを再インストール
npm install vite@7 vitest@3 @biomejs/biome@1 --save-dev

# 3. git で設定ファイルを復元
git checkout -- vite.config.ts biome.json package.json

# 4. 動作確認
npm run dev
npm run test
npm run lint
```

## 注意事項

- **Alpha 段階であるため、安定版リリース（v1.0 見込み: 2026年Q4〜）まではリスクを認識した上で運用する**
- Phase 1（Vite 8 アップグレード）は Vite+ とは独立して実施可能。先行して着手してもよい
- `Dockerfile` 内のビルドコマンドも `vp build` に更新が必要
- GitHub Actions の CI ワークフローで `vp` CLI のインストールステップが必要になる
