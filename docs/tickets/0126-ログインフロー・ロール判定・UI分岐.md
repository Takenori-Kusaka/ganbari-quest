### 概要

Email/Password でログインし、JWT のロール情報に基づいて管理画面または子供画面に振り分けるフロー。共用アカウント（parent）の場合は子供選択画面を経由する。

### フロー

#### owner がログインする場合

```
1. /auth/login でメール + パスワード入力
2. MFA 設定済みなら MFA コード入力
3. Cognito 認証成功 → JWT 取得
4. JWT から role=owner を判定
5. → /admin にリダイレクト
```

#### parent（共用アカウント）がログインする場合

```
1. /auth/login でメール + パスワード入力
2. Cognito 認証成功 → JWT 取得
3. JWT から role=parent を判定
4. → /admin/select-child（子供選択画面）にリダイレクト
5. 「管理画面へ」or「○○として使う」を選択
   - 管理画面 → /admin
   - 子供として → /child/{childId}
```

#### child がログインする場合（ティーン向け）

```
1. /auth/login でメール + パスワード入力
2. Cognito 認証成功 → JWT 取得
3. JWT から role=child, childId を判定
4. → /child/{childId} にリダイレクト（管理画面アクセス不可）
```

```json
{
  "sub": "u-xxxxxxxx",
  "email": "parent@example.com",
  "custom:tenantId": "t-xxxxxxxx",
  "custom:role": "owner",
  "iss": "https://cognito-idp.ap-northeast-1.amazonaws.com/...",
  "exp": 1234567890
}
```

### hooks.server.ts の認証フロー（AUTH_MODE=cognito）

```typescript
// 1. Cookie から JWT 取得
```
