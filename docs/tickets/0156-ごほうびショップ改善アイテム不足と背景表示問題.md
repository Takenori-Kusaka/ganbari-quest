## 問題 1: 既存 DB にサウンド・演出カテゴリが存在しない

### 現状

`seed.ts` は 33 件のアイテムを定義している:
- `background` × 8
- `frame` × 6
- `effect` × 5
- `sound` × 7（カスタム MP3 は `static/sounds/custom/` に全 6 種類存在）
- `celebration` × 7（`CelebrationEffect.svelte` は実装済み・ホーム画面で利用中）

しかし `seed.ts` の migration コードは **「アプリ初回起動時」または `npm run seed` 実行時にのみ动作する**。

`#0119` の実装コミット以前から運用している DB（NUC サーバーの本番 DB を含む）では、
`sound` / `celebration` カテゴリのアイテムが `avatar_items` テーブルに存在しない可能性が高い。
→ ショップに「きろくおん」「たっせいえんしゅつ」タブが表示されても、アイテムが 0 件になる。

### 解決策

専用マイグレーションスクリプト `scripts/add-shop-items-sound-celebration.cjs` を作成し、
既存 DB に不足しているアイテムを安全に追加する。

```js
// 既存のコードが存在しないアイテムのみ INSERT（冪等）
const missingItems = soundAndCelebrationItems.filter(item =>
  !db.prepare("SELECT 1 FROM avatar_items WHERE code = ?").get(item.code)
);
for (const item of missingItems) {
  db.prepare("INSERT INTO avatar_items (...) VALUES (...)").run(...);
}
```

**NUC サーバーへの適用手順**（CLAUDE.md 安全手順に従う）:
1. コンテナを停止
2. スクリプトを実行
3. コンテナを再起動

## 問題 2: カスタムアバター画像がある場合に背景アイテムが事実上無意味

### 現状

`AvatarDisplay.svelte` の描画構造:

```
[avatar-wrap 36px]
  └── [avatar-ring 36px]  ← background: {bgCss} を適用
        └── [img 30px]    ← (36 - 6 = 30px) 画像が背景を覆う
```

ヘッダーのアバター（36px）では画像が背景をほぼすべて覆い、  
背景グラデーションは **幅 3px の隙間** にしか表示されない。  
100〜500pt を払って購入しても視覚的な変化がほとんどない。

**フレーム（`border`）** は動作 ✅  
**エフェクト（CSS アニメーション）** は動作 ✅  
**背景（`background`）** は事実上不可視 ⚠️

### 解決策

カスタムアバター画像がある場合の背景を「背景グラデーション帯」として可視化する。

#### 案 A（推奨）: アバター画像の外側に背景を帯として表示

```
[avatar-wrap 40px]  ← 少し拡大
  ├── [bg-band 40px]  ← background: {bgCss} を全面に表示
  │     opacity: 0.35 で画像との融合
  └── [avatar-ring 36px]  ← 中央に画像
```

具体的には: 背景を `avatar-ring` の `background` から、`avatar-wrap` の `::before` 擬似要素に移動し、画像の背後に見えるようにする。

#### 案 B（UX 的に明快）: 「背景はカスタムアバター未使用時のみ有効」を UI で明示

ショップの背景タブに「カスタム写真を使用中のときは効果が薄くなります 📷」という注意書きを表示。  
実装コストが低く、ユーザーへの誠実な情報提供になる。

#### 採用方針

**案 B（注意書き追加）** を短期対応として実施。  
**案 A（視覚的改善）** を中期対応として別 PR で実施（AvatarDisplay のサイズ変更はヘッダー・ステータス画面等に波及するため慎重に）。
