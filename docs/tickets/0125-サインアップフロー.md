### 概要

新規ユーザーがライセンスキーを使ってサインアップし、家族グループ（テナント）を作成するフロー。

### フロー

```
1. ユーザーがサインアップ画面にアクセス
2. 入力: メールアドレス、パスワード、ライセンスキー
3. Cognito Pre Sign-up トリガー:
   - ライセンスキーを DynamoDB で検証
   - 無効 or 使用済み → エラー
4. Cognito がメール確認コード送信
5. ユーザーが確認コード入力
6. Cognito Post Confirmation トリガー:
   - 新規テナント（家族グループ）作成
   - ユーザーを owner ロールで登録
   - ライセンスキーをテナントに紐付け（使用済みに）
   - Cognito カスタム属性に tenantId, role を設定
7. 自動ログイン → /admin にリダイレクト
```

### ライセンスキー設計

```
フォーマット: GQ-XXXX-XXXX-XXXX（16文字、英数字）
DynamoDB キー: LICENSE#GQ-XXXX-XXXX-XXXX / META

属性:
- licenseKey: string
- status: 'unused' | 'active' | 'expired' | 'revoked'
- plan: 'monthly' | 'yearly' | 'lifetime'
- stripeSubscriptionId?: string
- tenantId?: string（使用後に紐付け）
- createdAt: string
- activatedAt?: string
- expiresAt?: string
```

### 完了条件

1. ライセンスキーを使ってサインアップできる
2. サインアップ後に家族グループが自動作成される
3. 初回ユーザーは owner ロールで登録される
4. 無効なライセンスキーではサインアップできない
5. メール確認が完了しないとログインできない
