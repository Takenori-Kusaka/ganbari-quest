# #0098 AI活動追加機能の改善

## Status: Done
## 優先度: 中
## 種別: 機能改善

---

## 概要

親の管理画面の「✨ AI追加」機能で、自然言語から活動を提案する際に以下の問題がある:

1. **アイコンが適切に設定されない** — AIが返すアイコンが正しく反映されていない可能性
2. **漢字表記・ふりがな表記が自動入力されない** — `nameKana`/`nameKanji` がnullで返ってきている
3. **フォールバック（キーワードベース推定）の品質が低い** — Gemini APIが使えない場合、アイコンはカテゴリの固定1つ目のみ、ふりがな/漢字は常にnull

## 現状の実装

### フロー

```
UI: aiInput入力 → suggestFromAI() → POST /api/v1/activities/suggest
  → activity-suggest-service.ts
    ├─ Gemini API利用可能 → suggestWithGemini()
    └─ API利用不可 or エラー → suggestByKeywords()（フォールバック）
```

### Gemini API (suggestWithGemini)

- **プロンプト**: カテゴリ・ポイント・アイコン（mainIcon/subIcon）・活動名3表記（name/nameKana/nameKanji）を指示
- **JSON抽出**: `responseText.match(/\{[^}]+\}/)` で最初のJSONオブジェクトを抽出
- **アイコン処理**: `mainIcon` + `subIcon` → `joinIcon()` で複合アイコンを生成

#### 問題点

1. **JSON抽出の正規表現が脆弱**: `/\{[^}]+\}/` はネストされたJSONに対応しない。`nameKanji` に `}` が含まれるケースや、Geminiの応答にJSONブロック外のテキストが混ざると抽出失敗する
2. **`nameKana`/`nameKanji` がnullで返される可能性**: プロンプトで「nameと同じならnull」と指示しているため、Geminiが積極的にnullを返す傾向がある。入力が既にひらがなだと `name` = `nameKana` と判断してnullになりやすい
3. **アイコンのフォールバック**: Geminiが `mainIcon`/`subIcon` を返さない場合、`parsed.icon` → `'📝'` にフォールバックし、活動内容と無関係なアイコンになる
4. **エラーハンドリング**: JSON parse失敗時にフォールバックに落ちるが、ユーザーにはフォールバックが使われたことが分からない

### フォールバック (suggestByKeywords) 

- **アイコン**: カテゴリごとの固定配列の先頭1つのみ（例: うんどう→常に🤸）。入力内容に関わらず同じアイコンになる
- **nameKana/nameKanji**: 常にnull（推定ロジックなし）
- **キーワードマッチ**: 最初にヒットしたルールで確定。「ピアノ」が `べんきょう` にマッチするなど分類精度に課題
- **ポイント**: カテゴリ毎に固定値のみ（3 or 5）

## 改善案

### 1. Gemini プロンプトの改善

| 項目 | 現状 | 改善案 |
|------|------|--------|
| nameKana/nameKanji | 「nameと同じならnull」→ nullが多い | 「必ず3つとも返す。同一でも省略しない」に変更 |
| JSON抽出 | `/\{[^}]+\}/` | ```` ```json ``` ```` ブロック抽出 or `JSON.parse` のリトライ |
| アイコン指示 | mainIcon/subIconの2つ | 具体的な絵文字候補をカテゴリ別に提示 |
| レスポンス形式 | 「JSONのみで回答」 | structured output（JSON mode）を利用 |

### 2. フォールバックの品質向上

| 項目 | 改善案 |
|------|--------|
| アイコン | キーワードごとに個別アイコンをマッピング（例: サッカー→⚽、ピアノ→🎹） |
| nameKana | 入力テキストに漢字が含まれていれば、入力をnameKanjiに、ひらがな変換をnameKanaに |
| nameKanji | 入力テキストがひらがなのみなら、nameKanaに入力をそのまま設定 |
| カテゴリ判定 | 複数キーワードのスコアリング方式に変更（先勝ちでなく最高スコアのカテゴリ） |
| ポイント | キーワードの種類で細かく分類（3/5/8/10） |

### 3. UI改善

| 項目 | 改善案 |
|------|--------|
| AI結果の表示 | 提案結果をプレビュー表示し、「この内容で追加」「修正する」を選択可能に |
| ローディング | アニメーション付きの待機表示（現状は「考え中...」テキストのみ） |
| フォールバック通知 | Gemini APIが使えずフォールバックが使われた場合にユーザーに通知 |
| 複数候補 | 1つの入力から複数の候補を提案し、選択可能に |

### 4. アイコン推定の強化

- Geminiプロンプトでカテゴリ別の絵文字リストを明示的に提示し、その中から選ばせる
- フォールバックでもキーワード→アイコンの詳細マッピングを用意
- `CompoundIcon`（mainIcon+subIcon）の組み合わせパターンを増やす
