## 概要

つよさ画面のステータス表示を、現在のプログレスバー形式からレーダーチャート（五角形）に変更する。
RPGのステータス画面を模した表現にすることで、よりゲーム的な体験を実現する。

## 背景

現在のつよさ画面は5カテゴリそれぞれのプログレスバーを縦に並べた表示。
機能的には十分だが、RPGの「ステータス画面」としての視覚的インパクトが弱い。
5カテゴリのバランスを一目で把握できるレーダーチャートの方がゲーム感が強い。

## 要件

### 1. レーダーチャート描画

- SVGベースで実装（Canvas不要、SSR互換）
- 5頂点（うんどう、べんきょう、せいかつ、こうりゅう、そうぞう）
- 値は年齢別maxValueで正規化（0〜100%）
- 塗りつぶし領域: 半透明のテーマカラー
- 外枠: 点線で最大値のペンタゴン
- 補助線: 25%, 50%, 75% の同心ペンタゴン

### 2. カテゴリラベル表示

- 各頂点の外側にカテゴリ名 + 星評価を表示
- トレンドアイコン（📈📉➡️）をラベル近くに配置

### 3. ステータス詳細（折りたたみ）

- レーダーチャートの下に「くわしくみる」ボタン
- タップで現在のプログレスバー形式の詳細表示を展開
- 数値と星評価を確認可能

### 4. アニメーション

- ページ表示時にレーダーチャートが0%から実際の値まで広がるアニメーション
- Svelte の `tweened` or CSS transition で実装

### 5. レスポンシブ対応

- チャートサイズはビューポート幅に応じて調整
- 最小: 240px、最大: 360px

### SVG座標計算

```typescript
// 五角形の各頂点座標（上から時計回り）
function getPoint(index: number, value: number, maxRadius: number): { x: number; y: number } {
  const angle = (Math.PI * 2 * index) / 5 - Math.PI / 2; // 上から開始
  const radius = (value / 100) * maxRadius;
  return {
    x: centerX + radius * Math.cos(angle),
    y: centerY + radius * Math.sin(angle)
  };
}
```

## 不具合改修履歴（2026-03-08）

### 修正1: カテゴリラベルの見切れ

- **現象**: SVG の viewBox がチャート描画領域と同サイズだったため、外側に配置されるカテゴリ名・星評価・トレンドアイコンがクリップされて読めなかった
- **原因**: `viewBox="0 0 {size} {size}"` でラベル領域の余白がなかった
- **修正**: viewBox に `padding = size * 0.2` の余白を四方に追加（`viewBox="{-padding} {-padding} {viewBoxSize} {viewBoxSize}"`）
- **対象ファイル**: `src/lib/ui/components/RadarChart.svelte`

### 修正2: レーダーチャートのプロットと星評価の乖離

- **現象**: ★4（偏差値58以上）のカテゴリでもチャート上のプロットが極端に小さく表示されていた
- **原因**: 星は偏差値ベース（相対評価）で算出されるのに対し、レーダーチャートのプロットは `value / maxValue`（絶対評価）で正規化していた。年齢別 maxValue は理論上の累積最大値（例: 5歳=500）のため、始めたばかりの子供の絶対値は必然的に小さい
- **修正**: レーダーチャートのプロットを偏差値ベースに変更。偏差値 [20, 70] → [0%, 100%] にマッピングし、星評価と視覚的に整合するようにした
- **対象ファイル**:
  - `src/lib/ui/components/RadarChart.svelte` — `CategoryData` に `deviationScore` 追加、正規化ロジック変更
  - `src/routes/(child)/kinder/status/+page.svelte` — `deviationScore` をデータに含める
  - `src/routes/(child)/baby/status/+page.svelte` — 同上
