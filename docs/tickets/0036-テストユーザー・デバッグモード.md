### 概要

さまざまな表示パターンや状態を手動確認するために、テストケース別のダミーユーザーを登録し、それらを切り替えて表示できるデバッグモードを実装する。年齢帯別UIの違い（baby/kinder/elementary 等）も一覧で体験できるようにする。

### 背景

- 現状はシードデータに子供1人（4歳・kinder）のみ
- baby モード（0-2歳）や小学生以上のUIを確認するには、DBを手動編集するかシードを書き換える必要がある
- あらゆる状態（レベル高/低、実績多/少、ポイント多/少）を操作で再現するのが大変

### 対応内容

**Phase 1: テストユーザーのシードデータ追加**
- 各年齢帯の代表的な子供を追加:
  - baby (1歳) — baby モード確認用
  - kinder (4歳) — 既存 + 進捗バリエーション
  - elementary_lower (7歳) — 小学低学年向けUI確認用
  - elementary_upper (11歳) — 小学高学年向けUI確認用
  - middle_school (14歳) — 中学生向けUI確認用
- 各テストユーザーにそれぞれ異なるステータス・実績・ポイント残高を付与

**Phase 2: デバッグパネル（開発時のみ）**
- `/switch` 画面にデバッグモード切り替え（開発環境のみ表示）
- デバッグモード有効時:
  - 全テストユーザーが切り替え画面に表示
  - 各ユーザーの状態（レベル、ポイント等）をサマリ表示
  - ワンクリックで状態リセット（activity_logs クリア、ポイント初期化等）

### 対象ファイル

- `src/lib/server/db/seed.ts` — テストユーザー追加
- `src/routes/(child)/switch/+page.svelte` — デバッグ切り替えUI
- `src/routes/(child)/switch/+page.server.ts` — デバッグ用アクション

### 受け入れ条件

- [ ] 各年齢帯（baby/kinder/elementary）のテストユーザーが `/switch` に表示される
- [ ] 各テストユーザーでそれぞれの年齢帯UIが正しく表示される
- [ ] 開発環境でのみデバッグ機能が有効（本番ビルドでは非表示）
- [ ] テストユーザーの状態を簡単にリセットできる
