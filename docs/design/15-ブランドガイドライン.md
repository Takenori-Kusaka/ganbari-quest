# ブランドガイドライン

## 1. ブランドコンセプト

**「がんばりクエスト」= 子供の成長を冒険に変える**

| 要素 | 方向性 |
|------|-------|
| キーワード | 成長・冒険・がんばり・達成・家族 |
| トーン | 温かい・楽しい・信頼感・やさしい |
| ターゲット印象 | 子供: わくわく / 親: 安心・信頼 |
| タグライン | こどもの がんばりを ぼうけんに |

## 2. シンボルマーク

かわいい勇者キャラクター（ヘルメット＋マント＋魔法の杖）のモチーフ。「クエスト（冒険・RPG）」と「がんばり（挑戦・達成）」を直感的に表現する。

### デザイン選定の経緯

- 2026-03-28: Gemini 3 Pro Image で6コンセプト×3バリエーション（計18候補）を生成
- コンセプト: A(芽と手), B(山頂の旗), C(星の塔), D(勇者キャラ), E(家族の木), F(輝く星)
- 最終選定: **D3（勇者キャラ・簡略版）** — 「がんばりクエスト」の名前とRPG世界観に最も直結
- 選定理由: 0-18歳の幅広い年齢に対応、ヘルメットで性別を特定しないデザイン、「やらなきゃいけない大変なこと」ではなく「冒険・楽しさ」を訴求
- 候補画像アーカイブ: `docs/design/logo-candidates/`

### 使用規定

- **最小サイズ**: 16×16px（これ以下ではキャラクターが潰れるため使用不可）
- **余白**: シンボルの周囲にシンボル幅の 25% 以上の余白を確保
- **背景制約**: 白または薄い背景での使用を推奨。favicon版は薄青背景付き
- **変形禁止**: アスペクト比の変更、回転、色の部分変更は禁止
- **キャラクター改変禁止**: 表情・ポーズ・装備の変更はブランドチームの承認が必要

## 3. ロゴ

### バリエーション

| バリエーション | ファイル | 用途 |
|--------------|---------|------|
| フルロゴ | `static/logo.svg` | ログイン画面、ランディングページ |
| コンパクトロゴ | `static/logo-compact.svg` | ヘッダー、ナビゲーション |
| シンボルのみ | `static/favicon.svg` | Favicon、PWA、小スペース |

### Logo コンポーネント

`src/lib/ui/components/Logo.svelte` で統一的に使用する。

```svelte
<!-- フルロゴ -->
<Logo variant="full" />

<!-- コンパクトロゴ -->
<Logo variant="compact" />

<!-- シンボルのみ（サイズ指定可能） -->
<Logo variant="symbol" size={48} />
```

| Props | 型 | デフォルト | 説明 |
|-------|---|---------|------|
| `variant` | `'symbol' \| 'full' \| 'compact'` | `'symbol'` | ロゴのバリエーション |
| `size` | `number` | variant依存 | サイズ指定（symbol=48, full=width 240, compact=width 160） |
| `class` | `string` | `''` | 追加CSSクラス |

## 5. タイポグラフィ

### ロゴタイプフォント

ロゴ内のテキスト「がんばりクエスト」には以下のフォントスタックを使用:

```
'M PLUS Rounded 1c', 'Rounded Mplus 1c', 'Hiragino Maru Gothic Pro', 'BIZ UDPGothic', sans-serif
```

- **推奨**: `M PLUS Rounded 1c`（Google Fonts、全プラットフォーム対応）
- SVG 内で `@import url('https://fonts.googleapis.com/css2?family=M+PLUS+Rounded+1c:wght@700;400')` を使用
- 丸ゴシック系で、子供向けの柔らかい印象を与える

### アプリ内フォント

UI設計書のデザイントークンに準拠（`app.css` で定義）。

## 7. アイコン再生成手順

`static/favicon.svg` を更新した場合、以下のコマンドでPNGアイコンを再生成する:

```bash
node scripts/generate-icons.cjs
```

生成されるファイル:
- `static/favicon-32x32.png`
- `static/icons/icon-192.png`
- `static/icons/icon-512.png`
- `static/icons/icon-maskable-512.png`
- `static/icons/apple-touch-icon.png`
