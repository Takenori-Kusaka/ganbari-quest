## 1. 企画概要

### サービスコンセプト

子供の日常活動（運動・勉強・お手伝い・コミュニケーション等）を RPG 風ゲーミフィケーションで動機づけし、成長を可視化する家庭向け Web アプリケーション。

### 一言でいうと

**「子供の"がんばり"を冒険に変える、家族だけのクエストボード」**

活動を記録するとポイントが貯まり、レベルが上がり、シールやキャラクターが成長する。親は子供の成長を客観的に把握でき、子供は自分の努力が形になる達成感を得られる。

### ビジョン（34-V2MOM より）

> **子供が自分から がんばりたくなる — 日本一やさしい 子育てゲーミフィケーションアプリ**

### Values（価値観）

1. **子供ファースト** — 全ての機能は子供の成長と楽しさを最優先に設計する
2. **親の負担ゼロ** — アプリの管理に時間を取らせない。設定は最小限、自動化を最大化
3. **ポジティブのみ** — 罰則・減点は一切なし。がんばりを認め、褒め、励ます
4. **活動の動機づけ以外は有害** — アプリ内滞在時間の最大化ではなく、現実世界の活動を促す
5. **シンプルに保つ** — 機能を増やすより、コア体験を磨く

## 2. 背景・課題

### 家庭内の課題

| 課題 | 詳細 |
|------|------|
| 動機づけの難しさ | 成長に繋がる活動は「楽しくない」と感じがち。楽なことに逃げたくなる気持ちは自然 |
| 成果の見えにくさ | 日々の小さな積み重ねが、どれだけ価値があるか子供自身が実感しづらい |
| お小遣い管理 | 年齢に応じた適切な報酬基準が不明確。単なる定額制では動機づけに繋がらない |
| 記録の継続性 | 紙やホワイトボードでの管理は親の負担が大きく、継続しにくい |

### 既存手段の問題点

| 手段 | 問題点 |
|------|--------|
| 紙のシール帳 | 振り返りが困難、定量化できない、紛失リスク |
| LINE メモ | 子供が直接操作できない、ゲーム性がない |
| 既存お手伝いアプリ | 機能偏重（お手伝いのみ or 金融のみ）で、運動・勉強を含む横断管理ができない |
| 口約束 | 記録が残らない、基準が曖昧、一貫性が保てない |

詳細な競合分析: [competitive_analysis.md](competitive_analysis.md)

### ユーザーが得られるメリット

| 対象 | メリット |
|------|----------|
| 子供 | 自分の成長を「冒険」として楽しめる。がんばりが目に見える形で評価される |
| 子供 | シール・レベル・実績で達成感を獲得。お小遣いという実利的な報酬 |
| 親 | 子供の活動を定量的に把握できる |
| 親 | お小遣いの基準を客観的に設定できる |

### Before / After

| 観点 | Before | After |
|------|--------|-------|
| 動機づけ | 「やりなさい」と言うだけ | 子供が自発的に「記録したい！」 |
| 成果の実感 | 漠然とした褒め言葉 | レベル・シール・キャラクターで視覚化 |
| お小遣い | 定額 or 曖昧な基準 | 活動量に応じた歩合制（1P＝1 円） |

## 5. サービス概要

### 提供形態（デュアルモデル）

本プロジェクトは、同一のコードベースを **2 つの形態** で提供する:

| 形態 | 対象 | 配布 | 運用 |
|------|------|------|------|
| **クラウド SaaS 版** | P1（一般家庭） | `ganbari-quest.com` | AWS サーバレスで運用、Stripe サブスクリプション課金 |
| **OSS セルフホスト版** | P2（エンジニア） | GitHub リポジトリ | ユーザー自身が Docker / NUC 等に展開、全機能無料 |

両形態は同一コードから分岐なくビルドされ、セルフホスト版もクラウド SaaS 版と同等の機能を利用できる（[19-プライシング戦略書.md](19-プライシング戦略書.md) §1.1 「セルフホスト版は全機能無料」原則）。

### アーキテクチャ概要（クラウド SaaS 版）

```
[ユーザー]
    │
    ▼
[CloudFront CDN] ─── キャッシュ ─── [Lambda (SvelteKit SSR, Docker)]
    │                                    │
    │                                    ├── [DynamoDB (シングルテーブル)]
    │                                    ├── [S3 (アバター・バックアップ)]
    │                                    ├── [Cognito (Google OAuth)]
    │                                    └── [Stripe (サブスク課金)]
    │
[Route 53] → CloudFront（ganbari-quest.com + ACM 証明書）
    │
[SES] → 問い合わせ受信パイプライン
```

- **フロント＋バックエンド**: SvelteKit 2 + Svelte 5（Runes）+ Ark UI Svelte（[24-ソフトウェアアーキテクチャ設計書.md](24-ソフトウェアアーキテクチャ設計書.md)）
- **インフラ**: AWS Lambda（Docker 実行、Lambda Web Adapter）+ DynamoDB シングルテーブル + Cognito + S3 + CloudFront（[13-AWSサーバレスアーキテクチャ設計書.md](13-AWSサーバレスアーキテクチャ設計書.md)）
- **認証**: Cognito User Pool + Google OAuth（ADR-0013）
- **DB 設計**: DynamoDB シングルテーブル（ADR-0012）、`TENANT#...` パーティションで家族単位に分離
- **課金**: Stripe Checkout / Customer Portal / Webhook によるサブスクリプション管理
- **デプロイ**: GitHub Actions → ECR → Lambda（CDK）

詳細は [13-AWSサーバレスアーキテクチャ設計書.md](13-AWSサーバレスアーキテクチャ設計書.md) および [25-デプロイパイプライン設計書.md](25-デプロイパイプライン設計書.md) を参照。

### 主要機能一覧

| # | 機能 | 対象 | 概要 |
|---|------|------|------|
| F1 | 活動記録 | 子供 | 2 タップで活動を記録。アイコンベース UI |
| F2 | 活動履歴 | 子供 | 今週／今月／今年の活動を振り返り |
| F3 | ステータス表示 | 子供 | レベル、ステータス値、キャラクター表示 |
| F4 | シール／実績獲得 | 子供 | 活動に応じてシール（N/R/SR/UR）と実績をゲット |
| F5 | 週次評価 | 自動 | 週次で活動を評価しポイント付与 |
| F6 | 活動管理 | 親 | 活動の追加・編集・表示／非表示 |
| F7 | ポイント変換 | 親 | ポイント → お小遣い変換（500P 単位） |
| F8 | 年齢モード切替 | 親 | 5 段階の年齢帯（baby / preschool / elementary / junior / senior）でUI 最適化 |
| F9 | キャラクター生成 | 自動 | Gemini API でステータスに応じたキャラ生成 |
| F10 | サブスク管理 | 親 | Stripe Customer Portal 連携でプラン変更・解約 |
| F11 | チュートリアル／デモ | 親 | 未登録でも体験できるデモモード |

詳細な機能仕様は [06-UI設計書.md](06-UI設計書.md) / [07-API設計書.md](07-API設計書.md) / [26-ゲーミフィケーション設計書.md](26-ゲーミフィケーション設計書.md) を参照。

## 7. 競合・代替手段

### 市場の空白地帯

競合 15 サービスの調査の結果、以下の組み合わせを持つサービスは **一つも存在しない**:

- お手伝い・勉強・運動を横断的に管理
- RPG 型ゲーミフィケーション（レベルアップ、シール、キャラクター成長）
- 5 段階の年齢帯対応 UI（3〜15 歳）
- デュアルモデル（クラウド SaaS + OSS セルフホスト）

### がんばりクエストの差別化

| 差別化要素 | 詳細 |
|-----------|------|
| **デュアルモデル** | クラウド SaaS とセルフホスト OSS を同一コードで両立。P1（一般家庭）と P2（エンジニア）の両方にリーチ |
| **深いゲーミフィケーション** | レベル、シール、実績、キャラクター。単なるポイント制ではない |
| **5 段階年齢 UI** | 3〜15 歳まで同じアプリで成長と共に進化する |
| **運動〜勉強の横断評価** | お手伝いだけでなく全活動を統合して評価 |
| **ワンコイン以下** | スタンダード ¥500 /月、ファミリー ¥780 /月 |
| **プライバシー配慮** | Cognito + DynamoDB、COPPA 対応方針に準拠（[29-COPPA対応方針書.md](29-COPPA対応方針書.md)） |
| **OSS 版は全機能無料** | セルフホスト版はプラン制限なし |

詳細な競合分析: [competitive_analysis.md](competitive_analysis.md)
