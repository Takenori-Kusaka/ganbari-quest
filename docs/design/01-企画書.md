## 1. 企画概要

### サービスコンセプト

子供の日常活動（運動・勉強・お手伝い・コミュニケーション等）をゲーミフィケーションで動機付けし、成長を可視化する家庭内専用Webアプリケーション。

### 一言でいうと

**「子供の"がんばり"を冒険に変える、家族だけのクエストボード」**

活動を記録するとポイントが貯まり、レベルが上がり、キャラクターが成長する。自分の成長を楽しみながら、1ポイント＝1円のお小遣いにもなる。

---

## 2. 背景・課題

### 家庭内の課題

| 課題 | 詳細 |
|------|------|
| 動機付けの難しさ | 成長に繋がる活動は「楽しくない」と感じがち。楽なことに逃げたくなる気持ちは自然 |
| 成果の見えにくさ | 日々の小さな積み重ねが、どれだけ価値があるか子供自身が実感しづらい |
| お小遣い管理 | 年齢に応じた適切な報酬基準が不明確。単なる定額制では動機付けに繋がらない |
| 記録の継続性 | 紙やホワイトボードでの管理は親の負担が大きく、継続しにくい |

### 既存手段の問題点

| 手段 | 問題点 |
|------|--------|
| 紙のシール帳 | 振り返りが困難、定量化できない、紛失リスク |
| LINEメモ | 子供が直接操作できない、ゲーム性がない |
| 既存お手伝いアプリ | 月額課金、個人情報のクラウド送信、4歳児向けUIでない |
| 口約束 | 記録が残らない、基準が曖昧、一貫性が保てない |

## 3. ターゲットユーザー

### 想定家族構成

- 父（35歳、IT技術者）：アプリの開発・管理・メンテナンス担当
- 母：子供と一緒に活動を記録、管理画面での操作
- 娘（4歳、保育園児）：メインユーザー。文字がまだ読めない
- 息子（0歳、乳児）：当面は対象外。成長に合わせて段階的に追加

### ペルソナ：お嬢様（4歳）

- 保育園に通う活発な女の子
- お絵描きとごっこ遊びが好き
- ひらがなは一部読めるが、まだ文字に頼ったUIは使えない
- 「できた！」という達成感に大きく反応する
- 色鮮やかで可愛いキャラクターが好き

### 利用シーン

| シーン | 詳細 |
|--------|------|
| 朝の支度後 | 「着替えできた！」→ アプリで記録 → ポイントゲット |
| 保育園帰宅後 | 「今日は友達と遊んだ！」→ コミュニケーション記録 |
| お手伝い後 | 「食器を運んだ！」→ お手伝い記録 → レベルアップ演出 |
| 週末 | 今週のがんばりを振り返り、ステータス確認。親と一緒に成長を実感 |
| 月末 | 貯まったポイントをお小遣いに変換 |

## 4. 提供価値・解決策

### ユーザーが得られるメリット

| 対象 | メリット |
|------|----------|
| 子供 | 自分の成長を「冒険」として楽しめる。頑張りが目に見える形で評価される |
| 子供 | 一般市場と比較して「自分がどれくらいすごいか」を実感できる |
| 子供 | ポイントが貯まる楽しさ。お小遣いという実利的な報酬 |
| 親 | 子供の活動を定量的に把握できる |
| 親 | お小遣いの基準を客観的に設定できる |
| 親 | 子供の成長記録として長期的に活用できる |

### Before / After

| 観点 | Before | After |
|------|--------|-------|
| 動機付け | 「やりなさい」と言うだけ | 子供が自発的に「記録したい！」 |
| 成果の実感 | 漠然とした褒め言葉 | レベル・ステータス・キャラクターで視覚化 |
| お小遣い | 定額 or 曖昧な基準 | 活動量に応じた歩合制（1P=1円） |
| 成長の記録 | 記録なし or 紙ベース | デジタルで蓄積、振り返り可能 |
| 市場比較 | 「普通」の基準がわからない | 同年齢の一般平均と客観的に比較 |

### 主要機能一覧

| # | 機能 | 対象 | 概要 |
|---|------|------|------|
| F1 | 活動記録 | 子供 | 2タップで活動を記録。アイコンベースUI |
| F2 | 活動履歴 | 子供 | 今週/今月/今年の活動を振り返り |
| F3 | ステータス表示 | 子供 | レベル、ステータス値、キャラクター表示 |
| F4 | 市場比較 | 子供/親 | 一般市場の標準偏差と比較 |
| F5 | 週次評価 | 自動 | 週次で活動を評価しポイント付与 |
| F6 | 活動管理 | 親 | 活動の追加・編集・表示/非表示 |
| F7 | ポイント変換 | 親 | ポイント→お小遣い変換（500P単位） |
| F8 | 年齢別自動非表示 | 自動 | 年齢に適さない活動を自動非表示 |
| F9 | ステータス減少 | 自動 | 未活動期間に応じたステータス減少 |
| F10 | キャラクター生成 | 自動 | Gemini APIでステータスに応じたキャラ生成 |

## 6. KPI・成功指標

| KPI | 目標 | 測定方法 |
|-----|------|----------|
| 日次アクティブ記録 | 1日1件以上の活動記録 | activity_logs テーブル |
| 週次継続率 | 4週連続で記録がある | 週次集計 |
| ポイント変換率 | 月1回以上のポイント変換 | points テーブル |
| ステータス成長率 | 月次でステータスが上昇傾向 | statuses テーブル |
| 子供の自発性 | 親に言われずに自分で記録する頻度 | 定性評価（親の所感） |

### 段階的な成功定義

| フェーズ | 期間 | 成功基準 |
|----------|------|----------|
| MVP | 1ヶ月 | お嬢様が毎日1回以上アプリを開く |
| 定着 | 3ヶ月 | 週5日以上の活動記録が習慣化 |
| 成長 | 6ヶ月 | ステータスに明確な上昇トレンド |
| 拡張 | 1年 | 坊ちゃまも参加開始 |

### 市場の空白地帯

競合15サービスの調査の結果、以下の組み合わせを持つサービスは**一つも存在しない**:

- お手伝い・勉強・運動を横断的に管理
- RPG型ゲーミフィケーション（レベルアップ、キャラクター成長）
- 統計ベース（偏差値）の客観的成長評価
- プライバシー重視のローカル完結型

> 詳細な競合分析レポート: docs/design/competitive_analysis.md

### がんばりクエストの差別化

| 差別化要素 | 詳細 |
|-----------|------|
| **完全プライベート** | 家庭内LANのみ。個人情報をクラウドに送信しない |
| **市場比較ステータス** | 一般市場の標準偏差と比較する独自のステータスシステム |
| **深いゲーミフィケーション** | レベル、キャラクター、ヒーロー演出。単なるポイント制でない |
| **4歳児対応UI** | アイコン中心で文字に頼らない設計 |
| **運動〜勉強の総合評価** | お手伝いだけでなく全活動を横断的に評価 |
| **ステータス減少メカニクス** | 「やらないと下がる」という自然な動機付け |
| **無料・無課金** | 自家製のため月額費用なし |
| **データ冗長化** | SQLite + Git による安全なデータ管理 |

## 10. リスク・懸念事項と対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 4歳児がアプリに興味を持たない | 高 | MVP段階で娘と一緒にテスト。フィードバック反映 |
| 市場比較データの不足 | 中 | 段階的にデータ蓄積。初期はハードコード値でスタート |
| ステータス減少が厳しすぎてモチベ低下 | 中 | 減少係数を調整可能に設計。親が介入できる仕組み |
| NUCサーバーの障害 | 低 | Git による自動バックアップ。SQLiteファイルの定期コピー |
| Gemini API の仕様変更・障害 | 低 | フォールバック画像を用意。API非依存で基本機能は動作 |

## 11. まとめ・意思決定事項

### 確定事項

- サービス名: がんばりクエスト
- 技術スタック: SvelteKit 2 + Svelte 5 + Ark UI + SQLite + Drizzle ORM
- デプロイ先: NUCサーバー（家庭内LAN限定）
- 認証: 親の管理画面のみPINコード認証
- 対象ユーザー: お嬢様（4歳）がメイン。坊ちゃま（0歳）は将来対応
- ポイント: 1ポイント＝1円、500ポイント単位で変換

### 今後検討が必要な事項

- 市場比較ロジックの具体的なアルゴリズム（ブレスト実施予定）
- ステータス減少の係数設計（学術調査が必要）
- 初期活動リストの策定
- キャラクターデザインの方向性
