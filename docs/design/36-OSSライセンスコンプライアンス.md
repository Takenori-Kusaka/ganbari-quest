## 1. 概要

がんばりクエストが使用するオープンソースソフトウェアのライセンスコンプライアンス方針を定義する。

---

## 2. ライセンス分類

### 2.1 許容ライセンス

| ライセンス | 商用利用 | 条件 |
|-----------|---------|------|
| MIT | ✅ | 著作権表示の保持 |
| Apache 2.0 | ✅ | 著作権表示 + 変更点の明示 |
| BSD (2-clause, 3-clause) | ✅ | 著作権表示の保持 |
| ISC | ✅ | 著作権表示の保持 |
| CC0 / Unlicense | ✅ | なし |

### 2.2 注意が必要なライセンス

| ライセンス | 商用利用 | 注意点 |
|-----------|---------|--------|
| LGPL | ✅（条件付き） | ライブラリとしてリンクする場合は OK、改変した場合はソース公開 |
| MPL 2.0 | ✅（条件付き） | 改変ファイルのソース公開が必要 |

### 2.3 禁止ライセンス

| ライセンス | 理由 |
|-----------|------|
| GPL (v2, v3) | 派生物のソース公開義務 |
| AGPL | ネットワーク越しの利用でもソース公開義務 |
| SSPL | 商用 SaaS での利用制限 |
| BSL (Business Source License) | 商用利用制限の可能性 |

---

## 3. 監査方法

### 3.1 定期監査

```bash
# 依存パッケージのライセンス一覧を取得
npx license-checker --summary
npx license-checker --csv > docs/licenses.csv
```

### 3.2 CI での自動チェック

`dependency-review.yml` ワークフローで PR 時に新規依存パッケージのライセンスを自動検証。

### 3.3 ライセンス表示

`/licenses` ページまたは LP のフッターに、使用している OSS のライセンス一覧を表示（将来対応）。

## 5. 新規パッケージ追加時のルール

1. `npm install` 前にライセンスを確認
2. 禁止ライセンスのパッケージは導入しない
3. LGPL/MPL のパッケージは導入前にレビュー
4. PR の dependency-review で自動チェックされることを確認
