## 1. 概要

がんばりクエストは3-15歳の子供が主要ユーザーであり、COPPA (Children's Online Privacy Protection Act) および日本の個人情報保護法における子供データ保護への対応方針を定義する。

### 1.1 関連設計書

| 設計書 | 関連箇所 |
|--------|---------|
| 14-セキュリティ設計書 | 認証・認可全般 |
| 17a-データ保護影響評価書 | DPIA |
| 30-同意管理設計書 | 保護者同意フロー |

## 2. 適用範囲

### 2.1 対象データ

| データ種別 | 収集有無 | 管理者 | 保存先 |
|-----------|---------|--------|--------|
| 子供の名前（ニックネーム） | あり | 保護者が入力 | DB |
| 子供の生年月日 | あり | 保護者が入力 | DB |
| 子供のアバター設定 | あり | 子供が選択 | DB |
| 活動記録（ポイント・レベル等） | あり | 子供の操作で生成 | DB |
| メールアドレス | なし | - | - |
| 位置情報 | なし | - | - |
| 写真・動画 | なし | - | - |
| デバイス識別子 | なし | - | - |

### 2.2 非収集データ（意図的に除外）

- 子供のメールアドレスは収集しない
- 位置情報は収集しない
- カメラ・マイクへのアクセスは行わない
- 広告トラッキングは行わない
- 第三者への子供データの提供・販売は行わない

---

## 3. 保護者同意

### 3.1 同意取得フロー

1. **アカウント作成は保護者のみ** — 子供はアカウントを作成できない
2. **子供プロフィール作成は保護者操作** — 管理画面から保護者が子供情報を入力
3. **利用規約・プライバシーポリシーへの同意** — サインアップ時に保護者が明示的に同意
4. **子供データの閲覧・削除は保護者のみ** — 管理画面から操作可能

### 3.2 保護者の権利

| 権利 | 実装 |
|------|------|
| 子供データの閲覧 | 管理画面 → 子供プロフィール |
| 子供データの修正 | 管理画面 → 子供編集 |
| 子供データの削除 | 管理画面 → 子供削除 |
| アカウント全体の削除 | 管理画面 → アカウント削除 |
| データエクスポート | 管理画面 → エクスポート機能 |
| 同意の撤回 | アカウント削除で全データ消去 |

---

## 4. データ最小化原則

### 4.1 収集するデータの正当性

| データ | 収集理由 | 最小化対策 |
|-------|---------|----------|
| 名前（ニックネーム） | 画面表示・報酬パーソナライズ | 本名不要、ニックネームで可 |
| 生年月日 | 年齢モード自動選択・誕生日機能 | 年齢のみでも機能する |
| 活動記録 | サービスのコア機能 | 最小限のメタデータのみ |

### 4.2 データ保持期間

| データ | 保持期間 | 削除方法 |
|-------|---------|---------|
| 子供プロフィール | アカウント存続中 | 子供削除 or アカウント削除 |
| 活動記録 | アカウント存続中 | アカウント削除で一括消去 |
| バックアップ | 30日 | 自動期限切れ |

---

## 5. 技術的保護措置

### 5.1 アクセス制御

- 子供データへのアクセスはテナント ID で分離（マルチテナント）
- 子供は他テナントのデータにアクセス不可
- 管理画面は保護者のみ（PIN / Cognito 認証）

### 5.2 暗号化

- 通信: TLS 1.2+ (CloudFront)
- 保存: DynamoDB サーバーサイド暗号化 (AES-256)
- SQLite: ファイルシステムレベル（LAN 内限定）

### 5.3 ログ

- 子供の個人情報をログに出力しない
- アクセスログにはテナント ID のみ記録（子供名は含まない）

---

## 6. 第三者サービス

| サービス | 子供データへのアクセス | 対策 |
|---------|-------------------|------|
| AWS (DynamoDB, S3) | データ保存先 | サーバーサイド暗号化、東京リージョン |
| Stripe | なし | 保護者の課金情報のみ |
| Google OAuth | なし | 保護者のメールアドレスのみ |
| SES | なし | 保護者へのメール送信のみ |

---

## 7. インシデント対応

子供データの漏洩・不正アクセスが発生した場合:

1. 即時にアクセス遮断
2. 72時間以内に影響範囲を特定
3. 影響を受けた保護者に通知
4. 必要に応じて個人情報保護委員会に報告
5. ADR としてインシデントレポートを記録
