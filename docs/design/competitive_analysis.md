# 競合サービス調査レポート

調査日: 2026-02-19

## 調査目的

「がんばりクエスト」の企画にあたり、子供の活動をゲーミフィケーションで動機付けするアプリ（お手伝い・勉強・運動の記録、ポイント制お小遣い管理）の競合サービスを調査する。

---

## 1. 日本国内の競合サービス

### 1.1 ファミポイ（FamiPoi）

| 項目 | 内容 |
|------|------|
| **開発元** | AppSeed Inc. |
| **対象** | 幼児～小学生 |
| **料金** | 無料 |
| **プラットフォーム** | iOS / Android |

**概要**: 子どものお手伝いや自立を応援するポイント管理アプリ。お手伝い完了時にポイントを付与し、貯めたポイントをご褒美と交換できる。

**主な機能**:
- 毎日やることリストの追加・編集・削除
- 達成項目ごとのポイント付与（毎日リセット）
- 今日の獲得ポイント・達成率表示
- 合計ポイント表示
- ポイント交換アイテムリストの管理
- 親による承認フロー

**がんばりクエストとの比較**:
- ポイント制のみでRPG的ゲーミフィケーション要素なし
- 勉強・運動の記録機能なし（お手伝い特化）
- 標準偏差との比較機能なし
- クラウド型（プライバシー面で差別化可能）

## 4. 市場分析と洞察

### 4.1 市場の傾向

1. **金融教育偏重**: 海外アプリの大半は「子供向け銀行・デビットカード」をコアとし、お手伝いはお金を稼ぐ手段として位置付けている。ゲーミフィケーションは副次的。

2. **日本市場の空白**: 日本のアプリは「お手伝いポイント管理」か「お小遣い帳」に二分化されており、本格的なRPG型ゲーミフィケーションを備えたアプリはほぼ存在しない。

3. **勉強・運動の統合管理は未開拓**: お手伝い・勉強・運動を横断的に記録・評価するアプリは調査範囲内で見当たらない。

4. **プライバシー意識の高まり**: 特に幼児の活動データをクラウドに送ることへの懸念は高まっているが、ローカル完結型のアプリは存在しない。

### 4.2 がんばりクエストの差別化要因の有効性

| 差別化ポイント | 競合状況 | 有効性 |
|----------------|----------|--------|
| **家庭内LANのみ（プライバシー重視）** | 全競合がクラウド型。ローカル完結型は皆無 | **極めて高い** - 唯一無二のポジション |
| **標準偏差ベースのステータスシステム** | 客観的な統計比較を提供する競合なし | **極めて高い** - 完全に独自の機能 |
| **RPGゲーミフィケーション（レベルアップ・キャラクター獲得）** | ChoreMonster（モンスター収集）、Levelty（アバター・コイン）が部分的に類似。ひばクエストがRPG型だが子供向けUIなし | **高い** - 子供専用RPGとしては差別化可能 |
| **4歳児対応のアイコンUI** | Cheermee（3歳～）、Levelty（4歳～）が低年齢対応だがアイコン特化ではない | **中～高** - アイコンUI特化は差別化になる |
| **データ冗長化（ローカル+GitHub）** | 全競合がクラウド一元管理。ローカル+Git冗長化は皆無 | **極めて高い** - 技術者向けの独自アプローチ |
| **お手伝い・勉強・運動の統合管理** | 全競合がお手伝い特化または金融教育特化 | **極めて高い** - 活動横断型は未開拓 |

### 4.3 潜在的リスク

1. **対象ユーザーの技術リテラシー**: 家庭内LAN構成やGitHub連携は技術的ハードルが高く、一般家庭への展開には課題がある。ただし、対象が開発者家庭であれば問題なし。

2. **ネットワーク効果の欠如**: クラウド型アプリはランキングやコミュニティ機能で再訪率を高めるが、ローカル完結型ではこれが難しい。家庭内の完結性をメリットとして打ち出す必要がある。

3. **ChoreMonsterのモンスター収集要素**: ゲーミフィケーション面で最も近い競合。キャラクター収集の楽しさで対抗する必要がある。

---

## 5. 結論と提言

### がんばりクエストの市場ポジション

がんばりクエストは以下の点で、調査した全競合と明確に差別化されている:

1. **唯一のローカル完結型アプリ** - プライバシー重視の家庭に対する独占的ポジション
2. **唯一の統計ベースステータスシステム** - 子供の成長を客観的に可視化
3. **唯一のお手伝い・勉強・運動統合型** - 子供の活動を包括的にカバー
4. **RPGゲーミフィケーション + 低年齢UI** - この組み合わせを実現している競合なし

### 最も注視すべき競合

1. **ChoreMonster** - ゲーミフィケーション（モンスター収集）面で最も類似
2. **Levelty** - 対象年齢（4歳～）とゲーム要素の組み合わせが近い
3. **ひばクエスト** - RPG型ゲーミフィケーションのコンセプトが類似（ただし子供向けではない）
4. **チアミー** - 低年齢対応と習慣化支援で部分的に競合

### 提言

- がんばりクエストの差別化要因は市場で非常に強い。特に「プライバシー重視」「統計ベースの成長可視化」「活動統合」の3点は、競合が全くカバーしていない領域である。
- 開発者家庭をコアターゲットとし、技術的な構成をメリットとして訴求する戦略は妥当。
- 将来的にユーザー層を広げる場合は、セットアップの簡素化（ワンクリックインストーラー等）を検討する価値がある。
