## §1 設計背景（Why）

### 1.1 現状の課題

がんばりクエストは Pre-PMF 段階にあり、初期の親契約 ~10 件を獲得して PMF 仮説を検証する局面にある。
PO（個人開発者・営業リソース極小）には次の 4 つの獲得経路候補があるが、いずれも具体施策が描けていない。

| 経路 | 略号 | PO 自己評価 |
|------|------|------------|
| SNS（Twitter/X / Instagram） | A | 補助。アカウントは存在するがフォロワーゼロ近似 |
| 育児メディア（コノビー / Tarzan / リンネル ベビー / ママリ 等） | B | **コネクションがない**。営業優先度高だが現実的施策が描けていない |
| ママ友コミュニティ（mixi / LINE オープンチャット / Facebook Group / 学校 PTA 等） | C | 押し売り感とブランド毀損のバランスが不明 |
| Google 検索 SEO（オーガニック） | E | LP は存在するが「HP アクセスゼロ」が現状。即実行できる施策の優先度が不明 |

PO の営業優先度は **B > C > E > A**（C-Q12 PO 回答）。本書はこの優先度の妥当性を Discovery（調査・仮説立案）で検証し、後続 Issue として実行可能な単位に切り出す。

### 1.2 制約条件

- **個人開発予算**: 月次マーケティング予算は数千円〜数万円規模を上限と想定。CAC 高チャネル（Google / Meta 広告）は PMF 後に明確に区切る。
- **時間リソース**: 開発・運用と兼務のため、**自動化・低タッチ運用**が前提。週 2-3 時間以下で運用できない施策は Pre-PMF では採用しない。
- **ADR-0010 (Pre-PMF) 整合**: 過剰防衛・スケール先行投資は禁止。「初期 ~10 親契約まで全員と直接対話」（ADR-0023 §3.7）が前提。
- **ADR-0012 (Anti-engagement) 整合**: 子供側 UI の滞在時間最適化は禁忌。獲得文言も「依存させる」訴求は不採用。販促コピーも同基準で審査する。
- **ADR-0013 (LP は実装真実から)**: 未実装機能を獲得文言に書かない。「やりたいこと」と「動くこと」を区別する。

### 1.3 本書の目的

- 各経路（B / C / E / A）について、**Pre-PMF 段階で実行可能な現実的施策**をコスト・期待 ROI・工数の 3 軸で比較する
- 即時着手 / PMF 後検討 / 不採用 の **3 段階優先度**を確定する
- 後続の **実行 Issue 起票テンプレート**を用意する

## §2 設計原則（How）

### 2.1 経路評価の 5 軸

各チャネル候補は次の 5 軸で評価する。

| 軸 | 評価方法 | 重要度 |
|----|---------|------|
| リーチ規模 | S（数十〜数百）/ M（数百〜数千）/ L（数千以上） | 中 |
| CAC（顧客獲得単価） | 低（< ¥500）/ 中（¥500〜¥3000）/ 高（> ¥3000） | 高 |
| 個人開発実現可能性 | ○（自分の手で完結）/ △（半外注 or 関係構築必要）/ ×（スケール不可） | 高 |
| 期待 LTV 親和性 | 子育て層への適合度（高 / 中 / 低） | 中 |
| ブランド毀損リスク | 押し売り・依存誘発・COPPA 等の倫理リスク（低 / 中 / 高） | 高（ADR-0012 整合） |

### 2.2 Pre-PMF 優先度の 3 段階

| 段階 | 採用基準 | 例 |
|------|---------|------|
| **即時着手 (Q2 2026)** | CAC 低 + 個人開発実現可能性 ○ + ブランド毀損リスク低 | SEO 改善、founder 直発信、LINE 公式 |
| **PMF 後検討 (Q3-Q4 2026 以降)** | リーチ大だが CAC 中以上、関係構築コスト中以上 | 育児メディア記事広告、SNS インフルエンサー、Google/Meta 広告 |
| **不採用** | 個人開発スケール不可（実現性 ×） | PTA 一斉配布、TV 広告、大規模ブース出展 |

### 2.3 ブランド毀損リスクの境界線（C 経路特有）

ママ友コミュニティ活用には「押し売り感」のリスクがある。次の境界線を守る。

- **OK**: 自分の子育てエピソードの自然な共有 → 興味を持たれた時のみ案内
- **OK**: コミュニティ運営者へ「事例提供してもよいですか」と打診 → 承諾を得てから記事化
- **NG**: 無差別な紹介リンク投稿 / DM 営業 / 連投宣伝
- **NG**: 「実体験」を装った第三者アカウントでの投稿（ステマ規制違反）

### 2.4 計測（KPI）

| 指標 | データソース | 評価頻度 |
|------|------------|------|
| 訪問 → signup CVR | Google Search Console + DynamoDB analytics（#1591） | 月次 |
| signup → 7 日継続 | 既存 Cron（#1601 lifecycle-emails） | 月次 |
| LP 構造化データ / OG | `scripts/measure-lp-dimensions.mjs` | PR 毎 |
| 経路別 signup 流入 | UTM パラメータ + DynamoDB analytics | 月次 |

**B 経路の即時着手アクション**:
1. **PR Times 1 回配信を試行**（¥30,000）— ローンチ時 / 大型機能追加時の節目で。配信文面は ADR-0013 整合（実装真実）で記述。
2. **こどもまなび☆ラボ等の知育メディアへ取材依頼**（無料）— 「個人開発者の Pre-PMF プロダクト紹介」企画として打診。
3. **子育てブロガー 5 人にレビュー依頼**（サンプル提供）— note / はてな で「子供 ご褒美 アプリ」を継続発信しているブロガーをリスト化。

**B 経路の PMF 後検討アクション**:
- コノビー / ママリ / Tarzan の純広告 / 記事広告
- リンネル ベビー / OZmall タイアップ

#### C 経路: ママ友コミュニティ

| コミュニティ | 規模 | 活用方法 | ブランド毀損リスク | 推奨度 |
|------------|------|--------|-----------------|------|
| LINE オープンチャット（子育て系） | 100〜数千人 / 部屋 | 自然な雑談で言及 / 運営者と関係構築後の事例提供 | 中（押し売り NG） | **即時着手（自然参加）** |
| mixi コミュニティ | 数百〜数千人 / コミュ | 同上。アクティブ度は LINE より低い | 中 | 補助 |
| Facebook Group（子育て / 知育系） | 数百〜数万人 | 同上。ピン投稿が運営者承認制 | 中 | 補助 |
| ママリコミュニティ | 大規模 | 質問への回答経由で自然紹介 | 中 | **即時着手（Q&A 回答）** |
| 学校 PTA / 習い事教室 | 数十人 / 単位 | PO 自身の子供の関係から自然紹介 | 低（顔が見える関係） | **即時着手（founder ネットワーク）** |
| 子育てサークル（地域） | 数十人 / 単位 | 同上 | 低 | 即時着手 |

**C 経路の即時着手アクション**:
1. **PO 自身が子育て系 LINE オープンチャットに参加**（自分の子育てメンバーとして）— 押し売り禁止、雑談中で自然に言及できる関係性を作る
2. **ママリで「子供のやる気」「お手伝い」関連質問への回答**（月 5-10 件） — 回答末尾に「自分はこういうの作ってます」と一行リンク（ステマ規制整合）
3. **PO の子供の習い事 / PTA 関係への founder 直接紹介**（#1594 「直接相談」CTA 動線と連動）

**C 経路の制約（ADR-0012 整合）**:
- 投稿は「依存させる訴求」ではなく「親の声かけ負担を減らす」訴求に統一
- 紹介報酬制度（口コミ煽り）は不採用 — Pre-PMF では「自然紹介のみ」

**C 経路の PMF 後検討アクション**:
- 紹介リンク機能の導入（#1594 founder 動線が安定後）
- 専用 Discord サーバー / コミュニティ運用（既存 `23-Discordサーバー設計書.md` 拡張）

#### E 経路: Google 検索 SEO（オーガニック）

既存の `39-SEO戦略書.md` が技術 SEO（meta / OGP / sitemap.xml / robots.txt / 構造化データ）を定義済み。本書はその上に乗る **コンテンツ SEO + 短期施策**を追加する。

| 施策 | 概要 | 工数 | 期待効果 | 推奨度 |
|------|------|------|---------|------|
| LP の meta description / h1 / structured data 見直し | `39-SEO戦略書.md` 既存施策の点検 | 4-8h | LP CTR 改善 | **即時着手** |
| 主要キーワード CTR 改善（「子供 ご褒美 アプリ」「お手伝い 記録 アプリ」「シール帳 デジタル」） | title / description のキーワード明示 | 2-4h | 検索順位上昇 | **即時着手** |
| ブログ記事 5-10 本（note / Zenn / 自社 LP の `/blog`） | 「子育て × ゲーミフィケーション」「親の声かけ疲れ」「習慣化の科学」等 | 各 2-4h × 10 = 20-40h | 累積流入 | **即時着手（月 2 本ペース）** |
| FAQ ページ拡充 | LP の FAQ を構造化データ（FAQ schema）化 | 4h | リッチリザルト表示 | **即時着手** |
| 競合 SEO 分析 | 「子供 ご褒美 アプリ」上位 10 サイトの構造分析 | 4h | キーワード戦略改善 | **即時着手** |
| Google Search Console / GA4 登録 + 計測ループ確立 | `39-SEO戦略書.md §3.1` に手順あり | 2-4h | 計測基盤 | **即時着手（前提条件）** |
| LP の Core Web Vitals 改善 | LCP / CLS / INP の点検 | 8-16h | 検索順位 | PMF 後（既存 `37-パフォーマンス基準書.md` で対応） |

**E 経路の即時着手アクション**:
1. **SEO 改善 PR**: meta description / h1 / structured data の見直し（LP 仕上げ R シリーズ #1664 と統合可能）
2. **competitor SEO 分析 Issue**: 上位 10 サイト分析
3. **blog 戦略 Issue**: 月 2 本ペースで note / Zenn 記事を配信
4. **FAQ 構造化データ Issue**: 既存 LP の FAQ セクションに FAQ schema 適用

**E 経路の PMF 後検討アクション**:
- Google 広告（リスティング）— CAC 高
- 大規模コンテンツ拡充（オウンドメディア化）
