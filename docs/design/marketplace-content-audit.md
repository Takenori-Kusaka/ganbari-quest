# マーケットプレイスコンテンツ監査レポート

> **目的**: がんばりクエストのマーケットプレイス（活動パック / ごほうび / 持ち物チェックリスト）のコンテンツ刷新方針を、文部科学省・厚生労働省・スポーツ庁・国立成育医療研究センター等の公的機関の出典に基づき定義する。
>
> **対象読者**: PO / コンテンツ設計担当 / Dev
> **作成日**: 2026-04-20
> **関連ドキュメント**: `docs/design/marketplace-persona-research.md`, `docs/design/asset-catalog.md`, `docs/DESIGN.md` §8

## 1. エグゼクティブサマリ

本監査では、マーケットプレイス収録の活動パック計 32 ファイル（`marketplace/activity-packs/` 14、`activity-packs/` 18）を、文部科学省「幼稚園教育要領」「小・中・高等学校学習指導要領」、厚生労働省「健康づくりのための睡眠ガイド 2023」、スポーツ庁「全国体力・運動能力等調査」、こども家庭庁「乳幼児身体発育調査」等を SSOT として精査した。

**主要な所見:**

1. **乳幼児期 (baby/kinder) は概ね適切**だが、5 領域（健康・人間関係・環境・言葉・表現）への明示的紐付けが弱く、特に「環境」（自然との関わり、数量・図形）と「言葉」（語彙の獲得）の活動が不足。
2. **小学生 (elementary) は教科カバレッジに偏りあり**。生活科・社会・理科・家庭科の活動が手薄で、「学習指導要領」の特別活動視点（人間関係形成・社会参画・自己実現）の活動が言語化されていない。
3. **中高生 (junior/senior) は実装が elementary と一部重複**し、「総合的な探究の時間」「キャリア教育」要素が浅い。`junior-high-challenge.json` は 10〜12 歳向けと定義しながら `gradeLevel: middle_school` を付与しており、対象とラベルが矛盾している。
4. **性別差を理由とした活動分岐は科学的根拠が薄く、ステレオタイプ強化のリスクあり**。`baby-boy.json` の「ボールであそんだ」と `baby-girl.json` の「おどってあそんだ」のような分岐は、国立成育医療研究センターの研究が示す「性別差は項目によっては存在するが、活動推奨を分けるほどの差ではない」という知見と整合しない。性別中立を基本とし、性別パックは「服装・ごっこ遊びの趣向」程度の軽い差別化に留めるべき。
5. **持ち物チェックリストのコンテンツ化が皆無**。マーケットプレイス上には季節別・行事別の持ち物テンプレートが存在せず、`#1164` で対応中の「持ち物チェックリスト LP」と整合する実装が必要。
6. **ごほうび設計は年齢相応の動機付け設計が未分化**。現状はポイント数のみで差別化されており、baby の感覚刺激 / kinder のシール・承認 / elementary の達成記録 / junior/senior の自己効力感、という発達段階別の動機構造が反映されていない。

**推奨アクション**: §6 の gap 監査に基づき、(a) 学習指導要領 5 領域 / 教科系統と紐付けたメタデータ追加、(b) 性別パックの段階的廃止または「趣向タグ」への再分類、(c) 持ち物チェックリストパックの新規追加、(d) ごほうび設計の年齢別 4 段階モデル導入、を提案する。

### 性別中立の原則

国立成育医療研究センターの研究では、運動発達の達成時期に項目別の性別差が一部見られるものの、活動の「推奨内容」自体を性別で分ける根拠とはならない。本アプリでは以下の方針を取る:

- **性別中立を基本**とし、男児/女児パックは「アイコンのバリエーション」と「ごっこ遊びの題材選択肢の例示」程度に留める
- 「ボールあそび=男児」「ダンス=女児」のような割当は廃止し、**両方の選択肢を中立パックに含める**
- 個別カスタマイズは「子供本人が選ぶ」ことを優先し、性別を理由としたデフォルト分岐は最小化

## 4. ごほうび設計指針（年齢相応の動機構造）

> 出典: 発達心理学（Erik Erikson の心理社会的発達理論）、文部科学省「ゲーミフィケーション」指針なし → 国立教育政策研究所の自己肯定感研究、スポーツ庁の運動習慣調査における内発的動機の言及

現状のごほうびは `basePoints` の数値のみで差別化されているが、以下の 4 段階モデルを導入することで、年齢相応の動機付けが実現できる。

### 4.1 baby (0-2 歳): 感覚刺激 / 共感応答

- **動機構造**: Erikson 第 1-2 段階「基本的信頼 / 自律性 vs 恥」
- **報酬設計**:
  - 親の笑顔・拍手・ハイタッチ（アプリ内では効果音 + 大きなアニメーション）
  - シール画像は使わず、**シンプルな色の変化 + 音 + 振動**
  - ポイント値は内部処理用（子供本人には見せない）
- **避けるべき**: 数値競争、他者比較、長期目標
- **実装例**: 活動完了時に画面全体が虹色に光り、「やったね！」音声が流れる

### 4.2 kinder (3-5 歳): 承認 / 物理的シール / コレクション初歩

- **動機構造**: Erikson 第 3 段階「自発性 vs 罪悪感」、外発的動機が中心
- **報酬設計**:
  - **シール画像が中核**（asset-catalog.md §シールマスタ N/R/SR/UR と整合）
  - 親や先生からの「すごいね」「がんばったね」フィードバック
  - 1 日の頑張り表 / シール台紙風 UI
  - レベル / ランクは「冒険のたまご→ひよこ→こども」程度の単純な進化
- **避けるべき**: 複雑な数値評価、長期目標、勝敗
- **実装例**: 既存のスタンプシステム（emoji 16 種）を画像化（asset-catalog 推奨）

### 4.3 elementary (6-12 歳): 達成記録 / コレクション / バッジ

- **動機構造**: Erikson 第 4 段階「勤勉性 vs 劣等感」、コレクション性が動機を最大化
- **報酬設計**:
  - **バッジ・称号の収集**（asset-catalog.md §実績バッジ・称号アイコン）
  - 連続記録（streak）の可視化、月間達成カレンダー
  - レアリティ（N/R/SR/UR）による収集欲の刺激
  - 自分の成長グラフ（過去の自分との比較）
- **注意**: 他者比較は内発的動機を損なう（自己決定理論）。**家族内ランキングのみ**とし、公開ランキングは行わない
- **実装例**: 既存のシール + 称号 + バッジを並列配置、月次サマリ

### 4.4 junior (13-15 歳): 自己効力感 / 数値目標 / 連続記録

- **動機構造**: Erikson 第 5 段階「同一性 vs 役割の混乱」、自己効力感（Bandura）が中心
- **報酬設計**:
  - **数値目標の設定と達成**（テスト勉強 30 時間 / 月、運動 420 分 / 週 等）
  - 連続記録の自己ベスト更新
  - 自分が選んだ目標に対する進捗可視化
  - 親の介入は最小化、本人の自律性を尊重
- **避けるべき**: 親からの過度な称賛（思春期は逆効果）、強制的な目標設定
- **実装例**: 「今月のチャレンジ」を本人が選ぶ、数値ダッシュボード、SNS 風シェア（家族内のみ）

### 4.5 senior (16-18 歳): 自己決定 / 探究 / 外的成果との連結

- **動機構造**: Erikson 第 5 段階継続、内発的動機への移行期
- **報酬設計**:
  - **目標を自分で立てる**（探究テーマ・受験計画・資格目標）
  - 進路・進学・資格取得などの外的成果と連動
  - アプリは「習慣形成のサポートツール」に徹する
  - ポイント・バッジは控えめ、振り返りログを重視
- **避けるべき**: 子供っぽい演出、強制感、過度なゲーミフィケーション
- **実装例**: 学習時間ログ、習慣カレンダー、目標達成までの逆算スケジュール

### 4.6 ごほうびメタデータの提案

各活動・パックの JSON に以下を追加することを推奨:

```json
{
  "rewardStyle": "sensory" | "approval" | "collection" | "self-efficacy" | "autonomy",
  "feedbackTone": "celebratory" | "warm" | "neutral" | "respectful"
}
```

これにより、UI 側で年齢に応じた演出を自動切替できる。

## 6. 既存実装の gap 監査

### 6.1 全体傾向

| パスカテゴリ | ファイル数 | 主な問題 |
|------------|-----------|---------|
| `marketplace/activity-packs/` | 14 | 性別中立。学習指導要領との明示的紐付け無し。年齢パックとテーマパックが混在 |

### 6.2 個別パックの所見

#### `marketplace/activity-packs/baby-first.json` (16 活動)

- **適切**: 5 領域に近い構造（運動・生活・学び・表現・交流）
- **不足**: 「環境」領域の自然観察（虫・植物・季節）が弱い。「言葉」の語彙獲得（指差し・物の名前）が「あいさつ」のみで不十分
- **過剰**: 「あんよした」「からだをうごかした」「おそとにでた」が undou に集中（4/16）→ 1 つに統合可
- **推奨追加**: 「しなを指さした」「もののなまえをいった」「むしをみつけた」「おはなをみつけた」

#### `marketplace/activity-packs/life-skills-baby.json` (8 活動)

- **適切**: 生活スキルに特化、トイレが含まれる（baby-first に無い）
- **不足**: スコープが狭く、`baby-first` と統合してテーマタグで切り替える方が UX 良
- **重複**: `baby-first` と「ごはん・はみがき・おきがえ・てあらい・おやすみ・おかたづけ」が重複

#### `marketplace/activity-packs/kinder-starter.json` (20 活動)

- **適切**: 5 領域全カバー。バランスが良い
- **不足**: 「環境」（自然観察・形・量）の活動が「すうじをかぞえた」のみ。「健康」のうち食育（朝ごはん・好き嫌いをへらす）が無い
- **推奨追加**: 「あさごはんをたべた」「むしや はなを みつけた」「かたちを みつけた」

#### `marketplace/activity-packs/elementary-challenge.json` (16 活動)

- **適切**: 国語・算数・体育・家庭科が含まれる
- **不足**: **理科・社会・図工・音楽の独立活動が無い**（「がっきのれんしゅう」のみ）。特別活動（委員会・係）が無い
- **過剰**: 「うんどうした」「なわとびした」「すいえいした」が並列で運動偏重
- **推奨追加**: 「理科の観察をした」「社会・地図を調べた」「図工で作品を作った」「委員会・係活動をした」「当番をした」

#### `marketplace/activity-packs/junior-high-challenge.json` (16 活動)

- **重大な矛盾**: パック名「中学生チャレンジ」、`targetAgeMin/Max: 10/12`、`gradeLevel: middle_school` がすべて異なる時期を指している
  - 10-12 歳 = 小学校高学年
  - 13-15 歳 = 中学校
  - **修正必須**: パックを 2 つに分割するか、ラベルと年齢範囲を一致させる
- **適切**: 予習復習・テスト勉強・部活・家事自立などの中学生向け活動
- **不足**: 特別活動（生徒会・委員会）、キャリア教育（職場体験・進路調べ）が無い
- **推奨追加**: 「進路・職業について調べた」「生徒会・委員会活動した」「ボランティア活動した」

#### `marketplace/activity-packs/senior-high-challenge.json` (16 活動)

- **重大な矛盾**: パック名「高校生チャレンジ」、`targetAgeMin/Max: 13/18`、`gradeLevel: high_school`
  - 13-14 歳 = 中学校なのに `high_school` ラベルは不適切
  - **修正必須**: `targetAgeMin: 15` に修正
- **適切**: 受験・自立生活・ボランティア・進路など高校生らしい活動
- **不足**: **「総合的な探究の時間」関連が無い**（テーマ設定・情報収集・発表）。これは 2022 年度から必履修
- **推奨追加**: 「探究テーマを決めた・調べた」「データを整理して発表した」「ニュースに自分の意見を持った」

#### `marketplace/activity-packs/exam-prep.json` (8 活動)

- **適切**: 受験生に特化、睡眠 7 時間以上が含まれる
- **過小**: 8 活動と少ない。「過去問分析」「弱点ノート」「集団勉強」など多様化推奨

#### `marketplace/activity-packs/study-master.json` (10 活動)

- **適切**: 教科横断的
- **不足**: 「英語」が「えいごのべんきょう」のみで、4 技能（読・書・聞・話）に分割されると指導要領的に正確
- **推奨追加**: 「えいごをきいた」「えいごではなした」

### 6.3 性別パック（`activity-packs/` 配下）の重大課題

`baby-boy/girl`, `kinder-boy/girl`, `elementary-boy/girl`, `junior-boy/girl`, `senior-boy/girl` の 10 ファイルを比較した結果:

#### 中身がほぼ同一の事例

- `baby-boy.json` vs `baby-girl.json`: 17 活動中 15 活動が同一。差分は「ボールであそんだ（男児）」vs「おどってあそんだ（女児）」、「つみき（男児）」vs「おにんぎょう（女児）」のみ
- `kinder-boy.json` vs `kinder-girl.json`: 同様にほぼ同一。差分は「かけっこ・ボール・のぼりぼう（男児）」vs「ダンス・からだをうごかした・おままごと（女児）」のみ

#### 問題点

1. **科学的根拠の欠如**: 国立成育医療研究センターの研究では、運動発達の項目別に性別差は一部存在するが、**活動を分けるほどの差ではなく**、むしろ環境（季節・月齢）の影響が大きい
2. **ステレオタイプ強化**: 「男児はボール、女児は人形・ダンス」は社会的構築物であり、子供の興味の幅を狭める
3. **保守コスト**: 中身がほぼ同一の 5 ペア（10 ファイル）を保守し続けるコストが高い
4. **インクルーシビティ**: 性別二分法は LGBTQ+ や非典型的な子供の体験を排除する

#### 推奨対応

- **段階 1（短期）**: 性別パックの差分活動（「ボール」「ダンス」「人形」「つみき」等）をすべて中立パックに統合
- **段階 2（中期）**: 性別パックを「アクティブ系」「クリエイティブ系」「自然系」「学び系」のような**趣向タグ**に再分類
- **段階 3（長期）**: 性別パック (`*-boy.json`, `*-girl.json`) を deprecation 予告付きで廃止、`gender-neutral` パックに統合

### 6.4 メタデータ統一の提案

すべての活動 JSON に以下を追加することで、検索・フィルタ・推薦の品質が向上する:

```json
{
  "name": "活動名",
  "categoryCode": "benkyou",
  "curriculumDomain": "kindergarten:5-domain:language" | "elementary:subject:japanese" | "junior:special-activity:career" | ...,
  "developmentalGoal": "literacy" | "motor-skill" | "social-skill" | "self-management" | "creativity",
  "rewardStyle": "sensory" | "approval" | "collection" | "self-efficacy" | "autonomy",
  "estimatedMinutes": 5 | 10 | 30 | 60,
  "frequency": "daily" | "weekly" | "monthly" | "occasional"
}
```

これにより、PO 視点で「elementary の言語領域カバレッジが何 % か」「kinder の 5 領域がバランス取れているか」などを定量評価できる。
