## 1. 概要

がんばりクエストの新規開発者向けオンボーディングガイド。ローカル開発環境のセットアップからコード構造の理解まで。

## 2. 前提条件

| ツール | バージョン | 用途 |
|-------|-----------|------|
| Node.js | 22.x | ランタイム |
| npm | 10.x | パッケージマネージャ |
| Git | 最新 | バージョン管理 |
| Docker | 最新 | NUC デプロイ用 |
| gh (GitHub CLI) | 最新 | Issue/PR 管理 |

---

## 3. セットアップ手順

```bash
# 1. リポジトリクローン
git clone https://github.com/Takenori-Kusaka/ganbari-quest.git
cd ganbari-quest

# 2. 依存関係インストール
npm install

# 3. 環境変数設定
cp .env.example .env
# .env の AUTH_MODE=local を確認

# 4. DB初期化
npm run db:push

# 5. 開発サーバー起動
npm run dev

# 6. ブラウザで http://localhost:5173 を開く
```

## 5. 開発フロー

```
1. Issue を確認 (gh issue view <番号>)
2. ブランチ作成 (git checkout -b feat/xxx)
3. 実装
4. テスト実行 (npx biome check . && npx svelte-check && npx vitest run && npx playwright test)
5. Draft PR 作成 (gh pr create --draft)
6. CI 通過確認
7. Ready に変更 (gh pr ready <番号>)
8. レビュー → マージ
```

---

## 6. 注意事項

- `docs/tickets/` にファイルを作らない（GitHub Issues で管理）
- routes 配下で hex カラーを直書きしない
- `$lib/ui/primitives/` にあるコンポーネントを再実装しない
- 5年齢モードの並行実装に注意（1つ変更したら5つ全て確認）
- 設計書を更新せずに PR を Done にしない
