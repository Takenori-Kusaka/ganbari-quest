## 1. 設計方針

### デュアルバックエンド構成

本アプリケーションは2つのデプロイ環境に対応するため、DB バックエンドを抽象化している:

| 環境 | DB | 用途 |
|------|-----|------|
| AWS Lambda（本番 SaaS） | DynamoDB | シングルテーブル設計、マルチテナント対応 |
| NUC サーバー（ローカル） | SQLite | 単一ファイルDB、単一テナント |

### アーキテクチャ

```
src/lib/server/db/
  ├── interfaces/          # リポジトリインターフェース（17ファイル）
  ├── dynamodb/            # DynamoDB 実装
  │   ├── keys.ts          # PK/SK 生成ヘルパー
  │   ├── counter.ts       # ID 自動採番
  │   └── *-repo.ts        # エンティティ別リポジトリ
  ├── sqlite/              # SQLite 実装（Drizzle ORM）
  │   └── *-repo.ts
  ├── factory.ts           # 環境変数で実装を切り替え
  ├── schema.ts            # SQLite スキーマ定義
  └── *-repo.ts            # ファサード（factory からの re-export）
```

- `DB_BACKEND` 環境変数で `dynamodb` / `sqlite` を切り替え
- 全リポジトリに `tenantId: string` パラメータが必須
- SQLite 実装では tenantId を受け取るが内部では無視（単一テナント）

### SQLite 固有の方針

- **Drizzle ORM**: TypeScript型安全なスキーマ定義
- **WAL モード**: 読み書き並行性を向上
- **バックアップ**: 日次で git push（DBファイルのコピー）

## 7. DynamoDB シングルテーブル設計（本番 SaaS）

### 7.1 テーブル構成

| 項目 | 値 |
|------|-----|
| テーブル名 | `ganbari-quest` |
| パーティションキー | `PK` (String) |
| ソートキー | `SK` (String) |
| 課金モード | オンデマンド |
| 暗号化 | AWS managed key |

### 7.2 GSI（グローバルセカンダリインデックス）

| インデックス名 | PK | SK | 用途 |
|--------------|-----|-----|------|
| GSI1 | `SK` | `PK` | 逆引きクエリ（全テナントの特定 SK を検索） |
| GSI2 | `GSI2PK` | `GSI2SK` | カテゴリ別活動クエリ、時系列クエリ |

### 7.3 テナント分離

全テナントスコープエンティティの PK に `T#<tenantId>#` プレフィックスを付与:

```
T#<tenantId>#CHILD#<childId>   // テナント内の子供
T#<tenantId>#ACTIVITY#<actId>  // テナント内の活動
T#<tenantId>#SETTING           // テナント設定
T#<tenantId>#COUNTER           // テナント内 ID カウンタ
```

グローバルエンティティ（CATEGORY, ACHIEVEMENT, TITLE, BENCH）はプレフィックスなし。

### 7.4 エンティティ別キー設計

#### テナントスコープエンティティ

| エンティティ | PK | SK | 説明 |
|------------|-----|-----|------|
| 子供プロフィール | `T#<tid>#CHILD#<cId>` | `PROFILE` | 子供の基本情報 |
| 活動マスタ | `T#<tid>#ACTIVITY#<aId>` | `MASTER` | 活動定義 |
| 活動ログ | `T#<tid>#CHILD#<cId>` | `LOG#<date>#<logId>` | 活動記録 |
| ポイント台帳 | `T#<tid>#CHILD#<cId>` | `POINT#<ts>#<ledgerId>` | ポイント増減記録 |
| ポイント残高 | `T#<tid>#CHILD#<cId>` | `BALANCE` | 集計済み残高 |
| ステータス | `T#<tid>#CHILD#<cId>` | `STATUS#<catId>` | カテゴリ別ステータス値 |
| ステータス履歴 | `T#<tid>#CHILD#<cId>` | `STATHIST#<catId>#<ts>#<id>` | ステータス変更履歴 |
| 実績獲得 | `T#<tid>#CHILD#<cId>` | `ACHV#<achvId>#<milestone>` | 子供の獲得実績 |
| 称号獲得 | `T#<tid>#CHILD#<cId>` | `TITLE#<titleId>` | 子供の獲得称号 |

| 評価 | `T#<tid>#CHILD#<cId>` | `EVAL#<weekStart>` | 週次評価 |
| 特別報酬 | `T#<tid>#CHILD#<cId>` | `REWARD#<ts>#<rewardId>` | マイルストーン報酬 |
| デイリーミッション | `T#<tid>#CHILD#<cId>` | `MISSION#<date>#<aId>` | 日替わりミッション |
| 活動ピン留め | `T#<tid>#CHILD#<cId>` | `ACTPREF#<aId>` | ピン留め設定 |
| キャラクター画像 | `T#<tid>#CHILD#<cId>` | `IMG#<type>#<hash>` | 生成画像 |
| チェックリストテンプレート | `T#<tid>#CHILD#<cId>` | `CKTPL#<tplId>` | チェックリスト定義 |
| チェックリスト項目 | `T#<tid>#CKTPL#<tplId>` | `ITEM#<sort>#<itemId>` | テンプレート内の項目 |
| チェックリストログ | `T#<tid>#CHILD#<cId>` | `CKLOG#<tplId>#<date>` | 日次チェック記録 |
| チェックリスト上書き | `T#<tid>#CHILD#<cId>` | `CKOVER#<date>#<overId>` | チェック上書き |

| 設定 | `T#<tid>#SETTING` | `<key>` | テナント設定（キー/値） |
| ID カウンタ | `T#<tid>#COUNTER` | `<entity>` | エンティティ別自動採番 |

#### グローバルエンティティ（テナント横断共有）

| エンティティ | PK | SK | 説明 |
|------------|-----|-----|------|
| カテゴリマスタ | `CATEGORY#<catId>` | `MASTER` | 活動カテゴリ定義 |
| 実績マスタ | `ACHIEVEMENT#<achvId>` | `MASTER` | 実績定義 |
| 称号マスタ | `TITLE#<titleId>` | `MASTER` | 称号定義 |
| 市場ベンチマーク | `BENCH#<age>` | `CAT#<catId>` | 年齢×カテゴリ別平均値 |

### 7.5 GSI2 の利用パターン

| GSI2PK | GSI2SK | 用途 |
|--------|--------|------|
| `T#<tid>#CAT#<catId>` | `ACT#<sort>#<actId>` | カテゴリ別活動一覧（ソート済み） |

### 7.6 ID 自動採番

DynamoDB には auto-increment がないため、`COUNTER` エンティティで原子的に ID を採番:

```
PK = T#<tenantId>#COUNTER
SK = <entityName>   (例: "child", "activity", "activityLog")
currentId = <最新ID>
```

`counter.ts` の `nextId()` は DynamoDB の `UpdateCommand` + `ADD` で原子的にインクリメント。

### 7.7 認証エンティティ（auth テーブル）

認証関連データは別途のキー構造で管理（`src/lib/server/db/dynamodb/auth-repo.ts`）:

| PK | SK | 説明 |
|-----|-----|------|
| `AUTH#<userId>` | `PROFILE` | ユーザープロフィール（email, cognitoSub） |
| `AUTH#<userId>` | `MEMBER#<tenantId>` | テナントメンバーシップ（role） |
| `TENANT#<tenantId>` | `PROFILE` | テナント情報（プラン, ownerId） |
| `INVITE#<code>` | `INVITE` | 招待コード（tenantId, role, 有効期限） |

### 7.8 バックアップ

- AWS Backup で日次バックアップ（3日保持）
- バックアップ時間: 毎日 18:00 UTC
- PITR は無効（コスト最適化）
