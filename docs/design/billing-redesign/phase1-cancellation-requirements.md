## 既存実装で大部分充足済み

- `account-deletion-flow.md §0` で **解約 ≠ 退会の区別は SSOT 化済み**
- `cancellation-service.ts` で **3分類 + 自由記述 (全プラン必須)** 実装済み (ADR-0012 引き止め UI 出さない意図明記)
- 退会猶予 (free 即時/standard 7日/family 30日) 既存。retention (ADR-0049: free 90日/standard 1年/family 無制限) は別軸

→ 新規決定は主に Open question (PO 判断) のみ。要件自体は Stripe 公式・日本法令と整合、大きな再設計は不要。

## 解約と退会の区別 (account-deletion-flow.md §0 SSOT)

| | 解約 (cancel) | 退会 (account deletion) |
|---|---|---|
| 対象 | Stripe Subscription | アカウント + 全データ |
| 結果 | 無料プランに移行 (**データ保持**) | 全データ完全削除 |
| 入口 | Customer Portal / 解約導線 | 設定「アカウント削除」(本人ログイン必須) |
| 復旧 | 再課金で復元可 | 不可 (猶予経過後物理削除) |

## 機能要件 (FR)

| ID | 要件 | 設計意図 + 根拠 |
|----|------|----------------|
| FR-1 | **解約は期末解約をデフォルト** (`cancel_at_period_end=true`)、支払済期間は有料機能のまま利用継続 | proration 事故回避。Stripe 公式推奨 ("less disruptive")。返金不要で運用単純 |
| FR-2 | **年額途中解約は返金なし・期末まで利用** | Stripe デフォルト無返金。返金ロジック不持で Pre-PMF 複雑性抑制。解約料規制 (算定根拠説明努力義務) 回避 |
| FR-3 | **解約 (無料に戻る/データ保持) と退会 (全削除) を別動線・別用語で完全分離** | クレーム防止。既存 SSOT 固定 |
| FR-4 | 解約理由収集は **解約確定後に任意収集 (skip 可)** (PO 確定 2026-05-27、既存「必須」から変更)。解約ボタン → 即解約確定 → 事後に 3分類 + 自由記述を任意提示 | 特商法ダークパターン規制 (解約を困難にする追加ステップ) 回避 + churn データ取得を両立。**既存 cancellation-service の必須化を skip 可に変更要**。Stripe Portal reason とは二重取得しない |
| FR-5 | **解約導線はログイン後 1〜2 クリックで到達、手順を 1 ページに** | 消費者契約法 努力義務 + 特商法ダークパターン回避 |
| FR-6 | **退会はログイン本人実行のみ** (CS 代行不可)、確認文字列ガード | なりすまし防止 (account-deletion §5.1) |
| FR-7 | 解約後データは無料 capability + retention (free 90日) に従う。退会後は猶予経過で物理削除 | サブスク軸 (解約) とアカウント軸 (退会) を混同させない (ADR-0049) |

## 非機能要件 (NFR)

- NFR-1: **解約フローに引き止め UI (retention coupon / 連続確認) を出さない**。Stripe Portal「顧客維持クーポン」OFF | ADR-0012 + 特商法ダークパターン規制の二重根拠
- NFR-2: webhook が SSOT (解約は Stripe 確定 → webhook で反映)
- NFR-3: 解約/退会文言は LP・利用規約 (terms.html 第7条/第13条) と一致 (ADR-0013)
- NFR-4: 解約理由自由記述は PII を含みうる → retention/削除対象に含める検討 (ADR-0049 拡張表に未掲載、#2538/法務 #2541 連動)

## ユーザーストーリー

1. 有料不要になったら解約。支払済の残期間は使えて安心 (FR-1)
2. 解約しても子供の記録は無料プランで残す (退会と違うと理解) (FR-3/FR-7)
3. 解約手順が分かりやすく数クリックで完結 (FR-5)
4. 完全にやめる時は退会してデータも消す (FR-3/FR-6)
5. 運営は解約理由を分類集計し churn 改善に使う (引き止めには使わない) (FR-4/NFR-1)

## 解約 (期末/即時) 推奨

**期末解約デフォルト、即時解約オプションは出さない** — Stripe 推奨 (less disruptive) / 即時=返金期待の誤解回避 / 特商法上の公平性 (支払分は使える) / ADR-0012 は「引き止めない」であって「即時に切る」ではない。
