## 機能要件 (FR)

| ID | 要件 | 設計意図 + 根拠 |
|----|------|----------------|
| FR-L1 | **最終確認画面で 5 項目表示**: ①分量 ②販売価格 ③支払時期・方法 ④役務提供期間/自動更新 ⑤解約・撤回方法 | サブスク誤認購入防止。特商法第12条の6 (令和4年6月施行) |
| FR-L2 | **トライアル→有料移行の明示**: family 固定7日・終了時期・カード登録要否を明示。**カード登録なし=7日後自動課金されず本人が checkout で有料化** (現 tokushoho と整合) | 誤認表示禁止。特商法第12条の6第2項 |
| FR-L3 | **自動更新の明示** (月/年の更新周期) を最終確認画面に | 特商法 |
| FR-L4 | **解約方法の明示と確実な導線** (請求管理ページ/設定からいつでも、電話のみ等は不可) | 特商法第12条の6。設計確定「期末・引き止めなし」と整合 |
| FR-L5 | **ライセンスキー言及の全削除** (上記リスト) | account-first 化で「キーをメール送信」は実装乖離=ADR-0013 違反・特商法 提供時期不実表示リスク |
| FR-L6 | **退会時 Stripe Customer 削除を privacy/terms に明記** | PIPC 第35条 削除請求実効性。決済 PII 残存防止 |
| FR-L7 | **データ保持期間の subscription 整合** (free 90日/standard 1年/family 無制限を terms/tokushoho/privacy で統一) | 文書間齟齬解消。PIPC 第22条 |

## 非機能要件 (NFR)

- NFR-L1: 最終確認画面の 5 項目は **1 画面内**で確認可能 (別ページ誘導のみ不可)
- NFR-L2: 法務 3 文書 + 最終確認画面 + LP の記載が相互矛盾しない (SSOT、不実表示排除)
- NFR-L3: 文書改訂時に最終更新日を更新、重要変更は利用者周知 (terms 第16条)

## ユーザーストーリー

1. 申込最終画面で「いつ・いくら・どの周期・自動更新・どこで解約」を 1 画面で確認 (FR-L1〜L4)
2. トライアル中無料で、7日後どうなるか (カード未登録なら自動課金なし) を明確に知る (FR-L2)
3. 退会したら Stripe に残ったカード顧客情報も消えると保証されたい (FR-L6)
4. 解約はメニューから自分で完結、引き止め・電話強制なし (FR-L4)

## 特商法 最終確認画面 (Stripe Checkout 兼用 + 補完)

Stripe Checkout は 5 項目のうち①②③④を自動表示できるが、**⑤解約方法明示 + 特商法表記リンクは事業者側補完が必須** (単体では不足)。Stripe の Commerce disclosure ページ設定で特商法表記をリンク。

## 既存文書の改訂

- **tokushoho.html (最重要)**: 「支払時期」「無料トライアル」欄を family 固定7日・カード登録なし・7日後挙動で更新。現「トライアル後自動課金なし」は**カード登録なし設計と整合** (確認済)
- **terms.html**: 第7条 (自動更新 記載済、微修正) / 第8条 (family 固定7日明示) / 第13条 (データ保持期間とアカウント削除猶予の 2 概念を書き分け、#2538 連動)
- **privacy.html**: §6 に退会時 Stripe Customer 削除追記 (微修正)。越境移転 (§28)・削除権・未成年保護は既存充実
