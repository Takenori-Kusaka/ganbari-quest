# 18. 個人開発SaaS展開ガイド

> **目的**: がんばりクエストの開発・運用を通じて得たナレッジを集約し、個人開発SaaS展開の実践ガイドとする。
> **対応チケット**: #0079
> **運用方法**: 各チケット完了時に該当カテゴリへ知見を追記する（チケットテンプレートの「ナレッジ」セクション参照）。

---

## 1. 法的・手続き

### 1.1 開業届・税務

- 個人事業主として開業届を税務署に提出
- 青色申告を選択（65万円控除のため e-Tax 利用推奨）
- インボイス制度: 課税売上1,000万円以下でも任意登録可能。BtoC メインなら慎重に判断

### 1.2 特商法・プライバシー

- 特定商取引法に基づく表記が必要（バーチャルオフィスの住所を活用可能）
- プライバシーポリシーは子供のデータを扱う場合、特に慎重な設計が必要
- GDPR 対応: EU ユーザーがいなくても将来の拡張を見据えて基本対応しておくと良い
  - DPIA（データ保護影響評価）を実施 → `docs/design/17-データ保護影響評価書.md` (#0121)
  - データ最小化原則を設計段階から組み込む

### 1.3 利用規約

- 利用規約とプライバシーポリシーはサインアップ時に明示的同意を取得
- 同意記録（タイムスタンプ・IP・UA）を保存 (#0121)
- 消費者契約法: 故意・重過失は免責不可

## 2. コスト管理

### 2.1 AWS インフラ

- **AWS Free Tier の活用**: Lambda 100万リクエスト/月、DynamoDB 25GB、S3 5GB が無料枠
- **Lambda + DynamoDB**: サーバーレスで固定費ゼロスタート。月額費用はアクセス量に比例
- **CloudFront**: 静的アセットの配信にCDNを活用。無料枠1TB/月
- **SES**: メール送信は月62,000通まで無料（EC2からの送信時）
- **Cost Explorer API**: $0.01/リクエスト。キャッシュして呼び出し回数を抑える (#0176)

### 2.2 外部サービス

- **ドメイン**: `.com` で年額約1,500円
- **GitHub**: 無料プランでプライベートリポジトリ利用可能。GitHub Actions 2,000分/月
- **Stripe**: 決済手数料 3.6%。月額固定費なし
- **Discord**: コミュニティ運営に無料で利用可能。Webhook で通知連携

### 2.3 コスト監視

- AWS リソース固定費用の監査チケットを定期的に実施 (#0202)
- 不要リソース（未使用の DynamoDB テーブル、S3 バケット等）の棚卸し

## 3. セキュリティ・リスク管理

### 3.1 認証・認可

- Cognito を利用した認証基盤（OAuth 2.0 / OIDC）
- ローカル開発モードでは PIN コード認証（LAN 内限定）
- Bearer トークンによる運営 API の保護 (#0176)

### 3.2 ファイルセキュリティ

- アップロード画像は Sharp で再エンコードして EXIF/polyglot ペイロードを除去 (#0243)
- MP3 は ID3v2 タグをストリップ
- Content-Type ホワイトリストで安全な MIME タイプのみ配信
- CSP ヘッダ（`object-src 'none'` 等）でブラウザ側の防御も追加

### 3.3 データ保護

- DynamoDB の暗号化（AWS managed key）
- S3 バケットのパブリックアクセスブロック
- CloudFront 経由のみでのアセット配信（S3 直接アクセス禁止）

### 3.4 Stripe セキュリティ

- Webhook 署名検証による改ざん防止
- ライセンスキー方式で決済とアプリ認証を分離 (#0247)
- Restricted API Key の使用（必要最小限の権限）

## 4. マーケティング

### 4.1 コミュニティ

- Discord サーバーでユーザーコミュニティを構築
- チャンネル分離: お知らせ / アップデート / 質問 / 要望 / 不具合報告
- Webhook で自動通知（新規サインアップ、問い合わせ等）

### 4.2 プロダクト

- 「作っている過程」を発信する価値（ビルドインパブリック）
- 子育てコミュニティへのリーチ（Twitter/X、子育てブログ）

## 5. 技術的知見

### 5.1 SvelteKit + AWS Lambda

- SvelteKit 2 + Svelte 5 (Runes) でフルスタック開発
- `adapter-node` でビルド → Docker (ARM64) → ECR → Lambda Web Adapter
- GitHub Actions で main push 時に自動デプロイ
- ローカル開発は SQLite、本番は DynamoDB のデュアルDB構成

### 5.2 DynamoDB シングルテーブル設計

- PK/SK パターンで複数エンティティを1テーブルに集約
- GSI を活用した逆引きクエリ（例: メールアドレスからテナント検索）
- テナント分離: PK に `T#<tenantId>#` プレフィックスを付与 (#0127)

### 5.3 ライセンスキー設計

- `GQ-XXXX-XXXX-XXXX` 形式（曖昧文字 0/O/1/I を除外）
- Stripe Checkout → Webhook → キー自動発行 → メール送信のパイプライン (#0247)
- サインアップ時にオプションで入力可能（無料トライアルと購入済みの両フロー対応）

### 5.4 メール配信

- SES でトランザクションメール（ウェルカム、ライセンスキー、解約通知等）
- HTML + テキスト両方のボディを送信（メールクライアント互換性）
- ローカルモードではログ出力のみ（SES 不使用）

### 5.5 テスト戦略

- Vitest でユニットテスト（1100+ テスト）
- Playwright で E2E テスト（222 テスト、デスクトップ + モバイル）
- `data-testid` 属性で要素特定（CSS セレクタに依存しない）
- CI で biome lint + svelte-check + vitest + playwright を毎回実行
