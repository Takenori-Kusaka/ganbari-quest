## 1. デザイン原則

### 最優先原則: 4歳児が使える

- **文字に頼らない**: アイコン・色・形でナビゲーション
- **大きなタップターゲット**: 最低 64x64px（推奨 80x80px）
- **即座のフィードバック**: タップしたら即反応（色変化、スケールアニメーション）
- **シンプルな遷移**: 最大2タップで目的達成
- **明確な視覚的区別**: 子供別のテーマで「自分のページ」を実感

- スクロールで全カテゴリ表示
- ヘッダーにポイント残高を常時表示

## 5. アニメーション・演出

### レベルアップ演出

| 要素 | 仕様 |
|------|------|
| トリガー | ステータス更新でレベルが上がった時 |
| 演出 | 画面中央にキャラクター拡大表示 + キラキラパーティクル |
| 時間 | 3秒間 |
| テキスト | 「レベルアップ！ Lv.X → Lv.Y」（大きなひらがな） |
| 音 | なし（家庭環境を考慮） |

### 記録完了アニメーション

| 要素 | 仕様 |
|------|------|
| トリガー | 活動記録完了時 |
| 演出 | ポイント数字がバウンスしながら表示 + 星が散る |
| 時間 | 1.5秒間 |

### タップフィードバック

| 要素 | 仕様 |
|------|------|
| アイコンタップ | scale(0.95) → scale(1.0)、0.15秒 |
| ボタンタップ | 背景色が10%暗くなる、0.1秒 |

## 6. レスポンシブ対応

| 画面幅 | 対象 | グリッド列数 | 特記事項 |
|--------|------|-------------|---------|
| 768px〜1024px | タブレット（メイン） | 3列 | 最適化対象 |
| 1024px〜 | PC | 4列 | 親の管理画面用 |
| 〜768px | スマートフォン | 2列 | 親の管理画面用 |

## 7. アクセシビリティ

| 項目 | 対応 |
|------|------|
| コントラスト比 | WCAG AA (4.5:1) 以上 |
| タップターゲット | 最低 48x48px、推奨 64x64px以上 |
| フォーカスインジケータ | キーボード操作時にリング表示 |
| aria-label | 全アイコンボタンに付与 |
| 色だけに依存しない | アイコン + 色で情報を伝達 |
