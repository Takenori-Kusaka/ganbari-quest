## 1. デザイン原則

### 最優先原則: 4歳児が使える

- **文字に頼らない**: アイコン・色・形でナビゲーション
- **大きなタップターゲット**: 最低 64x64px（推奨 80x80px）
- **即座のフィードバック**: タップしたら即反応（色変化、スケールアニメーション）
- **シンプルな遷移**: 最大2タップで目的達成
- **明確な視覚的区別**: 子供別のテーマで「自分のページ」を実感

### 4.8 スタンプカード（ログインボーナス）

**正仕様（ADR-0004 参照）:**

#### スタンプカード構造
- **一週間単位**のスタンプカード（月曜〜日曜）
- スタンプは一週間に**最大5個**まで押印可能（**曜日に依存しない**）
- 一日一回ログインボーナス表示時にスタンプカードへ押印アニメーションとともに押印

#### スタンプデザイン
- `static/assets/stamps/` の**イラスト画像アセット**（6種）を使用
- おみくじランク: 大大吉(UR) / 大吉(SR) / 中吉(R) / 小吉(R) / 吉(N) / 末吉(N)
- レア度確率: N=60%, R=25%, SR=12%, UR=3%

#### スタンプカードUI
```
┌──────────── スタンプカード ─── 4/7〜4/13 ─┐
│                                             │
│   ┌─ ─ ─┐  ┌─ ─ ─┐  ┌─ ─ ─┐            │  ← 上段3スロット
│   │ 🎋  │  │ 🌟  │  │  3  │            │     (画像 or 番号)
│   └─ ─ ─┘  └─ ─ ─┘  └─ ─ ─┘            │
│        ┌─ ─ ─┐  ┌─ ─ ─┐                 │  ← 下段2スロット
│        │  4  │  │  5  │                 │     (staggered)
│        └─ ─ ─┘  └─ ─ ─┘                 │
│                                             │
│   ████████████░░░░░░░░  2/5              │  ← 進捗バー
│                                             │
│   ✨ あと3回でコンプリート！                 │
└─────────────────────────────────────────────┘
```

**レイアウト:** 3+2のスタガード配置（リアルなスタンプカード風）

#### ポイント計算

| タイミング | 計算方法 | 例 |
|-----------|---------|-----|
| **押印時（即時）** | `基本5pt × レア度倍率` | N:5pt, R:10pt, SR:20pt, UR:40pt |
| **翌週redeem** | `スタンプ数 × 10pt + コンプリートボーナス` | 5枠コンプリート: 50+50=100pt |

- レア度倍率: N=1, R=2, SR=4, UR=8
- コンプリートボーナス: 50pt（5/5達成時のみ）
- redeem時はレア度**無関係**（スタンプ個数のみ）

#### 押印オーバーレイ演出（StampPressOverlay）
```
┌─────────────────────────────────┐
│                                 │
│     今週 3回目！                │
│                                 │
│  [🎋] [🌟] [   ] [   ] [   ]  │  ← 5スロットミニ表示
│                                 │
│       ┌──────────────┐         │
│       │   🌟 大吉    │         │  ← 当選画像（大きく表示）
│       └──────────────┘         │
│                                 │
│         +20pt                   │  ← 即時ポイント
│     3にちれんぞく！              │
│     あと2回でコンプリート！       │
│                                 │
│     [ やったね！ ]               │
│                                 │
└─────────────────────────────────┘
```

| フェーズ | 演出 | 時間 |
|---------|------|------|
| card | スタンプカードミニ表示、今日のスロットがパルス | 0-0.4s |
| press | スタンプ画像が大きくバウンス登場、SR/URはパーティクル | 0.4-1.2s |
| points | ポイント表示、連続日数、コンプリート状況 | 1.2s〜 |
| weekly | 前週の自動redeem結果（月曜のみ） | ユーザー操作後 |

**コンポーネント:**
- `StampCard.svelte` — ヘッダーのスタンプカードダイアログ
- `StampPressOverlay.svelte` — ログイン時の押印アニメーション
- `stamp-card-service.ts` — ビジネスロジック
- `stamp-image.ts` — おみくじランク定義・画像パスヘルパー

## 5. アニメーション・演出

| 要素 | 仕様 |
|------|------|
| トリガー | ホーム画面アクセス時（当日未押印） |

| 演出Phase3 | ポイント加算アニメーション |
| SR/URエフェクト | パーティクル散布 + グロー + 特別SE |
| 連続ログイン表示 | 日数とマルチプライヤー表示 |

### レベルアップ演出

| 要素 | 仕様 |
|------|------|
| トリガー | ステータス更新でレベルが上がった時 |
| 演出 | 画面中央にキャラクター拡大表示 + キラキラパーティクル |
| 時間 | 3秒間 |
| テキスト | 「レベルアップ！ Lv.X → Lv.Y」（大きなひらがな） |
| 音 | なし（家庭環境を考慮） |

### 記録完了アニメーション

| 要素 | 仕様 |
|------|------|
| トリガー | 活動記録完了時 |
| 演出 | ポイント数字がバウンスしながら表示 + 星が散る |
| 時間 | 1.5秒間 |
| ストリーク表示 | 2日以上連続の場合、炎マーク（🔥）が左から右へ並ぶアニメーション + ボーナスポイント表示 |
| ストリーク時間 | +0.8秒（基本アニメーションの後） |

### タップフィードバック

| 要素 | 仕様 |
|------|------|
| アイコンタップ | scale(0.95) → scale(1.0)、0.15秒 |
| ボタンタップ | 背景色が10%暗くなる、0.1秒 |

年齢に応じたデフォルト表示設定と、保護者によるカスタマイズ機能。

### 8.1 カードサイズプリセット

| サイズ | minWidth | iconSize | textSize | 対象 |
|--------|----------|----------|----------|------|
| `large` | 120px | 4rem (xl) | 13px | Baby (0-2歳) |
| `medium` | 90px | 3rem (lg) | 11px | Kinder (3-5歳) |
| `small` | 70px | 2.5rem (md) | 10px | Elementary (6歳以上) |

### 8.2 年齢別デフォルト

| 年齢帯 | カードサイズ | 表示件数/カテゴリ | 折りたたみ |
|--------|------------|-----------------|----------|
| Baby (0-2歳) | large | 6件 | あり |
| Kinder (3-5歳) | medium | 無制限 (0) | なし |
| Elementary (6歳以上) | small | 8件 | あり |

### 8.3 折りたたみ動作

- `collapsible=true` かつ `itemsPerCategory > 0` かつ実際のアイテム数が上限を超える場合に折りたたみ表示
- `max-height` + `overflow: hidden` で折りたたみ領域を制御
- 下部にグラデーションフェードオーバーレイ表示
- 「▼ もっとみる（のこり N こ）」ボタンで展開/折りたたみ切替

### 8.4 データモデル

`children` テーブルの `display_config` カラム（TEXT, nullable）に JSON で保存:

```json
{
  "cardSize": "medium",
  "itemsPerCategory": 0,
  "collapsible": false
}
```

`null` または未設定の場合は年齢からデフォルト値を自動算出。

### 8.5 関連ファイル

- `src/lib/domain/display-config.ts` — 型定義、デフォルト算出、パーサー
- `src/lib/ui/components/CategorySection.svelte` — 折りたたみ表示の実装
- `src/lib/ui/components/ActivityCard.svelte` — カードサイズ対応

## 9. UIモード（年齢別表示切替）

子供の年齢に応じて5つのUIモードを切り替える。`children.ui_mode` に保存。

| UIモード | 対象年齢 | ルートパス | 特徴 |
|----------|---------|-----------|------|
| `baby` | 0-2歳 | `/(child)/baby/` | 最大サイズのカード、文字なし、キャラクター中心 |
| `kinder` | 3-5歳 | `/(child)/kinder/` | 大きなカード、ひらがなのみ、キャラクター付き |
| `lower` | 6-9歳 | `/(child)/lower/` | 標準カード、ひらがな+漢字ルビ、称号表示あり |
| `upper` | 10-12歳 | `/(child)/upper/` | コンパクトカード、漢字混じり、称号表示あり |
| `teen` | 13-15歳 | `/(child)/teen/` | リスト表示、漢字中心、称号表示あり |

### 各モード共通ページ構成

| ページ | パス | 概要 |
|--------|------|------|
| ホーム | `/{mode}/home` | 活動一覧（カテゴリ別グリッド） |
| きろく（履歴） | `/{mode}/(character)/history` | 活動履歴（日別） |
| つよさ（ステータス） | `/{mode}/(character)/status` | カテゴリ別レベル・XP |
| じっせき（実績） | `/{mode}/(character)/achievements` | 獲得実績一覧 |
| しょうごう（称号） | `/{mode}/(character)/titles` | 獲得称号一覧（baby以外） |

### 子供用追加ページ

| ページ | パス | 概要 |
|--------|------|------|
| チェックリスト | `/(child)/checklist` | 持ち物チェックリスト |
| バトル | `/(child)/[uiMode]/(character)/battle` | 日次バトルアドベンチャー |
