# アイコン・ラベル統一規約

## 概要

本ドキュメントは、がんばりクエスト内で使用するアイコン（絵文字）とラベル（テキスト）の統一規約を定める。
#0289 の総点検で発見された不整合の再発を防止するための恒久的なルール。

## アイコン定数

全てのナビゲーション・セクションアイコンは `src/lib/domain/icons.ts` に一元定義されている。
新しいアイコンを追加する際は**必ずこのファイルに定数を追加**し、コンポーネントから参照すること。

### ナビゲーションアイコン

| 定数名 | 値 | 用途 |
|--------|-----|------|
| `ICON_HOME` | 🏠 | ホーム画面 |
| `ICON_STATUS` | ⭐ | つよさ / ステータス |
| `ICON_HISTORY` | 📋 | きろく / 記録（履歴） |
| `ICON_ACHIEVEMENTS` | 🏆 | じっせき / 実績 |
| `ICON_SWITCH` | 🔄 | きりかえ / 切替 |

### セクション・機能アイコン

| 定数名 | 値 | 用途 |
|--------|-----|------|
| `ICON_RECORD_SUMMARY` | 📝 | 今日の記録サマリー |
| `ICON_TITLES` | 🎖️ | しょうごう / 称号 |
| `ICON_POINTS` | ⭐ | ポイント表示 |
| `ICON_STAMP` | 💮 | スタンプカード |

### 使用禁止のアイコン

| アイコン | 理由 |
|---------|------|
| 🛡️ | 「防御」の意味。ステータスには不適切 |
| 👤 | 「プロフィール」の意味。「切替」には不適切 |
| 📊 | 「データ分析」の意味。子供向けUIには不適切（親管理画面のみ許容） |
| 🏅 | 「メダル」の意味。「称号」とは微妙にズレ |

## 年齢モード別ラベル規約

## 本番・デモの整合性

- ボトムナビの項目数・構成は本番とデモで**必ず一致**させること
- デモは「本番の模擬体験」であり、構造的な差異は許容しない
- アイコン・ラベルの変更は両方に同時適用すること

## SSOT 完全化達成記録 (#1465 / #1683)

### Phase B Priority 4 達成 (PR #1680, 2026-04-29)

`scripts/check-hardcoded-strings.mjs` の Svelte ハードコード違反 baseline が **226 件 → 0 件**（100% 削減）に到達。`local/no-hardcoded-jp-text` ESLint ルールでの誤検知 (`<style>` ブロック内 JP コメント) を全英語化することで、SSOT 100% 化を達成した。

#### 経緯と Phase 区分

#### CI gate 運用ルール（継続）

- **ratchet rule**: `scripts/hardcoded-strings-baseline.json` の `count` を上げる（増加方向）変更は禁止
- 新規追加コードでハードコード JP を増やすと CI red (`scripts/check-hardcoded-strings.mjs`)
- LP 側の同等 gate: `scripts/check-lp-ssot.mjs` (`scripts/lp-ssot-baseline.json`) も baseline 0 維持

#### 残されたフォローアップ

- TypeScript ファイル（非 Svelte / 非 HTML）の hardcoded JP は scope 外。`scripts/check-hardcoded-strings.mjs` は `src/**/*.svelte` のみを検査対象とする
- ESLint `local/no-hardcoded-jp-text` の `<style>` ブロック除外: 別 Issue で対応推奨（今回は実コメントの英語化で実質回避）

#### i18n 対応への展望 (#1683 / V2MOM Q3+)

baseline 0 達成により、`labels.ts` を `i18n` 対応辞書化する基盤が整った。英語版 LP / 多言語化が必要となった段階で、`labels.ts` の構造を `{ ja: {...}, en: {...} }` 形式に拡張することで対応可能。
