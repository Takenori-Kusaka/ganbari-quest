## 1. 概要

保護者同意の取得・記録・撤回フローを定義する。子供データを扱うサービスとして、保護者の明示的な同意を全ての前提とする。

### 1.1 関連設計書

| 設計書 | 関連箇所 |
|--------|---------|
| 29-COPPA対応方針書 | 子供データ保護方針 |
| 14-セキュリティ設計書 | 認証・認可 |
| 07-API設計書 | consent 関連 API |

---

## 2. 同意の種別

| 同意種別 | 取得タイミング | 必須 | 撤回方法 |
|---------|-------------|------|---------|
| 利用規約同意 | サインアップ時 | ✅ | アカウント削除 |
| プライバシーポリシー同意 | サインアップ時 | ✅ | アカウント削除 |
| 子供データ処理同意 | 子供プロフィール作成時 | ✅ | 子供削除 |
| メール通知同意 | 設定画面 | ❌ | 設定変更 |
| Web Push 通知同意 | 設定画面 | ❌ | ブラウザ設定 |
| Cookie 同意 | 初回アクセス時 | ✅（SaaS版） | Cookie 削除 |

---

## 3. 同意取得フロー

### 3.1 サインアップ時

```
1. メールアドレス入力
2. パスワード設定
3. [✅] 利用規約に同意します（リンクで全文表示）
4. [✅] プライバシーポリシーに同意します（リンクで全文表示）
5. アカウント作成ボタン
```

- チェックボックスを ON にしないとアカウント作成不可
- 同意日時を `consents` テーブルに記録

### 3.2 子供プロフィール作成時

```
1. 子供の名前入力
2. 生年月日入力
3. 「お子さまの情報は利用規約・プライバシーポリシーに基づき管理されます」表示
4. 作成ボタン
```

- 保護者がログイン済みであることが前提（暗黙的同意）
- サインアップ時の同意が子供データ処理の根拠

### 3.3 Cookie 同意（SaaS版のみ）

初回アクセス時にバナー表示:
```
「当サイトでは認証やサービス提供のために Cookie を使用しています。
詳細は Cookie ポリシーをご覧ください。」
[同意する] [詳細を見る]
```

## 4. 同意の記録

### 4.1 データモデル

`consents` テーブル（settings KV ストア内）:

| キー | 値 | 説明 |
|------|-----|------|
| `consent_tos_accepted_at` | ISO 8601 | 利用規約同意日時 |
| `consent_tos_version` | string | 同意時の利用規約バージョン |
| `consent_privacy_accepted_at` | ISO 8601 | プライバシーポリシー同意日時 |
| `consent_privacy_version` | string | 同意時の PP バージョン |
| `consent_email_enabled` | boolean | メール通知同意 |
| `consent_push_enabled` | boolean | Push 通知同意 |

### 4.2 記録の不変性

- 同意記録は上書きではなく追記
- 撤回時は新しいレコードを追加（撤回日時を記録）
- 監査証跡として過去の同意履歴を保持

---

## 5. 同意の撤回

### 5.1 撤回方法

| 同意種別 | 撤回方法 | 影響 |
|---------|---------|------|
| 利用規約/PP | アカウント削除 | 全データ消去 |
| 子供データ処理 | 子供削除 | 該当子供のデータ消去 |
| メール通知 | 設定画面で OFF | メール送信停止 |
| Push 通知 | ブラウザ設定 or 設定画面 | 通知停止 |
| Cookie | ブラウザの Cookie 削除 | セッション終了 |

### 5.2 アカウント削除フロー

```
1. 管理画面 → 設定 → アカウント削除
2. 削除理由の選択（任意）
3. パスワード再入力（確認）
4. 30日間の猶予期間（復元可能）
5. 猶予期間後に完全削除
```

API: `POST /api/v1/account/delete`

---

## 6. 規約変更時の再同意

利用規約またはプライバシーポリシーが変更された場合:

1. 次回ログイン時に変更通知バナーを表示
2. 「変更点を確認して同意する」ボタン
3. 同意しない場合はサービス利用を制限（閲覧のみ可能）
4. 新しい同意日時とバージョンを記録
