## 3. コスト最適化施策

### 3.1 Lambda 最適化

| 施策 | 効果 |
|------|------|
| ARM64 (Graviton) | x86 比 20% 安価 |
| Docker イメージ最小化 | コールドスタート短縮 → 実行時間削減 |
| メモリ 512MB | コスト/パフォーマンスの最適バランス |
| Provisioned Concurrency なし | コスト回避（コールドスタートは許容） |

### 3.2 DynamoDB 最適化

| 施策 | 効果 |
|------|------|
| オンデマンドキャパシティ | 使った分だけ課金 |
| シングルテーブル設計 | テーブル数最小化 |
| TTL 活用 | 不要データの自動削除 |

### 3.3 CloudFront 最適化

| 施策 | 効果 |
|------|------|
| 静的アセットのキャッシュ | オリジンリクエスト削減 |
| Brotli 圧縮 | 転送量削減 |
| キャッシュポリシー最適化 | ヒット率 80% 以上を目標 |
