# 17. ログ設計書

## 1. ログシステム概要

### 1.1 ログ出力先

| 環境 | 出力先 | 保持期間 |
|------|--------|----------|
| Lambda (本番) | CloudWatch Logs (stdout/stderr) | デフォルト永続 |
| NUC サーバー (ローカル) | ファイル (`data/logs/app-YYYY-MM-DD.log`) + stdout | 30日（手動管理） |
| 開発 (npm run dev) | stdout のみ | セッション中 |

### 1.2 ログフォーマット

JSON Lines 形式。1行1エントリ。

```json
{"timestamp":"2026-03-29T10:00:00.000Z","level":"INFO","message":"[AUTH] ユーザーログイン成功","requestId":"a1b2c3d4-...","tenantId":"tenant_xxxx","method":"POST","path":"/api/v1/auth/login","status":200,"durationMs":42}
```

## 2. ログレベル定義（5段階）

| レベル | 数値 | 用途 | Discord通知 | CloudWatch |
|--------|------|------|-------------|------------|
| `DEBUG` | 10 | 開発時のみ。本番は抑制 | - | dev のみ |
| `INFO` | 20 | 正常フローのマイルストーン | - | ✅ |
| `WARN` | 30 | 期待外だが継続可能 | - | ✅ |
| `ERROR` | 40 | 5xx エラー・業務ロジック失敗 | ✅ 即時 | ✅ |
| `CRITICAL` | 50 | システム停止レベル | ✅ @everyone | ✅ |

### ログレベル制御

環境変数 `LOG_LEVEL` で本番の最小ログレベルを制御:
- 未設定時: 本番=`info`, 開発=`debug`
- 設定例: `LOG_LEVEL=warn` → warn/error/critical のみ出力

## 3. LogEntry スキーマ

```typescript
interface LogEntry {
  timestamp: string;        // ISO 8601
  level: LogLevel;          // 'debug'|'info'|'warn'|'error'|'critical'
  message: string;

  // HTTP リクエスト情報
  method?: string;          // HTTP メソッド
  path?: string;            // リクエストパス
  status?: number;          // HTTP ステータス
  durationMs?: number;      // レスポンス時間

  // エラー情報
  error?: string;           // エラーメッセージ
  stack?: string;           // スタックトレース

  // 相関・識別情報
  requestId?: string;       // UUID v4（リクエスト単位の相関ID）
  tenantId?: string;        // テナント識別子
  userId?: string;          // Cognito userId
  service?: string;         // モジュール分類

  // 追加コンテキスト
  context?: Record<string, unknown>;
}
```

## 4. requestId 相関ID設計

### 生成タイミング

`hooks.server.ts` の `handle` 関数冒頭で `crypto.randomUUID()` により生成し、`event.locals.requestId` にセット。

### 伝搬経路

```
hooks.server.ts (生成)
  → event.locals.requestId
  → logger.request() / logger.requestError() に自動付与
  → handleError() でアラートに含まれる
  → サービス層では必要に応じてログコンテキストとして渡す
```

### 利用方法

CloudWatch Logs Insights で requestId を検索:
```
fields @timestamp, @message
| filter requestId = "a1b2c3d4-..."
| sort @timestamp asc
```

## 5. Discord アラート仕様

### 5.1 通知トリガー条件

| 条件 | 通知レベル | メンション |
|------|-----------|-----------|
| HTTP 500+ エラー | ERROR | なし |
| `logger.critical()` 呼び出し | CRITICAL | @everyone |

### 5.2 通知フォーマット

**ERROR レベル**:
- 赤色 embed
- フィールド: Endpoint, Status, RequestId, TenantId, Error, Stack (先頭3行)

**CRITICAL レベル**:
- 暗赤色 embed + `@everyone` メンション
- フィールド: Message, Service

### 5.3 スロットリング仕様

- キー: `${path}:${errorSummary}` の組み合わせ
- ウィンドウ: 5分間
- スレッショルド: 3件
- 動作:
  - 1〜2件: 個別通知
  - 3件目: まとめ通知（「多発検知」）
  - 4件目以降: 無音
  - 5分経過後: リセット

### 5.4 Webhook URL

| 環境変数 | 用途 |
|----------|------|
| `DISCORD_ALERT_WEBHOOK_URL` | アラート専用（優先） |
| `DISCORD_WEBHOOK_INCIDENT` | フォールバック |

## 6. サービス別ログガイドライン

### 認証 (auth)

```typescript
logger.info('[AUTH] ユーザーログイン成功', { tenantId, userId });
logger.warn('[AUTH] 無効な PIN 試行', { tenantId, context: { attempts: 3 } });
logger.error('[AUTH] Cognito トークン検証失敗', { tenantId, error });
```

### 活動記録 (activity)

```typescript
logger.info('[ACTIVITY] 活動記録完了', { tenantId, context: { childId, activityId, points } });
logger.warn('[ACTIVITY] 日次制限到達', { tenantId, context: { childId, activityId } });
```

### 決済 (stripe)

```typescript
logger.info('[STRIPE] Checkout 完了', { tenantId, context: { plan } });
logger.error('[STRIPE] Webhook 処理失敗', { tenantId, error });
logger.critical('[STRIPE] Webhook Secret 未設定', { service: 'stripe' });
```

### DB

```typescript
logger.error('[DB] DynamoDB 書き込み失敗', { tenantId, context: { table, error } });
logger.critical('[DB] 全テナント DynamoDB 接続不可', { service: 'db', error });
```

## 7. インシデント調査フロー

### 7.1 Discord 通知受信 → CloudWatch Logs 検索

1. Discord 通知の `RequestId` をコピー
2. CloudWatch Logs Insights で該当ロググループを開く
3. クエリ実行:
   ```
   fields @timestamp, level, message, method, path, status, error
   | filter requestId = "コピーしたID"
   | sort @timestamp asc
   ```

### 7.2 テナント別調査

特定テナントで問題が多発している場合:
```
fields @timestamp, level, message, path, error
| filter tenantId = "tenant_xxxx"
| filter level in ["ERROR", "CRITICAL"]
| sort @timestamp desc
| limit 50
```

## 8. ログ保持ポリシー

| 環境 | 保持期間 | 管理方法 |
|------|----------|----------|
| CloudWatch Logs (Lambda) | 無期限（デフォルト） | AWS コンソールで変更可能 |
| NUC ローカルファイル | 30日（手動削除） | cron job 推奨 |
| 開発環境 | セッション中 | stdout のみ |
