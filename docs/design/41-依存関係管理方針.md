# 依存関係管理方針

| 項目 | 内容 |
|------|------|
| 版数 | 1.0 |
| 作成日 | 2026-04-09 |

---

## 1. 概要

npm パッケージの依存関係管理方針を定義する。セキュリティと安定性を維持しつつ、メンテナンスコストを最小化する。

---

## 2. 更新ポリシー

### 2.1 Dependabot 設定

| 更新種別 | 自動マージ | レビュー |
|---------|----------|---------|
| patch | ✅ CI 通過時 | 不要 |
| minor | ✅ CI 通過時 | 不要 |
| major | ❌ | 手動レビュー必須 |

### 2.2 更新頻度

- Dependabot: weekly（毎週月曜）
- セキュリティアップデート: 即時（Dependabot Security Alerts）

---

## 3. パッケージ追加ルール

| ステップ | 内容 |
|---------|------|
| 1 | ライセンス確認（→ 36-OSSライセンスコンプライアンス） |
| 2 | バンドルサイズ影響確認（`npx bundlephobia` または npm サイト） |
| 3 | メンテナンス状態確認（最終更新、Issue 数、ダウンロード数） |
| 4 | 代替手段の検討（標準 API で実現可能か） |

---

## 4. ロックファイル管理

- `package-lock.json` はコミット対象
- `npm ci` でインストール（CI 環境）
- `npm install` は開発環境でのみ使用
- ロックファイルのコンフリクトは `npm install` で再生成して解決

---

## 5. セキュリティ

- `npm audit` を定期実行（CI に含む）
- CodeQL で脆弱性スキャン（週次）
- `dependency-review.yml` で PR 時に新規依存の脆弱性チェック
