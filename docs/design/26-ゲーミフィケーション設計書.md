## 1. 概要

がんばりクエストのゲーミフィケーションシステムを定義する。
子供の活動を RPG 風の報酬メカニクスで動機付けし、「やりたくなる→やる→褒められる→もっとやりたくなる」のコアループを回す。

### 1.1 設計原則

1. **活動記録の動機づけ以外は有害** — コレクション閲覧・着せ替え等のアプリ内滞在目的機能は禁止
2. **親が褒める仕組み** — アプリはきっかけを作るだけ。最終的な報酬は親からの言葉
3. **ポジティブトーンのみ** — 罰則・減点・ネガティブフィードバックは一切なし
4. **年齢に応じた体験** — 5つの年齢モードで UI とメッセージを最適化

## 2. コアループ

```
[活動選択] → [記録] → [ポイント獲得] → [シール獲得] → [レベルアップ]
                                              ↓
                                     [親が通知を受信]
                                              ↓
                                     [おうえんメッセージ]
                                              ↓
                                     [子供が次もやりたくなる]
```

## 3. ポイントシステム

### 3.1 ポイント獲得

| 項目 | 仕様 |
|------|------|
| 基本ポイント | 活動定義ごとに設定（1〜100） |
| ストリークボーナス | 連続記録日数に応じた倍率（下記参照） |
| コンボボーナス | 同日複数カテゴリ記録で倍率加算 |
| ログインボーナス | 日次初回アクセス時に付与 |

### 3.2 ストリーク倍率

| 連続日数 | 倍率 |
|---------|------|
| 1-2日 | ×1.0 |
| 3-6日 | ×1.2 |
| 7-13日 | ×1.5 |
| 14-29日 | ×1.8 |
| 30日以上 | ×2.0 |

### 3.3 コンボシステム

同日中に異なるカテゴリの活動を記録するとコンボ倍率が加算される。

| コンボ数 | 追加倍率 |
|---------|---------|
| 2カテゴリ | +0.1 |
| 3カテゴリ | +0.2 |
| 4カテゴリ | +0.3 |
| 5カテゴリ（全制覇） | +0.5 |

### 3.4 ポイント減衰

長期間記録がない場合にポイントが緩やかに減衰する（設定で ON/OFF 可能）。

| 未記録期間 | 減衰率 |
|-----------|--------|
| 1-6日 | なし |
| 7-13日 | -1%/日 |
| 14日以上 | -2%/日（下限: 最大ポイントの50%） |

## 4. レベルシステム

### 4.1 レベル計算

累計ポイントから閾値テーブルを参照してレベルを決定する。

```typescript
// src/lib/domain/level-calc.ts
function calculateLevel(totalPoints: number): number
```

### 4.2 レベル閾値（抜粋）

| レベル | 必要累計ポイント | 到達目安 |
|-------|----------------|---------|
| 1 | 0 | 初期 |
| 2 | 50 | 1週間 |
| 3 | 150 | 2週間 |
| 5 | 500 | 1ヶ月 |
| 10 | 2,000 | 3ヶ月 |
| 20 | 10,000 | 6ヶ月 |
| 50 | 100,000 | 2年以上 |

### 4.3 レベルアップ演出

レベルアップ時に `LevelUpOverlay` を表示。

| 要素 | 仕様 |
|------|------|
| 表示条件 | レベルが上がった瞬間 |
| アニメーション | 紙吹雪 + レベル数字の拡大表示 |
| 効果音 | ファンファーレ（音声設定が ON の場合） |
| 自動閉じ | 5秒後 or タップ |

## 5. スタンプカードシステム

### 5.1 概要

活動記録のたびにスタンプ（シール）を1枚獲得し、カードに貼る。
カードが完成すると新しいカードが発行される。

### 5.2 スタンプガチャ

| レアリティ | 出現率 | シール枠の色 |
|-----------|--------|------------|
| N (Normal) | 55% | シルバー |
| R (Rare) | 30% | ゴールド |
| SR (Super Rare) | 12% | レインボー |
| UR (Ultra Rare) | 3% | ホログラム |

```typescript
// src/lib/domain/stamp-gacha.ts
function rollStampGacha(): { stampId: number; rarity: Rarity }
```

### 5.3 スタンプカード仕様

| 項目 | 値 |
|------|-----|
| カードあたりスロット数 | 5 |
| スタンプ獲得条件 | 活動記録1回につき1枚 |
| カード完成ボーナス | なし（カード完成自体が報酬） |
| 表示 | 子供ホーム画面の上部に現在のカード表示 |

### 5.4 スタンプマスタ（16種）

## 7. チャレンジシステム

### 7.1 デイリーミッション

日替わりのミッション。達成でボーナスポイント。

| 項目 | 値 |
|------|-----|
| ミッション数/日 | 3 |
| 生成ロジック | 子供の活動傾向から自動選定 |
| ボーナス | 各 +20ポイント |

サービス: `daily-mission-service.ts`

### 7.2 ウィークリーチャレンジ

週次の自動生成チャレンジ。

| 項目 | 値 |
|------|-----|
| チャレンジ数/週 | 1 |
| 期間 | 月曜〜日曜 |
| ステータス | active → completed / expired |
| ボーナス | +100ポイント |
| 生成ロジック | 前週の未達成カテゴリを優先 |

テーブル: `auto_challenges`
サービス: `auto-challenge-service.ts`
