## 1. 概要

がんばりクエストのパフォーマンス目標と計測方法を定義する。子供向けアプリとして、待ち時間の少ない快適な体験を提供する。

## 2. Core Web Vitals 目標

| 指標 | 目標 | 説明 |
|------|------|------|
| LCP (Largest Contentful Paint) | < 2.5秒 | メインコンテンツの表示完了 |
| INP (Interaction to Next Paint) | < 200ms | 操作に対する応答性 |
| CLS (Cumulative Layout Shift) | < 0.1 | レイアウトの安定性 |

### 2.1 ページ別目標

| ページ | LCP | INP | 備考 |
|-------|-----|-----|------|
| 子供ホーム | < 2.0秒 | < 100ms | 最も利用頻度が高い |
| 活動記録結果 | < 1.5秒 | < 100ms | ダイアログ表示 |
| 管理画面ホーム | < 2.5秒 | < 200ms | データ集計あり |
| LP | < 2.0秒 | < 100ms | ファーストインプレッション |

## 3. API パフォーマンス目標

| エンドポイント | p50 | p95 | p99 |
|-------------|-----|-----|-----|
| POST /activities/log | < 100ms | < 300ms | < 1,000ms |
| GET /home (SSR) | < 200ms | < 500ms | < 2,000ms |
| GET /api/v1/children | < 100ms | < 300ms | < 500ms |
| POST /api/v1/settings | < 100ms | < 200ms | < 500ms |

---

## 4. Lambda パフォーマンス

### 4.1 コールドスタート

| 条件 | 目標 |
|------|------|
| 初回起動 | < 3,000ms |
| 通常リクエスト | < 200ms |

### 4.2 最適化施策

| 施策 | 効果 |
|------|------|
| ARM64 (Graviton) | コールドスタート短縮 |
| Docker イメージ最小化 | 起動時間短縮 |
| ESM バンドル | ロード時間短縮 |
| 接続プール | DB 接続時間削減 |

---

## 5. フロントエンドパフォーマンス

### 5.1 バンドルサイズ目標

| 対象 | 目標 |
|------|------|
| 初期 JS バンドル | < 100KB (gzip) |
| 初期 CSS | < 30KB (gzip) |
| ルートごとの追加 JS | < 50KB (gzip) |

### 5.2 画像最適化

| 対象 | フォーマット | サイズ |
|------|-----------|--------|
| スタンプ画像 | PNG (128×128) | < 20KB |
| アバター | PNG (256×256) | < 50KB |
| 背景画像 | WebP | < 100KB |
| アイコン | SVG | < 5KB |
