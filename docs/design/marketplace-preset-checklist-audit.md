## 要約 (TL;DR)

1. **総チェックリスト項目数**: 121 (年齢別 15×平均 6.5 = 98 + イベント 3×平均 11 = 33)
2. **CRITICAL 所見**: **0 件**（活動/ごほうびで見られた「アプリ内機能の二重計上」に相当するものは持ち物側にはない）
3. **件数非対称**: baby 5 → elementary 8 の緩やかな段階的増加で、年齢の行動レパートリー拡大に即しており妥当。年齢別 vs イベント系の 5-8 vs 10-12 件差は「イベント特有の持ち物網羅性」として必要な差分
4. **イベント系 3 件の扱い**: 活動/ごほうびのテーマ型 (screen-time 等、嗜好分岐型) とは**性質が異なる**。持ち物リストにおけるイベント型は「普段使わない特殊持ち物の可視化」に本質的価値があり、廃止/追加インポート分離は不適切
5. **漢字/ひらがな遷移**: baby-elementary = ひらがな、junior-senior = 漢字中心、で一貫。遷移点は elementary (6-9) → junior (10-12) で小5-6 の漢字習得時期と一致し妥当
6. **LOW PRIORITY**: イベント系バリエーション不足（習い事/旅行/お泊り等）、persona 依存項目 (スマホ/SNS) の persona タグ精度

## §2 CRITICAL 所見 — 該当なし

活動プリセット監査 (#107) では「メタ活動 (アプリ内機能の二重計上)」を検出し、ごほうびプリセット監査では「メタごほうび (stamp/badge 二重計上)」2 件を検出・是正した。**持ち物プリセットでは同種の欠陥は確認されなかった**。

### 理由

- 持ち物はすべて**物理的なアイテム**(歯ブラシ・ランドセル・水着・タオル等) または**物理的なアクション**(おふろに入る・パジャマに着替える) であり、アプリ内で自動付与される抽象概念 (stamp / badge / point) との境界が明確
- ユーザが「ポイントで交換するアイテム」と誤解する構造的リスクが持ち物の文脈には存在しない

---

## §3 運用方針所見 — イベント系 3 件は維持が妥当

### 背景

- 活動プリセット: memory「プリセット顧客付加価値原則」に基づきテーマ型 9 件廃止（15 パック純化、task #106 完了）
- ごほうびプリセット: テーマ型 5 件の扱いは PO 判断待ち（reward 監査 §3）
- 持ち物プリセット: **イベント系 3 件 (field-trip / pool / school-start) は性質が異なる**

### 活動/ごほうびのテーマ型と持ち物イベント型の差異

| 観点 | 活動/ごほうびのテーマ型 | 持ち物のイベント型 |
|---|---|---|
| 廃止候補の性質 | 嗜好分岐（ゲーム好きか否か、creative か outdoor か） | 場面特化（普段使わない特殊持ち物） |
| 年齢別パックで代替可 | 可（活動/ごほうびは毎日選ぶ性質） | 不可（プール用品・入学用品は年齢別 morning/evening には入れられない） |
| 追加インポートに分離する価値 | 中（嗜好ごとに選ぶと恩恵） | 低（イベント時にだけ ON する一時的リスト、インポートコストが高い） |
| ユーザ体験 | 年齢別だけで 80% 充足 | イベント発生時に **その場で使える状態**であることが価値 |

### 結論

**イベント系 3 件は年齢別 15 パックと同列に同梱継続**。活動/ごほうびの「テーマ型は追加インポート化」方針と**意図的に非対称にする**のが正。

## §4 LOW PRIORITY 所見 (follow-up 候補)

### §4.1 イベント系の網羅性不足

現状 3 件のイベントタイプは以下のみ。家族生活で頻出する他のイベント型チェックリストが未整備。

| 未整備イベント | 想定シーン | 優先度 |
|---|---|---|
| お泊り会・旅行 | 着替え・歯ブラシ・タオル・充電器 | 中（家族外出頻度高） |
| 習い事 | 道具・飲み物・連絡帳 | 中（週次発生） |
| 運動会・発表会 | ユニフォーム・お弁当・水筒 | 低（年 1-2 回） |
| 病院受診 | 保険証・母子手帳・着替え | 低（必要時のみ） |

→ **本 PR 対応外。follow-up Issue 化候補として記録のみ。**

### §4.2 persona 依存項目の persona タグ精度

`evening-junior` の「スマホを充電場所に置く」、`evening-senior` の「SNSチェック終了」は、**スマホを持っている家庭前提**の項目。personas に `self-care` / `academic-oriented` だけ付与されているが、スマホ非所持家庭にとってはノイズ。

改善案:
- persona に `smartphone-user` 等の新タグ追加
- または持ち物内の optional フラグ付与（schema 拡張が必要）

→ 軽微・schema 変更を要するため本 PR 対応外。

### §4.3 weekend-baby の能動性

`weekend-baby` (5 items) のうち「おそとで あそぶ」「おひるね」「おやつタイム」「かぞくじかん」は**子供の主体的チェック対象ではなく、保護者が主導する活動**。0-2 歳の発達段階上これは妥当だが、ファーストステップとして「子供が chek-off する」体験を意識するなら、該当ペルソナ `first-child` 向けに **保護者側が記録する運用前提**である旨を description に明示する改善余地あり。

→ 軽微。文言調整レベル。
