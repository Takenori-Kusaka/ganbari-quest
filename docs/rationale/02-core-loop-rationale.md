# コアループ設計経緯 — L1/L2/L3 3 層モデルと親子ポイント経済モデル

> **関連 Issue**: #1328 (C1 umbrella)、#1334 (C1-DOC-RATIONALE-COREFLOW)

---

## 議論の発端

- **日時**: 2026-04-21（B6 ブレスト）
- **発端 Issue / セッション**: #1328（C1 umbrella）、#1262 ADR 大刷新 B6 派生
- **問題意識**:
  1. 企画書 01 には F7（親による 500P 単位のお小遣い変換）のみ記載。**親子ポイント経済モデル**（親子両視点統合、疑似ショッピング体験設計）がどこにも明文化されていなかった
  2. **L1/L2/L3 3 層モデル**（活動層 / メタ習慣層 / 経済層）が未定義のため、並列フィードバック系（おみくじ / レベル / 称号 / 実績 / バトル）が独立成長し、ポイント経済が歪む状態が続いていた
  3. LP に「シールガチャ」文言が混入したまま、実装は日 1 回 cap の `loginStamp` omikuji という食い違いが発生（#1307 B9）。明示的な設計原則がなかったため、retrofit（後付け合理化）が繰り返されてきた
  4. ごほうびショップの**子供側 UI が未実装**（親管理画面 `/admin/rewards` のみ）で、疑似ショッピング体験が成立していなかった（#501 問題提起から放置）

---

## 検討した代替案

| 案 | 概要 | 検討した理由 |
|----|------|-----------|
| 案 A: 現状維持（並列フィードバック系を独立のまま） | おみくじ・レベル・称号・実績・バトルを独立した報酬系として維持する | 既存実装を最小変更で維持できる |
| 案 B: ポイント廃止・評価軸多元化 | ポイントを廃止し、各軸（運動・勉強・生活など）を独立スコアで表現する | 単一ポイントの「お小遣い化」に対する親の抵抗感を下げられる |
| 案 C: ガチャ中心モデル（射幸心活用） | スタンプガチャを主軸に置き、レア報酬への期待感で継続を動機づける | 子供向けアプリの一般的なリテンション手法 |
| **採用案: L1/L2/L3 3 層 + ポイント統一原則** | 活動→スタンプカード→ごほうびショップの 3 層で設計し、ポイントを唯一の通貨とする | 親子ポイント経済モデルと Anti-engagement 原則を両立できる |

---

## 棄却理由

- **案 A（現状維持）棄却理由**: 並列フィードバック系の独立成長がポイント経済を歪め、「ポイントが唯一の通貨」という前提が崩れる。また、設計意図の不在がガチャ混入のような誤方向への drift を構造的に許してしまう
- **案 B（ポイント廃止）棄却理由**: 多元スコアでは「子供自身が貯めて交換する」疑似ショッピング体験を設計しにくい。また、親がごほうびを「陳列」し子が「交換申請」する親子連携フローを自然に表現できない
- **案 C（ガチャ中心）棄却理由**: ADR-0012（Anti-engagement 原則）に直接抵触する。「連続ガチャ / 射幸心訴求」は子供側 UI の滞在時間を増やす設計であり、親の購入動機（現実活動の充実）と反する。LP と実装の乖離を生んだ直接原因でもあった

---

## 採用案とその理由

### L1/L2/L3 3 層モデル

```
L1 活動層    : 個別活動完了 → ポイント付与
L2 メタ習慣層: 「毎日正しく入力する」→ スタンプカード (rarity = anti-habituation) → ポイント付与
L3 経済層    : 累積ポイント → ごほうびショップ (唯一の出口) → 実物/お金/特権
```

**採用理由**:
- L1 は直接的な活動フィードバック（活動実績が即ポイントに反映されることで子供が実感できる）
- L2 は「毎日継続する」行動それ自体への報酬。個別活動の内容ではなく**継続習慣の形成**を促す。スタンプのレア度分散は射幸心ではなく anti-habituation（習慣化阻害への対策）として設計される
- L3 は「貯めた成果を自分で使う」体験が、子供の自己効力感と親子コミュニケーションを同時に充足する

### 親子ポイント経済モデル

```
親 (供給側): 活動プリセット登録 + ごほうび陳列
子 (需要側): 活動実行 + ポイント蓄積 + ごほうび交換申請
                              ↓
        親による承認 → 受取（実物 / お金 / 特権）
```

**採用理由**:
- 単なる「記録アプリ」でなく「家族内の疑似経済圏」を形成することで、子供が「がんばった → 報われた」を実体験できる
- 「ごほうびショップ = 唯一の出口」とすることで、ポイントの価値が親管理画面でのみ供給・消費される閉じた経済圏が成立する
- 親が陳列を管理するため、ADR-0012 の Anti-engagement 原則（子供側 UI の滞在時間制限）と両立できる

### ポイント統一原則

並列フィードバック系の位置づけ:

| フィードバック系 | L3 経済層での役割 |
|----------------|----------------|
| スタンプカード (L2) | ポイント供給（メタ習慣の報酬） |
| おみくじ（日 1 回 cap） | ポイント供給（日次サプライズ。ADR-0012 許容範囲） |
| レベル・称号・実績 | 表現層（ポイント供給なし。現実活動結果の見える化） |
| バトル / クエスト | 現実活動記録完了への一回性報酬（ポイント供給） |

**採用理由**:
- 各フィードバック系が「ポイント供給か表現層か」を明確化することで、将来の機能追加時の位置づけが即判断できる
- ポイントが唯一の通貨であることで、各フィードバック系が独立した「もう一つの通貨」になる drift を防ぐ
