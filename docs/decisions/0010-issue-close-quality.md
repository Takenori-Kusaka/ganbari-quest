# 0010. Issue 起票・クローズの品質基準

| 項目 | 内容 |

## コンテキスト

Issue を closes する PR が、Issue の Acceptance Criteria を部分的にしか満たしていないケースが頻発した。
また、Issue 作成時に根本原因分析や構造的解決案がなく、対症療法的な修正が繰り返されていた。

## 決定

1. **Issue 作成時**: 根本原因分析と構造的解決策の提案を必須とする
2. **Issue クローズ時**: PR で `closes #N` を使用する場合、Issue の AC を全て満たしていること
3. **部分実装の場合**: `closes` ではなく `ref #N` を使い、未達 AC は別 Issue に切り出してからクローズ

## 結果

- PR レビューで AC 充足を確認するチェックポイントが追加される
- 「closes したが AC 未達」という状態がレビューで検出される
