# 0012. Anti-engagement 原則（滞在時間 = 価値毀損）

- **Status**: Accepted
- **Date**: 2026-04-21
- **Related Issue**: #1309 (#1307 umbrella 派生 B9-ADR-ENGAGEMENT)

## コンテキスト

本プロダクトは「親の願い（子供の現実活動の充実）の可視化」を存在目的とする。一方、従来の子供向けアプリ / ソシャゲ / 動画サービスは DAU × 滞在時間を KPI にした engagement 最適化設計を共有する（ガチャ / インフィニットスクロール / 通知過多 / 自動再生）。

この設計哲学は子供が **アプリと現実活動のどちらに時間を使うか** のトレードオフで現実活動を減らす方向に働く。親が購入を判断するプロダクトにおいて、子供の現実時間を奪う設計は親の購入動機と反する。

実際に、#1307 B9 で判明した通り、明示的 ADR が無かった結果、LP に「シールガチャ」文言が混入したまま (a) 実装は日 1 回 cap の `loginStamp` omikuji、(b) LP 上の描画は連続ガチャ的、という食い違いが 3 年以上放置された。哲学レベルでの判断原則として永続化し、将来の機能追加・コピー提案の即時審査に使う。

## 決定

### 1. アプリ滞在時間は価値毀損指標（親管理画面を除く）

子供側 UI は「記録する → 数秒で閉じる」を最短経路として設計する。セッション長を延ばす動機設計は採用しない。

### 2. 禁止する設計パターン（Anti-engagement 原則の下で原則不採用）

- 連続ガチャ / ランダム報酬を引き続けられる UI（2 回目以降のドロー動線を置かない）
- インフィニットスクロール
- 通知による再入店誘導（親の明示的同意がない限り）
- 動画視聴の連続自動再生
- 「次は何が出るか」心理を駆動するサプライズ系演出の濫用
- LP / 販促文言での「毎日遊べる」「連続ログインでレア獲得」等、滞在時間延伸を暗示する語彙

### 3. 許容する pattern

- **日 1 回 cap の短時間 interaction**（例: ログインボーナス、おみくじ — 数秒で完結するもの）
- **現実活動記録をトリガとする短時間フィードバック**（数秒の祝福演出）
- **親管理画面の滞在**（情報把握が目的なので制限なし）

### 4. KPI 設計方針

- DAU は追う（家族内での日次起動率）
- **セッション長は追わない** — むしろ短くなる方向が善
- 現実活動記録数 / チェックリスト達成率は追う（アプリ外時間の産物）

### 5. 新機能審査における使い方

新機能提案時は本 ADR に照らして「滞在時間を延ばす動機設計か」を必ず審査する。該当するなら採用しない（PMF 後でも原則変えない）。LP / パンフレット / プライシング戦略書等の販促文言も同じ審査対象。

## 結果

- 新機能追加時の判断軸に「子供の現実時間を奪う設計か」が恒久的に追加される
- LP / 販促文言のレビュー時に射幸性・engagement 増大系語彙が即検出対象になる（`scripts/measure-lp-dimensions.mjs` の `forbiddenTerms` 拡張を #1313 で実装）
- KPI ダッシュボードから子供側セッション長指標を外す（親側のみ残す）
- ガチャ系 UI 提案 / 通知連打系 UI 提案は初動で却下できる

### トレードオフ

- 一般的な子供向けアプリのベストプラクティス（例: daily streak を視覚的に誇張する）の一部は採用できず、短期 retention は相対的に弱い
- 「親向けに販促しにくい代わりに子供が自発的に長時間使う機能」も不採用となり、一部 CV 機会を放棄する

## Alternatives considered

- **一般的な engagement 重視戦略**: 本プロダクトの親層購入動機と反するため不採用
- **無制限自由設計（原則なし）**: 滞在時間最適化の誘惑に構造的に勝てず、個別判断で崩れるため不採用
