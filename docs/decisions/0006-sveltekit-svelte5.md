## コンテキスト

子供向け活動記録アプリのフレームワーク選定。要件: PWA対応、SSR可能、TypeScript対応、学習コストが低い、バンドルサイズが小さい。

## 検討した選択肢

### 選択肢 A: Next.js (React)
- メリット: エコシステム最大、情報が多い
- デメリット: バンドルサイズ大、PWA対応に追加設定必要

### 選択肢 B: SvelteKit 2 + Svelte 5
- メリット: 軽量、コンパイル時最適化、Runes による直感的リアクティビティ、PWA ビルトインサポート
- デメリット: エコシステムが React より小さい

### 選択肢 C: Nuxt (Vue)
- メリット: Vue のシンプルさ
- デメリット: Svelte ほど軽量ではない

## 決定

SvelteKit 2 + Svelte 5 (Runes) を採用。理由:
- 子供向けアプリでモバイル PWA がメイン → バンドルサイズの軽さが最重要
- Runes (`$state`, `$derived`, `$effect`) が直感的で保守性が高い
- ファイルベースルーティングが年齢モード別ルート設計に適合

## 結果

- `$state` / `$derived` / `$effect` を状態管理の基本とする
- stores は最小限に抑制
- Svelte 4 の `$:` リアクティブ宣言は使用禁止
