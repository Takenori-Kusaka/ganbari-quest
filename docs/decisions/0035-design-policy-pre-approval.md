# 0035. 設計ポリシー先行確認フロー — 新機能 / 新 interface は実装前に PO 合意を必須化

| 項目 | 内容 |
|------|------|
| ステータス | accepted |
| 日付 | 2026-04-17 |
| 起票者 | Takenori Kusaka |
| 関連 Issue | #1023, #1022 |

## コンテキスト

2026-04-16 のセッションで、以下のパターンが繰り返し発生した:

1. Issue が「やる」前提で起票され、Dev が着手
2. 実装完了後、PR レビュー段階で PO 設計ポリシーと照合 → 不一致発覚 → 撤回
3. Dev の作業（interface 設計 / SQLite 実装 / DynamoDB 実装 / テスト / 設計書更新）が全て無駄に

具体例:
- #804 (license_event) / #820 (ops_audit_log) — チケット時点で「やる」前提だったが、Pre-PMF 段階でのコスト対効果が見合わず撤回
- PR #1003 (closed), PR #1017 (closed) — 90% 方向転換 → close

根本原因: **実装着手前に「このフェーズでこの機能が本当に必要か」を PO と確認するフローがなかった**。

## 検討した選択肢

### 選択肢 A: PR レビュー時に設計ポリシーチェックを強化

- メリット: 既存フローの変更が最小
- デメリット: 実装完了後の撤回が減らない。Dev の無駄作業は継続

### 選択肢 B: 着手前に PO 設計ポリシー合意を必須化

- メリット: 着手後の撤回・大幅方向転換が激減。Dev のリソース無駄遣いを防止
- デメリット: 着手までのリードタイムが増加（PO の応答待ち）

### 選択肢 C: 全 Issue に PO 承認を必須化

- メリット: 漏れがない
- デメリット: bug fix / refactor にまで承認が必要になり、オーバーヘッドが大きすぎる

## 決定

**選択肢 B** を採用する。

以下のカテゴリに該当する Issue / PR は、**実装着手前に PO 設計ポリシー合意を必須**とする:

| カテゴリ | 該当例 |
|---------|-------|
| 新テーブル / 新スキーマ | `license_events`, `ops_audit_log`, 新 DynamoDB テーブル |
| 新 interface 追加 | `$lib/server/db/interfaces/*.interface.ts` の新ファイル |
| セキュリティ機能 | レート制限、ブルート検知、WAF |
| 課金・ライフサイクル変更 | Stripe 連携の新イベント、retention 変更 |
| AWS リソース追加 | S3 bucket、SNS topic、Cognito group など |
| 3 人日以上の工数 | 工数見積が一定以上 |

### PO 設計ポリシー合意の形式（いずれか 1 つ）

- Issue に **「PO 設計承認済み」ラベル** 追加
- ADR を先行起票し、Issue からリンク
- PO が Issue 本文にコメントで明示同意

### PR 本文テンプレートへの追加

`.github/PULL_REQUEST_TEMPLATE.md` に「設計ポリシー確認」セクションを追加し、該当 PR では合意の根拠リンクを記載させる。

### Reviewer 側の確認責務

- 新 interface / 新テーブルを含む PR を受け取ったら、まず「PO 設計合意の根拠」を確認
- 根拠が示されていない場合は **レビューを開始せず** Dev に差し戻し
- 「コードは良いが設計ポリシーと合わない」を実装完了後に指摘するパターンを根絶

### 免除条件

- `type:fix` の既存機能バグ修正（新規スキーマ / interface を伴わないもの）
- `type:refactor` の既存機能リファクタリング
- `type:docs` のドキュメント変更
- `priority:critical` で PO が緊急承認を口頭で出した場合（事後 ADR 必須）

## 結果

- Dev は着手前に PO 合意の存在を確認する手順が増えるが、着手後の撤回リスクが大幅に低減
- PO は Issue 起票時に設計ポリシー合意を明示する責務が増えるが、レビュー段階での方向転換コストが削減
- Reviewer は PO 合意の根拠を確認するチェック項目が増えるが、設計不一致の指摘が減る
- ADR-0018 (Issue 起票品質基準) と ADR-0023 (Pre-PMF Issue 優先度判断基準) を補完する位置づけ
