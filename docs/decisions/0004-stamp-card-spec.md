# ADR-0004: スタンプカード正仕様

|------|------|
| ステータス | accepted |
| 日付 | 2026-04-09 |
| 関連 Issue | #607 |

## コンテキスト

スタンプカードの仕様が #0284 で正しく実装された後、#387 のリデザインで全面的にデグレした（画像→絵文字、5枠→7枠、リアルデザイン→横並び、redeem にレア度倍率混入）。設計書に仕様が記載されていなかったため、4回にわたり同じ仕様を説明する事態になった。

## 決定（正仕様）

### 構造
1. **一週間単位**のスタンプカード
2. スタンプは一週間に**最大5個**まで押印可能（**曜日に依存しない**）
3. 一日一回ログインボーナス表示時にスタンプカードへ**押印アニメーション**とともに押印

### デザイン
4. `static/assets/stamps/` の**イラスト画像アセット**（6種: 大大吉/大吉/中吉/小吉/吉/末吉）を使用
5. 世の中の**リアルなスタンプカード**のデザインを参考にした、**単調な横並びではない**レイアウト

### ポイント
6. **押印時（即時）**: おみくじ6種をレア度確率で抽選 → **レア度に応じたポイントを即時付与**
7. **翌週（redeem）**: 前週の**スタンプ個数に応じたポイント**を付与（レア度は**無関係**）

### 画像アセット
- `static/assets/stamps/daidaikichi.png` — 大大吉
- `static/assets/stamps/daikichi.png` — 大吉
- `static/assets/stamps/chukichi.png` — 中吉
- `static/assets/stamps/shokichi.png` — 小吉
- `static/assets/stamps/kichi.png` — 吉
- `static/assets/stamps/suekichi.png` — 末吉

## 結果

この仕様に反する実装（7スロット、絵文字表示、redeem にレア度倍率）は全てバグとして修正する。

## 教訓

- **仕様は設計書に書く** — Issue やPRの本文だけでは、後のリデザインで上書きされる
- **画像アセットがある機能を絵文字に戻すのは明確なデグレ** — asset-catalog.md の「絵文字NG」基準を確認する
