# 0005. テスト品質 ratchet

- **Status**: Accepted
- **Date**: 2026-04-20
- **Related Issue**: #1262 / #1265
- **統合元**: 旧 ADR-0017 + ADR-0020（本 PR で削除、詳細は git 履歴）

## コンテキスト

> 旧 ADR-0017（テスト品質ラチェット方針）と ADR-0020（強制プロセス化）を統合。ADR 10 枠再構成（#1262）の一環。

2026-04-01 にカバレッジ閾値を lines: 48 / functions: 37 に引き上げたが、**48 時間以内に 2 回引き下げられた**（#0216, #0276-#0278 の機能追加時）。コメントには「テスト追加で再度引き上げ予定」と記載されたが、引き上げは一度も実行されていない。

同時期に以下の品質劣化パターンが確認された:

- Critical 修正（#543）に E2E テストなし
- `clearDialogGhosts()` でアプリバグをテスト側で隠蔽
- `test.skip()` 濫用（dialog-queue.spec.ts）
- activity-log-service.test.ts がサービス層をテストしていない（19 テスト全てが DB 直接操作）
- レアリティ分布テストが自己参照（実装の `RARITY_WEIGHTS` を呼ばず独自乱数シミュレーション）

**結果**: 4 回の既知リグレッション（#543 / #611 / #671 / #673）全てがテストで検出不可能。

## 決定

### 1. カバレッジ閾値の引き下げを CI で禁止

- `vite.config.ts` の `thresholds` を引き下げる PR は **CI で自動リジェクト**（`scripts/check-coverage-threshold.js`）
- 引き下げる唯一の方法は、ADR にその理由を記録し復元計画（日付つき）を同時にコミットすること

### 2. 機能追加 PR にカバレッジ差分チェック

- PR で追加されたファイル群のカバレッジが全体閾値以上であることを CI で検証
- 「テストは後で追加」は禁止。テストなしの機能追加 PR はドラフトのまま据え置き

### 3. テスト回避パターンの禁止

| 禁止パターン | 検出 | あるべき姿 |
|-------------|------|----------|
| `clearDialogGhosts()` 等のバグ隠蔽ヘルパー | PR レビューで `[must]` | アプリ側のバグを修正 |
| `test.skip()` の安易な使用 | CI で skip 数計測、増加時に warn | テストが通るようアプリ修正、または条件分岐 |
| カバレッジ閾値の引き下げ | CI で thresholds 変更検出 | テストを追加してからマージ |
| サービスを呼ばないサービステスト | PR レビュー | テスト対象の公開 API を直接呼ぶ |
| テスト内で実装ロジックを再実装 | PR レビュー | 実装関数を呼んで結果を検証 |
| `waitForTimeout()` の E2E 新規使用 | PR レビュー | 明示的な待機条件に置換 |

## 結果

- 機能追加のたびにテスト品質が下がる「品質負債の雪だるま」が構造的に防止される
- 「テストは後で」「閾値は一時的に下げる」が CI レベルで不可能になる
- テストが「存在するが機能しない」状態から「リグレッションを検出できる」状態に回復
