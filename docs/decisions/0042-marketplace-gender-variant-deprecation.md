## Context

`src/lib/data/activity-packs/` 配下に 10 件の性別バリアント JSON が存在する:

| 性別パック | 統合先候補 (中立) |
|----------|---------------|
| `baby-boy.json`, `baby-girl.json` | `baby-first.json` |
| `kinder-boy.json`, `kinder-girl.json` | `kinder-starter.json` |
| `elementary-boy.json`, `elementary-girl.json` | `elementary-challenge.json` |
| `junior-boy.json`, `junior-girl.json` | `junior-high-challenge.json` |
| `senior-boy.json`, `senior-girl.json` | `senior-high-challenge.json` |

### 初稿 (2026-04-20 初版) での判断と PO からの指摘

初稿では「性別ステレオタイプ解消」「内閣府方針整合」「海外トレンド」を根拠に**廃止 (選択肢 a)** を単独推奨した。

PO からの指摘 (2026-04-20):
> 単にトレンドとして性差を扱うべきではないという風潮はあれど日本社会における実家庭の割合でどうかを知りたい。単に海外の風潮から日本国内でもそういった声や SNS 発信が増えて目立つようになっていることは承知しているが、具体的な家庭でそうだという裏付けにはならない。

→ この指摘は的確であり、初稿の根拠は SNS/海外トレンド/政策方針に偏り、**日本の実家庭レベルの定量データが不足**していた。

### 再検討のために実施した市場調査

`docs/design/gender-segmentation-market-research.md` に 150+ 出典の一次資料を整理 (詳細は該当ドキュメントを参照)。主要発見:

**(A) 中立化圧力のデータ**:
- 国内競合子育て/お手伝い/習慣化アプリ **13 件中、活動プリセットを性別で分岐する UI 採用は 0 件** (ファミポイ/ママペイ/やることカード/ぴよログ/しまじろう/ワンダーボックス/スマイルゼミ/進研ゼミ 他)
- 日本玩具協会「日本おもちゃ大賞」が 2021 年に **ボーイズ/ガールズ部門を廃止**
- 日本トイザらスが 2021 年に店頭の男女別売り場を融合
- 電通ダイバーシティ・ラボ 2022 (n=681): 親の **24.5% が「性別にふさわしくない」で商品購入を拒否**した経験あり
- 電通 2022: **88.5% が子に性別に縛られない教育機会を望む**
- 電通 2022: **37.9% が「性別なのに」発言にモヤモヤ**、発言者 60.9% が祖父母
- 電通総研 2022 (n=3,000): 性差撤廃広告への肯定率 **74.5%**
- 博報堂生活総研「こども 20 年変化」/ 日本財団「こども 1 万人意識調査」: 現役子世代は明確に中立志向

**(B) 性別差残存のデータ**:
- 内閣府 R3: 親世代の **76.3% が何らかの性別固定観念を保有**
- ランドセル色選好 2026: 男児黒 54.4% / 女児紫・薄紫 30.0%
- 習い事の極端な男女偏り: バレエ **98% 女性** / サッカー **4.58 倍男子** / プログラミング **男 81.8% vs 女 18.2%**
- 性別特化育児書 (『男の子の一生を決める〜』等) が Amazon ベストセラー継続
- Instagram「#男の子ママ」「#女の子ママ」= 投稿数百万件規模の巨大コミュニティ
- 七五三・こいのぼり・ひな祭りの制度的性別差

**(C) データが示す両義性**:
**「性別軸をそのまま UI に直結させる需要は減っている一方、興味/生活実態の性別偏りは依然として強い」** というまだら状態。

重要な洞察: **習い事の男女偏り (バレエ 98%、サッカー 4.58 倍、プログラミング 4.5 倍) はジャンル偏りであって、日常タスク (歯磨き/お片付け/挨拶) の性別差ではない**。本アプリの活動プリセットは後者を主に扱うため、性別ではなく興味軸での分岐のほうが実データに忠実。

## 選択肢と推奨度

### 選択肢 (a): 性別バリアント廃止 (中立統合)

10 件の性別パックを 5 件の中立パックに統合する。差分活動は中立パックに全て吸収。

**データ裏付け**:
- 国内競合 0/13 件採用 → 業界標準と整合
- 玩具業界の部門撤廃 (2021-) トレンドと整合
- 電通 88.5% の親心理と整合
- 実装コスト低 (統合作業 + LEGACY_PACK_ID_MAP のみ)

**反対データ**:
- 性別特化育児書のベストセラー層 (支持派) には訴求力を下げる
- ランドセル色選好等の「実家庭でも一部性別差は受容されている」実態と一部不整合

**推奨度**: **中〜高** (シンプルで業界標準だが、支持派を無視した形になる)

### 選択肢 (b): 性別バリアント維持

現状の 15 バリアント構造を維持。ただし `elementary-boy` に「プログラミング」がないなど既存の非対称は修正。

**データ裏付け**:
- 内閣府 76.3% の性別固定観念保有層は現実に存在
- 習い事の実態偏り (バレエ 98%/サッカー 4.58 倍等) は実データ
- 性別特化育児書ベストセラー / SNS 巨大コミュニティの存在

**反対データ**:
- 国内競合 0/13 件で採用なし → 業界常識から逸脱
- 電通 24.5% の購入拒否層には逆差別化 (不快感) リスク
- 玩具業界の部門撤廃トレンドに逆行
- 日常タスクの性別差エビデンスは弱い (習い事とは別)
- メンテナンスコスト高 (15 バリアント保守)

**推奨度**: **低** (実データは「性別差あり」だが、それは興味ジャンル偏りであって日常タスクの性別差ではない。維持の根拠が弱い)

### 選択肢 (c): 再設計 — 興味軸セグメント

性別 UI を廃し、**興味/生活実態タグ**で活動パックを再編成。子自身が好きなタグを選ぶ設計。

**タグ候補 (5-8 粒度)**: うんどう派 / げいじゅつ派 / べんきょう派 / おてつだい派 / どうぶつずき派 / ぼうけん派 / テック派

**データ裏付け**:
- 習い事の男女偏り (§5) は**ジャンル偏り**なので、興味タグで賄える (バレエ → げいじゅつ派、サッカー → うんどう派、プログラミング → テック派)
- 電通 88.5% の「性別に縛られない教育機会を望む」親心理と整合
- 電通 63.3% の「子の意見尊重」と「子自身が興味タグを選ぶ UX」が合致
- 国内競合 0/13 件の採用 → 興味タグは**業界差別化要素**
- 性別特化育児書支持層 (§7) も「個性に合わせた子育て」文脈で受容可能

**反対データ**:
- 初回セットアップで子に興味を選ばせる UX 追加 (学習コスト)
- タグ粒度設計の新規決定事項
- 既存ユーザー (`pack_id='baby-boy'` 等で DB 保存) の移行設計

**推奨度**: **最高** (すべての実データと整合、業界差別化、親の 88.5% 心理と一致、習い事ジャンル偏りの実態とも一致)

## Decision (PO 承認待ち)

**推奨順**:
1. **優先**: 選択肢 (c) 再設計 — 興味軸セグメント
2. **代替**: 選択肢 (a) 廃止 — (c) の実装コストが Pre-PMF 段階で過大な場合
3. **非推奨**: 選択肢 (b) 維持

### PO 判断が (c) 再設計の場合 — 実装方針

1. **興味タグ定義** (新設): `src/lib/domain/activity-interest-tag.ts`
   - `type InterestTag = 'sports' | 'art' | 'study' | 'household' | 'nature' | 'adventure' | 'tech'`
   - labels.ts で日本語ラベル (「うんどう派」等) と絵文字を紐付け
2. **パック JSON のメタデータ拡張**: `interestTags: InterestTag[]` を追加
3. **パック構造**: 5 世代 × 5-7 興味タグ = 25-35 パック
   - 既存 5 中立パック (baby-first 等) は維持し、興味タグで追加推薦を表示
   - または世代 × タグの組合せで新規パック生成
4. **初回セットアップ UX**: 子のプロフィール作成時に「何が好き？」(複数選択可) を追加
5. **既存ユーザー migration**: `pack_id='baby-boy'` → `baby-first` + default interest tags (経験的マッピング) に透過変換
6. **ADR-0041 命名戦略**と整合: 「おすすめパック」セクションが興味タグベースで並ぶ

### PO 判断が (a) 廃止 (単純中立統合) の場合 — 実装方針

前版 ADR-0042 の記述と同等:
1. 10 件の性別パックを 5 件の中立パックに統合、差分活動を吸収
2. `src/lib/data/marketplace/legacy-pack-id-map.ts` (新設) で 10 件の旧 ID → 新 ID 透過解決
3. `LEGACY_URL_MAP` に 1 entry (`/activity-packs` → `/marketplace`)
4. `tests/e2e/legacy-url-redirect.spec.ts` + `tests/unit/marketplace-pack-coverage.test.ts` で CI 検証

### PO 判断が (b) 維持の場合 — 実装方針

1. 現行 15 バリアント構造を維持
2. `elementary-boy` に「プログラミング」、`elementary-girl` に「プログラミング」を追加等、既存の非対称だけ修正
3. LEGACY_URL_MAP / LEGACY_PACK_ID_MAP は不要
4. 選択 UI に「性別で選ぶのはイヤ、という方は中立パックも選べます」の導線を追加 (反対派への配慮)

## Consequences

### (c) 再設計を選んだ場合

**Positive**:
- 日本の実家庭データにもっとも整合
- 業界差別化ポイント (国内競合 0 件)
- 既存ユーザーの「好きな活動」が変わらないこと
- 「うちの娘がサッカー、息子がバレエ」のような非典型家庭にも自然にフィット
- 七五三・こいのぼり等の伝統文化は「年中行事コレクション」として別レイヤーで扱える (日常タスクの性別差は持ち込まない)

**Negative**:
- 世代 × タグで 25-35 パックに増加 (現 15 から +10-20)
- 興味タグの粒度設計・ラベリングが追加の設計議論
- 子 UI で「好きなこと」を選ばせる画面の新設

### (a) 廃止を選んだ場合

**Positive**:
- 実装コスト最小
- ジャンダーフリーを明確にブランド化
- マスタファイル 18 → 8 に減少

**Negative**:
- 支持派 (性別特化育児書購読層・SNS コミュニティ) へのアピールが弱まる
- ランドセル色・七五三等の実文化に残る性別差を無視する形

### (b) 維持を選んだ場合

**Positive**:
- 変更ゼロ
- 性別特化育児書支持層には違和感なし

**Negative**:
- 国内競合業界標準と逆行
- 電通 24.5% 購入拒否層にネガティブシグナル
- メンテコスト高止まり
- 玩具業界トレンドと逆行
