# ADR-0002: オーバーレイは同時に1つだけ表示する（ダイアログキュー必須）

| 項目 | 内容 |
|------|------|
| ステータス | accepted |
| 日付 | 2026-04-09 |
| 関連 Issue | #543, #611 |

## コンテキスト

活動記録後にレベルアップ・特別報酬・スタンプ押印・誕生日ボーナスなど複数のオーバーレイが同時に `open=true` になると、1つ閉じても下のオーバーレイが残り画面が操作不能になる。

#543 で対症療法（タップガード）のみ実装し、根本対策（ダイアログキュー）は未実装のまま Issue をクローズした結果、不具合が継続した。

## 決定

1. **OverlaysSection.svelte にダイアログキューを実装する** — 同時に表示されるオーバーレイは常に1つだけ。閉じたら次を表示
2. **ガード条件は防御層として残す** — キューがあってもガードは保険として必要
3. **Critical バグ修正では Issue の全提案を実装してからクローズ** — 部分実装での closes 禁止

## 結果

- 複数の報酬が同時発生しても、ユーザーは1つずつ順番に確認できる
- 「次のダイアログがある」ことを示すUIがあるとなお良い（将来拡張）
- E2E テストで回帰を防止する

## 教訓

- **対症療法だけでは根本解決にならない** — ガード条件は「入口を塞ぐ」だけで、サーバーレスポンスが複数ダイアログを同時にトリガーするケースは防げない
- **Issue の「提案」を部分実装で closes するのは不可** — 未実装の対策がある場合は Issue に残すか、別 Issue に切り出す
- **Critical バグの修正には E2E テストを必須とする** — テストがなければ次のリファクタリングで消える
- **セルフレビュー + Copilot のみで Critical をマージしない** — 人間の目によるレビューが必要
