# ADR-0036: マーケットプレイス公開アクセス設計

> **Archived (2026-04-20)**: マーケットプレイス公開アクセス。実装完了、運用段階

- Status: accepted
- Date: 2026-04-18
- Issue: #1099

## Context

LP からマーケットプレイスへのリンクが2箇所存在する。
マーケットプレイスの認可設計が明示的に文書化されていなかった。

選択肢:
- A) パブリック公開 — 閲覧もインポートも認証不要
- B) 認証必須 — ログイン壁で LP からの導線が dead end
- C) 閲覧パブリック / インポート認証必須

## Decision

**選択肢 C を採用**: 閲覧はパブリック、インポートは認証必須。

- `/marketplace` および `/marketplace/[type]/[itemId]` は `isPublicRoute()` に追加し、未認証ユーザーでも閲覧可能
- インポート操作（POST）は認証済みユーザーのみ実行可能（既存の API 認証ミドルウェアで保護）
- 未認証ユーザーには画面下部に「がんばりクエストに登録してインポート」CTA を表示（実装済み）

## Consequences

- LP → マーケットプレイス → 「良さそう」→ サインアップの導線が成立
- テンプレート数（現在49件）の閲覧でサービスの価値が伝わる
- `isPublicRoute()` に `/marketplace` を明示的に追加（暗黙のデフォルト許可に依存しない）
