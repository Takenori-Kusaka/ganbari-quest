## 決定の要点

**性別バリアント (10 JSON ファイル) を維持する。廃止しない。** ただし内容は市場データで精査し、以下を修正する:
- 既存の非対称 (例: `elementary-boy` にプログラミングがあり `elementary-girl` にない) を解消
- 全活動を 1 個ずつ市場実施率・教育推奨根拠で裏付けし、根拠不足の活動は入れ替え

## Context

`src/lib/data/activity-packs/` 配下に 10 件の性別バリアント JSON が存在する:

| 性別バリアント | 対となる中立パック |
|----------|---------------|
| `baby-boy.json`, `baby-girl.json` | `baby-first.json` |
| `kinder-boy.json`, `kinder-girl.json` | `kinder-starter.json` |
| `elementary-boy.json`, `elementary-girl.json` | `elementary-challenge.json` |
| `junior-boy.json`, `junior-girl.json` | `junior-high-challenge.json` |
| `senior-boy.json`, `senior-girl.json` | `senior-high-challenge.json` |

### 初稿 / 改訂 1 の判断プロセス (反省と是正)

**初稿**: 「性別ステレオタイプ解消」「内閣府方針」「海外トレンド」を根拠に廃止を単独推奨。

**PO 指摘 (2026-04-20 前半)**:
> 単にトレンドとして性差を扱うべきではないという風潮はあれど、日本社会における実家庭の割合でどうかを知りたい。単に海外の風潮から日本国内でもそういった声や SNS 発信が増えて目立つようになっていることは承知しているが、具体的な家庭でそうだという裏付けにはならない。

→ 初稿の根拠が表明データ (SNS / 調査 stated preference) に偏り、**行動データ (revealed preference) が不足**。実家庭データを再調査。

**改訂 1 (3 選択肢 open 型)**:
- (a) 廃止 / (b) 維持 / (c) 興味タグで再設計 — の 3 案
- (c) を推奨とした (興味タグで性別 UI を置換、「複雑度最小化」に寄与するとの解釈)

**PO 指摘 2 (2026-04-20 後半)**:
> 現在の 5 カテゴリを増やすかどうか、という話？増やす必要はない。プログラミングは勉強に入れるなど現在のカテゴリにまとめてください。これは情報複雑度の話で、要素が増えることは海外ゲームでよくあるが、多くの日本向けゲームではより要素を減らすことの方が求められる。多くのペルソナ像にとってはライトゲームユーザ以下であることを前提にすると複雑度を上げるのはあり得ない選択肢。

→ 興味タグ新設は複雑度増加であり却下。

**PO 指摘 3 (2026-04-20 最終)**:
> 性別バリアント廃止を承認した覚えはない。(中略) 複雑度を最小化したいのはユーザ目線。子供にとっては登録された活動プリセットがすべてなのでプリセットが多く用意されていることはまったく子供にとっての複雑度を高めていない。親が選ぶ際の複雑度は増すが、セットアップフローで追加したい子供の年齢と性別を入力し、それに伴ったプリセットが自動適用されるため初期段階でその複雑性にあたることはない。(中略) 私が先ほど調べてくれ、といったのは日本の家庭における性別バリアントの付加価値がどの程度あるか。日本社会における多くの家庭で有効、であれば維持、なければ中性中立な年齢別プリセットだけになる。

→ 本 ADR の問うべき核心は「**日本の家庭において性別バリアントに付加価値があるか**」。リサーチ結果を**価値の有無**という観点で読み直した結果が以下。

### リサーチ結果の読み直し (価値の有無 という観点)

#### 表明された価値観 (stated preference)

| 指標 | 値 | 出典 |
|------|---|-----|
| 「性別にとらわれない教育が望ましい」 | 88.5% | 電通ダイバーシティ・ラボ 2022 (n=681) |
| 性別発言に「もやもや」経験 | 37.9% | 同上 |
| 性別理由で購入を拒否した経験 | 24.5% | 同上 |
| 性差撤廃広告への肯定率 | 74.5% | 電通総研 2022 (n=3,000) |
| 競合アプリの性別セグメント採用 | 0/13 件 | `marketplace-competitor-analysis.md` |

→ **表明レベルでは「押し付けないでほしい」が優勢**

#### 実際の行動 (revealed preference)

| 指標 | 値 | 出典 |
|------|---|-----|
| バレエ教室女児比率 | 98% | 文科省 社会教育調査 |
| 少年サッカー男児比率 | 82% (男:女 4.58:1) | JFA 登録データ |
| 小学生プログラミング男児比率 | 82% (男:女 4.5:1) | 総務省統計 |
| `#男の子ママ` Instagram 投稿数 | 480 万件超 | Instagram 2026-04 |
| `#女の子ママ` 同 | 520 万件超 | 同上 |
| 性別別育児書 (『男の子の育て方』等) | 継続ベストセラー | Amazon 育児書カテゴリ |
| 七五三・こいのぼり・ランドセル色 | 性別慣習残存 | 業界統計 |
| 親世代の性別固定観念保有率 | 76.3% | 内閣府 R3 |

→ **行動レベルでは性差が明確に存在**

### 判断: 日本の家庭における性別バリアントの付加価値は「ある」

- 行動データ (習い事比率、コミュニティ規模、育児書市場、伝統文化) は性差の実在を強く支持
- 表明データ (88.5% が「押し付け反対」) は、**選択肢としての存在を否定していない**。否定されているのは「押し付け」「ふさわしくない発言」「購入強制」
- 日本の家庭には「性差を当然視する層」と「中立を望む層」が**共存**している

### 現行 UI はこの二層構造に両対応している

- 性別必須ではない: セットアップで性別を選ばずに中立パック (`baby-first` 等) だけを受けることも可能
- 中立パックはそのまま並存: 性別バリアントを廃止しなくても中立選好派は問題なく使える
- 子供が見るのは**適用済みプリセット**のみ: 「性別パックが存在すること」自体は子供 UX の複雑度に影響しない

**したがって維持が妥当。** ただし中身の現状には以下の問題があり是正を要する:

1. **非対称**: `elementary-boy` にプログラミングがあり `elementary-girl` にはない / `elementary-girl` にダンス・手芸があり `elementary-boy` にはない。**教育機会の男女平等**観点で不整合
2. **エビデンス不足**: 各活動の登録に「なぜこの活動を入れたか」の市場根拠が記録されていない。Pre-PMF での初期体験クオリティを担保できない
3. **世代カバー穴**: `junior` / `senior` 中立パックの中身が薄い (課題設定がほぼ学習一本)

## Decision (accepted)

### 1. 性別バリアント 15 構造 (中立 5 + 男の子 5 + 女の子 5) を維持する

- 廃止しない
- 統合もしない
- `LEGACY_URL_MAP` / `LEGACY_PACK_ID_MAP` も不要 (既存 ID を維持するため)

### 2. 全活動の中身を市場データで精査する

- `docs/design/marketplace-preset-activity-audit.md` を新設し、全活動を以下の軸で監査:
  - 日本の家庭実施率 (ベネッセ/博報堂こども研/厚労省/文科省 等の一次資料)
  - 教育的推奨根拠 (学習指導要領 / 保育所保育指針 / 小児科学会指針)
  - 年齢適合性 (この年齢帯に妥当か)
  - 性別差エビデンス (統計的有意差があるか、文化的慣習か)
- 判定: KEEP / MOVE_TIER / MOVE_CATEGORY / REMOVE / ADD (新規)

### 3. 既存の非対称を是正する

以下の修正原則を適用:

| 活動種別 | 扱い |
|---------|------|
| **教育機会 (プログラミング/外国語/STEM)** | 男女パック両方に収録。男女差データ (プログラミング 82%:18% 等) は社会環境の結果であり、本アプリは機会平等を提供する立場 |
| **スポーツジャンル (サッカー/野球/バレエ/ダンス)** | 男女の実施率偏り (サッカー 4.58:1 男 / バレエ 98% 女) は revealed preference として反映。ただし「女の子にもサッカー」「男の子にもダンス」の項目は両パックに少量含めて非典型家庭にも配慮 |
| **生活習慣 (歯磨き/着替え/挨拶)** | 性別差なし。全パック共通 |
| **文化慣習 (七五三/こいのぼり/ひな祭り/ランドセル)** | 伝統文化としての性別慣習を反映。男女パックで扱いを分ける。ただし年中行事コレクションは別レイヤーで扱う可能性あり (別 ADR で議論) |
| **創作・表現 (手芸/工作/絵画)** | 男女パック両方に収録。手芸を女の子パック限定にしていた既存設計は男女差エビデンス薄く、教育機会観点で非対称是正 |

### 4. 改訂しない項目

- 子プロフィールの性別属性: **残す** (セットアップで自動パック適用に必要)
- 中立パックの存在: **残す** (中立選好家庭用)
- 性別選択 UI の必須化: **しない** (性別を「選ばない」選択肢を残す)

## 実装方針

1. `docs/design/marketplace-preset-activity-audit.md` の監査結果に基づき 15 JSON を精査 (deep-research 実施中)
2. 監査で判定された ADD / REMOVE / MOVE を JSON に反映
3. 非対称解消:
   - `elementary-girl.json` にプログラミング追加
   - `elementary-boy.json` にダンス/表現活動追加
   - `junior-girl.json` / `junior-boy.json` のダブルスタンダード解消
   - `senior-girl.json` / `senior-boy.json` のキャリア/STEM 機会均等化
4. 監査完了版の 15 JSON を `marketplace-overhaul-spec.md` §5.4 に正式リスト化
5. LEGACY_PACK_ID_MAP は**不要** (ID が変わらないため)

## Consequences

### Positive

- 日本市場の **stated + revealed preference の両層**に対応
- 中立選好家庭: 中立パックを選べる
- 性別特化育児書層・SNS #男の子ママ/#女の子ママ 巨大コミュニティ層: 性別パックで刺さる
- 非典型家庭 (娘がサッカー / 息子がバレエ): 男女パック両方から選ぶ or 中立パック選択可能
- プログラミング機会の男女平等化: 社会問題として認知された教育機会格差に対し、プロダクトとして機会平等を提供する立場を明示

### Negative

- 15 バリアントのメンテナンス負担 (ただし内部管理のみで子供 UX 影響なし)
- 初期コンテンツ作成コスト: 活動 1 個ずつの市場根拠調査が必要 (`marketplace-preset-activity-audit.md` として成果化)
- 電通 24.5% の「性別理由で購入拒否」層へのリーチは限定的 (中立パック誘導の UX 強化で緩和)

### Neutral

- 命名戦略 (ADR-0041「みんなのテンプレート」) とは独立 — 並行して進行可能
- 既存ユーザーの `pack_id='baby-boy'` 等のデータは移行不要 (ID 不変)

## 教訓 (meta)

この ADR 自体が「複雑度最小化」の誤適用事例となった:

- **誤用 1 (初稿)**: 海外・SNS トレンドに引きずられ「日本の実家庭」を調査せず廃止推奨
- **誤用 2 (改訂 1)**: 複雑度最小化原則を「枠を減らす」と機械的に解釈し、興味タグ新設という**真の複雑度増加**を推奨
- **誤用 3 (改訂 2 直前)**: 「(c) を選択」= 性別廃止と勝手に結び付け PO 未承認事項を決定済み扱い

是正: 複雑度最小化は**ユーザ目線に限定** (子供が面する UI / 親がセットアップで面する入力)。内部バリアント数は別次元。ADR 起票時は stated / revealed 両データを並置し、どちらが優先されるべきかの根拠まで含めて記述する。
