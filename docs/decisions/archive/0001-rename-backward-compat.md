## コンテキスト

#571 で URL パスをリネームした際、旧 URL からの 301 リダイレクトを設定しなかった。
結果、ブックマーク・PWA ショートカット・外部リンクが全て 404 になるインシデントが発生。

## 決定

URL をリネーム・廃止する場合は、必ず `src/lib/server/routing/legacy-url-map.ts` の `LEGACY_URL_MAP` にエントリを追加する。個別ページで `redirect()` を書くことは禁止（認可チェックやセッションリダイレクトを除く）。

## 結果

- 旧 URL からの永続リダイレクトが保証される
- リダイレクトの一元管理により、棚卸し・ログ・テストが容易になる
- `LEGACY_URL_MAP` のエントリは永久に残す（ブックマークが生き続けるため）
