# src/routes/ — UI 実装ルール

## デザインシステム（3層トークンアーキテクチャ）
- **hex カラー直書き禁止**: `--color-brand-500` や `--color-action-primary` などのトークンを使用。
- **primitives 必須**: ボタンは `$lib/ui/primitives/Button.svelte` 等を使用し、タグ直書きを避ける。
- **Ark UI 直接 import 禁止**: 必ず primitives ラッパを経由する。
- **スタイル制限**: `<style>` ブロックは50行以下を推奨。

## 用語管理
- **用語辞書の SSOT**: `src/lib/domain/labels.ts` を唯一の正解とする (ADR-0009)。
- **年齢帯 variant**: 分岐をコンポーネントに散らさず、ラベル層 (`getLabel`) に寄せる (ADR-0015)。

## チュートリアル修正ルール
1. **全ステップ通し操作**: デスクトップ・モバイル両方で崩れがないか確認。
2. **証跡**: PR に全ステップのスクリーンショットを添付。

## 旧 URL 廃止ルール (#578)
- `src/lib/server/routing/legacy-url-map.ts` にエントリを追加する。
- ページ個別での `redirect()` は禁止。

## UI 変更の Done 基準
- **実機検証**: ブラウザで実際に開き、デザインシステム準拠と崩れがないかを確認。
- **モバイル対応**: レスポンシブモードで主要ブレークポイントを確認。
- **LP 同期**: 必要に応じて `site/` 側の文言も更新。
