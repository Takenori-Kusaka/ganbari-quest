# tests/ — テスト品質ルール

## テスト品質の維持 (ADR-0005)
- **閾値の維持**: `vite.config.ts` のカバレッジ閾値を下げない。
- **回避禁止**: `test.skip()` や `waitForTimeout()` の安易な使用を禁止。
- **サービス経由**: サービステストは必ず公開 API を経由し、DB 直接操作を避ける。

## スキーマ変更時の必須テスト (ADR-0031)
- **カラム追加**: `default()` の設定と、マイグレーションスクリプトでのバックフィル。
- **NULL 安全性**: NULL 混在行での動作検証テストを 1 件以上追加。

## E2E ガイダンス
- **全年齢モード**: baby/kinder/lower/upper/teen の全モードでデータを用意。
- **プラン別検証**: `DEBUG_PLAN` 環境変数を使用してフリー/スタンダード/ファミリープランの挙動を確認。

## cron テスト
- `tests/e2e/helpers.ts` の `getCronHeaders()` を使用し、認証を適切にハンドリングする。
